
# 前端项目初始化笔记

## 1. 先说结论

如果是现在开始学习和新建项目，推荐组合如下：

- Vue 项目：`Vue 3 + Vite + pnpm`
- React 项目：`React 19 + Vite + pnpm`
- Vue 2：主要用于学习旧项目和维护历史项目
- 包管理器：入门先会 `npm`，实际开发优先 `pnpm`
- `cnpm`：了解即可，主要用于镜像加速场景

---

## 2. Vue 2、Vue 3、React 当前主流版本

> 这里说的是“主流版本线”，不是“必须安装的唯一精确版本”。

| 技术 | 当前适合写进笔记的主流版本 | 说明 |
|---|---|---|
| Vue 2 | `2.7.x` | Vue 2 的最终 minor 版本线，适合老项目维护 |
| Vue 3 | `3.5.x` | 当前稳定主流版本线，适合新项目 |
| React | `19.x` | 当前主流 major 版本线，适合新项目 |

### 2.1 更细一点的理解

- Vue 2：如果你想写精确版本，可以写 `2.7.16`
- Vue 3：笔记里更建议写 `3.5.x`
- React：笔记里更建议写 `19.x`

### 2.2 为什么不直接写最新版本

因为初始化项目时，很多时候我们更关心的是：

- 当前稳定版本线
- 生态是否成熟
- 教程和社区是否容易对齐
- 是否适合学习和企业项目

所以写成：

- Vue 2：`2.7.x`
- Vue 3：`3.5.x`
- React：`19.x`

通常比死盯最新补丁号更实用。

---

## 3. Vue 2 项目怎么创建

### 3.1 主流方式：Vue CLI

Vue 2 常见的创建方式是：

```bash
npm install -g @vue/cli
vue create my-vue2-app
```

创建时在交互选项中选择：

- `Vue 2`

### 3.2 特点

- 构建工具通常是 `webpack`
- 常见于旧项目
- 适合学习历史项目结构
- 不适合作为现在新项目的首选方案

### 3.3 注意

- Vue 2 已于 **2023-12-31** 官方 EOL
- Vue 2 的最终 minor 版本线是 `2.7.x`
- 如果是新项目，不推荐优先学习 Vue 2 作为主线

---

## 4. Vue 3 项目怎么创建

### 4.1 官方推荐方式：create-vue

```bash
npm create vue@latest
```

或者：

```bash
pnpm create vue@latest
```

然后按提示选择：

- TypeScript 是否启用
- Router 是否启用
- Pinia 是否启用
- ESLint / Prettier 是否启用

### 4.2 另一种常见方式：直接用 Vite 创建

```bash
npm create vite@latest my-vue3-app -- --template vue
```

或者：

```bash
pnpm create vite my-vue3-app --template vue
```

### 4.3 Vue 3 的特点

- 当前新项目首选
- 通常搭配 Vite
- 性能更好
- 生态更现代
- 更适合配合 TypeScript、Pinia、组合式 API

---

## 5. React 项目怎么创建

> React 官方现在不推荐再用 Create React App（CRA），CRA 已被弃用。  
> 学习和普通 SPA 项目，最常见的是 `React + Vite`。

### 5.1 用 Vite 创建 React 项目

```bash
npm create vite@latest my-react-app -- --template react
```

或者：

```bash
pnpm create vite my-react-app --template react
```

如果想直接创建 TypeScript 版本：

```bash
npm create vite@latest my-react-app -- --template react-ts
```

或者：

```bash
pnpm create vite my-react-app --template react-ts
```

### 5.2 创建完成后启动项目

```bash
cd my-react-app
npm install
npm run dev
```

或者：

```bash
cd my-react-app
pnpm install
pnpm dev
```

### 5.3 React 新项目的常见选择

- 学习基础 React：`React + Vite`
- 做全栈 React：通常会考虑 `Next.js` 或 `React Router` 方案
- 纯前端 SPA：`Vite` 仍然非常常见

## 6. npm、cnpm、pnpm 的使用

## 6.1 npm

npm 是 Node.js 默认的包管理器。

常用命令：

```bash
npm init
npm install
npm install axios
npm install -D vite
npm run dev
npm run build
```

适合：

- 初学者
- 通用项目
- 所有 Node 前端工程基础入门

---

## 6.2 cnpm

`cnpm` 本质上是面向镜像源的 npm 客户端，命令风格和 npm 很接近。

安装：

```bash
npm install cnpm -g --registry=https://registry.npmmirror.com
```

使用：

```bash
cnpm install
cnpm install axios
cnpm install -D vite
cnpm run dev
```

适合：

- 网络环境导致 npm 安装慢
- 需要使用镜像源加速

注意：

- 现在更常见的做法是直接给 `npm` 或 `pnpm` 配镜像
- 所以 `cnpm` 需要了解，但不一定要作为主力工具

---

## 6.3 pnpm

pnpm 是现在非常流行的包管理器。

特点：

- 安装更快
- 更省磁盘空间
- 对 monorepo / workspace 更友好
- 更适合现代前端工程

常用命令：

```bash
pnpm install
pnpm add axios
pnpm add -D vite
pnpm dev
pnpm build
```

如果本机没安装 pnpm，可以先用 Corepack：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

---

## 6.4 三者对比

| 工具 | 特点 | 适合谁 |
|---|---|---|
| npm | 官方默认，兼容性最好 | 初学者、通用项目 |
| cnpm | 镜像友好，命令接近 npm | 网络较慢环境 |
| pnpm | 快、节省空间、工程化体验更好 | 现代前端项目 |

### 建议

- 入门先掌握 `npm`
- 实际开发优先使用 `pnpm`
- 网络慢时了解 `cnpm` 或镜像配置

---

## 7. Vite 是什么

Vite 是一个现代前端构建工具，可以理解为：

- 开发阶段：启动本地开发服务器
- 构建阶段：打包项目用于上线

### 7.1 Vite 的优点

- 启动快
- 热更新快
- 配置相对更简单
- 非常适合 Vue 3 和 React 新项目

### 7.2 常用命令

```bash
npm run dev
npm run build
npm run preview
```

或者：

```bash
pnpm dev
pnpm build
pnpm preview
```

### 7.3 Vite 的注意点

- `index.html` 在项目根目录
- Vite 适合现代浏览器开发
- 现在 Vue 3 和 React 新项目都很常用它

├─ public/
├─ src/

```text

├─ src/

│  ├─ pages/

└─ vite.config.js / vite.config.ts
```

## 9. 如果不想用“最新版本”，该怎么写依赖

这里要分清两件事：

- `create-vue@latest` / `create-vite@latest`：这是脚手架工具版本
- `vue` / `react`：这是你项目真正依赖的框架版本

也就是说：

- 创建项目时可以用最新脚手架
- 但项目依赖可以固定到你想要的主流版本线

### 9.1 版本写法示例

#### Vue 2

```json
{
  "dependencies": {
    "vue": "2.7.16"
  }
}
```

#### Vue 3

```json
{
  "dependencies": {
    "vue": "~3.5.0"
  }
}
```

#### React

```json
{
  "dependencies": {
    "react": "~19.2.0",
    "react-dom": "~19.2.0"
  }
}
```

### 9.2 `^` 和 `~` 的区别

- `^3.5.0`：允许升级到 `3.x` 内更高 minor / patch
- `~3.5.0`：只允许升级到 `3.5.x` 内更高 patch
- `3.5.34`：完全固定到某一个精确版本

### 9.3 学习阶段推荐写法

- 想稳一点：用 `~`
- 想完全固定：用精确版本
- 不想频繁漂移到新小版本：少用 `^`
