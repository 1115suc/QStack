## Docker 部署数据库

### Docker 部署 MySQL

```shell
mkdir -p /SuChan/docker/volume/MySQL/log
mkdir -p /SuChan/docker/volume/MySQL/data
mkdir -p /SuChan/docker/volume/MySQL/conf
mkdir -p /SuChan/docker/volume/MySQL/mysql-files
```

```shell
# 启动临时容器
docker run -d --name mysql_temp mysql:5.7

# 拷贝默认配置文件到宿主机
docker cp mysql_temp:/etc/mysql/my.cnf /SuChan/docker/volume/MySQL/conf/my.cnf

# 停止并删除临时容器
docker stop mysql_temp && docker rm mysql_temp
```

- 修改配置文件关键项：

```shell
vim /SuChan/docker/volume/MySQL/conf/my.cnf
```

```properties
[mysqld]
# 字符集
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
# 时区
default-time-zone='+8:00'
# 最大连接数
max_connections=1000
# 日志
general_log=ON
general_log_file=/var/log/mysql/mysql.log
slow_query_log=ON
slow_query_log_file=/var/log/mysql/slow.log
long_query_time=2
# InnoDB 缓冲池（按实际内存调整）
innodb_buffer_pool_size=512M
[client]
default-character-set=utf8mb4
[mysql]
default-character-set=utf8mb4
```

```shell
docker run -d \
      --name MySQL \
      --restart unless-stopped \
      --privileged=true \
      -p 3306:3306 \
      -e TZ=Asia/Shanghai \
      -e MYSQL_CHARACTER_SET_SERVER=utf8mb4 \
      -e MYSQL_COLLATION_SERVER=utf8mb4_unicode_ci \
      -e MYSQL_DEFAULT_TIME_ZONE='+8:00' \
      -e MYSQL_ROOT_PASSWORD=24364726 \
      -v /SuChan/docker/volume/MySQL/log:/var/log/mysql \
      -v /SuChan/docker/volume/MySQL/data:/var/lib/mysql \
      -v /SuChan/docker/volume/MySQL/conf:/etc/mysql/conf.d \
      -v /SuChan/docker/volume/MySQL/mysql-files:/var/lib/mysql-files/ \
      mysql:5.7
```

> **关键参数说明：**
> - `--privileged=true`：赋予容器与宿主机相同的权限，允许访问宿主机所有设备
> - `MYSQL_ROOT_PASSWORD`：root 账号密码，**请替换为强密码**
> - `3306`：MySQL 默认端口

### Docker 部署 MySQL 8+

```shell
mkdir -p /SuChan/docker/volume/MySQL8/log
mkdir -p /SuChan/docker/volume/MySQL8/data
mkdir -p /SuChan/docker/volume/MySQL8/conf
mkdir -p /SuChan/docker/volume/MySQL8/mysql-files
```

```shell
# 启动临时容器
docker run -d --name mysql8_temp mysql:8.0

# 拷贝默认配置文件到宿主机
docker cp mysql8_temp:/etc/mysql/my.cnf /SuChan/docker/volume/MySQL8/conf/my.cnf

# 停止并删除临时容器
docker stop mysql8_temp && docker rm mysql8_temp
```

修改配置文件关键项：

```shell
vim /SuChan/docker/volume/MySQL8/conf/my.cnf
```

```properties
[mysqld]
# 字符集（MySQL 8 默认已是 utf8mb4，建议显式声明）
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
# 时区
default-time-zone='+8:00'
# 最大连接数
max_connections=1000
# MySQL 8 新认证方式（若旧客户端/驱动不兼容，可改为 mysql_native_password）
default_authentication_plugin=caching_sha2_password
# 日志
general_log=ON
general_log_file=/var/log/mysql/mysql.log
slow_query_log=ON
slow_query_log_file=/var/log/mysql/slow.log
long_query_time=2
binlog_expire_logs_seconds=604800
# InnoDB 缓冲池（按实际内存调整）
innodb_buffer_pool_size=512M
[client]
default-character-set=utf8mb4
[mysql]
default-character-set=utf8mb4
```

```shell
docker run -d \
      --name MySQL8 \
      --restart unless-stopped \
      --privileged=true \
      -p 3306:3306 \
      -e TZ=Asia/Shanghai \
      -e MYSQL_CHARACTER_SET_SERVER=utf8mb4 \
      -e MYSQL_COLLATION_SERVER=utf8mb4_unicode_ci \
      -e MYSQL_DEFAULT_TIME_ZONE='+8:00' \
      -e MYSQL_ROOT_PASSWORD=24364726 \
      -v /SuChan/docker/volume/MySQL8/log:/var/log/mysql \
      -v /SuChan/docker/volume/MySQL8/data:/var/lib/mysql \
      -v /SuChan/docker/volume/MySQL8/conf:/etc/mysql/conf.d \
      -v /SuChan/docker/volume/MySQL8/mysql-files:/var/lib/mysql-files/ \
      mysql:8.0
```

> **与 MySQL 5.7 的主要差异：**
> - `default_authentication_plugin=caching_sha2_password`：MySQL 8 默认认证方式改为 `caching_sha2_password`，若 Navicat
    等旧客户端连接报错，可改为 `mysql_native_password`
> - `binlog_expire_logs_seconds`：MySQL 8 使用该参数替代 5.7 的 `expire_logs_days` 控制 binlog 过期时间
> - MySQL 8 默认字符集已是 `utf8mb4`，无需像 5.7 一样额外强制指定，但显式声明更稳妥

```shell
mkdir -p /SuChan/docker/volume/Redis/data
mkdir -p /SuChan/docker/volume/Redis/logs
mkdir -p /SuChan/docker/volume/Redis/config
```

```shell
# 启动临时容器
docker run -d --name redis_temp redis:latest

# 拷贝默认配置文件到宿主机
docker cp redis_temp:/etc/redis/redis.conf /SuChan/docker/volume/Redis/config/redis.conf

# 停止并删除临时容器
docker stop redis_temp && docker rm redis_temp
```

> 若拷贝失败（部分镜像无默认配置文件），可从 Redis 官方下载对应版本的 redis.conf：
> https://redis.io/docs/management/config/

修改配置文件关键项：

```shell
vim /SuChan/docker/volume/Redis/config/redis.conf
```

```properties
# 允许所有 IP 访问（默认只监听 127.0.0.1）
bind 0.0.0.0
# 关闭保护模式（允许外部连接）
protected-mode no
# 设置密码
requirepass 24364726
# 开启持久化 AOF
appendonly yes
appendfilename "appendonly.aof"
# 日志文件路径
logfile /data/logs/redis.log
# 最大内存限制（按实际情况调整）
maxmemory 512mb
maxmemory-policy allkeys-lru
```

```shell
docker run -d \
      --name Redis \
      --restart unless-stopped \
      --privileged=true \
      -p 6379:6379 \
      -e TZ=Asia/Shanghai \
      -v /SuChan/docker/volume/Redis/data:/data \
      -v /SuChan/docker/volume/Redis/logs:/data/logs \
      -v /SuChan/docker/volume/Redis/config/redis.conf:/etc/redis/redis.conf \
      redis:latest \
      redis-server /etc/redis/redis.conf
```

### Docker 部署 Redis Stack

```shell
# 创建挂载目录 (Redis Stack 不需要复杂的日志和备份目录映射，主要是持久化数据和配置)
mkdir -p /SuChan/docker/volume/RedisStack/data
mkdir -p /SuChan/docker/volume/RedisStack/config
```

```shell
# 启动临时容器
docker run -d --name redis_temp redis/redis-stack:latest

# 拷贝默认配置文件到宿主机
# 注意：Redis Stack 的核心配置通常通过环境变量传递，这里的 conf 留作深度定制化使用
docker cp redis_temp:/etc/redis-stack.conf /SuChan/docker/volume/RedisStack/config/redis-stack.conf

# 停止并删除临时容器
docker stop redis_temp && docker rm redis_temp
```

```shell
# 启动正式容器
docker run -d \
      --name RedisStack \
      --restart unless-stopped \
      -v /SuChan/docker/volume/RedisStack/data:/data \
      -v /SuChan/docker/volume/RedisStack/config/redis-stack.conf:/etc/redis-stack.conf \
      -p 6379:6379 \
      -p 8001:8001 \
      --privileged=true \
      -e REDIS_ARGS="--requirepass 24364726 --appendonly yes" \
      -e TZ=Asia/Shanghai \
      redis/redis-stack:latest 
```

> 参数与核心配置解析
> **--privileged=true \**  
> 当使用 `--privileged=true` 选项运行容器时，Docker 会赋予容器几乎与主机相同的权限。具体来说，这个选项做了以下两件事情：
> 1. 给容器添加了所有的 capabilities。
> 2. 允许容器访问主机的所有设备。（对于 Redis 这种需要极高性能进行内存与磁盘 IO 交换的数据库，此参数能避免一些宿主机权限带来的性能瓶颈）。
>
> **-p 8001:8001 \**  
> Redis Stack 特有参数。它内置了 **RedisInsight (Web UI 控制台)**。  
> 部署成功后，你可以直接在浏览器访问 `http://宿主机IP:8001`。**强烈推荐使用它**，在这个可视化界面里，你可以直接看到你存入的 LangChain4j 向量数据，排查维度问题非常方便！
>
> **-e REDIS_ARGS="--requirepass 24364726 --appendonly yes" \**  
> Redis Stack 推荐使用 `REDIS_ARGS` 环境变量来传递启动参数：
> * `--requirepass 24364726`：设置 Redis 的连接密码（也就是你之前的 24364726）。
> * `--appendonly yes`：开启 AOF 增量持久化。防止容器重启后，你存入的向量数据和聊天记忆丢失。

### Docker 部署 Elasticsearch & Kibana

```shell
mkdir -p /SuChan/docker/volume/Elasticsearch/data
mkdir -p /SuChan/docker/volume/Elasticsearch/logs
mkdir -p /SuChan/docker/volume/Elasticsearch/plugins
mkdir -p /SuChan/docker/volume/Elasticsearch/config
mkdir -p /SuChan/docker/volume/Kibana/config
```

1. 拉取镜像

```shell
# 拉取 Elasticsearch
docker pull elasticsearch:7.17.5

# 拉取 Kibana
docker pull kibana:7.17.5
```

2. 创建 Docker 网络（ES 与 Kibana 需在同一网络下通信）

```shell
docker network create es-net
```

3. 启动临时容器，拷贝默认配置文件

```shell
# 启动临时容器
docker run -d --name es_temp elasticsearch:7.17.5

# 拷贝默认配置文件到宿主机
docker cp es_temp:/usr/share/elasticsearch/config/elasticsearch.yml /SuChan/docker/volume/Elasticsearch/config/elasticsearch.yml

# 停止并删除临时容器
docker stop es_temp && docker rm es_temp
```

```shell
# 启动临时容器
docker run -d --name kibana_temp kibana:7.17.5

# 拷贝默认配置文件到宿主机
docker cp kibana_temp:/usr/share/kibana/config/kibana.yml /SuChan/docker/volume/Kibana/config/kibana.yml

# 停止并删除临时容器
docker stop kibana_temp && docker rm kibana_temp
```

4. 修改 Elasticsearch 配置文件

```shell
vim /SuChan/docker/volume/Elasticsearch/config/elasticsearch.yml
```

```yaml
# 集群名称
cluster.name: "es-cluster"

# 节点名称
node.name: es-node-1

# 允许所有 IP 访问
network.host: 0.0.0.0

# 单节点模式
discovery.type: single-node

# 关闭 xpack 安全认证（开发环境，生产环境建议开启）
xpack.security.enabled: false

# 跨域配置（Kibana 连接需要）
http.cors.enabled: true
http.cors.allow-origin: "*"
http.cors.allow-headers: Authorization,X-Requested-With,Content-Length,Content-Type
```

5. 修改 Kibana 配置文件

```shell
vim /SuChan/docker/volume/Kibana/config/kibana.yml
```

```yaml
# Kibana 服务端口
server.port: 5601

# 允许所有 IP 访问
server.host: "0.0.0.0"

# Elasticsearch 地址（使用容器名通信）
elasticsearch.hosts: [ "http://elasticsearch:9200" ]

# 设置中文界面
i18n.locale: "zh-CN"
```

6. 修改宿主机 vm.max_map_count（ES 启动必须）

```shell
# 临时生效
sysctl -w vm.max_map_count=262144

# 永久生效
echo "vm.max_map_count=262144" >> /etc/sysctl.conf
sysctl -p
```

7. 启动 Elasticsearch

```shell
docker run -d \
      --name Elasticsearch \
      --restart unless-stopped \
      --privileged=true \
      --network es-net \
      -p 9200:9200 \
      -p 9300:9300 \
      -e TZ=Asia/Shanghai \
      -e "ES_JAVA_OPTS=-Xms512m -Xmx512m" \
      -e "discovery.type=single-node" \
      -v /SuChan/docker/volume/Elasticsearch/config/elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml \
      -v /SuChan/docker/volume/Elasticsearch/data:/usr/share/elasticsearch/data \
      -v /SuChan/docker/volume/Elasticsearch/logs:/usr/share/elasticsearch/logs \
      -v /SuChan/docker/volume/Elasticsearch/plugins:/usr/share/elasticsearch/plugins \
      elasticsearch:7.17.5
```

8. 启动 Kibana

```shell
docker run -d \
      --name Kibana \
      --restart unless-stopped \
      --privileged=true \
      --network es-net \
      -p 5601:5601 \
      -e TZ=Asia/Shanghai \
      -e ELASTICSEARCH_HOSTS=http://elasticsearch:9200 \
      -v /SuChan/docker/volume/Kibana/config/kibana.yml:/usr/share/kibana/config/kibana.yml \
      kibana:7.17.5
```

9. 验证部署

```shell
# 查看容器运行状态
docker ps | grep -E "Elasticsearch|Kibana"

# 查看 Elasticsearch 日志
docker logs -f Elasticsearch

# 查看 Kibana 日志
docker logs -f Kibana

# 测试 Elasticsearch 是否正常响应
curl http://localhost:9200

# 查看集群健康状态
curl http://localhost:9200/_cluster/health?pretty
```

> 启动成功后浏览器访问：
> - Elasticsearch：`http://192.168.150.128:9200`
> - Kibana：`http://192.168.150.128:5601`

> **端口说明：**
> | 端口 | 说明 |
> |------|------|
> | `9200` | ES REST API 端口，客户端读写数据使用 |
> | `9300` | ES 集群节点间通信端口 |
> | `5601` | Kibana 可视化面板端口 |

> **启动顺序：** Elasticsearch → Kibana，需等待 ES 完全启动后再启动 Kibana

### Docker 部署 PostgreSQL

```shell
mkdir -p /SuChan/docker/volume/PostgreSQL/data
mkdir -p /SuChan/docker/volume/PostgreSQL/logs
mkdir -p /SuChan/docker/volume/PostgreSQL/config
mkdir -p /SuChan/docker/volume/PostgreSQL/backup
```

1. 拉取镜像

```shell
docker pull postgres:latest
```

2. 启动临时容器，拷贝默认配置文件

```shell
# 启动临时容器
docker run -d --name postgres_temp \
      -e POSTGRES_PASSWORD=24364726 \
      postgres:latest

# 等待容器完全启动后拷贝配置文件
docker cp postgres_temp:/var/lib/PostgreSQL/data/postgresql.conf /SuChan/docker/volume/PostgreSQL/config/postgresql.conf
docker cp postgres_temp:/var/lib/PostgreSQL/data/pg_hba.conf /SuChan/docker/volume/PostgreSQL/config/pg_hba.conf

# 停止并删除临时容器
docker stop postgres_temp && docker rm postgres_temp
```

3. 修改 postgresql.conf 配置文件

```shell
vim /SuChan/docker/volume/PostgreSQL/config/postgresql.conf
```

```properties
# 监听所有 IP
listen_addresses='*'
# 端口
port=5432
# 最大连接数
max_connections=200
# 时区
timezone='Asia/Shanghai'
log_timezone='Asia/Shanghai'
# 内存配置（按实际内存调整）
shared_buffers=256MB
work_mem=4MB
maintenance_work_mem=64MB
# 日志配置
logging_collector=on
log_directory='/var/log/postgresql'
log_filename='postgresql-%Y-%m-%d.log'
log_rotation_age=1d
log_min_duration_statement=2000
```

4. 修改 pg_hba.conf 配置文件（允许外部连接）

```shell
vim /SuChan/docker/volume/PostgreSQL/config/pg_hba.conf
```

```properties
# TYPE  DATABASE  USER  ADDRESS       METHOD
# 允许本地连接
local all       all                 trust
# 允许本机 IPv4 连接
host all       all   127.0.0.1/32  scram-sha-256
# 允许所有外部 IPv4 连接（生产环境建议限制 IP 段）
host all       all   0.0.0.0/0     scram-sha-256
# 允许所有外部 IPv6 连接
host all       all   ::/0          scram-sha-256
```

5. 正式部署

```shell
docker run -d \
      --name PostgreSQL \
      --restart unless-stopped \
      --privileged=true \
      -p 5432:5432 \
      -e TZ=Asia/Shanghai \
      -e POSTGRES_USER=root \
      -e POSTGRES_PASSWORD=24364726 \
      -e POSTGRES_DB=postgres \
      -e PGDATA=/var/lib/postgresql/data \
      -v /SuChan/docker/volume/PostgreSQL/data:/var/lib/postgresql/data \
      -v /SuChan/docker/volume/PostgreSQL/logs:/var/log/postgresql \
      -v /SuChan/docker/volume/PostgreSQL/backup:/backup \
      -v /SuChan/docker/volume/PostgreSQL/config/postgresql.conf:/etc/postgresql/postgresql.conf \
      -v /SuChan/docker/volume/PostgreSQL/config/pg_hba.conf:/etc/postgresql/pg_hba.conf \
      postgres:latest \
      postgres -c config_file=/etc/postgresql/postgresql.conf \
               -c hba_file=/etc/postgresql/pg_hba.conf
```

6. 验证部署

```shell
# 查看容器运行状态
docker ps | grep PostgreSQL

# 查看启动日志
docker logs -f PostgreSQL

# 进入容器，使用 psql 连接
```

### Docker 部署 RocketMQ

```shell
mkdir -p /SuChan/docker/volume/RocketMQ/nameserver/logs
mkdir -p /SuChan/docker/volume/RocketMQ/nameserver/store
mkdir -p /SuChan/docker/volume/RocketMQ/broker/logs
mkdir -p /SuChan/docker/volume/RocketMQ/broker/store
mkdir -p /SuChan/docker/volume/RocketMQ/broker/config
```

1. 拉取镜像

```shell
# 拉取 RocketMQ 主程序（NameServer + Broker）
docker pull apache/rocketmq:5.3.1

# 拉取 RocketMQ Dashboard 可视化面板
docker pull apacherocketmq/rocketmq-dashboard:latest
```

2. 创建 Broker 配置文件

```shell
vim /SuChan/docker/volume/RocketMQ/broker/config/broker.conf
```

```properties
# Broker 名称
brokerName=broker-a
# 0 表示 Master，>0 表示 Slave
brokerId=0
# NameServer 地址（替换为宿主机实际 IP）
namesrvAddr=192.168.150.128:9876
# Broker 对外暴露的 IP（替换为宿主机实际 IP）
brokerIP1=192.168.150.128
# 默认 Topic 队列数
defaultTopicQueueNums=4
# 是否允许自动创建 Topic（生产环境建议关闭）
autoCreateTopicEnable=true
# 是否允许自动创建订阅组
autoCreateSubscriptionGroup=true
# 存储路径
storePathRootDir=/home/rocketmq/store
storePathCommitLog=/home/rocketmq/store/commitlog
# 消息保留时间（小时）
fileReservedTime=48
# 刷盘方式：ASYNC_FLUSH 异步（性能好）/ SYNC_FLUSH 同步（可靠性高）
flushDiskType=ASYNC_FLUSH
```

3. 启动 NameServer

```shell
docker run -d \
      --name RocketMQ-NameServer \
      --restart unless-stopped \
      --privileged=true \
      -p 9876:9876 \
      -e TZ=Asia/Shanghai \
      -v /SuChan/docker/volume/RocketMQ/nameserver/logs:/home/rocketmq/logs \
      -v /SuChan/docker/volume/RocketMQ/nameserver/store:/home/rocketmq/store \
      apache/rocketmq:5.3.1 \
      sh mqnamesrv
```

4. 启动 Broker

```shell
docker run -d \
      --name RocketMQ-Broker \
      --restart unless-stopped \
      --privileged=true \
      -p 10909:10909 \
      -p 10911:10911 \
      -p 10912:10912 \
      -e TZ=Asia/Shanghai \
      -e JAVA_OPT_EXT="-Xms512m -Xmx512m" \
      -v /SuChan/docker/volume/RocketMQ/broker/logs:/home/rocketmq/logs \
      -v /SuChan/docker/volume/RocketMQ/broker/store:/home/rocketmq/store \
      -v /SuChan/docker/volume/RocketMQ/broker/config/broker.conf:/home/rocketmq/conf/broker.conf \
      apache/rocketmq:5.3.1 \
      sh mqbroker -c /home/rocketmq/conf/broker.conf
```

5. 启动 RocketMQ Dashboard

```shell
docker run -d \
      --name RocketMQ-Dashboard \
      --restart unless-stopped \
      -p 8180:8080 \
      -e TZ=Asia/Shanghai \
      -e JAVA_OPTS="-Drocketmq.namesrv.addr=192.168.150.128:9876" \
      apacherocketmq/rocketmq-dashboard:latest
```

6. 验证部署

```shell
# 查看容器运行状态
docker ps | grep RocketMQ

# 查看 NameServer 日志
docker logs -f RocketMQ-NameServer

# 查看 Broker 日志
docker logs -f RocketMQ-Broker
```

> 启动成功后，浏览器访问 Dashboard：`http://192.168.150.128:8180`

> **端口说明：**
> | 端口 | 说明 |
> |------|------|
> | `9876` | NameServer 端口，Producer/Consumer 通过此端口发现 Broker |
> | `10911` | Broker 监听端口，用于收发消息 |
> | `10909` | Broker VIP 通道端口（10911 - 2）|
> | `10912` | Broker HA 高可用同步端口 |
> | `8180` | Dashboard 可视化面板端口 |

> **启动顺序：** NameServer → Broker → Dashboard，关闭顺序相反
