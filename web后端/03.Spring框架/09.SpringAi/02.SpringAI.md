## 2. Advisor 机制

`Advisor` 类似 AI 调用链上的拦截器，可以在调用模型前后做增强：

```text
用户输入
  -> Advisor 前置处理：注入记忆、RAG 上下文、改写问题、安全检查
  -> ChatModel 调用
  -> Advisor 后置处理：保存记忆、记录日志、处理响应
  -> 返回最终答案
```

常用内置 Advisor：

| Advisor | 作用 |
|---|---|
| `MessageChatMemoryAdvisor` | 把历史消息按消息列表方式注入上下文 |
| `PromptChatMemoryAdvisor` | 把历史消息拼接进 Prompt |
| `QuestionAnswerAdvisor` | 简单 RAG，把向量检索结果注入上下文 |
| `RetrievalAugmentationAdvisor` | 更完整的 RAG 流程，适合复杂检索增强 |
| `SafeGuardAdvisor` | 基于规则的输入安全拦截 |

## 3. 内存聊天记忆

内存记忆适合学习和 Demo，应用重启后会丢失。

```java
import org.springframework.ai.chat.memory.ChatMemory;
import org.springframework.ai.chat.memory.MessageWindowChatMemory;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class ChatMemoryConfig {

    @Bean
    public ChatMemory chatMemory() {
        return MessageWindowChatMemory.builder()
                .maxMessages(20)
                .build();
    }
}
```

使用 `MessageChatMemoryAdvisor`：

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.client.advisor.MessageChatMemoryAdvisor;
import org.springframework.ai.chat.client.advisor.AbstractChatMemoryAdvisor;
import org.springframework.ai.chat.memory.ChatMemory;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/ai/memory")
public class MemoryChatController {

    private final ChatClient chatClient;

    public MemoryChatController(ChatClient.Builder builder, ChatMemory chatMemory) {
        this.chatClient = builder
                .defaultSystem("你是一个专业的 Java 后端开发助手。")
                .defaultAdvisors(MessageChatMemoryAdvisor.builder(chatMemory).build())
                .build();
    }

    @PostMapping("/chat")
    public String chat(@RequestBody ChatRequest request) {
        return chatClient.prompt()
                .user(request.message())
                .advisors(spec -> spec.param(
                        AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                        request.conversationId()))
                .call()
                .content();
    }

    public record ChatRequest(String conversationId, String message) {
    }
}
```

测试：

```bash
curl -X POST http://localhost:8080/ai/memory/chat \
  -H "Content-Type: application/json" \
  -d '{"conversationId":"u001","message":"我叫张三，是Java开发者"}'

curl -X POST http://localhost:8080/ai/memory/chat \
  -H "Content-Type: application/json" \
  -d '{"conversationId":"u001","message":"你还记得我叫什么吗"}'
```

## 4. 会话隔离

聊天记忆依赖 `conversationId` 隔离。生产环境不要完全信任前端传入的用户 ID，推荐从登录态生成。

```java
import jakarta.servlet.http.HttpServletRequest;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.client.advisor.AbstractChatMemoryAdvisor;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/ai/user")
public class UserChatController {

    private final ChatClient chatClient;

    public UserChatController(ChatClient chatClient) {
        this.chatClient = chatClient;
    }

    @PostMapping("/chat")
    public String chat(@RequestBody String message, HttpServletRequest request) {
        String conversationId = request.getSession().getId();

        return chatClient.prompt()
                .user(message)
                .advisors(spec -> spec.param(
                        AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                        conversationId))
                .call()
                .content();
    }
}
```

如果集成 Spring Security，通常使用：

```java
String conversationId = SecurityContextHolder.getContext()
        .getAuthentication()
        .getName();
```

多会话场景可以拼接业务维度：

```java
String conversationId = "user:" + userId + ":conversation:" + conversationNo;
```

## 5. JDBC 持久化记忆

JDBC 记忆适合 MySQL、PostgreSQL 等关系型数据库。

依赖：

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-chat-memory-repository-jdbc</artifactId>
</dependency>

<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>
```

配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/spring_ai?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root
    driver-class-name: com.mysql.cj.jdbc.Driver
  ai:
    chat:
      memory:
        repository:
          jdbc:
            initialize-schema: always
```

手动配置示例：

```java
import org.springframework.ai.chat.memory.ChatMemory;
import org.springframework.ai.chat.memory.MessageWindowChatMemory;
import org.springframework.ai.chat.memory.repository.jdbc.JdbcChatMemoryRepository;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class JdbcMemoryConfig {

    @Bean
    public ChatMemory chatMemory(JdbcChatMemoryRepository repository) {
        return MessageWindowChatMemory.builder()
                .chatMemoryRepository(repository)
                .maxMessages(20)
                .build();
    }
}
```

## 8. MongoDB 持久化记忆

MongoDB 记忆适合文档型存储场景。Spring AI 1.1.6 中使用 repository 命名：

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-chat-memory-repository-mongodb</artifactId>
</dependency>
```

配置示例：

```yaml
spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/spring_ai
  ai:
    chat:
      memory:
        repository:
          mongo:
            create-indices: true
            ttl: 86400
```

手动配置示例：

```java
import org.springframework.ai.chat.memory.ChatMemory;
import org.springframework.ai.chat.memory.MessageWindowChatMemory;
import org.springframework.ai.chat.memory.repository.mongodb.MongoChatMemoryRepository;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.mongodb.core.MongoTemplate;

@Configuration
public class MongoMemoryConfig {

    @Bean
    public ChatMemory mongoChatMemory(MongoTemplate mongoTemplate) {
        MongoChatMemoryRepository repository = MongoChatMemoryRepository.builder()
                .mongoTemplate(mongoTemplate)
                .build();

        return MessageWindowChatMemory.builder()
                .chatMemoryRepository(repository)
                .maxMessages(20)
                .build();
    }
}
```

## 9. Tool Calling 基础

Tool Calling 允许模型在需要时调用 Java 方法，例如查数据库、查订单、查天气。

定义工具：

```java
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

import org.springframework.ai.tool.annotation.Tool;
import org.springframework.ai.tool.annotation.ToolParam;
import org.springframework.stereotype.Component;

@Component
public class CommonTools {

    @Tool(description = "获取当前日期和时间")
    public String currentTime() {
        return LocalDateTime.now()
                .format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    }

    @Tool(description = "查询指定城市的天气")
    public Weather weather(@ToolParam(description = "城市名称，例如北京、上海") String city) {
        return new Weather(city, "晴", 26, "东南风3级");
    }

    public record Weather(String city, String condition, int temperature, String wind) {
    }
}
```

注册工具：

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/ai/tools")
public class ToolController {

    private final ChatClient chatClient;

    public ToolController(ChatClient.Builder builder, CommonTools commonTools) {
        this.chatClient = builder
                .defaultSystem("你是一个智能助手，可以在需要时调用工具。")
                .defaultTools(commonTools)
                .build();
    }

    @GetMapping("/chat")
    public String chat(@RequestParam String message) {
        return chatClient.prompt()
                .user(message)
                .call()
                .content();
    }
}
```

测试：

```bash
curl "http://localhost:8080/ai/tools/chat?message=现在几点"
curl "http://localhost:8080/ai/tools/chat?message=北京天气怎么样"
```

## 10. 动态注册工具

有些工具只应在特定场景开放，可以按请求动态传入。

```java
@GetMapping("/dynamic")
public String dynamic(@RequestParam String message,
                      @RequestParam boolean enableWeather) {
    ChatClient.ChatClientRequestSpec spec = chatClient.prompt().user(message);

    if (enableWeather) {
        spec = spec.tools(commonTools);
    }

    return spec.call().content();
}
```

## 11. ToolContext 传递安全上下文

`ToolContext` 用于传递不应该交给模型决定的上下文，例如当前登录用户 ID、租户 ID。

```java
import java.math.BigDecimal;
import java.util.List;

import org.springframework.ai.chat.model.ToolContext;
import org.springframework.ai.tool.annotation.Tool;
import org.springframework.stereotype.Component;

@Component
public class OrderTools {

    @Tool(description = "查询当前登录用户的订单列表")
    public List<OrderSummary> myOrders(ToolContext toolContext) {
        String userId = (String) toolContext.getContext().get("userId");
        if (userId == null) {
            throw new IllegalStateException("用户未登录");
        }

        return List.of(new OrderSummary("ORD-10001", "已发货", new BigDecimal("99.00")));
    }

    public record OrderSummary(String orderId, String status, BigDecimal amount) {
    }
}
```

调用时传入：

```java
return chatClient.prompt()
        .user("我的订单发货了吗")
        .tools(orderTools)
        .toolContext(Map.of("userId", currentUserId))
        .call()
        .content();
```

## 12. 工具设计原则

1. 工具方法名称和描述要清晰，描述会影响模型是否调用工具。
2. 参数尽量使用简单类型或 record，方便 JSON 序列化。
3. 工具内部必须做权限校验，不能只相信模型传入的参数。
4. 对有副作用的工具，例如下单、退款、删除数据，建议加入二次确认流程。
5. 工具返回值不要过大，必要时分页或摘要。

## 13. 自定义 Advisor：日志记录

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.ai.chat.client.advisor.api.AdvisedRequest;
import org.springframework.ai.chat.client.advisor.api.AdvisedResponse;
import org.springframework.ai.chat.client.advisor.api.CallAroundAdvisor;
import org.springframework.ai.chat.client.advisor.api.CallAroundAdvisorChain;
import org.springframework.core.Ordered;
import org.springframework.stereotype.Component;

@Component
public class LoggingAdvisor implements CallAroundAdvisor {

    private static final Logger log = LoggerFactory.getLogger(LoggingAdvisor.class);

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest request, CallAroundAdvisorChain chain) {
        long start = System.currentTimeMillis();
        log.info("[AI Request] {}", request.userText());

        AdvisedResponse response = chain.nextAroundCall(request);

        long cost = System.currentTimeMillis() - start;
        log.info("[AI Response] cost={}ms", cost);
        return response;
    }

    @Override
    public String getName() {
        return "LoggingAdvisor";
    }

    @Override
    public int getOrder() {
        return Ordered.HIGHEST_PRECEDENCE;
    }
}
```

注册：

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class ChatClientConfig {

    @Bean
    public ChatClient chatClient(ChatClient.Builder builder,
                                 LoggingAdvisor loggingAdvisor,
                                 CommonTools commonTools) {
        return builder
                .defaultSystem("你是一个专业助手。")
                .defaultAdvisors(loggingAdvisor)
                .defaultTools(commonTools)
                .build();
    }
}
```

## 14. 自定义 Advisor：输入拦截

```java
import java.util.List;

import org.springframework.ai.chat.client.advisor.api.AdvisedRequest;
import org.springframework.ai.chat.client.advisor.api.AdvisedResponse;
import org.springframework.ai.chat.client.advisor.api.CallAroundAdvisor;
import org.springframework.ai.chat.client.advisor.api.CallAroundAdvisorChain;
import org.springframework.stereotype.Component;

@Component
public class KeywordBlockAdvisor implements CallAroundAdvisor {

    private static final List<String> BLOCKED = List.of("敏感词1", "敏感词2");

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest request, CallAroundAdvisorChain chain) {
        String userText = request.userText();
        boolean blocked = BLOCKED.stream().anyMatch(userText::contains);
        if (blocked) {
            throw new IllegalArgumentException("输入包含不允许的内容");
        }
        return chain.nextAroundCall(request);
    }

    @Override
    public String getName() {
        return "KeywordBlockAdvisor";
    }

    @Override
    public int getOrder() {
        return 0;
    }
}
```

## 15. 常见问题

### 15.1 依赖下载不到

优先检查是否误用了这些旧名称：

```text
spring-ai-model-chat-memory-jdbc
spring-ai-model-chat-memory-redis
spring-ai-model-chat-memory-mongodb
```

在 Spring AI 1.1.6 中，推荐使用官方 repository starter，例如：

```text
spring-ai-starter-model-chat-memory-repository-jdbc
spring-ai-starter-model-chat-memory-repository-cassandra
spring-ai-starter-model-chat-memory-repository-neo4j
spring-ai-starter-model-chat-memory-repository-mongodb
```

### 15.2 记忆串话

原因通常是多个用户复用了同一个 `conversationId`。生产环境要从认证上下文生成 ID，例如：

```java
String conversationId = "user:" + userId + ":conv:" + conversationIdFromRequest;
```

### 15.3 工具被错误调用

优化 `@Tool(description = "...")` 和 `@ToolParam(description = "...")`。描述越具体，模型选择工具越准确。

### 15.4 工具方法有副作用

例如付款、退款、删除订单，不要让模型一步执行。推荐流程：

1. 模型先生成操作计划。
2. 后端返回待确认操作。
3. 用户确认后，再调用真正的业务接口。
