# Spring AI 进阶使用指南

## 1. Advisor 机制概览

`Advisor` 是 Spring AI 的核心扩展点，类似 AOP 切面，用于在 AI 调用前后注入逻辑。

```
用户输入
  │
  ▼
[Advisor 链（Pre 阶段）]  ← 注入上下文、历史记忆、RAG 文档等
  │
  ▼
[ChatModel 调用]
  │
  ▼
[Advisor 链（Post 阶段）] ← 保存记忆、日志记录、结果过滤等
  │
  ▼
最终响应
```

Spring AI 内置多个 Advisor：

| Advisor | 作用 |
|---|---|
| `MessageChatMemoryAdvisor` | 将对话历史注入为消息列表 |
| `PromptChatMemoryAdvisor` | 将对话历史注入为 System Prompt 文本 |
| `VectorStoreChatMemoryAdvisor` | 从向量数据库检索长期记忆注入 System Prompt |
| `QuestionAnswerAdvisor` | RAG：从向量数据库检索相关文档注入用户问题上下文 |
| `RetrievalAugmentationAdvisor` | 高级 RAG Advisor（1.1+ 推荐） |
| `SafeGuardAdvisor` | 关键词过滤安全守卫 |

## 2. 聊天记忆——基础使用

### 2.1 内存记忆（MessageWindowChatMemory）

`MessageWindowChatMemory` 是最简单的内存实现，保留最近 N 条消息窗口。

```java
@Configuration
public class ChatMemoryConfig {

    /**
     * 配置默认 ChatMemory（内存实现）
     * 保留最近 20 条消息（10 轮对话）
     */
    @Bean
    public ChatMemory chatMemory() {
        return MessageWindowChatMemory.builder()
                .maxMessages(20)          // 保留最近 20 条消息
                .build();
    }
}
```

```java
@RestController
@RequestMapping("/ai/memory")
public class MemoryChatController {

    private final ChatClient chatClient;

    public MemoryChatController(ChatClient.Builder builder, ChatMemory chatMemory) {
        this.chatClient = builder
                .defaultSystem("你是一个专业的 Java 编程助手，请记住用户告诉你的信息。")
                // 挂载记忆 Advisor（默认使用 MessageChatMemoryAdvisor）
                .defaultAdvisors(MessageChatMemoryAdvisor.builder(chatMemory).build())
                .build();
    }

    /**
     * 带记忆的聊天接口
     * POST /ai/memory/chat
     * Body: {"sessionId": "session-001", "message": "我叫张三"}
     */
    @PostMapping("/chat")
    public String chatWithMemory(@RequestBody ChatWithMemoryRequest request) {
        return chatClient.prompt()
                .user(request.message())
                // 通过 Advisor 参数指定会话 ID（记忆隔离的关键）
                .advisors(advisorSpec -> advisorSpec
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               request.sessionId())
                )
                .call()
                .content();
    }

    record ChatWithMemoryRequest(String sessionId, String message) {}
}
```

## 3. 用户间聊天记忆的隔离

### 3.1 基于会话 ID 的隔离策略

Spring AI 的记忆隔离依赖 **conversationId**，每个用户/会话使用唯一 ID 即可天然隔离。

```java
@RestController
@RequestMapping("/ai/user-chat")
public class UserIsolatedChatController {

    private final ChatClient chatClient;

    /**
     * 推荐：使用 Spring Security 的当前用户 ID 作为 conversationId
     * 或者 HttpSession ID、JWT 中的 userId 等
     */
    public UserIsolatedChatController(ChatClient.Builder builder, ChatMemory chatMemory) {
        this.chatClient = builder
                .defaultAdvisors(MessageChatMemoryAdvisor.builder(chatMemory).build())
                .build();
    }

    /**
     * 生产级用户聊天接口
     * conversationId 从 JWT Token / Security Context 中获取，不依赖前端传参
     */
    @PostMapping("/chat")
    public String chat(@RequestBody String message,
                       HttpServletRequest request) {
        // 方案一：使用 HttpSession ID（会话级隔离）
        String conversationId = request.getSession().getId();

        // 方案二：使用当前登录用户 ID（用户级隔离，推荐）
        // String conversationId = SecurityContextHolder.getContext()
        //         .getAuthentication().getName();  // 获取 username

        // 方案三：用户 ID + 对话主题（多对话场景）
        // String conversationId = userId + ":topic:" + topicId;

        return chatClient.prompt()
                .user(message)
                .advisors(spec -> spec.param(
                        AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                        conversationId))
                .call()
                .content();
    }

    /**
     * 清除指定用户的聊天记忆
     */
    @DeleteMapping("/memory/{userId}")
    public void clearMemory(@PathVariable String userId,
                             ChatMemory chatMemory) {
        chatMemory.clear(userId);
    }

    /**
     * 查看指定会话的历史记录
     */
    @GetMapping("/history/{conversationId}")
    public List<Message> getHistory(@PathVariable String conversationId,
                                     ChatMemory chatMemory) {
        return chatMemory.get(conversationId);
    }
}
```

### 4.4 完整的持久化 + 隔离方案（生产级示例）

```java
@RestController
@RequestMapping("/api/v1/chat")
@RequiredArgsConstructor
public class ProductionChatController {

    private final ChatClient chatClient;
    private final ChatMemory chatMemory;

    /**
     * 生产级聊天接口（集成 JWT 认证 + MongoDB/Redis 持久化）
     */
    @PostMapping
    public ResponseEntity<ChatMessageResponse> chat(
            @RequestBody ChatMessageRequest request,
            @AuthenticationPrincipal UserDetails userDetails) {

        // 用户 ID 来自认证上下文，不信任前端传参
        String userId = userDetails.getUsername();
        String conversationId = buildConvId(userId, request.conversationId());

        String reply = chatClient.prompt()
                .system("你是一名专业的客服助手。")
                .user(request.content())
                .advisors(spec -> spec
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               conversationId)
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_RETRIEVE_SIZE_KEY,
                               20)  // 最多携带 20 条历史
                )
                .call()
                .content();

        return ResponseEntity.ok(new ChatMessageResponse(conversationId, reply));
    }

    /**
     * 获取对话历史
     */
    @GetMapping("/history")
    public List<MessageVO> history(@RequestParam String conversationId,
                                    @AuthenticationPrincipal UserDetails userDetails) {
        String fullConvId = buildConvId(userDetails.getUsername(), conversationId);
        return chatMemory.get(fullConvId).stream()
                .map(msg -> new MessageVO(msg.getMessageType().name(), msg.getText()))
                .toList();
    }

    /**
     * 清除对话历史
     */
    @DeleteMapping("/history")
    public void clearHistory(@RequestParam String conversationId,
                              @AuthenticationPrincipal UserDetails userDetails) {
        chatMemory.clear(buildConvId(userDetails.getUsername(), conversationId));
    }

    private String buildConvId(String userId, String conversationId) {
        return "user:" + userId + ":conv:" + conversationId;
    }

    record ChatMessageRequest(String conversationId, String content) {}
    record ChatMessageResponse(String conversationId, String reply) {}
    record MessageVO(String type, String content) {}
}
```

## 5. Tool 工具调用

Tool Calling 允许模型在需要时调用 Java 方法，获取实时信息或执行操作。

### 5.1 基础工具定义（注解方式，推荐）

```java
/**
 * 工具类：提供天气查询和时间查询能力
 * 注意：工具类的方法参数和返回值必须能被 JSON 序列化
 */
@Component
public class CommonTools {

    /**
     * 获取当前时间（让模型知道当前时间）
     */
    @Tool(description = "获取当前日期和时间，当用户询问时间相关问题时使用")
    public String getCurrentDateTime() {
        return LocalDateTime.now()
                .format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    }

    /**
     * 查询天气信息
     *
     * @param city 城市名称（中文或英文）
     * @return 天气信息字符串
     */
    @Tool(description = "查询指定城市的实时天气信息，当用户询问天气时使用")
    public WeatherInfo getWeather(
            @ToolParam(description = "需要查询天气的城市名称，例如：北京、上海") String city) {
        // 实际生产中调用真实天气 API
        // 此处为演示模拟数据
        return new WeatherInfo(city, "晴天", 25, "东南风 3级", 60);
    }

    /**
     * 查询数据库中的订单信息
     */
    @Tool(description = "根据订单号查询订单详情，当用户询问订单状态时使用")
    public OrderInfo queryOrder(

        // 实际业务中注入 OrderService 查询数据库
        return new OrderInfo(orderId, "已发货", "2024-01-15", "张三", 299.00);
    }

    // 工具返回的数据结构
    record WeatherInfo(String city, String condition, int temperature,
```

### 5.2 在 ChatClient 中注册工具

```java
@RestController
@RequestMapping("/ai/tools")
public class ToolChatController {

    private final ChatClient chatClient;
    private final CommonTools commonTools;

    public ToolChatController(ChatClient.Builder builder, CommonTools commonTools) {
        this.commonTools = commonTools;
        // 方式一：全局注册（所有请求都携带这些工具）
        this.chatClient = builder
                .defaultSystem("你是一个智能助手，可以查询天气、时间和订单信息。")
                .defaultTools(commonTools)    // 注册工具对象
                .build();
    }

    /**
     * 全局工具调用聊天
     */
    @GetMapping("/chat")
    public String chatWithTools(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .call()
                .content();
    }

    /**
     * 方式二：按请求动态注册工具（精细控制）
     */
    @GetMapping("/dynamic-chat")
    public String chatWithDynamicTools(@RequestParam String question,
                                        @RequestParam(required = false) boolean needWeather) {
        var promptSpec = chatClient.prompt().user(question);

        if (needWeather) {
            // 只在需要时注册天气工具
            promptSpec = chatClient.prompt()
                    .user(question)
                    .tools(commonTools);  // 覆盖或追加工具
        }

        return promptSpec.call().content();
    }
}
```

### 6.1 工具方法使用 Spring Bean 注入

```java
@Component
public class DatabaseTools {
```

### 6.2 工具执行上下文（ToolContext）

`ToolContext` 允许在工具调用时传递额外的上下文信息（如当前用户 ID、租户 ID 等），这些信息不通过模型传递，而是通过 Spring 传递。

```java
@Component
public class SecureTools {

    private final OrderService orderService;

    public SecureTools(OrderService orderService) {
        this.orderService = orderService;
    }

    /**
     * 带安全上下文的工具：只能查询当前用户自己的订单
     * ToolContext 参数由 Spring AI 框架注入，不是模型参数
     */
    @Tool(description = "查询当前登录用户的订单列表")
    public List<OrderSummary> getMyOrders(ToolContext toolContext) {
        // 从 ToolContext 中获取框架传入的上下文信息
        String currentUserId = (String) toolContext.getContext().get("currentUserId");

        if (currentUserId == null) {
            throw new SecurityException("用户未登录");
        }

        return orderService.findByUserId(currentUserId).stream()
                .map(o -> new OrderSummary(o.getId(), o.getStatus(), o.getAmount()))
                .toList();
    }

    record OrderSummary(String id, String status, BigDecimal amount) {}
}
```

## 7. 自定义 Advisor

### 7.1 实现请求日志 Advisor

```java
/**
 * 自定义 Advisor：记录每次 AI 请求的日志
 */
@Component
public class LoggingAdvisor implements CallAroundAdvisor {

    private static final Logger log = LoggerFactory.getLogger(LoggingAdvisor.class);

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest advisedRequest,
                                       CallAroundAdvisorChain chain) {
        // Pre 处理：请求前日志
        String userMessage = advisedRequest.userText();
        log.info("[AI Request] 用户输入: {}", userMessage);
        long startTime = System.currentTimeMillis();

        // 执行实际的 AI 调用
        AdvisedResponse response = chain.nextAroundCall(advisedRequest);

        // Post 处理：响应后日志
        long elapsed = System.currentTimeMillis() - startTime;
        String content = response.response().getResult().getOutput().getText();
        log.info("[AI Response] 耗时: {}ms, 回复长度: {} 字符", elapsed, content.length());

        return response;
    }

    @Override
    public String getName() {
        return "LoggingAdvisor";
    }

    @Override
    public int getOrder() {
        return Ordered.HIGHEST_PRECEDENCE;  // 最先执行
    }
}
```

### 7.2 关键词安全守卫 Advisor

```java
/**
 * 自定义 Advisor：敏感关键词过滤
 */
@Component
public class ContentSafetyAdvisor implements CallAroundAdvisor {

    private static final List<String> BLOCKED_KEYWORDS = List.of(
            "违禁词1", "违禁词2", "敏感内容"
    );

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest advisedRequest,
                                       CallAroundAdvisorChain chain) {
        String userText = advisedRequest.userText();

        // 检查用户输入是否包含违禁词
        boolean hasBlockedContent = BLOCKED_KEYWORDS.stream()
                .anyMatch(userText::contains);

        if (hasBlockedContent) {
            // 直接返回拒绝响应，不调用模型
            throw new IllegalArgumentException("您的输入包含不当内容，请重新输入。");
        }

        return chain.nextAroundCall(advisedRequest);
    }

    @Override
    public String getName() {
        return "ContentSafetyAdvisor";
    }

    @Override
    public int getOrder() {
        return 0;  // 在日志 Advisor 之后执行
    }
}
```

```java
    public ChatClient chatClient(ChatClient.Builder builder,
                                  ChatMemory chatMemory,
                                  LoggingAdvisor loggingAdvisor,
                                  ContentSafetyAdvisor safetyAdvisor) {
        return builder
                .defaultSystem("你是一个专业助手。")
                .defaultAdvisors(
                        loggingAdvisor,          // 日志（最先）
                        safetyAdvisor,           // 安全检查
                        MessageChatMemoryAdvisor.builder(chatMemory).build()  // 记忆（最后）
                )
                .build();
    }
}
```
