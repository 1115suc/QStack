# Spring AI Alibaba 进阶使用指南

## 1. Advisor 机制详解

Advisor 是 Spring AI（含 Alibaba）的 AOP 扩展点，在 AI 调用链前后注入自定义逻辑。Spring AI Alibaba 完全兼容 Spring AI 的 Advisor 体系。

```
用户请求
   │
   ▼
┌─────────────────────────────────────┐
│  Advisor 链（Pre 阶段）             │
│  ① SimpleLoggerAdvisor（日志）     │
│  ② ContentSafetyAdvisor（安全）    │
│  ③ MessageChatMemoryAdvisor（记忆）│
└─────────────────────────────────────┘
   │
   ▼
 DashScopeChatModel（通义千问）
   │
   ▼
┌─────────────────────────────────────┐
│  Advisor 链（Post 阶段）            │
│  ③ 保存对话到记忆存储              │
│  ① 记录响应日志                    │
└─────────────────────────────────────┘
   │
   ▼
返回响应
```

### Spring AI Alibaba 内置 Advisor 一览

| Advisor | 作用 |
|---|---|
| `SimpleLoggerAdvisor` | 自动打印请求/响应日志（调试利器） |
| `MessageChatMemoryAdvisor` | 将对话历史注入为消息列表（短期记忆） |
| `PromptChatMemoryAdvisor` | 将对话历史注入到 System Prompt 文本 |
| `VectorStoreChatMemoryAdvisor` | 从向量数据库检索长期记忆 |
| `QuestionAnswerAdvisor` | RAG：检索向量库相关文档注入上下文 |
| `RetrievalAugmentationAdvisor` | 高级 RAG（含查询扩展/重排序） |
| `SafeGuardAdvisor` | 关键词安全过滤守卫 |

## 2. 聊天记忆——基础使用

### 2.1 内存记忆（MessageWindowChatMemory）

```java
@Configuration
public class ChatMemoryConfig {

    /**
     * 内存聊天记忆（保留最近 20 条消息，即 10 轮对话）
     * 应用重启后数据丢失，开发/测试环境使用
     */
    @Bean
    public ChatMemory inMemoryChatMemory() {
        return MessageWindowChatMemory.builder()
                .maxMessages(20)
                .build();
    }
}
```

```java
@RestController
@RequestMapping("/ai/memory")
public class MemoryChatController {

    private final ChatClient chatClient;

    public MemoryChatController(ChatClient.Builder builder,
                                 ChatMemory chatMemory) {
        this.chatClient = builder
                .defaultSystem("你是一个专业的通义千问助手，请记住用户告诉你的信息，保持对话连贯。")
                // ⚠️ 挂载记忆 Advisor（核心步骤）
                .defaultAdvisors(
                        new SimpleLoggerAdvisor(),    // 日志（可选）
                        MessageChatMemoryAdvisor.builder(chatMemory).build()
                )
                .build();
    }

    /**
     * 带记忆的对话接口
     * POST /ai/memory/chat
     * Body: {"conversationId": "conv-001", "message": "我叫张三，是一名 Java 工程师"}
     */
    @PostMapping("/chat")
    public String chatWithMemory(@RequestBody MemoryChatRequest request) {
        return chatClient.prompt()
                .user(request.message())
                // 通过 Advisor 参数指定会话 ID（记忆隔离的关键！）
                .advisors(spec -> spec
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               request.conversationId())
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_RETRIEVE_SIZE_KEY,
                               20)   // 最多携带 20 条历史消息
                )
                .call()
                .content();
    }

    /**
     * 流式记忆对话
     */
    @PostMapping(value = "/stream-chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> streamChatWithMemory(@RequestBody MemoryChatRequest request) {
        return chatClient.prompt()
                .user(request.message())
                .advisors(spec -> spec
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               request.conversationId())
                )
                .stream()
                .content();
    }

    record MemoryChatRequest(String conversationId, String message) {}
}
```

## 3. 用户间聊天记忆的隔离

### 3.1 基于 conversationId 的隔离策略

Spring AI Alibaba 的记忆隔离依赖 **`conversationId`** 参数，每个用户或对话线程使用唯一 ID 即可天然隔离。

```
用户 A（userId=1001）
  └─ conversationId = "user:1001:conv:default"
      └─ 历史：["你好", "我叫张三"]

用户 B（userId=1002）
  └─ conversationId = "user:1002:conv:default"
      └─ 历史：["Hello", "My name is Li Si"]

用户 A 的对话 2（同一用户多个对话）
  └─ conversationId = "user:1001:conv:work-assistant"
      └─ 历史：["帮我写一份会议纪要"]
```

```java
@RestController
@RequestMapping("/api/v1/chat")
@RequiredArgsConstructor
public class UserIsolatedChatController {

    private final ChatClient chatClient;
    private final ChatMemory chatMemory;

    /**
     * 生产级用户聊天接口
     * conversationId 从 JWT / Security Context 中获取，不信任前端传参！
     */
    @PostMapping
    public ResponseEntity<ChatReply> chat(
            @RequestBody ChatMessage request,
            @AuthenticationPrincipal UserDetails userDetails) {

        String userId = userDetails.getUsername();

        // 方案一：单对话（用户级隔离）
        String conversationId = "user:" + userId;

        // 方案二：多对话（用户 + 对话编号隔离，支持同一用户多个对话线程）
        // String conversationId = "user:" + userId + ":conv:" + request.conversationId();

        String reply = chatClient.prompt()
                .user(request.content())
                .advisors(spec -> spec
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               conversationId)
                )
                .call()
                .content();

        return ResponseEntity.ok(new ChatReply(conversationId, reply));
    }

    /**
     * 获取指定会话的历史记录
     */
    @GetMapping("/history")
    public List<MessageVO> getHistory(
            @RequestParam String conversationId,
            @AuthenticationPrincipal UserDetails userDetails) {

        String fullConvId = "user:" + userDetails.getUsername()
                + ":conv:" + conversationId;

        return chatMemory.get(fullConvId).stream()
                .map(msg -> new MessageVO(
                        msg.getMessageType().name(),
                        msg.getText()))
                .toList();
    }

    /**
     * 清除指定会话记忆
     */
    @DeleteMapping("/history")
    public void clearHistory(
            @RequestParam String conversationId,
            @AuthenticationPrincipal UserDetails userDetails) {

        String fullConvId = "user:" + userDetails.getUsername()
                + ":conv:" + conversationId;
        chatMemory.clear(fullConvId);
    }

    record ChatMessage(String conversationId, String content) {}
    record ChatReply(String conversationId, String reply) {}
    record MessageVO(String type, String content) {}
}
```

### 3.2 多对话管理服务

```java
/**
 * 对话管理服务：同一用户支持多个独立对话线程
 */
@Service
@RequiredArgsConstructor
public class ConversationManagerService {

    private final ChatClient chatClient;
    private final ChatMemory chatMemory;

    /**
     * 构建标准 conversationId：用户 ID + 对话主题
     */
    public String buildConversationId(String userId, String topicOrConvId) {
        return "user:" + userId + ":conv:" + topicOrConvId;
    }

    /**
     * 发送消息（带记忆隔离）
     */
    public String chat(String userId, String conversationId, String message) {
        String fullConvId = buildConversationId(userId, conversationId);

        return chatClient.prompt()
                .user(message)
                .advisors(spec -> spec
                        .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               fullConvId)
                )
                .call()
                .content();
    }

    /**
     * 获取指定对话历史
     */
    public List<Message> getConversationHistory(String userId, String conversationId) {
        return chatMemory.get(buildConversationId(userId, conversationId));
    }

    /**
     * 清除对话记忆
     */
    public void clearConversation(String userId, String conversationId) {
        chatMemory.clear(buildConversationId(userId, conversationId));
    }
}
```

## 5. Tool 工具调用

Tool Calling 允许通义千问模型在推理时调用 Java 方法获取实时数据或执行操作。

### 5.1 定义工具（@Tool 注解）

```java
/**
 * 通用工具集
 * 注意：工具类可以注入 Spring Bean，但应保证线程安全
 */
@Component
public class CommonTools {

    /**
     * 获取当前日期时间
     * 通义千问不知道当前时间，需要通过工具获取
     */
    @Tool(description = "获取当前日期和时间，当用户询问时间、日期相关问题时使用")
    public String getCurrentDateTime() {
        return LocalDateTime.now()
                .format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    }

    /**
     * 查询城市天气
     */
    @Tool(description = "查询指定城市的当前天气情况，包括温度、天气状况、风力等信息")
    public WeatherInfo getWeather(
            @ToolParam(description = "需要查询天气的城市名称，如：北京、上海、杭州") String city) {
        // 实际项目中调用真实天气 API（如高德天气、和风天气）
        return new WeatherInfo(city, "晴转多云", 28, "东南风 3级", 65);
    }

    /**
     * 汇率查询
     */
    @Tool(description = "查询两种货币之间的实时汇率，当用户询问货币换算时使用")
    public String getExchangeRate(
            @ToolParam(description = "源货币代码，如：CNY、USD、EUR") String from,
            @ToolParam(description = "目标货币代码，如：CNY、USD、EUR") String to) {
        // 实际项目调用汇率 API
        return String.format("1 %s = 7.25 %s（参考汇率，实时数据请查询银行）", from, to);
    }

    /**
     * 计算器工具
     */
    @Tool(description = "执行数学计算，支持加减乘除和基础数学运算")
    public double calculate(
            @ToolParam(description = "数学表达式，如：100 * 0.85 - 20") String expression) {
        try {
            // 实际项目使用 Aviator 或 JEP 等表达式引擎
            return Double.parseDouble(expression); // 简化示例
        } catch (Exception e) {
            throw new RuntimeException("计算表达式无效：" + expression);
        }
    }

    record WeatherInfo(String city, String condition, int temperature,
                       String wind, int humidity) {}
}
```

### 5.2 注册工具并使用

```java
@RestController
@RequestMapping("/ai/tools")
@RequiredArgsConstructor
public class ToolChatController {

    private final ChatClient chatClient;
    private final CommonTools commonTools;

    /**
     * 方式一：全局注册（所有请求都携带这些工具）
     */
    @Bean
    public ChatClient toolChatClient(ChatClient.Builder builder, CommonTools tools) {
        return builder
                .defaultSystem("""
                        你是一个智能助手，可以查询实时天气、时间、汇率等信息。
                        请在合适时主动调用工具获取准确信息，不要凭空猜测。
                        """)
                .defaultTools(tools)    // 注册工具对象
                .build();
    }

    /**
     * 工具调用对话
     * GET /ai/tools/chat?question=北京今天天气怎么样？
     */
    @GetMapping("/chat")
    public String chatWithTools(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .call()
                .content();
    }

    /**
     * 方式二：按请求动态注册工具（精细控制，避免暴露不必要工具）
     * GET /ai/tools/dynamic?question=现在几点了？&needTime=true
     */
    @GetMapping("/dynamic")
    public String chatWithDynamicTools(
            @RequestParam String question,
            @RequestParam(defaultValue = "false") boolean needTime,
            @RequestParam(defaultValue = "false") boolean needWeather) {

        var promptBuilder = chatClient.prompt().user(question);

        // 仅按需注册工具（减少 Token 消耗，避免工具滥用）
        List<Object> tools = new ArrayList<>();
        if (needTime) tools.add(commonTools);
        if (needWeather) tools.add(commonTools);

        if (!tools.isEmpty()) {
            promptBuilder = chatClient.prompt()
                    .user(question)
                    .tools(tools.toArray());
        }

        return promptBuilder.call().content();
    }
}
```

## 6. 工具调用进阶

### 6.1 工具注入 Spring Bean（访问业务数据库）

```java
/**
 * 业务数据库工具：注入 Repository 查询数据
 */
@Component
@RequiredArgsConstructor
public class BusinessDatabaseTools {

    private final OrderRepository orderRepository;
    private final ProductService productService;
    private final UserService userService;

    @Tool(description = "根据订单号查询订单状态、金额、发货信息。格式：ORD-XXXXXXXX")
    public OrderDetail queryOrder(
            @ToolParam(description = "订单编号，格式：ORD-XXXXXXXX") String orderId) {
        return orderRepository.findById(orderId)
                .map(order -> new OrderDetail(
                        order.getId(),
                        order.getStatus().getDescription(),
                        order.getTotalAmount(),
                        order.getTrackingNumber(),
                        order.getEstimatedDelivery()))
                .orElseThrow(() -> new RuntimeException("订单不存在：" + orderId));
    }

    @Tool(description = "查询商品库存、价格、规格信息，支持商品名称或商品ID查询")
    public ProductInfo queryProduct(
            @ToolParam(description = "商品名称或商品ID") String productQuery) {
        return productService.findByNameOrId(productQuery)
                .map(p -> new ProductInfo(p.getName(), p.getStock(),
                                          p.getPrice(), p.getSpecs()))
                .orElse(null);
    }

    record OrderDetail(String id, String status, BigDecimal amount,
                       String trackingNo, String estimatedDelivery) {}
    record ProductInfo(String name, int stock, BigDecimal price, String specs) {}
}
```

### 6.2 使用 ToolContext 传递安全上下文

`ToolContext` 允许在工具调用时安全传递后端上下文（如当前用户 ID、租户 ID），这些参数不通过模型传递，避免提示注入攻击。

```java
/**
 * 需要鉴权的工具：只允许用户查询自己的数据
 */
@Component
@RequiredArgsConstructor
public class SecureUserTools {

    private final OrderRepository orderRepository;

    /**
     * ToolContext 参数由 Spring AI 框架注入，不是模型参数
     * 模型无法控制这个参数的值，确保安全
     */
    @Tool(description = "查询当前登录用户的所有订单列表，无需提供用户ID")
    public List<OrderSummary> getMyOrders(ToolContext toolContext) {
        // 从 ToolContext 获取后端传入的用户 ID（不信任模型提供）
        String userId = (String) toolContext.getContext().get("currentUserId");

        if (userId == null) {
            throw new SecurityException("用户未登录，无法查询订单");
        }

        return orderRepository.findByUserId(userId).stream()
                .map(o -> new OrderSummary(o.getId(), o.getStatus().name(),
                                           o.getTotalAmount()))
                .toList();
    }

    record OrderSummary(String id, String status, BigDecimal amount) {}
}
```

```java
/**
 * 在 Controller 中注入安全上下文到工具调用
 */
@PostMapping("/secure-chat")
public String secureChatWithContext(
```

## 7. 自定义 Advisor

### 7.1 请求响应日志 Advisor

```java
/**
 * 自定义 Advisor：记录完整的 AI 调用日志
 */
@Component
@Slf4j
public class AiAuditLogAdvisor implements CallAroundAdvisor {

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest advisedRequest,
                                       CallAroundAdvisorChain chain) {
        String conversationId = (String) advisedRequest.adviseContext()
                .getOrDefault(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY, "unknown");

        // Pre：记录请求
        log.info("[AI-AUDIT] 请求开始 | conversationId={} | 用户输入={}",
                conversationId,
                abbreviate(advisedRequest.userText(), 100));

        long startTime = System.currentTimeMillis();

        // 执行 AI 调用
        AdvisedResponse response = chain.nextAroundCall(advisedRequest);

        // Post：记录响应
        long elapsed = System.currentTimeMillis() - startTime;
        String content = response.response().getResult().getOutput().getText();
        Usage usage = response.response().getMetadata().getUsage();

        log.info("[AI-AUDIT] 请求完成 | conversationId={} | 耗时={}ms | " +
                        "promptTokens={} | completionTokens={} | 响应长度={}",
                conversationId,
                elapsed,
                usage != null ? usage.getPromptTokens() : 0,
                usage != null ? usage.getGenerationTokens() : 0,
                content.length());

        return response;
    }

    @Override
    public String getName() {
        return "AiAuditLogAdvisor";
    }

    @Override
    public int getOrder() {
        return Ordered.HIGHEST_PRECEDENCE;   // 最先执行
    }

    private String abbreviate(String text, int maxLen) {
        return text != null && text.length() > maxLen
                ? text.substring(0, maxLen) + "..."
                : text;
    }
}
```

### 7.2 内容安全 Advisor（敏感词过滤）

```java
/**
 * 内容安全 Advisor：检测并拦截敏感内容
 */
@Component
public class ContentSafetyAdvisor implements CallAroundAdvisor {

    // 敏感词库（实际项目可从数据库或配置中心加载）
    private static final List<String> SENSITIVE_WORDS = List.of(
            "违禁词A", "违禁词B", "敏感内容C"
    );

    // 正则模式（检测变体写法）
    private static final List<Pattern> PATTERNS = List.of(
            Pattern.compile("\\b(admin|root)\\s+password\\b", Pattern.CASE_INSENSITIVE)
    );

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest advisedRequest,
                                       CallAroundAdvisorChain chain) {
        String userText = advisedRequest.userText();

        // 检查敏感词
        for (String word : SENSITIVE_WORDS) {
            if (userText.contains(word)) {
                throw new ContentSafetyException("您的输入包含不当内容，请重新表述。");
            }
        }

        // 检查正则模式
        for (Pattern pattern : PATTERNS) {
            if (pattern.matcher(userText).find()) {
                throw new ContentSafetyException("检测到潜在安全风险，请勿尝试获取敏感信息。");
            }
        }

        return chain.nextAroundCall(advisedRequest);
    }

    @Override
    public String getName() {
        return "ContentSafetyAdvisor";
    }

    @Override
    public int getOrder() {
        return 10;   // 在日志 Advisor 之后执行
    }

    public static class ContentSafetyException extends RuntimeException {
        public ContentSafetyException(String message) {
            super(message);
        }
    }
}
```

```java
/**
 * Token 用量监控 Advisor：集成 Micrometer 指标
 */
@Component
@RequiredArgsConstructor
public class TokenUsageAdvisor implements CallAroundAdvisor {

    private final MeterRegistry meterRegistry;

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest advisedRequest,
                                       CallAroundAdvisorChain chain) {
        AdvisedResponse response = chain.nextAroundCall(advisedRequest);

        // 提取 Token 用量
        Usage usage = response.response().getMetadata().getUsage();
        String model = response.response().getMetadata().getModel();

        if (usage != null) {
            // 记录输入 Token 指标
            meterRegistry.counter("qwen.token.usage",
                            "type", "prompt",
                            "model", model != null ? model : "unknown")
                    .increment(usage.getPromptTokens());

            // 记录输出 Token 指标
            meterRegistry.counter("qwen.token.usage",
                            "type", "completion",
                            "model", model != null ? model : "unknown")
                    .increment(usage.getGenerationTokens());
        }

        return response;
    }

    @Override
    public String getName() {
        return "TokenUsageAdvisor";
    }

    @Override
    public int getOrder() {
        return Ordered.LOWEST_PRECEDENCE;   // 最后执行
    }
}
```

### 8.2 文生图使用

```java
@RestController
@RequestMapping("/ai/image")
@RequiredArgsConstructor
public class ImageGenerationController {

    private final ImageModel imageModel;  // 自动注入 DashScopeImageModel

    /**
     * 文生图接口
     * POST /ai/image/generate
     * Body: {"prompt": "一只可爱的熊猫在竹林中玩耍，动漫风格，高清"}
     */
    @PostMapping("/generate")
    public ImageGenerateResult generateImage(@RequestBody ImageGenerateRequest request) {
        ImageOptions options = DashScopeImageOptions.builder()
                .withModel("wanx2.1-t2i-turbo")
                .withN(request.count())
                .withWidth(1024)
                .withHeight(1024)
                .build();

        ImageResponse response = imageModel.call(
                new ImagePrompt(request.prompt(), options)
        );

        List<String> imageUrls = response.getResults().stream()
                .map(result -> result.getOutput().getUrl())
                .toList();

        return new ImageGenerateResult(request.prompt(), imageUrls);
    }

    record ImageGenerateRequest(String prompt, int count) {}
    record ImageGenerateResult(String prompt, List<String> imageUrls) {}
}
```
