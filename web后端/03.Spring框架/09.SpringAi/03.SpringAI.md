# Spring AI 高级使用指南

## 1. 模型上下文协议（MCP）

MCP（Model Context Protocol）是由 Anthropic 主导的开放标准，让 AI 模型以统一协议访问外部工具和资源。Spring AI 1.1 对 MCP 提供了完整支持。

### 1.1 MCP 架构说明

```
┌─────────────────────────────────────────────┐
│              Spring AI 应用                  │
│                                             │
│  ┌─────────────┐    ┌──────────────────┐   │
│  │  ChatClient  │───▶│  MCP Client      │   │
│  └─────────────┘    │  (工具桥接层)    │   │
│                      └────────┬─────────┘   │
└───────────────────────────────┼─────────────┘
                                │ MCP 协议（STDIO / SSE / HTTP）
          ┌─────────────────────┼──────────────────────┐
          │                     │                      │
   ┌──────▼──────┐    ┌────────▼────────┐   ┌────────▼────────┐
   │ MCP Server  │    │  MCP Server     │   │  MCP Server     │
   │ （本地 JS） │    │  （Python 服务）│   │  （Java 服务）  │
   │ Filesystem  │    │  数据库查询     │   │  企业系统工具   │
   └─────────────┘    └─────────────────┘   └─────────────────┘
```

### 1.2 作为 MCP Client 消费外部工具

#### 依赖配置

```xml
<!-- MCP Client 依赖 -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-mcp-client-spring-boot-starter</artifactId>
</dependency>

<!-- 如果 MCP Server 通过 SSE 传输（WebFlux 项目） -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```

#### STDIO 方式连接 MCP Server（本地进程）

```yaml
# application.yml
spring:
  ai:
    mcp:
      client:
        enabled: true
        # 同步客户端（默认）
        type: SYNC
        stdio:
          connections:
            # 连接到文件系统 MCP Server（Node.js）
            filesystem-server:
              command: npx
              args:
                - "-y"
                - "@modelcontextprotocol/server-filesystem"
                - "/path/to/allowed/directory"

            # 连接到自定义 Python MCP Server
            python-tools:
              command: python
              args:
                - "/path/to/mcp_server.py"
              env:
                DATABASE_URL: "postgresql://localhost:5432/mydb"
```

#### SSE/HTTP 方式连接 MCP Server（远程服务）

```yaml
spring:
  ai:
    mcp:
      client:
        enabled: true
        type: SYNC
        sse:
          connections:
            # 连接到远程 MCP Server
            remote-tools:
              url: http://mcp-server.example.com/sse
            # 本地 Spring Boot MCP Server
            local-spring-server:
              url: http://localhost:8081/sse
```

#### 在代码中使用 MCP 工具

```java
@RestController
@RequestMapping("/ai/mcp")
public class McpChatController {

    private final ChatClient chatClient;

    /**
     * Spring AI 会自动将 MCP Server 暴露的工具注册到 ChatClient
     * 通过 McpToolCallbackProvider 桥接
     */
    public McpChatController(ChatClient.Builder builder,
                              McpToolCallbackProvider mcpToolCallbackProvider) {
        this.chatClient = builder
                .defaultSystem("你是一个智能助手，可以访问文件系统和数据库。")
                // 注册 MCP 工具（来自所有已连接的 MCP Server）
                .defaultTools(mcpToolCallbackProvider.getToolCallbacks())
                .build();
    }

    @GetMapping("/chat")
    public String chat(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .call()
                .content();
    }
}
```

## 2. 检索增强生成（RAG）

RAG 将外部知识库与 LLM 结合，解决模型知识截止问题，提高回答准确性。

### 2.1 RAG 工作流程

```
用户问题
   │
   ▼
[Embedding] ──▶ 用户问题向量化
   │
   ▼
[向量检索] ──▶ 从向量数据库检索 TopK 相关文档
   │
   ▼
[上下文注入] ──▶ 将检索结果拼接到 Prompt
   │
   ▼
[LLM 生成] ──▶ 基于上下文生成回答
   │
   ▼
最终回答
```

### 2.2 使用 QuestionAnswerAdvisor（简单 RAG）

```java
@Configuration
public class RagConfig {

    /**
     * 配置 RAG 的 QuestionAnswerAdvisor
     * 自动将向量检索结果注入到用户问题的上下文中
     */
    @Bean
    public QuestionAnswerAdvisor questionAnswerAdvisor(VectorStore vectorStore) {
        return QuestionAnswerAdvisor.builder(vectorStore)
                // 检索时的过滤条件（可选）
                .searchRequest(SearchRequest.builder()
                        .topK(5)              // 返回最相关的 5 个文档片段
                        .similarityThreshold(0.7)  // 相似度阈值（0-1）
                        .build())
                .build();
    }

    @Bean
    public ChatClient ragChatClient(ChatClient.Builder builder,
                                    QuestionAnswerAdvisor questionAnswerAdvisor) {
        return builder
                .defaultSystem("""
                        你是一个专业的知识助手。
                        请基于提供的上下文信息回答问题，如果上下文中没有相关信息，
                        请明确告知用户，不要编造答案。
                        """)
                .defaultAdvisors(questionAnswerAdvisor)
                .build();
    }
}
```

```java
@RestController
@RequestMapping("/ai/rag")
public class RagController {

    private final ChatClient ragChatClient;

    public RagController(ChatClient ragChatClient) {
        this.ragChatClient = ragChatClient;
    }

    /**
     * 基于知识库的问答
     */
    @PostMapping("/ask")
    public String ask(@RequestBody AskRequest request) {
        return ragChatClient.prompt()
                .user(request.question())
                // 动态过滤：只检索特定分类的文档
                .advisors(spec -> spec.param(
                        QuestionAnswerAdvisor.FILTER_EXPRESSION,
                        "category == '" + request.category() + "'"
                ))
                .call()
                .content();
    }

    record AskRequest(String question, String category) {}
}
```

### 3.5 向量数据库操作

```java
@Service
public class QdrantVectorService {

    private final VectorStore vectorStore;

    public QdrantVectorService(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    /**
     * 添加文档到向量数据库
     */
    public void addDocuments(List<String> textChunks, String category, String source) {
        List<Document> documents = textChunks.stream()
                .map(text -> Document.builder()
                        .text(text)
                        // 添加元数据，支持过滤查询
                        .metadata(Map.of(
                                "category", category,
                                "source", source,
                                "created_at", LocalDate.now().toString()
                        ))
                        .build())
                .toList();

        vectorStore.add(documents);
        log.info("成功添加 {} 个文档到 Qdrant", documents.size());
    }

    /**
     * 相似度检索
     */
    public List<Document> search(String query, int topK) {
        return vectorStore.similaritySearch(
                SearchRequest.builder()
                        .query(query)
                        .topK(topK)
                        .similarityThreshold(0.7)
                        .build()
        );
    }

    /**
     * 带过滤条件的检索（按元数据过滤）
     */
    public List<Document> searchWithFilter(String query, String category) {
        // Spring AI 的元数据过滤语法（SQL-like）
        FilterExpressionBuilder b = new FilterExpressionBuilder();
        Filter.Expression filterExpr = b.eq("category", category).build();

        return vectorStore.similaritySearch(
                SearchRequest.builder()
                        .query(query)
                        .topK(5)
                        .similarityThreshold(0.65)
                        .filterExpression(filterExpr)
                        .build()
        );
    }

    /**
     * 删除指定 ID 的文档
     */
    public void deleteDocuments(List<String> documentIds) {
        vectorStore.delete(documentIds);
    }
}
```

### 3.6 Qdrant + RAG 完整示例

```java
@RestController
@RequestMapping("/knowledge")
@RequiredArgsConstructor
public class KnowledgeBaseController {

    private final QdrantVectorService vectorService;
    private final ChatClient chatClient;
    private final VectorStore vectorStore;

    /**
     * 上传文档到知识库
     * POST /knowledge/ingest
     */
    @PostMapping("/ingest")
    public ResponseEntity<String> ingestDocument(
            @RequestParam MultipartFile file,
            @RequestParam String category) throws IOException {

        // 读取文件内容
        String content = new String(file.getBytes());

        // 文本分块（简单按段落分）
        List<String> chunks = splitIntoChunks(content, 500);

        // 存入 Qdrant
        vectorService.addDocuments(chunks, category, file.getOriginalFilename());

        return ResponseEntity.ok("成功导入 " + chunks.size() + " 个文档块");
    }

    /**
     * 基于知识库问答（RAG）
     * POST /knowledge/ask
     */
    @PostMapping("/ask")
    public String ask(@RequestBody AskRequest request) {
        // 1. 检索相关文档
        List<Document> relevant = vectorService.searchWithFilter(
                request.question(), request.category());

        // 2. 构建上下文
        String context = relevant.stream()
                .map(Document::getText)
                .collect(Collectors.joining("\n\n---\n\n"));

        // 3. 构建 Prompt 并调用模型
        return chatClient.prompt()
                .system("""
                        你是一个专业知识助手。
                        请严格根据以下上下文信息回答用户问题。
                        如果上下文中没有相关信息，请说"根据现有资料无法回答此问题"。
                        
                        上下文信息：
                        """ + context)
                .user(request.question())
                .call()
                .content();
    }

    private List<String> splitIntoChunks(String text, int chunkSize) {
        List<String> chunks = new ArrayList<>();
        String[] paragraphs = text.split("\n\n");
        StringBuilder current = new StringBuilder();

        for (String para : paragraphs) {
            if (current.length() + para.length() > chunkSize && !current.isEmpty()) {
                chunks.add(current.toString().trim());
                current = new StringBuilder();
            }
            current.append(para).append("\n\n");
        }

        if (!current.isEmpty()) {
            chunks.add(current.toString().trim());
        }

        return chunks;
    }

    record AskRequest(String question, String category) {}
}
```

## 4. 向量数据库集成——PgVector（PostgreSQL）

PgVector 是 PostgreSQL 的向量扩展，适合已有 PostgreSQL 基础设施的项目，无需额外部署新数据库。

### 4.1 PgVector 部署（Docker）

```bash
# 使用内置 pgvector 扩展的 PostgreSQL 镜像
docker run -d \
  --name pgvector \
  -e POSTGRES_DB=spring_ai \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=your_password \
  -p 5432:5432 \
  pgvector/pgvector:pg16
```

### 4.2 依赖配置

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-pgvector-store-spring-boot-starter</artifactId>
</dependency>

<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <scope>runtime</scope>
</dependency>
```

### 4.3 PgVector 配置

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/spring_ai
    username: postgres
    password: your_password

  ai:
    vectorstore:
      pgvector:
        initialize-schema: true     # 自动建表（首次）
        index-type: HNSW            # 索引类型：HNSW（推荐）或 IVFFLAT
        distance-type: COSINE_DISTANCE  # 距离计算方式
        dimensions: 1536            # 向量维度（与 Embedding 模型一致）
        schema-name: public         # 数据库 schema
        table-name: vector_store    # 向量存储表名
```

### 4.4 PgVector 自动建表结构

`initialize-schema: true` 会自动执行：

```sql
-- 启用 pgvector 扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 创建向量存储表
CREATE TABLE IF NOT EXISTS vector_store (
    id          UUID            DEFAULT gen_random_uuid() PRIMARY KEY,
    content     TEXT,
    metadata    JSON,
    embedding   vector(1536)    -- 向量列，维度与模型一致
);

-- 创建 HNSW 向量索引（提升检索速度）
CREATE INDEX IF NOT EXISTS vector_store_embedding_idx
    ON vector_store
    USING hnsw (embedding vector_cosine_ops)
    WITH (m = 16, ef_construction = 64);
```

## 5. ETL 文档处理管道

Spring AI 提供完整的文档处理（Extract-Transform-Load）框架，支持将各类文档转化为向量存储。

### 5.1 文档读取器（Document Readers）

```xml
<!-- PDF 读取器 -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-pdf-document-reader</artifactId>
</dependency>

<!-- Tika 通用读取器（支持 Word、Excel、PPT 等） -->
<dependency>

</dependency>
```

### 5.2 完整 ETL 管道示例

```java
@Service
@RequiredArgsConstructor
public class DocumentIngestionService {

    private final VectorStore vectorStore;
    private final TokenTextSplitter textSplitter;

    /**
     * 处理 PDF 文档
     */
    public void ingestPdf(Resource pdfResource, String category) {
        // 1. Extract：读取 PDF
        ParagraphPdfDocumentReader reader = new ParagraphPdfDocumentReader(
                pdfResource,
                PdfDocumentReaderConfig.builder()
                        .withPageTopMargin(0)
                        .withPageBottomMargin(0)
                        .build()
        );

        List<Document> rawDocs = reader.read();

        // 2. Transform：分块处理
        List<Document> splitDocs = textSplitter.split(rawDocs);

        // 3. 添加元数据
        splitDocs = splitDocs.stream()
                .map(doc -> Document.builder()
                        .text(doc.getText())
                        .metadata(new HashMap<>(doc.getMetadata()) {{
                            put("category", category);
                            put("source", pdfResource.getFilename());
                            put("ingested_at", Instant.now().toString());
                        }})
                        .build())
                .toList();

        // 4. Load：写入向量库
        vectorStore.add(splitDocs);
        log.info("PDF 文档 {} 处理完成，共 {} 块", pdfResource.getFilename(), splitDocs.size());
    }

    /**
     * 处理通用文件（Word/Excel/PPT/HTML 等）
     */
    public void ingestGenericFile(Resource resource, String category) {
        TikaDocumentReader reader = new TikaDocumentReader(resource);
        List<Document> docs = reader.read();

        List<Document> splitDocs = textSplitter.split(docs).stream()
                .map(doc -> Document.builder()
                        .text(doc.getText())
                        .metadata(Map.of("category", category,
                                         "source", resource.getFilename()))
                        .build())
                .toList();

        vectorStore.add(splitDocs);
    }

    /**
     * 处理网页内容
     */
    public void ingestUrl(String url, String category) {
        Document doc = Document.builder()
                .text(fetchWebContent(url))  // 实际使用 Jsoup 或 WebClient
                .metadata(Map.of("category", category, "source", url))
                .build();

        List<Document> splitDocs = textSplitter.split(List.of(doc));
        vectorStore.add(splitDocs);
    }

    private String fetchWebContent(String url) {
        // 使用 Jsoup 抓取网页文本
        try {
            return org.jsoup.Jsoup.connect(url).get().text();
        } catch (Exception e) {
            throw new RuntimeException("无法获取网页内容: " + url, e);
        }
    }
}
```

## 6. Spring AI Skills 使用

> **注意：** Spring AI 1.1 中的 "Skills" 概念在框架层面通过 **Tool 组合 + Advisor 模式** 体现，而非一个独立的 `@Skill` 注解。"Skills" 是指将一组相关工具、Advisor 和提示策略封装为可复用的能力模块。

### 6.1 Skills 设计模式

Skills 本质是将一组相关工具和 Advisor 按照业务能力聚合的设计模式：

```
Skills（能力模块）
├── Tool 集合（具体操作能力）
├── System Prompt（角色定义）
├── Advisor 链（记忆/RAG/安全）
└── ChatClient 实例（独立配置）
```

### 6.2 实现可复用的 Skill 模块

```java
/**
 * Skill 路由器：根据用户意图分发到不同 Skill
 */
@Service
@RequiredArgsConstructor
public class SkillRouter {

    private final CustomerServiceSkill customerServiceSkill;
    private final CodeAssistantSkill codeAssistantSkill;
    private final ChatClient routerClient;  // 轻量级路由模型

    /**
     * 智能路由：先识别意图，再分发到对应 Skill
     */
    public String route(String userId, String message) {
        // 意图识别
        Intent intent = routerClient.prompt()
                .system("""
                        判断用户消息属于哪个类别，只返回 JSON：
                        {"type": "customer_service" | "code_assistant" | "general"}
                        """)
                .user(message)
                .call()
                .entity(Intent.class);

        return switch (intent.type()) {
            case "customer_service" -> customerServiceSkill.assist(userId, message);
            case "code_assistant"   -> codeAssistantSkill.generateCode(message, "Java");
            default                 -> routerClient.prompt().user(message).call().content();
        };
    }

    record Intent(String type) {}
}
```

## 7. Agentic 工作流

### 7.1 顺序工作流（链式调用）

```java
/**
 * 顺序工作流：文章生成管道
 * 步骤：主题分析 → 大纲生成 → 内容生成 → 润色优化
 */
@Service
@RequiredArgsConstructor
public class ArticleGenerationWorkflow {

    private final ChatClient chatClient;

    public ArticleResult generateArticle(String topic) {
        // Step 1: 主题分析
        TopicAnalysis analysis = chatClient.prompt()
                .system("你是内容策略专家，分析给定主题的核心角度和目标读者。")
                .user("分析主题：" + topic)
                .call()
                .entity(TopicAnalysis.class);

        // Step 2: 大纲生成
        String outline = chatClient.prompt()
                .system("你是大纲规划专家，根据主题分析生成结构化文章大纲。")
                .user("主题：%s\n读者：%s\n核心角度：%s\n\n生成大纲"
                        .formatted(topic, analysis.targetAudience(), analysis.mainAngle()))
                .call()
                .content();

        // Step 3: 内容生成
        String draftContent = chatClient.prompt()
                .system("你是内容创作专家，根据大纲撰写高质量文章。")
                .user("大纲：\n" + outline + "\n\n请撰写完整文章。")
                .options(OpenAiChatOptions.builder().maxTokens(3000).build())
                .call()
                .content();

        // Step 4: 润色优化
        String polishedContent = chatClient.prompt()
                .system("你是资深编辑，负责文章润色、纠错和风格优化。")
                .user("请润色以下文章：\n\n" + draftContent)
                .call()
                .content();

        return new ArticleResult(topic, outline, polishedContent, analysis);
    }

    record TopicAnalysis(String mainAngle, String targetAudience, List<String> keyPoints) {}
    record ArticleResult(String topic, String outline, String content, TopicAnalysis analysis) {}
}
```

### 7.2 并行工作流（CompletableFuture）

```java
/**
 * 并行工作流：同时从多个角度分析同一问题
 */
@Service
@RequiredArgsConstructor
public class ParallelAnalysisWorkflow {

    private final ChatClient chatClient;

    public ComprehensiveAnalysis analyzeInParallel(String topic) {
        // 并行执行三个分析任务
        CompletableFuture<String> technicalFuture = CompletableFuture.supplyAsync(() ->
                chatClient.prompt()
                        .system("你是技术专家，从技术实现角度分析问题。")
                        .user(topic)
                        .call()
                        .content()
        );

        CompletableFuture<String> businessFuture = CompletableFuture.supplyAsync(() ->
                chatClient.prompt()
                        .system("你是商业分析师，从商业价值角度分析问题。")
                        .user(topic)
                        .call()
                        .content()
        );

        CompletableFuture<String> riskFuture = CompletableFuture.supplyAsync(() ->
                chatClient.prompt()
                        .system("你是风险管理专家，从风险控制角度分析问题。")
                        .user(topic)
                        .call()
                        .content()
        );

        // 等待所有任务完成
        CompletableFuture.allOf(technicalFuture, businessFuture, riskFuture).join();

        // 汇总分析结果
        String summary = chatClient.prompt()
                .system("你是综合分析专家，汇总多维度分析结果形成结论。")
                .user("""
                        请综合以下三个维度的分析，给出综合结论：
                        
                        【技术角度】
                        %s
                        
                        【商业角度】
                        %s
                        
                        【风险角度】
                        %s
                        """.formatted(
                        technicalFuture.join(),
                        businessFuture.join(),
                        riskFuture.join()
                ))
                .call()
                .content();

        return new ComprehensiveAnalysis(
                technicalFuture.join(),
                businessFuture.join(),
                riskFuture.join(),
                summary
        );
    }

    record ComprehensiveAnalysis(
            String technical,
            String business,
            String risk,
            String summary
    ) {}
}
```

```java
@Component
@RequiredArgsConstructor
public class TokenUsageMonitor {

    private final MeterRegistry meterRegistry;

    /**
     * 记录每次调用的 Token 用量
     */
    public void recordTokenUsage(ChatResponse response, String model) {
        Usage usage = response.getMetadata().getUsage();

        if (usage != null) {
            // 输入 Token 计数器
            meterRegistry.counter("ai.token.usage",
                            "type", "prompt",
                            "model", model)
                    .increment(usage.getPromptTokens());

            // 输出 Token 计数器
            meterRegistry.counter("ai.token.usage",
                            "type", "completion",
                            "model", model)
                    .increment(usage.getGenerationTokens());

            log.debug("Token 用量 - Prompt: {}, Completion: {}, Total: {}",
                    usage.getPromptTokens(),
                    usage.getGenerationTokens(),
                    usage.getTotalTokens());
        }
    }
}
```

```java
@RequiredArgsConstructor
public class TokenUsageAdvisor implements CallAroundAdvisor {

    private final TokenUsageMonitor monitor;

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest request,
                                       CallAroundAdvisorChain chain) {
        AdvisedResponse response = chain.nextAroundCall(request);
        monitor.recordTokenUsage(
                response.response(),
                response.response().getMetadata().getModel()
        );
        return response;
    }

    @Override
    public String getName() { return "TokenUsageAdvisor"; }

    @Override
    public int getOrder() { return Ordered.LOWEST_PRECEDENCE; }
}
```
