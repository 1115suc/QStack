# Spring AI 01：基础接入与稳定依赖

> 推荐版本：Spring AI `1.1.6`，Spring Boot `3.4.x` 或 `3.5.x`，JDK `17+`。  
> 本文统一使用 Spring AI 1.x 之后的新 starter 命名，避免旧版依赖名下载失败。

---

## 1. 依赖命名变化

Spring AI 1.x 后，很多 starter 名称已经从旧格式切换为新格式：

| 场景 | 旧写法，不推荐 | 新写法，推荐 |
|---|---|---|
| OpenAI | `spring-ai-openai-spring-boot-starter` | `spring-ai-starter-model-openai` |
| Ollama | `spring-ai-ollama-spring-boot-starter` | `spring-ai-starter-model-ollama` |
| Qdrant | `spring-ai-qdrant-store-spring-boot-starter` | `spring-ai-starter-vector-store-qdrant` |
| MCP Client | `spring-ai-mcp-client-spring-boot-starter` | `spring-ai-starter-mcp-client` |
| MCP Server | `spring-ai-mcp-server-spring-boot-starter` | `spring-ai-starter-mcp-server-webmvc` 或 `spring-ai-starter-mcp-server-webflux` |

建议只引入一个 `spring-ai-bom`，具体 starter 不要再手写版本号。

## 2. Maven 基础依赖

```xml
<properties>
    <java.version>17</java.version>
    <spring-ai.version>1.1.6</spring-ai.version>
</properties>

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.ai</groupId>
            <artifactId>spring-ai-bom</artifactId>
            <version>${spring-ai.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- OpenAI / 兼容 OpenAI 协议的模型服务 -->
    <dependency>
        <groupId>org.springframework.ai</groupId>
        <artifactId>spring-ai-starter-model-openai</artifactId>
    </dependency>
</dependencies>
```

如果使用 Spring Boot Parent：

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.5.0</version>
    <relativePath/>
</parent>
```

Spring Boot 版本不由 Spring AI BOM 管理，仍由 Boot Parent 或自己的 dependency management 管理。

## 4. OpenAI 配置

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      base-url: https://api.openai.com
      chat:
        options:
          model: gpt-4o-mini
          temperature: 0.7
          max-tokens: 2048
      embedding:
        options:
          model: text-embedding-3-small
```

Windows PowerShell 设置环境变量：

```powershell
$env:OPENAI_API_KEY="sk-xxxx"
```

Linux / macOS：

```bash
export OPENAI_API_KEY="sk-xxxx"
```

## 5. DeepSeek 配置

DeepSeek 使用 OpenAI 兼容接口，因此仍然引入：

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-openai</artifactId>
</dependency>
```

配置示例：

```yaml
spring:
  ai:
    openai:
      api-key: ${DEEPSEEK_API_KEY}
      base-url: https://api.deepseek.com
      chat:
        options:
          model: deepseek-chat
          temperature: 0.7
```

如果服务商要求路径包含 `/v1`，再写成服务商文档要求的地址。例如：

```yaml
spring:
  ai:
    openai:
      base-url: https://api.example.com/v1
```

## 7. 基础 ChatClient 用法

`ChatClient` 是 Spring AI 推荐的高层 API，写法类似 WebClient。

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/ai/chat")
public class ChatController {

    private final ChatClient chatClient;

    public ChatController(ChatClient.Builder builder) {
        this.chatClient = builder
                .defaultSystem("你是一个专业的 Java 后端开发助手，回答要简洁、准确。")
                .build();
    }

    @GetMapping
    public String chat(@RequestParam String message) {
        return chatClient.prompt()
                .user(message)
                .call()
                .content();
    }
}
```

测试：

```bash
curl "http://localhost:8080/ai/chat?message=什么是Spring AI"
```

## 8. 使用底层 ChatModel

需要更精细控制消息时，可以直接使用 `ChatModel`。

```java
import java.util.List;

import org.springframework.ai.chat.messages.Message;
import org.springframework.ai.chat.messages.SystemMessage;
import org.springframework.ai.chat.messages.UserMessage;
import org.springframework.ai.chat.model.ChatModel;
import org.springframework.ai.chat.model.ChatResponse;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.stereotype.Service;

@Service
public class ChatModelService {

    private final ChatModel chatModel;

    public ChatModelService(ChatModel chatModel) {
        this.chatModel = chatModel;
    }

    public String ask(String message) {
        List<Message> messages = List.of(
                new SystemMessage("你是一个 Java 编程专家。"),
                new UserMessage(message)
        );

        ChatResponse response = chatModel.call(new Prompt(messages));
        return response.getResult().getOutput().getText();
    }
}
```

## 9. 流式输出 SSE

流式接口需要 Reactor 的 `Flux`。普通 WebMVC 项目也可以返回 `Flux<String>` 并指定 SSE。

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.http.MediaType;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;
import reactor.core.publisher.Flux;

@RestController
@RequestMapping("/ai/stream")
public class StreamController {

    private final ChatClient chatClient;

    public StreamController(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    @GetMapping(value = "/sse", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> stream(@RequestParam String message) {
        return chatClient.prompt()
                .user(message)
                .stream()
                .content();
    }
}
```

前端接收：

```javascript
const source = new EventSource("/ai/stream/sse?message=介绍Spring AI");

source.onmessage = (event) => {
  document.querySelector("#output").textContent += event.data;
};

source.onerror = () => source.close();
```

## 10. 结构化输出

Spring AI 可以把模型输出直接映射为 Java 对象。

```java
import java.util.List;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.core.ParameterizedTypeReference;
import org.springframework.stereotype.Service;

@Service
public class StructuredOutputService {

    private final ChatClient chatClient;

    public StructuredOutputService(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    public Movie recommend(String genre) {
        return chatClient.prompt()
                .user("推荐一部" + genre + "类型的经典电影，返回结构化信息。")
                .call()
                .entity(Movie.class);
    }

    public List<Movie> recommendList(String genre) {
        return chatClient.prompt()
                .user("推荐3部" + genre + "类型的经典电影。")
                .call()
                .entity(new ParameterizedTypeReference<List<Movie>>() {});
    }

    public record Movie(String title, String director, int year, String reason) {
    }
}
```

## 11. 多模态图片输入

必须使用支持图片输入的模型，例如 `gpt-4o`、`gpt-4o-mini` 等。

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      chat:
        options:
          model: gpt-4o-mini
```

示例：

```java
import java.net.URI;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.util.MimeTypeUtils;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/ai/image")
public class ImageChatController {

    private final ChatClient chatClient;

    public ImageChatController(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    @PostMapping("/url")
    public String analyze(@RequestBody ImageRequest request) {
        return chatClient.prompt()
                .user(user -> user
                        .text(request.question())
                        .media(MimeTypeUtils.IMAGE_JPEG, URI.create(request.imageUrl())))
                .call()
                .content();
    }

    public record ImageRequest(String question, String imageUrl) {
    }
}
```

## 12. 常见下载失败排查

1. 不要混用旧 starter：优先检查是否还在使用 `spring-ai-openai-spring-boot-starter` 这类旧 artifactId。
2. 不要给 starter 单独写版本：统一由 `spring-ai-bom` 管理。
3. Spring AI 里程碑版本需要额外仓库，稳定版 `1.1.6` 不需要配置 Spring Milestone 仓库。
4. 国内网络如果 Maven Central 访问慢，优先配置 Maven 镜像，而不是换成不存在的依赖名。
5. 一个项目同时引入多个模型 starter 时，可能出现多个 `ChatModel` Bean，需要通过 `@Primary`、`@Qualifier` 或独立 `ChatClient` Bean 明确选择。
