# Spring AI 基础使用指南

> 基于 Spring AI **1.1.0 GA** | Spring Boot **3.4.x** | JDK **21**

---

## 1. 项目快速引入

### 1.1 通过 Spring Initializr 创建项目

访问 [https://start.spring.io](https://start.spring.io)，选择以下依赖：

- **Spring Web**
- **OpenAI**（或其他模型 Starter）
- **Spring Boot DevTools**（开发可选）

### 1.2 手动添加 Maven 依赖

Spring AI 使用 BOM（Bill of Materials）统一管理依赖版本，推荐方式如下：

```xml
<!-- pom.xml -->
<properties>
    <java.version>21</java.version>
    <spring-ai.version>1.1.0</spring-ai.version>
</properties>

<!-- BOM 依赖管理（必须添加） -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.ai</groupId>
            <artifactId>spring-ai-bom</artifactId>
            <version>${spring-ai.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<dependencies>
    <!-- Spring Web -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- OpenAI Starter（按需替换为其他 Provider） -->
    <dependency>
        <groupId>org.springframework.ai</groupId>
        <artifactId>spring-ai-openai-spring-boot-starter</artifactId>
    </dependency>

    <!-- 如使用 Ollama（本地模型） -->
    <!-- <dependency>
        <groupId>org.springframework.ai</groupId>
        <artifactId>spring-ai-ollama-spring-boot-starter</artifactId>
    </dependency> -->
</dependencies>
```

### 1.3 Gradle 方式

```groovy
// build.gradle
ext {
    set('springAiVersion', "1.1.0")
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.springframework.ai:spring-ai-openai-spring-boot-starter'
}

dependencyManagement {
    imports {
        mavenBom "org.springframework.ai:spring-ai-bom:${springAiVersion}"
    }
}
```

## 2. 模型配置详解

### 2.1 OpenAI 配置

```yaml
# application.yml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}            # 从环境变量读取
      base-url: https://api.openai.com      # 默认值，可替换为代理地址
      chat:
        options:
          model: gpt-4o                     # 默认聊天模型
          temperature: 0.7                  # 温度：0~2，越高越随机
          max-tokens: 2048                  # 最大输出 Token 数
          top-p: 1.0                        # 核采样
          frequency-penalty: 0.0           # 频率惩罚
          presence-penalty: 0.0            # 存在惩罚
      embedding:
        options:
          model: text-embedding-ada-002     # 默认 Embedding 模型
```

### 2.2 Ollama 配置（本地部署）

```yaml
spring:
  ai:
    ollama:
      base-url: http://localhost:11434      # Ollama 服务地址
      chat:
        options:
          model: llama3.2                  # 本地已拉取的模型
          temperature: 0.8
          num-predict: 1024                # 最大生成 Token
      embedding:
        options:
          model: nomic-embed-text         # 本地 Embedding 模型
```

### 2.3 DeepSeek 配置（兼容 OpenAI 协议）

```yaml
spring:
  ai:
    openai:
      api-key: ${DEEPSEEK_API_KEY}
      base-url: https://api.deepseek.com
      chat:
        options:
          model: deepseek-chat
          temperature: 0.7
```

### 2.4 多模型提供者共存（多 Provider 场景）

当项目中需要同时使用多个 Provider 时，通过属性指定激活哪个自动配置：

```yaml
spring:
  ai:
    chat:
      # 指定使用 openai 的 chat 实现（多 Provider 时消除歧义）
      client:
        enabled: true
    openai:
      api-key: ${OPENAI_API_KEY}
      chat:
        enabled: true       # 显式启用 openai chat
      embedding:
        enabled: true
    ollama:
      chat:
        enabled: false      # 显式禁用 ollama chat
```

### 2.5 使用 Java 配置类（精细控制）

```java
@Configuration
public class AiModelConfig {

    /**
     * 自定义 ChatModel Bean（以 OpenAI 为例）
     * 适用于需要动态参数或多实例场景
     */
    @Bean
    @Primary
    public OpenAiChatModel primaryChatModel(OpenAiConnectionProperties connectionProps) {
        OpenAiChatOptions options = OpenAiChatOptions.builder()
                .model("gpt-4o")
                .temperature(0.7)
                .maxTokens(2048)
                .build();

        return OpenAiChatModel.builder()
                .openAiApi(OpenAiApi.builder()
                        .apiKey(connectionProps.getApiKey())
                        .build())
                .defaultOptions(options)
                .build();
    }

    /**
     * 第二个 ChatModel 实例（例如用于特定任务）
     */
    @Bean("summaryModel")
    public OpenAiChatModel summaryModel(OpenAiConnectionProperties props) {
        return OpenAiChatModel.builder()
                .openAiApi(OpenAiApi.builder().apiKey(props.getApiKey()).build())
                .defaultOptions(OpenAiChatOptions.builder()
                        .model("gpt-4o-mini")
                        .temperature(0.3)
                        .maxTokens(512)
                        .build())
                .build();
    }
}
```

## 3. 基础聊天使用

Spring AI 提供两种聊天方式：低级的 `ChatModel` API 和高级的 `ChatClient` Fluent API，推荐使用 `ChatClient`。

### 3.1 使用 ChatClient（推荐）

```java
@RestController
@RequestMapping("/ai/chat")
public class BasicChatController {

    private final ChatClient chatClient;

    // 通过 Builder 构建 ChatClient
    public BasicChatController(ChatClient.Builder builder) {
        this.chatClient = builder
                // 设置系统消息（全局角色定义）
                .defaultSystem("你是一个专业的 Java 后端开发助手，回答简洁专业。")
                .build();
    }

    /**
     * 最简单的单次问答
     * GET /ai/chat/simple?question=什么是Spring AI?
     */
    @GetMapping("/simple")
    public String simpleChat(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .call()
                .content();  // 直接返回字符串内容
    }

    /**
     * 带自定义参数的聊天
     * POST /ai/chat/custom
     */
    @PostMapping("/custom")
    public ChatResponse customChat(@RequestBody ChatRequest request) {
        return chatClient.prompt()
                .system(request.systemPrompt())
                .user(request.userMessage())
                .options(OpenAiChatOptions.builder()
                        .model("gpt-4o")
                        .temperature(request.temperature())
                        .build())
                .call()
                .chatResponse();  // 返回完整响应（含 Token 用量等元数据）
    }

    record ChatRequest(String systemPrompt, String userMessage, Double temperature) {}
}
```

### 3.2 使用底层 ChatModel API

```java
@Service
public class ChatModelService {

    private final ChatModel chatModel;

    public ChatModelService(ChatModel chatModel) {
        this.chatModel = chatModel;
    }

    /**
     * 使用 Prompt + Message 构建请求
     */
    public String chat(String userInput) {
        // 构建消息列表
        List<Message> messages = List.of(
                new SystemMessage("你是一个 Java 编程专家。"),
                new UserMessage(userInput)
        );

        // 构建 Prompt
        Prompt prompt = new Prompt(messages, OpenAiChatOptions.builder()
                .temperature(0.7)
                .build());

        // 调用模型
        ChatResponse response = chatModel.call(prompt);

        // 获取结果
        return response.getResult().getOutput().getText();
    }

    /**
     * 使用 PromptTemplate 模板（参数化提示词）
     */
    public String chatWithTemplate(String topic, String language) {
        PromptTemplate template = new PromptTemplate("""
                请用 {language} 写一个关于 {topic} 的简单代码示例，
                并附上简要说明。
                """);

        Prompt prompt = template.create(Map.of(
                "topic", topic,
                "language", language
        ));

        return chatModel.call(prompt).getResult().getOutput().getText();
    }
}
```

### 3.3 多轮对话（手动维护上下文）

```java
@Service
public class MultiTurnChatService {

    private final ChatModel chatModel;
    // 简单内存存储（生产环境请使用 ChatMemory，见进阶篇）
    private final Map<String, List<Message>> sessions = new ConcurrentHashMap<>();

    public MultiTurnChatService(ChatModel chatModel) {
        this.chatModel = chatModel;
    }

    public String chat(String sessionId, String userInput) {
        // 获取或初始化会话历史
        List<Message> history = sessions.computeIfAbsent(sessionId, k ->
                new ArrayList<>(List.of(new SystemMessage("你是一个专业助手。")))
        );

        // 添加用户消息
        history.add(new UserMessage(userInput));

        // 调用模型（携带完整历史）
        ChatResponse response = chatModel.call(new Prompt(history));
        String assistantReply = response.getResult().getOutput().getText();

        // 将助手回复加入历史
        history.add(new AssistantMessage(assistantReply));

        return assistantReply;
    }
}
```

## 4. 流式输出

流式输出（Streaming）让模型逐 Token 返回结果，提升用户体验。Spring AI 使用 Reactor 的 `Flux` 实现流式响应。

### 4.1 Server-Sent Events（SSE）流式输出

```java
@RestController
@RequestMapping("/ai/stream")
public class StreamChatController {

    private final ChatClient chatClient;

    public StreamChatController(ChatClient.Builder builder) {
        this.chatClient = builder
                .defaultSystem("你是一个专业助手。")
                .build();
    }

    /**
     * SSE 流式输出（前端可用 EventSource 接收）
     * GET /ai/stream/sse?question=请介绍一下Spring AI
     */
    @GetMapping(value = "/sse", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> streamChat(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .stream()
                .content();  // 返回 Flux<String>，每个元素是一个 Token 片段
    }

    /**
     * 返回完整 ChatResponse 的流（含元数据）
     */
    @GetMapping(value = "/full", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ChatResponse> streamFullResponse(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .stream()
                .chatResponse();  // 返回 Flux<ChatResponse>
    }

    /**
     * 将流聚合为完整字符串（适合需要后处理的场景）
     */
    @GetMapping("/collect")
    public Mono<String> streamAndCollect(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .stream()
                .content()
                .collectList()
                .map(tokens -> String.join("", tokens));
    }
}
```

### 4.2 WebFlux 响应式配置

使用流式输出需要引入 WebFlux（或保持 WebMVC 但添加响应式支持）：

```xml
<!-- 如果使用 WebFlux -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```

> **注意：** 在 WebMVC 项目中也可以直接返回 `Flux<String>` 并设置 `produces = MediaType.TEXT_EVENT_STREAM_VALUE`，Spring MVC 会自动处理。

### 4.3 前端接收 SSE 示例（JavaScript）

```javascript
const eventSource = new EventSource('/ai/stream/sse?question=请介绍Spring AI');

eventSource.onmessage = (event) => {
    document.getElementById('output').textContent += event.data;
};

eventSource.onerror = () => {
    eventSource.close();
};
```

## 5. 多模态模型使用

Spring AI 支持将图片、音频等多媒体内容与文本一起发送给多模态模型（如 GPT-4o、Gemini）。

### 5.1 图片理解（URL 方式）

```java
@RestController
@RequestMapping("/ai/multimodal")
public class MultiModalController {

    private final ChatClient chatClient;

    public MultiModalController(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    /**
     * 通过图片 URL 进行图像理解
     */
    @PostMapping("/image-url")
    public String analyzeImageByUrl(@RequestBody ImageUrlRequest request) {
        return chatClient.prompt()
                .user(userSpec -> userSpec
                        .text(request.question())
                        .media(MimeTypeUtils.IMAGE_JPEG,
                               new URL(request.imageUrl()))
                )
                .call()
                .content();
    }

    record ImageUrlRequest(String question, String imageUrl) {}
}
```

### 5.2 图片理解（Base64 / 文件上传方式）

```java
/**
 * 通过上传文件进行图像理解
 * POST /ai/multimodal/upload
 */
@PostMapping("/upload")
public String analyzeUploadedImage(
        @RequestParam("file") MultipartFile file,
        @RequestParam("question") String question) throws IOException {

    // 读取图片字节数据
    byte[] imageBytes = file.getBytes();
    MimeType mimeType = MimeTypeUtils.parseMimeType(
            Objects.requireNonNull(file.getContentType()));

    return chatClient.prompt()
            .user(userSpec -> userSpec
                    .text(question)
                    .media(mimeType, new ByteArrayResource(imageBytes))
            )
            .call()
            .content();
}
```

### 5.3 多图片同时理解

```java
/**
 * 同时发送多张图片
 */
@PostMapping("/multi-images")
public String analyzeMultipleImages(@RequestBody MultiImageRequest request) {
    return chatClient.prompt()
            .user(userSpec -> {
                userSpec.text(request.question());
                request.imageUrls().forEach(url -> {
                    try {
                        userSpec.media(MimeTypeUtils.IMAGE_JPEG, new URL(url));
                    } catch (Exception e) {
                        throw new RuntimeException(e);
                    }
                });
            })
            .call()
            .content();
}

record MultiImageRequest(String question, List<String> imageUrls) {}
```

### 5.4 多模态配置说明

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      chat:
        options:
          # 必须使用支持多模态的模型
          model: gpt-4o          # 支持图片
          # model: gpt-4o-mini   # 也支持图片，成本更低
```

> **注意事项：**
> - 并非所有模型都支持多模态，需确认所选模型支持图像输入
> - 图片大小有限制（OpenAI 单张最大 20MB，Base64 最大 20MB）
> - 支持的格式：PNG、JPEG、GIF、WEBP

## 6. 结构化输出

Spring AI 支持将模型输出直接映射为 Java 对象（POJO），无需手动解析 JSON。

### 6.1 使用 BeanOutputConverter

```java
@Service
public class StructuredOutputService {

    private final ChatClient chatClient;

    public StructuredOutputService(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    // 定义返回结构
    record MovieRecommendation(
            String title,
            String director,
            int year,
            String genre,
            String reason
    ) {}

    /**
     * 让模型返回结构化的电影推荐
     */
    public MovieRecommendation getMovieRecommendation(String genre) {
        return chatClient.prompt()
                .user("推荐一部 " + genre + " 类型的经典电影，返回结构化信息。")
                .call()
                .entity(MovieRecommendation.class);  // 自动解析为 POJO
    }

    /**
     * 返回列表
     */
    public List<MovieRecommendation> getMovieList(String genre, int count) {
        return chatClient.prompt()
                .user("推荐 " + count + " 部 " + genre + " 类型的电影。")
                .call()
                .entity(new ParameterizedTypeReference<List<MovieRecommendation>>() {});
    }
}
```

### 6.2 结构化输出在 Controller 中的使用

```java
@GetMapping("/recommend")
public MovieRecommendation recommend(@RequestParam String genre) {
    return structuredOutputService.getMovieRecommendation(genre);
}
```

## 7. 常见问题

### 7.1 多个 ChatModel Bean 冲突

**问题：** 同时引入多个 Provider Starter 导致 `ChatModel` Bean 注入歧义。

**解决：** 在配置类中使用 `@Primary` 指定默认实现，或使用 `@Qualifier` 精确注入。

```java
@Autowired
@Qualifier("summaryModel")
private ChatModel summaryModel;
```

### 7.2 API Key 安全管理

**推荐方式：** 使用环境变量而非硬编码：

```bash
# Linux/Mac
export OPENAI_API_KEY=sk-xxxxx

# Windows
set OPENAI_API_KEY=sk-xxxxx
```

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}  # 引用环境变量
```

### 7.3 连接超时配置

```yaml
spring:
  ai:
    openai:
      # 连接超时和读取超时（毫秒）
      connect-timeout: 10000
      read-timeout: 60000
```

### 7.4 代理配置（网络受限环境）

```java
@Configuration
public class HttpClientConfig {

    @Bean
    public RestClient.Builder restClientBuilder() {
        return RestClient.builder()
                .requestFactory(new HttpComponentsClientHttpRequestFactory(
                        HttpClients.custom()
                                .setProxy(new HttpHost("proxy.example.com", 8080))
                                .build()
                ));
    }
}
```
