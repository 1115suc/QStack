# Spring AI Alibaba 高级使用指南

## 1. 模型上下文协议（MCP）

MCP（Model Context Protocol）是 Anthropic 主导的标准化协议，让 AI 模型以统一方式访问外部工具。Spring AI Alibaba 在 Spring AI MCP 基础上增加了 Nacos MCP Registry 支持，实现企业级分布式工具发现。

### 1.1 MCP 架构（Spring AI Alibaba 扩展）

```
┌──────────────────────────────────────────────────────┐
│               Spring AI Alibaba 应用                  │
│                                                      │
│  ┌──────────────┐   ┌────────────────────────────┐  │
│  │  ChatClient  │──▶│  McpToolCallbackProvider   │  │
│  └──────────────┘   │  (工具桥接到通义千问)       │  │
│                     └────────────┬───────────────┘  │
└──────────────────────────────────┼──────────────────┘
                                   │ MCP 协议
          ┌──────────┬─────────────┴────────────────┐
          │          │                              │
   ┌──────▼──────┐  ┌▼──────────────┐  ┌──────────▼─────────┐
   │ MCP Server  │  │  MCP Server   │  │  MCP Server        │
   │（STDIO 本地）│  │（SSE 远程）   │  │（Nacos 注册发现）  │
   └─────────────┘  └───────────────┘  └────────────────────┘
                                              ↑
                                       Nacos MCP Registry
                                       （阿里云特色）
```

#### 在 ChatClient 中使用 MCP 工具

```java
@RestController
@RequestMapping("/ai/mcp")
public class McpChatController {

    private final ChatClient chatClient;

    /**
     * Spring AI Alibaba 自动将 MCP Server 的工具注册到 ChatClient
     */
    public McpChatController(ChatClient.Builder builder,
                              DashScopeChatModel chatModel,
                              McpToolCallbackProvider mcpToolCallbackProvider) {
        this.chatClient = ChatClient.builder(chatModel)
                .defaultSystem("""
                        你是一个能够使用外部工具的通义千问助手。
                        当用户询问需要实时数据或特定操作时，请主动调用合适的工具。
                        """)
                // 注册来自所有 MCP Server 的工具
                .defaultTools(mcpToolCallbackProvider.getToolCallbacks())
                .build();
    }

    @GetMapping("/chat")
    public String chatWithMcpTools(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .call()
                .content();
    }

    /**
     * 流式 MCP 工具对话
     */
    @GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> streamWithMcpTools(@RequestParam String question) {
        return chatClient.prompt()
                .user(question)
                .stream()
                .content();
    }
}
```

## 2. Nacos MCP Registry（阿里云特色）

Spring AI Alibaba 提供 Nacos MCP Registry，实现 MCP Server 的分布式注册与发现，这是区别于标准 Spring AI 的重要企业级特性。

### 2.1 架构说明

```
微服务 A（业务工具服务）                微服务 B（AI 应用）
┌──────────────────────────┐          ┌──────────────────────────┐
│  MCP Server（工具提供方）  │          │  Spring AI Alibaba 应用   │
│  - 客户查询工具           │          │  - DashScopeChatModel     │
│  - 订单查询工具           │  注册    │  - McpToolCallbackProvider│
│  - 库存查询工具           │────────▶│  - ChatClient             │
└──────────────────────────┘          └──────────────────────────┘
            │                                      │
            │ 注册服务信息                          │ 发现可用 MCP Server
            ▼                                      ▼
     ┌─────────────────┐
     │  Nacos Registry  │
     │  MCP 服务注册中心 │
     └─────────────────┘
```

### 2.2 依赖配置

```xml
<!-- Nacos MCP Registry 依赖 -->
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-mcp-nacos</artifactId>
</dependency>
```

### 2.3 MCP Server 注册到 Nacos

## 3. 检索增强生成（RAG）

### 3.1 RAG 工作流程

```
用户问题
   │
   ▼ DashScope Embedding（text-embedding-v3）
向量化查询
   │
   ▼ Qdrant 向量相似度检索
TopK 相关文档
   │
   ▼ 注入到 Prompt 上下文
通义千问（qwen-plus）推理
   │
   ▼
基于知识库的精准回答
```

### 3.2 使用 QuestionAnswerAdvisor（简单 RAG）

```java
@Configuration
@RequiredArgsConstructor
public class RagConfig {

    private final VectorStore vectorStore;       // Qdrant VectorStore
    private final DashScopeChatModel chatModel;

    /**
     * 简单 RAG：QuestionAnswerAdvisor 自动检索并注入上下文
     */
    @Bean
    public QuestionAnswerAdvisor questionAnswerAdvisor() {
        return QuestionAnswerAdvisor.builder(vectorStore)
                .searchRequest(SearchRequest.builder()
                        .topK(5)                    // 返回最相关的 5 个文档块
                        .similarityThreshold(0.7)   // 相似度阈值（0-1）
                        .build())
                .build();
    }

    /**
     * RAG ChatClient：集成通义千问 + 向量检索
     */
    @Bean("ragChatClient")
    public ChatClient ragChatClient(QuestionAnswerAdvisor qaAdvisor,
                                    ChatMemory chatMemory) {
        return ChatClient.builder(chatModel)
                .defaultSystem("""
                        你是一个专业的知识助手，由通义千问提供支持。
                        请严格基于提供的上下文信息回答用户问题。
                        如果上下文中没有相关信息，请明确告知用户无法从知识库中找到答案，
                        不要凭空捏造内容。
                        """)
                .defaultAdvisors(
                        qaAdvisor,   // RAG 检索 Advisor
                        MessageChatMemoryAdvisor.builder(chatMemory).build()
                )
                .defaultOptions(DashScopeChatOptions.builder()
                        .withModel("qwen-plus")
                        .withTemperature(0.3)   // RAG 场景建议低温，保证准确性
                        .build())
                .build();
    }
}
```

```java
@RestController
@RequestMapping("/ai/rag")
@RequiredArgsConstructor
public class RagController {

    @Qualifier("ragChatClient")
    private final ChatClient ragChatClient;

    /**
     * 基于知识库的问答（带分类过滤）
     * POST /ai/rag/ask
     */
    @PostMapping("/ask")
    public String ask(@RequestBody RagAskRequest request) {
        return ragChatClient.prompt()
                .user(request.question())
                // 可通过 Advisor 参数动态过滤文档分类
                .advisors(spec -> {
                    spec.param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               request.conversationId());
                    if (request.category() != null) {
                        // 元数据过滤：只检索指定分类的文档
                        spec.param(QuestionAnswerAdvisor.FILTER_EXPRESSION,
                                   "category == '" + request.category() + "'");
                    }
                })
                .call()
                .content();
    }

    record RagAskRequest(String conversationId, String question, String category) {}
}
```

### 3.3 高级 RAG（RetrievalAugmentationAdvisor）

```java
@Configuration
@RequiredArgsConstructor
public class AdvancedRagConfig {

    private final VectorStore vectorStore;
    private final DashScopeChatModel chatModel;

    /**
     * 高级 RAG：包含查询扩展、文档重排序

                        .build())

                .build();
    }
}
```

## 4. 向量数据库集成——Qdrant

Qdrant 是高性能向量数据库，适合生产级 RAG 场景。Spring AI Alibaba 完全继承 Spring AI 的 Qdrant 集成能力。

### 4.1 Qdrant 部署（Docker）

```bash
# 启动 Qdrant
docker run -d \
  --name qdrant \
  -p 6333:6333 \   # HTTP REST API
  -p 6334:6334 \   # gRPC API（Spring AI 使用）
  -v $(pwd)/qdrant_data:/qdrant/storage \
  qdrant/qdrant:latest

# 验证运行
curl http://localhost:6333/health
# {"title":"qdrant - vector search engine","version":"1.x.x"}
```

### 4.2 依赖配置

```xml

<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-qdrant-store-spring-boot-starter</artifactId>
</dependency>
```

### 4.4 Qdrant 自定义配置类

```java
@Configuration
public class QdrantVectorStoreConfig {

    @Value("${spring.ai.vectorstore.qdrant.host:localhost}")
    private String host;

    @Value("${spring.ai.vectorstore.qdrant.port:6334}")
    private int port;

    @Value("${spring.ai.vectorstore.qdrant.collection-name:qwen_knowledge_base}")
    private String collectionName;

    /**
     * 主知识库 VectorStore（使用 DashScope text-embedding-v3）
     */
    @Bean
    @Primary
    public VectorStore qdrantVectorStore(EmbeddingModel embeddingModel) {
        QdrantClient qdrantClient = new QdrantClient(
                QdrantGrpcClient.newBuilder(host, port, false).build()
        );

        return QdrantVectorStore.builder(qdrantClient, embeddingModel)
                .collectionName(collectionName)
                .initializeSchema(true)
                .build();
    }

    /**
     * 用户长期记忆 VectorStore（独立 Collection）
     */
    @Bean("memoryVectorStore")
    public VectorStore memoryVectorStore(EmbeddingModel embeddingModel) {
        QdrantClient qdrantClient = new QdrantClient(
                QdrantGrpcClient.newBuilder(host, port, false).build()
        );

        return QdrantVectorStore.builder(qdrantClient, embeddingModel)
                .collectionName("user_long_term_memory")
                .initializeSchema(true)
                .build();
    }
```

### 4.5 Embedding 维度自定义

```java
/**
 * 使用 DashScope text-embedding-v3 的自定义维度
 * 减少维度可降低存储开销，同时保持较高检索质量
 */
@Configuration
public class DashScopeEmbeddingConfig {

    @Bean
    public EmbeddingModel dashScopeEmbeddingModel(DashScopeApi dashScopeApi) {
        DashScopeEmbeddingOptions options = DashScopeEmbeddingOptions.builder()
                .model("text-embedding-v3")
                .dimensions(512)   // 可选：1024/768/512/256/128/64
                .build();

        return new DashScopeEmbeddingModel(dashScopeApi, options);
    }

```

### 4.6 向量数据库操作服务

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class QdrantKnowledgeService {

    private final VectorStore vectorStore;

    /**
     * 批量添加文档到知识库
     */
    public void addDocuments(List<String> textChunks,
                              String category,
                              String sourceFile) {
        List<Document> documents = textChunks.stream()
                .map(chunk -> Document.builder()
                        .text(chunk)
                        .metadata(Map.of(
                                "category", category,
                                "source", sourceFile,
                                "created_at", LocalDate.now().toString(),
                                "version", "1.0"
                        ))
                        .build())
                .toList();

        vectorStore.add(documents);
        log.info("✅ 成功添加 {} 个文档块到 Qdrant，分类：{}", documents.size(), category);
    }

    /**
     * 语义相似度检索
     */
    public List<Document> search(String query, int topK) {
        return vectorStore.similaritySearch(
                SearchRequest.builder()
                        .query(query)
                        .topK(topK)
                        .similarityThreshold(0.7)
                        .build()
        );
    }

    /**
     * 带元数据过滤的检索（按文档分类）
     */
    public List<Document> searchWithCategoryFilter(String query,
                                                    String category,
                                                    int topK) {
        FilterExpressionBuilder b = new FilterExpressionBuilder();
        Filter.Expression filterExpr = b.eq("category", category).build();

        return vectorStore.similaritySearch(
                SearchRequest.builder()
                        .query(query)
                        .topK(topK)
                        .similarityThreshold(0.65)
                        .filterExpression(filterExpr)
                        .build()
        );
    }

    /**
     * 删除文档
     */
    public void deleteDocuments(List<String> documentIds) {
        vectorStore.delete(documentIds);
        log.info("🗑️ 已删除 {} 个文档", documentIds.size());
    }
}
```

### 4.7 完整 RAG 知识库管理接口

```java
@RestController
@RequestMapping("/knowledge")
@RequiredArgsConstructor
public class KnowledgeBaseController {

    private final QdrantKnowledgeService knowledgeService;
    private final TokenTextSplitter textSplitter;

    @Qualifier("ragChatClient")
    private final ChatClient ragChatClient;

    /**
     * 上传文档入库
     * POST /knowledge/ingest
     */
    @PostMapping("/ingest")
    public ResponseEntity<IngestResult> ingestDocument(
            @RequestParam("file") MultipartFile file,
            @RequestParam("category") String category) throws IOException {

        String content = new String(file.getBytes(), StandardCharsets.UTF_8);

        // 使用 TokenTextSplitter 分块（按 Token 数切分）
        List<Document> rawDocs = List.of(Document.builder().text(content).build());
        List<Document> chunks = textSplitter.split(rawDocs);

        List<String> chunkTexts = chunks.stream()
                .map(Document::getText)
                .toList();

        knowledgeService.addDocuments(chunkTexts, category, file.getOriginalFilename());

        return ResponseEntity.ok(new IngestResult(
                file.getOriginalFilename(), category, chunkTexts.size()));
    }

    /**
     * 知识库问答（RAG）
     * POST /knowledge/ask
     */
    @PostMapping("/ask")
    public String ask(@RequestBody KnowledgeAskRequest request) {
        return ragChatClient.prompt()
                .user(request.question())
                .advisors(spec -> {
                    spec.param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                               request.sessionId());
                    if (request.category() != null) {
                        spec.param(QuestionAnswerAdvisor.FILTER_EXPRESSION,
                                   "category == '" + request.category() + "'");
                    }
                })
                .call()
                .content();
    }

    /**
     * 流式知识库问答
     */
    @PostMapping(value = "/stream-ask", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> streamAsk(@RequestBody KnowledgeAskRequest request) {
        return ragChatClient.prompt()
                .user(request.question())
                .advisors(spec ->
                        spec.param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY,
                                   request.sessionId()))
                .stream()
                .content();
    }

    record IngestResult(String fileName, String category, int chunkCount) {}
    record KnowledgeAskRequest(String sessionId, String question, String category) {}
}
```

## 5. ETL 文档处理管道

Spring AI Alibaba 继承 Spring AI 的完整 ETL 框架，支持 PDF、Word、网页等多种格式文档处理。

### 5.1 PDF / Word / 网页文档读取

```xml
<!-- PDF 读取器 -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-pdf-document-reader</artifactId>
</dependency>

<!-- Tika 通用读取器（Word、Excel、PPT 等） -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-tika-document-reader</artifactId>
</dependency>
```

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class DocumentIngestionService {

    private final VectorStore vectorStore;

    @Bean
    public TokenTextSplitter tokenTextSplitter() {
        return new TokenTextSplitter(
                500,    // 每块最大 Token（DashScope 建议 256-512）
                80,     // 块间重叠 Token（保证上下文连续性）
                5,
                10000,
                true
        );
    }

    /**
     * 处理 PDF 文档
     */
    public void ingestPdf(Resource pdfResource, String category) {
        // 1. Extract：读取 PDF
        PagePdfDocumentReader reader = new PagePdfDocumentReader(
                pdfResource,
                PdfDocumentReaderConfig.builder()
                        .withPageTopMargin(0)
                        .withPageBottomMargin(0)
                        .build()
        );

        List<Document> rawDocs = reader.read();

        // 2. Transform：文本分块
        List<Document> splitDocs = tokenTextSplitter().split(rawDocs).stream()
                .map(doc -> Document.builder()
                        .text(doc.getText())
                        .metadata(new HashMap<>(doc.getMetadata()) {{
                            put("category", category);
                            put("file_type", "pdf");
                            put("source", pdfResource.getFilename());
                        }})
                        .build())
                .toList();

        // 3. Load：存入 Qdrant（DashScope 自动 Embedding）
        vectorStore.add(splitDocs);
        log.info("✅ PDF 文档 {} 处理完成，共 {} 块", pdfResource.getFilename(), splitDocs.size());
    }

    /**
     * 处理通用格式文件（Word / Excel / PPT / HTML 等）
     */
    public void ingestGenericFile(Resource resource, String category) {
        TikaDocumentReader reader = new TikaDocumentReader(resource);
        List<Document> docs = reader.read();

        List<Document> splitDocs = tokenTextSplitter().split(docs).stream()
                .map(doc -> Document.builder()
                        .text(doc.getText())
                        .metadata(Map.of("category", category,
                                         "source", resource.getFilename()))
                        .build())
                .toList();

        vectorStore.add(splitDocs);
    }

    /**
     * 批量处理目录下所有文件
     */
    public void ingestDirectory(Path directory, String category) throws IOException {
        try (Stream<Path> files = Files.walk(directory)) {
            files.filter(Files::isRegularFile)
                 .filter(p -> p.toString().matches(".*\\.(pdf|docx|txt|md)$"))
                 .forEach(file -> {
                     try {
                         Resource resource = new FileSystemResource(file);
                         if (file.toString().endsWith(".pdf")) {
                             ingestPdf(resource, category);
                         } else {
                             ingestGenericFile(resource, category);
                         }
                     } catch (Exception e) {
                         log.error("❌ 处理文件失败：{}", file, e);
                     }
                 });
        }
    }
}
```

### 6.4 Skill 路由器（智能分发）

```java
/**
 * Skill 路由器：根据用户意图智能分发到对应 Skill
 */
@Service
@RequiredArgsConstructor
@Slf4j
public class SkillRouter {

    private final CustomerServiceSkill customerServiceSkill;
    private final CodeAssistantSkill codeAssistantSkill;
    private final DashScopeChatModel chatModel;

    // 轻量路由模型（用 qwen-turbo 减少延迟和成本）
    private final ChatClient routerClient;

    public SkillRouter(CustomerServiceSkill csSkill,
                       CodeAssistantSkill codeSkill,
                       DashScopeChatModel chatModel) {
        this.customerServiceSkill = csSkill;
        this.codeAssistantSkill = codeSkill;
        this.chatModel = chatModel;
        this.routerClient = ChatClient.builder(chatModel)
                .defaultOptions(DashScopeChatOptions.builder()
                        .withModel("qwen-turbo")   // 路由用最轻量模型
                        .withTemperature(0.0)      // 意图识别需要确定性
                        .build())
                .build();
    }

    /**
     * 智能路由：先识别意图，再分发到对应 Skill
     */
    public String route(String userId, String message) {
        // 意图识别
        Intent intent = routerClient.prompt()
                .system("""
                        判断用户消息属于哪个类别，只返回 JSON 不要其他内容：
                        {"type": "customer_service" | "code_assistant" | "general"}
                        
                        分类规则：
                        - customer_service：订单、售后、产品咨询、退款等客服问题
                        - code_assistant：代码编写、Bug 修复、技术问题、架构设计
                        - general：日常对话、其他
                        """)
                .user(message)
                .call()
                .entity(Intent.class);

        log.info("用户 {} 意图识别：{}", userId, intent.type());

        return switch (intent.type()) {
            case "customer_service" -> customerServiceSkill.assist(userId, message);
            case "code_assistant"   -> codeAssistantSkill.generateCode(message);
            default                 -> routerClient.prompt().user(message).call().content();
        };
    }

    record Intent(String type) {}
}
```

### 7.2 构建顺序工作流（文章生成）

```java
/**
 * StateGraph 工作流：文章生成管道
 * 步骤：用户输入 → 主题分析 → 大纲生成 → 内容写作 → 润色优化 → 输出
 */
@Component
@RequiredArgsConstructor
public class ArticleGenerationWorkflow {

    private final DashScopeChatModel chatModel;

    /**
     * 构建并执行文章生成工作流
     */
    public ArticleResult generateArticle(String topic) throws Exception {
        // 定义工作流状态
        StateGraph<ArticleState> graph = new StateGraph<>(ArticleState::new)
                // 添加节点
                .addNode("analyze", this::analyzeTopicNode)
                .addNode("outline", this::generateOutlineNode)
                .addNode("write", this::writeContentNode)
                .addNode("polish", this::polishContentNode)
                // 定义执行顺序（边）
                .addEdge(START, "analyze")
                .addEdge("analyze", "outline")
                .addEdge("outline", "write")
                .addEdge("write", "polish")
                .addEdge("polish", END);

        // 编译并执行
        CompiledGraph<ArticleState> compiledGraph = graph.compile();

        ArticleState initialState = new ArticleState();
        initialState.put("topic", topic);

        ArticleState result = compiledGraph.invoke(initialState).get();

        return new ArticleResult(
                topic,
                (String) result.get("analysis"),
                (String) result.get("outline"),
                (String) result.get("polishedContent")
        );
    }

    // 节点：主题分析
    private Map<String, Object> analyzeTopicNode(ArticleState state) {
        String topic = (String) state.get("topic");
        ChatClient client = ChatClient.builder(chatModel).build();

        String analysis = client.prompt()
                .system("你是内容策略专家，分析主题的核心角度和目标读者。")
                .user("分析主题：" + topic + "\n请返回：目标读者、核心角度、3个关键论点")
                .call()
                .content();

        return Map.of("analysis", analysis);
    }

    // 节点：大纲生成
    private Map<String, Object> generateOutlineNode(ArticleState state) {
        ChatClient client = ChatClient.builder(chatModel).build();
        String outline = client.prompt()
                .system("你是大纲规划专家，生成结构清晰的文章大纲。")
                .user("基于以下主题分析生成文章大纲：\n" + state.get("analysis"))
                .call()
                .content();

        return Map.of("outline", outline);
    }

    // 节点：内容写作
    private Map<String, Object> writeContentNode(ArticleState state) {
        ChatClient client = ChatClient.builder(chatModel)
                .defaultOptions(DashScopeChatOptions.builder()
                        .withModel("qwen-max")
                        .withMaxTokens(3000)
                        .build())
                .build();

        String content = client.prompt()
                .system("你是专业内容创作者，按大纲撰写高质量文章。")
                .user("按照以下大纲撰写完整文章：\n" + state.get("outline"))
                .call()
                .content();

        return Map.of("draftContent", content);
    }

    // 节点：内容润色
    private Map<String, Object> polishContentNode(ArticleState state) {
        ChatClient client = ChatClient.builder(chatModel).build();
        String polished = client.prompt()
                .system("你是资深编辑，负责文章润色和风格优化。")
                .user("请润色以下文章，改善表达，提升可读性：\n" + state.get("draftContent"))
                .call()
                .content();

        return Map.of("polishedContent", polished);
    }

    // 工作流状态（继承 HashMap）
    static class ArticleState extends HashMap<String, Object> {}

    record ArticleResult(String topic, String analysis, String outline, String content) {}
}
```

### 7.3 带条件分支的工作流（智能客服）

```java
/**
 * 带条件边的工作流：智能问题分类路由
 */
@Component
@RequiredArgsConstructor
public class IntelligentServiceWorkflow {

    private final DashScopeChatModel chatModel;

    public String process(String userQuestion) throws Exception {
        StateGraph<ServiceState> graph = new StateGraph<>(ServiceState::new)
                .addNode("classify", this::classifyQuestionNode)
                .addNode("rag_answer", this::ragAnswerNode)
                .addNode("tool_answer", this::toolAnswerNode)
                .addNode("general_answer", this::generalAnswerNode)
                // 条件边：根据分类跳转到不同节点
                .addEdge(START, "classify")
                .addConditionalEdges("classify",
                        this::routeByClassification,
                        Map.of(
                                "knowledge_base", "rag_answer",
                                "tool_call", "tool_answer",
                                "general", "general_answer"
                        ))
                .addEdge("rag_answer", END)
                .addEdge("tool_answer", END)
                .addEdge("general_answer", END);

        CompiledGraph<ServiceState> compiled = graph.compile();
        ServiceState state = new ServiceState();
        state.put("question", userQuestion);

        ServiceState result = compiled.invoke(state).get();
        return (String) result.get("answer");
    }

    private Map<String, Object> classifyQuestionNode(ServiceState state) {
        // 问题分类
        return Map.of("classification", "knowledge_base");  // 示例
    }

    private String routeByClassification(ServiceState state) {
        return (String) state.getOrDefault("classification", "general");
    }

    private Map<String, Object> ragAnswerNode(ServiceState state) {
        return Map.of("answer", "基于知识库的回答：...");
    }

    private Map<String, Object> toolAnswerNode(ServiceState state) {
        return Map.of("answer", "通过工具调用获取的回答：...");
    }

    private Map<String, Object> generalAnswerNode(ServiceState state) {
        return Map.of("answer", "通用回答：...");
    }

    static class ServiceState extends HashMap<String, Object> {}
}
```
