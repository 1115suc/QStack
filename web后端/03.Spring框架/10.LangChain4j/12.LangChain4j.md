# LangChain4j 

## 一、聊天记忆（Chat Memory）

LangChain4j 的"记忆"本质是在每次请求时**将历史消息追加到上下文**中，分为两种实现：

| 实现类 | 说明 | 适用场景 |
|---|---|---|
| `MessageWindowChatMemory` | 保留最近 N 条消息（滑动窗口） | 普通多轮对话 |
| `TokenWindowChatMemory` | 保留最近 N 个 Token（超出整条删除） | Token 精确控制场景 |

### 1.1 最简单的内存记忆（单用户）

```java
@Bean
public ChatMemory chatMemory() {
    // 保留最近 20 条消息
    return MessageWindowChatMemory.withMaxMessages(20);
}
```

```java
@AiService
public interface MemoryAssistant {

    // 注入 ChatMemory Bean，自动管理对话历史
    @SystemMessage("你是一个乐于助人的 AI 助手，请记住我们对话的上下文。")
    String chat(String userMessage);
}
```

### 1.2 TokenWindow 记忆（精确 Token 控制）

```java
@Bean
public ChatMemory tokenWindowChatMemory(Tokenizer tokenizer) {
    return TokenWindowChatMemory.builder()
            .maxTokens(4096, tokenizer)   // 最多保留 4096 个 Token 的历史
            .build();
}

@Bean
public Tokenizer tokenizer() {
    // OpenAI 系列使用 OpenAiTokenizer
    return new OpenAiTokenizer("gpt-4o");
}
```

## 二、用户间记忆隔离与持久化

### 2.1 基于 @MemoryId 的多用户隔离（内存存储）

`@MemoryId` 注解让框架为每个 ID 维护一份独立的记忆，实现用户间完全隔离。

```java
@AiService
public interface MultiUserAssistant {

    @SystemMessage("你是一个专业的客服助手，请根据对话历史提供个性化服务。")
    String chat(@MemoryId String userId, @UserMessage String userMessage);
}
```

```java
@RestController
@RequestMapping("/api/memory-chat")
@RequiredArgsConstructor
public class MemoryChatController {

    private final MultiUserAssistant multiUserAssistant;

    @PostMapping
    public String chat(
            @RequestHeader("X-User-Id") String userId,
            @RequestBody ChatRequest request) {
        // 不同 userId 的对话历史完全隔离
        return multiUserAssistant.chat(userId, request.getMessage());
    }
}
```

### 2.2 ChatMemoryStore：自定义存储实现

默认情况下记忆存放在 JVM 内存中，重启后丢失。通过实现 `ChatMemoryStore` 接口可持久化到任意存储。

**接口定义：**

```java
public interface ChatMemoryStore {
    List<ChatMessage> getMessages(Object memoryId);
    void updateMessages(Object memoryId, List<ChatMessage> messages);
    void deleteMessages(Object memoryId);
}
```

**2.2.1 Redis 持久化实现**

```xml
<!-- 引入 Redis 依赖 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

```java
@Component
@RequiredArgsConstructor
public class RedisChatMemoryStore implements ChatMemoryStore {

    private final RedisTemplate<String, String> redisTemplate;
    private static final String KEY_PREFIX = "chat:memory:";
    private static final long EXPIRE_DAYS = 7; // 7天过期

    @Override
    public List<ChatMessage> getMessages(Object memoryId) {
        String key = KEY_PREFIX + memoryId;
        String json = redisTemplate.opsForValue().get(key);
        if (json == null || json.isEmpty()) {
            return new ArrayList<>();
        }
        // 使用 LangChain4j 提供的工具类序列化
        return ChatMessageDeserializer.messagesFromJson(json);
    }

    @Override
    public void updateMessages(Object memoryId, List<ChatMessage> messages) {
        String key = KEY_PREFIX + memoryId;
        String json = ChatMessageSerializer.messagesToJson(messages);
        redisTemplate.opsForValue().set(key, json, Duration.ofDays(EXPIRE_DAYS));
    }

    @Override
    public void deleteMessages(Object memoryId) {
        String key = KEY_PREFIX + memoryId;
        redisTemplate.delete(key);
    }
}
```

**2.2.2 MongoDB 持久化实现**

```java
// MongoDB 文档实体
@Document(collection = "chat_memories")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class ChatMemoryDocument {

    @Id
    private String memoryId;
    private String messagesJson;
    @LastModifiedDate
    private LocalDateTime updatedAt;
}
```

```java
public interface ChatMemoryRepository extends MongoRepository<ChatMemoryDocument, String> {}
```

```java
@Component
@RequiredArgsConstructor
public class MongoChatMemoryStore implements ChatMemoryStore {

    private final ChatMemoryRepository chatMemoryRepository;

    @Override
    public List<ChatMessage> getMessages(Object memoryId) {
        return chatMemoryRepository.findById(memoryId.toString())
                .map(doc -> ChatMessageDeserializer.messagesFromJson(doc.getMessagesJson()))
                .orElse(new ArrayList<>());
    }

    @Override
    public void updateMessages(Object memoryId, List<ChatMessage> messages) {
        String json = ChatMessageSerializer.messagesToJson(messages);
        ChatMemoryDocument doc = new ChatMemoryDocument(
                memoryId.toString(), json, LocalDateTime.now()
        );
        chatMemoryRepository.save(doc);
    }

    @Override
    public void deleteMessages(Object memoryId) {
        chatMemoryRepository.deleteById(memoryId.toString());
    }
}
```

### 2.3 注册持久化 Store 到 AI Service

```java
@Configuration
@RequiredArgsConstructor
public class AiServiceConfig {

    private final RedisChatMemoryStore redisChatMemoryStore;
    // 或 private final MongoChatMemoryStore mongoChatMemoryStore;

    @Bean
    public ChatMemoryProvider chatMemoryProvider() {
        return memoryId -> MessageWindowChatMemory.builder()
                .id(memoryId)
                .maxMessages(30)
                .chatMemoryStore(redisChatMemoryStore) // 替换为 Redis 存储
                .build();
    }
}
```

```java
// AI Service 使用 ChatMemoryProvider（而非单个 ChatMemory Bean）
@AiService
public interface PersistentAssistant {

    @SystemMessage("你是一个专业的 AI 助手，请记住用户的偏好和历史问题。")
    String chat(@MemoryId String userId, @UserMessage String userMessage);
}
```

### 2.4 多维度记忆隔离（用户 + 会话）

实际业务中，同一用户可能有多个独立会话（如多标签页），可以将 `userId + sessionId` 组合作为 `@MemoryId`：

```java
@AiService
public interface SessionAwareAssistant {

    @SystemMessage("你是一个专业的 AI 助手。")
    String chat(@MemoryId String sessionKey, @UserMessage String userMessage);
}
```

```java
@Service
@RequiredArgsConstructor
public class SessionChatService {

    private final SessionAwareAssistant sessionAwareAssistant;

    public String chat(String userId, String sessionId, String message) {
        // 组合 Key：用户 + 会话双重隔离
        String sessionKey = userId + ":" + sessionId;
        return sessionAwareAssistant.chat(sessionKey, message);
    }

    /**
     * 清除指定用户的指定会话记忆
     */
    public void clearSession(String userId, String sessionId, ChatMemoryStore store) {
        String sessionKey = userId + ":" + sessionId;
        store.deleteMessages(sessionKey);
    }
}
```

### 2.5 系统提示词 + 记忆注入顺序

```
系统消息（SystemMessage）
     ↓
历史对话消息（从 ChatMemoryStore 读取）
     ↓
本次用户消息（UserMessage）
     ↓
→ 发送给 LLM
```

## 三、工具调用（Tool Calling）

工具调用允许模型在生成响应时，自主决定调用预先定义的 Java 方法（Function），再将结果融入最终回答。

### 3.1 定义工具（@Tool 注解）

```java
@Component
public class WeatherTool {

    /**
     * 获取指定城市的天气
     *
     * @param city 城市名称，如"北京"、"上海"
     * @return 天气描述字符串
     */
    @Tool("根据城市名称获取当前天气信息")
    public String getWeather(@P("城市名称") String city) {
        // 实际应调用天气 API，这里模拟返回
        Map<String, String> weatherData = Map.of(
                "北京", "晴天，气温 28°C，湿度 45%",
                "上海", "多云，气温 25°C，湿度 78%",
                "广州", "小雨，气温 32°C，湿度 90%"
        );
        return weatherData.getOrDefault(city, "暂无 " + city + " 的天气数据");
    }

    /**
     * 获取未来 N 天的天气预报
     */
    @Tool("获取指定城市未来几天的天气预报")
    public String getWeatherForecast(
            @P("城市名称") String city,
            @P("预报天数（1-7）") int days) {
        return String.format("%s 未来 %d 天：预计晴天为主，温度 20-30°C", city, days);
    }
}
```

```java
@Component
public class CalculatorTool {

    @Tool("执行数学计算，支持加减乘除")
    public double calculate(
            @P("数字A") double a,
            @P("运算符：+、-、*、/") String operator,
            @P("数字B") double b) {
        return switch (operator) {
            case "+" -> a + b;
            case "-" -> a - b;
            case "*" -> a * b;
            case "/" -> {
                if (b == 0) throw new ArithmeticException("除数不能为零");
                yield a / b;
            }
            default -> throw new IllegalArgumentException("不支持的运算符：" + operator);
        };
    }

    @Tool("查询当前时间")
    public String getCurrentTime() {
        return LocalDateTime.now()
                .format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    }
}
```

### 3.2 将工具挂载到 AI Service

**方式一：注解声明（推荐，Spring Boot Starter 自动扫描）**

```java
@AiService(
    tools = {WeatherTool.class, CalculatorTool.class}   // 指定工具类
)
public interface ToolAssistant {

    @SystemMessage("""
            你是一个智能助手，拥有以下能力：
            1. 查询城市天气
            2. 执行数学计算
            3. 查询当前时间
            当用户需要这些信息时，请主动调用对应工具。
            """)
    String chat(@MemoryId String sessionId, @UserMessage String userMessage);
}
```

**方式二：自动扫描所有 @Tool Bean（全局注册）**

```yaml
# application.yml 开启自动工具扫描
langchain4j:
  spring-boot-starter:
    auto-tool-discovery: true
```

### 3.3 工具调用完整示例

```java
@RestController
@RequestMapping("/api/tools")
@RequiredArgsConstructor
public class ToolController {

    private final ToolAssistant toolAssistant;

    @PostMapping("/chat")
```

**对话示例：**
```

模型（内部）：调用 getWeather("北京")
工具返回：晴天，气温 28°C，湿度 45%
AI 回复：北京今天天气晴朗，气温 28°C，湿度适中（45%），适合外出活动。
```

### 3.4 工具调用异常处理

```java
@Component
public class DatabaseQueryTool {

    private final UserRepository userRepository;

    @Tool("根据用户 ID 查询用户信息")
    public String queryUser(@P("用户ID") Long userId) {
        try {
            return userRepository.findById(userId)
                    .map(user -> String.format("用户：%s，邮箱：%s", user.getName(), user.getEmail()))
                    .orElse("用户不存在，ID：" + userId);
        } catch (Exception e) {
            // 工具异常会被框架捕获并告知模型
            return "查询失败：" + e.getMessage();
        }
    }
}
```

### 3.5 工具调用的执行流程

```
用户消息
    ↓
LLM 判断是否需要工具
    ↓（需要）
生成 ToolExecutionRequest（工具名 + 参数）
    ↓
LangChain4j 框架调用对应 @Tool 方法
    ↓
工具返回 ToolExecutionResultMessage
    ↓
LLM 综合工具结果生成最终回复
    ↓
返回给用户
```

### 4.3 手动构建 JSON Schema（低级 API）

当需要更精细控制 Schema 时，使用 `ChatRequest` + `ResponseFormat`：

```java
@Service
@RequiredArgsConstructor
public class StructuredOutputService {

    private final ChatLanguageModel chatLanguageModel;

    public String generateStructuredJson(String userInput) {
        // 手动构建 JSON Schema
        JsonSchemaElement nameSchema = JsonStringSchema.builder()
                .description("人员姓名")
                .build();

        JsonSchemaElement ageSchema = JsonIntegerSchema.builder()
                .description("年龄，0-150 之间")
                .build();

        JsonSchemaElement emailSchema = JsonStringSchema.builder()
                .description("邮箱地址")
                .build();

        JsonObjectSchema personSchema = JsonObjectSchema.builder()
                .addProperty("name", nameSchema)
                .addProperty("age", ageSchema)
                .addProperty("email", emailSchema)
                .required("name", "age", "email")
                .build();

        // 构建 ResponseFormat
        ResponseFormat responseFormat = ResponseFormat.builder()
                .type(ResponseFormatType.JSON)
                .jsonSchema(JsonSchema.builder()
                        .name("PersonInfo")
                        .rootElement(personSchema)
                        .build())
                .build();

        // 构建 ChatRequest
        ChatRequest chatRequest = ChatRequest.builder()
                .messages(List.of(
                        SystemMessage.from("请从用户输入中提取人员信息，按 JSON 格式输出。"),
                        UserMessage.from(userInput)
                ))
                .responseFormat(responseFormat)
                .build();

        ChatResponse response = chatLanguageModel.chat(chatRequest);
        return response.aiMessage().text();
    }
}
```

### 5.2 动态提示词构建

```java
@Service
@RequiredArgsConstructor
public class DynamicPromptService {

    private final ChatLanguageModel chatLanguageModel;

    public String generateWithDynamicPrompt(String role, String context, String question) {
        PromptTemplate systemTemplate = PromptTemplate.from(
                "你是一名 {{role}}。请基于以下背景信息回答问题：\n背景：{{context}}"
        );

        Prompt systemPrompt = systemTemplate.apply(
                Map.of("role", role, "context", context)

                SystemMessage.from(systemPrompt.text()),
                UserMessage.from(question)
        );

        return chatLanguageModel.generate(messages).content().text();
    }
}
```

## 六、异常处理与可靠性

### 6.1 全局 AI 异常处理

```java
@RestControllerAdvice
public class AiExceptionHandler {

    @ExceptionHandler(dev.langchain4j.exception.HttpException.class)
    public ResponseEntity<String> handleHttpException(HttpException e) {
        if (e.statusCode() == 429) {
            return ResponseEntity.status(429).body("请求频率过高，请稍后重试");
        }
        if (e.statusCode() == 401) {
            return ResponseEntity.status(500).body("AI 服务认证失败，请检查 API Key");
        }
        return ResponseEntity.status(500).body("AI 服务异常：" + e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<String> handleGenericException(Exception e) {
        return ResponseEntity.status(500).body("系统错误：" + e.getMessage());
    }
}
```
