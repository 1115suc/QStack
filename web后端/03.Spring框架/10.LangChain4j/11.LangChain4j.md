# LangChain4j 
## 一、项目快速引入

### 1.1 BOM 统一版本管理（推荐）

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-bom</artifactId>
            <version>1.0.0-beta3</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

### 1.2 核心依赖

```xml
<!-- LangChain4j Spring Boot 核心 Starter（AI Services、注解驱动等） -->
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-spring-boot-starter</artifactId>
</dependency>

<!-- OpenAI 兼容协议 Starter（兼容 SiliconFlow / 阿里云百炼 / Azure OpenAI 等） -->
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-spring-boot-starter</artifactId>
</dependency>

<!-- 如需使用 Ollama 本地模型 -->
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-ollama-spring-boot-starter</artifactId>
</dependency>

<!-- 如需 Google Gemini -->
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-google-ai-gemini-spring-boot-starter</artifactId>
</dependency>
```

> **说明**：引入对应 Starter 后，Spring Boot 会自动完成 `ChatLanguageModel`、`StreamingChatLanguageModel`、`EmbeddingModel` 等 Bean 的注册，无需手动 `@Bean` 定义（除非需要多实例/自定义配置）。

### 1.3 完整 pom.xml 示例

```xml
<project>
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.3.5</version>
    </parent>

    <properties>
        <java.version>17</java.version>
        <langchain4j.version>1.0.0-beta3</langchain4j.version>
    </properties>

    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>dev.langchain4j</groupId>
                <artifactId>langchain4j-bom</artifactId>
                <version>${langchain4j.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-spring-boot-starter</artifactId>
        </dependency>
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-open-ai-spring-boot-starter</artifactId>
        </dependency>
        <!-- Lombok（可选） -->
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <optional>true</optional>
        </dependency>
    </dependencies>
</project>
```

### 2.4 Java 代码方式配置（多模型场景）

当需要同时使用多个不同提供商的模型时，需要关闭自动配置并手动定义 Bean：

```java
@Configuration
public class LangChain4jConfig {

    /**
     * 主对话模型（DeepSeek）
     */
    @Bean
    @Primary
    public ChatLanguageModel primaryChatModel(
            @Value("${ai.siliconflow.api-key}") String apiKey,
            @Value("${ai.siliconflow.base-url}") String baseUrl) {
        return OpenAiChatModel.builder()
                .apiKey(apiKey)
                .baseUrl(baseUrl)
                .modelName("deepseek-ai/DeepSeek-V3")
                .temperature(0.7)
                .maxTokens(4096)
                .logRequests(true)
                .logResponses(true)
                .build();
    }

    /**
     * 主流式对话模型
     */
    @Bean
    @Primary
    public StreamingChatLanguageModel primaryStreamingChatModel(
            @Value("${ai.siliconflow.api-key}") String apiKey,
            @Value("${ai.siliconflow.base-url}") String baseUrl) {
        return OpenAiStreamingChatModel.builder()
                .apiKey(apiKey)
                .baseUrl(baseUrl)
                .modelName("deepseek-ai/DeepSeek-V3")
                .temperature(0.7)
                .build();
    }

    /**
     * Embedding 模型（用于 RAG）
     */
    @Bean
    public EmbeddingModel embeddingModel(
            @Value("${ai.siliconflow.api-key}") String apiKey,
            @Value("${ai.siliconflow.base-url}") String baseUrl) {
        return OpenAiEmbeddingModel.builder()
                .apiKey(apiKey)
                .baseUrl(baseUrl)
                .modelName("BAAI/bge-m3")
                .build();
    }
}
```

## 三、基础聊天使用

LangChain4j 提供两种聊天方式：**低级 API**（`ChatLanguageModel`）和**高级 AI Services API**（`@AiService`）。

### 3.1 低级 API（ChatLanguageModel）

```java
@Service
@RequiredArgsConstructor
public class BasicChatService {

    private final ChatLanguageModel chatLanguageModel;

    /**
     * 最简单的单次问答
     */
    public String simpleChat(String userMessage) {
        return chatLanguageModel.generate(userMessage);
    }

    /**
     * 带系统提示词的对话
     */
    public String chatWithSystemPrompt(String systemPrompt, String userMessage) {
        List<ChatMessage> messages = List.of(
                SystemMessage.from(systemPrompt),
                UserMessage.from(userMessage)
        );
        AiMessage aiMessage = chatLanguageModel.generate(messages).content();
        return aiMessage.text();
    }

    /**
     * 多轮对话（手动维护历史）
     */
    public String multiTurnChat(List<ChatMessage> history, String userMessage) {
        List<ChatMessage> messages = new ArrayList<>(history);
        messages.add(UserMessage.from(userMessage));

        Response<AiMessage> response = chatLanguageModel.generate(messages);
        AiMessage aiMessage = response.content();

        // 调用方需将 userMessage 和 aiMessage 追加到 history 中
        return aiMessage.text();
    }

    /**
     * 获取完整响应元信息（token 使用量等）
     */
    public void chatWithMetadata(String userMessage) {
        Response<AiMessage> response = chatLanguageModel.generate(
                UserMessage.from(userMessage)
        );

        System.out.println("回复内容：" + response.content().text());
        System.out.println("结束原因：" + response.finishReason());
        System.out.println("Token 用量 - 输入：" + response.tokenUsage().inputTokenCount()
                + "，输出：" + response.tokenUsage().outputTokenCount());
    }
}
```

### 3.2 高级 AI Services API（推荐）

AI Services 是 LangChain4j 的核心特性，通过**接口 + 注解**即可完成复杂 AI 交互的定义，框架自动完成实现。

**第一步：定义 AI Service 接口**

```java
@AiService
public interface AssistantService {

    /**
     * 简单问答（无记忆）
     */
    String chat(String userMessage);

    /**
     * 带系统提示词
     */
    @SystemMessage("你是一名专业的 Java 技术专家，请用简洁专业的语言回答问题。")
    String askJavaExpert(String question);

    /**
     * 使用提示词模板
     */
    @SystemMessage("你是一名 {{role}} 领域的专家")
    @UserMessage("请解释以下概念：{{concept}}")
    String explainConcept(@V("role") String role, @V("concept") String concept);
}
```

**第二步：直接注入使用**

```java
@RestController
@RequestMapping("/api/chat")
@RequiredArgsConstructor
public class ChatController {

    private final AssistantService assistantService;

    @GetMapping("/simple")
    public String simpleChat(@RequestParam String message) {
        return assistantService.chat(message);
    }

    @GetMapping("/expert")
    public String askExpert(@RequestParam String question) {
        return assistantService.askJavaExpert(question);
    }

    @GetMapping("/explain")
    public String explainConcept(
            @RequestParam String role,
            @RequestParam String concept) {
        return assistantService.explainConcept(role, concept);
    }
}
```

> **核心注解说明：**
> | 注解 | 说明 |
> |---|---|
> | `@AiService` | 标记接口为 AI Service，Spring Boot 自动扫描并生成代理 |
> | `@SystemMessage` | 设置系统提示词，支持 `{{变量}}` 模板 |
> | `@UserMessage` | 设置用户消息模板，支持 `{{变量}}` 模板 |
> | `@V("name")` | 绑定方法参数到模板变量 |
> | `@MemoryId` | 标记聊天记忆 ID 参数（见进阶篇） |

## 四、流式输出

流式输出（Streaming）让模型的响应逐字/逐 Token 实时返回，适用于聊天界面、长文本生成等场景。

### 4.1 低级 API 流式输出

```java
@Service
@RequiredArgsConstructor
public class StreamingChatService {

    private final StreamingChatLanguageModel streamingChatLanguageModel;

    /**
     * 基础流式输出（CompletableFuture 方式）
     */
    public CompletableFuture<String> streamChat(String userMessage) {
        CompletableFuture<String> future = new CompletableFuture<>();
        StringBuilder sb = new StringBuilder();

        streamingChatLanguageModel.generate(userMessage, new StreamingResponseHandler<AiMessage>() {
            @Override
            public void onNext(String token) {
                sb.append(token);
                System.out.print(token); // 实时打印
            }

            @Override
            public void onComplete(Response<AiMessage> response) {
                future.complete(sb.toString());
            }

            @Override
            public void onError(Throwable error) {
                future.completeExceptionally(error);
            }
        });

        return future;
    }
}
```

### 4.2 与 Spring WebFlux SSE 整合（推荐前端对接方式）

```java
// 接口定义
@AiService(wiringMode = AiServiceWiringMode.EXPLICIT,
           streamingChatLanguageModel = "streamingChatLanguageModel")
public interface StreamingAssistantService {

    /**
     * 返回 TokenStream，可转换为 Flux<String>
     */
    @SystemMessage("你是一个乐于助人的 AI 助手。")
    TokenStream streamChat(@MemoryId String sessionId, @UserMessage String userMessage);
}
```

```java
@RestController
@RequestMapping("/api/stream")
@RequiredArgsConstructor
public class StreamController {

    private final StreamingAssistantService streamingAssistantService;

    /**
     * SSE 流式接口
     */
    @GetMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> streamChat(
            @RequestParam String sessionId,
            @RequestParam String message) {

        TokenStream tokenStream = streamingAssistantService.streamChat(sessionId, message);

        return Flux.create(emitter ->
                tokenStream
                        .onNext(emitter::next)
                        .onComplete(response -> emitter.complete())
                        .onError(emitter::error)
                        .start()
        );
    }
}
```

**前端 JavaScript 接入示例：**

```javascript
const eventSource = new EventSource(
    `/api/stream/chat?sessionId=user123&message=${encodeURIComponent('你好')}`
);

eventSource.onmessage = (event) => {
    document.getElementById('output').innerText += event.data;
};

eventSource.onerror = () => {
    eventSource.close();
};
```

### 4.3 返回 TokenStream 手动处理

```java
@Service
@RequiredArgsConstructor
public class ManualStreamService {

    private final StreamingChatLanguageModel streamingModel;

    public void streamWithFullControl(String prompt, Consumer<String> tokenConsumer) {
        List<ChatMessage> messages = List.of(
                SystemMessage.from("你是一个 AI 助手"),
                UserMessage.from(prompt)
        );

        streamingModel.generate(messages, new StreamingResponseHandler<AiMessage>() {
            @Override
            public void onNext(String token) {
                tokenConsumer.accept(token);
            }

            @Override
            public void onComplete(Response<AiMessage> response) {
                System.out.println("\n[流式完成] Token 总量: "
                        + response.tokenUsage().totalTokenCount());
            }

            @Override
            public void onError(Throwable error) {
                log.error("流式生成异常", error);
            }
        });
    }
}
```

### 5.2 图片 + 文字输入（URL 方式）

```java
@Service
@RequiredArgsConstructor
public class MultiModalService {

    private final ChatLanguageModel chatLanguageModel;

    /**
     * 通过图片 URL 进行图像理解
     */
    public String analyzeImageFromUrl(String imageUrl, String question) {
        UserMessage userMessage = UserMessage.from(
                ImageContent.from(imageUrl),          // 图片 URL
                TextContent.from(question)            // 用户问题
        );

        Response<AiMessage> response = chatLanguageModel.generate(
                List.of(userMessage)
        );
        return response.content().text();
    }

    /**
     * 通过 Base64 编码图片进行分析
     */
    public String analyzeImageFromBase64(byte[] imageBytes, String mimeType, String question) {
        String base64 = Base64.getEncoder().encodeToString(imageBytes);

        UserMessage userMessage = UserMessage.from(
                ImageContent.from(base64, mimeType),  // Base64 图片（如 "image/png"）
                TextContent.from(question)
        );

        Response<AiMessage> response = chatLanguageModel.generate(
                List.of(userMessage)
        );
        return response.content().text();
    }

    /**
     * 从本地文件读取图片进行分析
     */
    public String analyzeLocalImage(String filePath, String question) throws IOException {
        byte[] imageBytes = Files.readAllBytes(Path.of(filePath));
        // 根据文件扩展名判断 mimeType
        String mimeType = filePath.endsWith(".png") ? "image/png" : "image/jpeg";
        return analyzeImageFromBase64(imageBytes, mimeType, question);
    }
}
```

### 5.4 在 Controller 中接收上传图片

```java
@RestController
@RequestMapping("/api/vision")
@RequiredArgsConstructor
public class VisionController {

    private final MultiModalService multiModalService;

    /**
     * 上传图片并分析

        String result = multiModalService.analyzeImageFromUrl(imageUrl, question);
        return ResponseEntity.ok(result);
    }
}
```

## 六、常见问题与注意事项

### 6.1 自动配置与手动配置冲突

若同时在 `application.yml` 配置了参数**又**手动定义了相同类型的 `@Bean`，Spring Boot 的自动配置会被覆盖，以手动 Bean 为准。若不想被覆盖，使用 `@ConditionalOnMissingBean` 保护自动配置。

### 6.2 国内访问 OpenAI 超时

```yaml
langchain4j:
  open-ai:
    chat-model:
      timeout: 120s          # 默认 60s，复杂任务适当延长
      max-retries: 3         # 失败重试次数
```

### 6.3 Token 预算控制

```yaml
langchain4j:
  open-ai:
    chat-model:
      max-tokens: 2048       # 限制单次输出最大 Token
      # max-completion-tokens: 2048  # o1/o3 系列使用此参数
```

### 6.4 日志调试

```yaml
logging:
  level:
    dev.langchain4j: DEBUG   # 输出详细的请求/响应日志
```

### 6.5 核心接口速查

| 接口/类 | 说明 |
|---|---|
| `ChatLanguageModel` | 同步对话模型接口 |
| `StreamingChatLanguageModel` | 流式对话模型接口 |
| `EmbeddingModel` | 向量嵌入模型接口 |
| `UserMessage` | 用户消息 |
| `SystemMessage` | 系统提示词消息 |
| `AiMessage` | AI 回复消息 |
| `Response<T>` | 模型响应包装（含 TokenUsage、FinishReason） |
| `TokenStream` | AI Service 流式输出令牌流 |
| `@AiService` | 声明 AI Service 接口 |
| `@SystemMessage` | AI Service 系统提示词注解 |
| `@UserMessage` | AI Service 用户消息模板注解 |
| `@V` | 模板变量绑定注解 |
