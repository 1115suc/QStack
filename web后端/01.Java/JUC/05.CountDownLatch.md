# CountDownLatch 学习笔记

## 一、核心概念

**CountDownLatch（倒计数器）**：JUC 包中的同步辅助类，允许一个或多个线程等待其他线程完成操作后再继续执行。

### 工作原理

- 初始化时设置计数器值 N
- 每次调用 `countDown()` 方法，计数器减 1
- 当计数器归零时，所有调用 `await()` 的线程被唤醒
- **一次性使用**：计数器归零后无法重置

---

## 二、核心 API

```java
// 1. 创建计数器（N 表示需要等待的任务数）
CountDownLatch latch = new CountDownLatch(N);

// 2. 计数减 1（由完成任务的线程调用）
latch.countDown();

// 3. 阻塞等待，直到计数器为 0
latch.await();

// 4. 带超时的等待（推荐，避免无限期等待）
boolean completed = latch.await(long timeout, TimeUnit unit);

```

## 三、典型应用场景

### 场景 1：等待多个子任务完成

```java
CountDownLatch latch = new CountDownLatch(5);

for (int i = 1; i <= 5; i++) {
    new Thread(() -> {
        try {
            // 执行业务逻辑
            doSomething();
        } finally {
            latch.countDown(); // 必须在 finally 中调用
        }
    }).start();
}

latch.await(); // 主线程等待
System.out.println("所有任务完成");

```

### 场景 2：模拟并发测试

```java
int threadCount = 10;
CountDownLatch startLatch = new CountDownLatch(1); // 控制同时启动
CountDownLatch endLatch = new CountDownLatch(threadCount); // 等待完成

for (int i = 0; i < threadCount; i++) {
    new Thread(() -> {
        try {
            startLatch.await(); // 等待启动信号
            // 执行业务逻辑（如调用接口）
            callApi();
        } finally {
            endLatch.countDown();
        }
    }).start();
}

startLatch.countDown(); // 释放信号，所有线程同时启动
endLatch.await(); // 等待所有线程完成

```

### 场景 3：Netty 异步通信等待响应

```java
CountDownLatch responseLatch = new CountDownLatch(1);

// 发送请求
channel.writeAndFlush(requestMessage);

// 在 Handler 中接收响应
@Override
public void channelRead(ChannelHandlerContext ctx, Object msg) {
// 处理响应
    processResponse(msg);

}

// 等待响应（带超时）
boolean received = responseLatch.await(5, TimeUnit.SECONDS);
if(!received){
        throw new TimeoutException("等待响应超时");
}
```

## 四、CountDownLatch vs join() 对比

### 4.1 功能对比表

| 对比项      | CountDownLatch        | join()                |
|----------|-----------------------|-----------------------|
| **灵活性**  | ✅ 高，可在任意位置 countDown  | ❌ 低，只能等待线程结束          |
| **超时控制** | ✅ 支持 `await(timeout)` | ❌ 不支持（仅 join(millis)） |
| **复用性**  | ❌ 一次性使用               | -                     |
| **线程管理** | ✅ 无需保存线程引用            | ❌ 需保存所有线程引用           |
| **精细控制** | ✅ 可中途触发唤醒             | ❌ 只能等待线程自然结束          |
| **适用场景** | 等待多个线程/事件             | 等待单个线程结束              |
| **底层实现** | AQS（CAS + 队列）         | Thread.join()         |

### 4.2 代码对比示例

#### 使用 CountDownLatch（推荐）

```java
public static void testCountDownLatch() throws InterruptedException {
    int taskCount = 5;
    CountDownLatch latch = new CountDownLatch(taskCount);

    for (int i = 1; i <= taskCount; i++) {
        final int taskId = i;
        new Thread(() -> {
            try {
                System.out.println("任务 " + taskId + " 执行中");
                Thread.sleep(1000);
                System.out.println("任务 " + taskId + " 完成");
            } catch (InterruptedException e) {
                e.printStackTrace();
            } finally {
                latch.countDown(); // 灵活控制，可在任意位置调用
            }
        }).start();
    }

    latch.await(); // 简洁明了
    System.out.println("所有任务完成");
}
```

#### 使用 join()（繁琐）

```java
public static void testJoin() throws InterruptedException {
    int taskCount = 5;
    Thread[] threads = new Thread[taskCount]; // 需要保存线程引用

    for (int i = 1; i <= taskCount; i++) {
        final int taskId = i;
        Thread thread = new Thread(() -> {
            try {
                System.out.println("任务 " + taskId + " 执行中");
                Thread.sleep(1000);
                System.out.println("任务 " + taskId + " 完成");
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        threads[i - 1] = thread;
        thread.start();
    }

    // 需要遍历所有线程并调用 join()
    for (Thread thread : threads) {
        thread.join(); // 必须按顺序等待
    }

    System.out.println("所有任务完成");
}
```

### 4.3 关键差异说明

**1. 灵活性差异**

```java
// CountDownLatch：可以在任何位置调用 countDown()
new Thread(() -> {
    step1();
    latch.countDown(); // 步骤1完成就通知
    step2(); // 步骤2可以继续执行
}).start();

// join()：只能等待整个线程执行完毕
thread.join(); // 必须等线程全部执行完

```

**2. 超时控制差异**

```java
// CountDownLatch：优雅的超时处理
boolean completed = latch.await(5, TimeUnit.SECONDS);
if (!completed) {
    log.warn("部分任务超时，但主线程继续执行");
}

// join()：超时后仍需处理异常
thread.join(5000);
if (thread.isAlive()) {
    log.warn("线程仍在运行"); // 但无法优雅处理
}

```

**3. 并发启动控制**

```java
// CountDownLatch：可以轻松实现多线程同时启动
CountDownLatch startLatch = new CountDownLatch(1);
for (int i = 0; i < 10; i++) {
    new Thread(() -> {
        startLatch.await(); // 所有线程在此等待
        // 同时执行业务逻辑
    }).start();
}
startLatch.countDown(); // 一键启动所有线程

// join()：无法实现此功能

```

## 五、注意事项

### ⚠️ 常见陷阱

1. **忘记调用 countDown()**

```java
// ❌ 错误示范：异常时未调用 countDown()
new Thread(() -> {
    doSomething(); // 如果抛异常，countDown() 不会执行
    latch.countDown();
}).start();

// ✅ 正确示范：在 finally 中调用
new Thread(() -> {
    try {
        doSomething();
    } finally {
        latch.countDown(); // 确保一定执行
    }
}).start();

```

2. **计数器不匹配**

```java
class Demo {
    public static void main(String[] args) {
        // ❌ 错误：创建了 5 个任务，但计数器是 3
        CountDownLatch latch = new CountDownLatch(3);
        for (int i = 0; i < 5; i++) {
            new Thread(() -> {
                latch.countDown();
            }).start();
        }
        // 结果：前 3 个任务完成后主线程就继续了
        // ✅ 正确：计数器与任务数一致
        CountDownLatch latch = new CountDownLatch(5);
    }
}
```

3. **无限期等待**

```java
// ❌ 危险：可能永远等待
latch.await();

// ✅ 安全：设置超时时间
boolean completed = latch.await(10, TimeUnit.SECONDS);
if(!completed){
    log.error("等待超时");
}
```

4. **重复使用**

```java
// ❌ CountDownLatch 不可重置
latch.await(); // 第一次使用
// latch.await(); // 第二次使用会立即返回，因为计数器已为 0

// ✅ 如需循环使用，改用 CyclicBarrier
CyclicBarrier barrier = new CyclicBarrier(5);
```

## 六、底层原理

### 6.1 AQS 实现机制

```
CountDownLatch 内部结构：
┌─────────────────────┐
│  Sync (extends AQS) │
│  - state = N        │  ← 计数器
│  - FIFO 等待队列     │  ← 存储等待的线程
└─────────────────────┘
```

**核心流程：**

1. **构造方法**：初始化 AQS 的 state 为 N
2. **countDown()**：
    - CAS 操作将 state 减 1
    - 如果 state == 0，唤醒所有等待线程
3. **await()**：
    - 如果 state > 0，将当前线程加入等待队列并阻塞
    - 如果 state == 0，直接返回

### 6.2 源码简化版

```java
public class CountDownLatch {
    private final Sync sync;

    public CountDownLatch(int count) {
        this.sync = new Sync(count);
    }

    public void await() throws InterruptedException {
        sync.acquireSharedInterruptibly(1);
    }

    public void countDown() {
        sync.releaseShared(1);
    }

    private static final class Sync extends AbstractQueuedSynchronizer {
        Sync(int count) {
            setState(count); // 初始化计数器
        }

        private int tryAcquireShared(int acquires) {
            return (getState() == 0) ? 1 : -1;
        }

        protected boolean tryReleaseShared(int releases) {
            for (; ; ) {
                int c = getState();
                if (c == 0) return false;
                int nextc = c - 1;
                if (compareAndSetState(c, nextc)) {
                    return nextc == 0; // 计数器归零时返回 true
                }
            }
        }
    }
```

## 七、最佳实践

### ✅ 推荐做法

1. **始终在 finally 中调用 countDown()**
2. **使用带超时的 await()**
3. **计数器与任务数保持一致**
4. **合理选择线程池而非手动创建线程**

### ❌ 避免做法

1. ~~忘记调用 countDown()~~
2. ~~在不确定的场景使用无超时 await()~~
3. ~~尝试重复使用 CountDownLatch~~
4. ~~计数器设置过大导致内存浪费~~

## 九、面试高频问题

### Q1: CountDownLatch 和 CyclicBarrier 的区别？

**A:**

- CountDownLatch：一个线程等待多个线程，一次性使用
- CyclicBarrier：多个线程互相等待，可循环使用

### Q2: CountDownLatch 的计数器能否增加？

**A:** 不能，只能递减。如需更灵活的控制，考虑使用 Semaphore。

### Q3: 如果某个线程没有调用 countDown() 会怎样？

**A:** 调用 await() 的线程会一直阻塞（除非使用超时版本）。这就是为什么要在 finally 中调用 countDown()。

### Q4: CountDownLatch 的底层实现原理？

**A:** 基于 AQS 实现，使用 state 作为计数器，通过 CAS 操作保证线程安全，等待的线程进入 FIFO 队列。

## 十、总结

**CountDownLatch 核心价值：**

- ✅ 简化多线程同步代码
- ✅ 提供灵活的超时控制
- ✅ 无需管理线程引用
- ✅ 适用于"一对多"等待场景

**使用口诀：**
> 创建计数器 → 子任务 countDown → 主线程 await → 注意 finally

掌握 CountDownLatch 能让你写出更优雅、更健壮的并发代码！

## CountDownLatch 与 Join 对比 Demo

```java
package course.QStack.JUCDemo;

import java.util.concurrent.CountDownLatch;
import java.util.concurrent.TimeUnit;

public class CountDownLatchDemo {

    public static void main(String[] args) throws InterruptedException {
        System.out.println("========== 场景1: CountDownLatch 基本用法 ==========");
        testCountDownLatch();

        System.out.println("\n========== 场景2: join() 实现相同功能 ==========");
        testJoin();

        System.out.println("\n========== 场景3: CountDownLatch 灵活控制（带超时） ==========");
        testCountDownLatchWithTimeout();

        System.out.println("\n========== 场景4: 模拟并发测试 ==========");
        testConcurrentSimulation();
    }

    /**
     * CountDownLatch 基本用法
     * 主线程等待5个子任务完成
     */
    private static void testCountDownLatch() throws InterruptedException {
        int taskCount = 5;
        CountDownLatch latch = new CountDownLatch(taskCount);

        for (int i = 1; i <= taskCount; i++) {
            final int taskId = i;
            new Thread(() -> {
                try {
                    System.out.println(Thread.currentThread().getName() + " - 任务 " + taskId + " 开始执行");
                    Thread.sleep((long) (Math.random() * 2000));
                    System.out.println(Thread.currentThread().getName() + " - 任务 " + taskId + " 执行完成");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    latch.countDown(); // 任务完成，计数器减1
                }
            }, "Task-" + i).start();
        }

        // 主线程等待所有任务完成
        latch.await();
        System.out.println("主线程：所有任务执行完毕，继续执行后续操作");
    }

    /**
     * 使用 join() 实现相同功能
     * 需要保存所有线程引用，不够灵活
     */
    private static void testJoin() throws InterruptedException {
        int taskCount = 5;
        Thread[] threads = new Thread[taskCount];

        for (int i = 1; i <= taskCount; i++) {
            final int taskId = i;
            Thread thread = new Thread(() -> {
                try {
                    System.out.println(Thread.currentThread().getName() + " - 任务 " + taskId + " 开始执行");
                    Thread.sleep((long) (Math.random() * 2000));
                    System.out.println(Thread.currentThread().getName() + " - 任务 " + taskId + " 执行完成");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }, "Join-Task-" + i);
            threads[i - 1] = thread;
            thread.start();
        }

        // 需要遍历所有线程并调用 join()
        for (Thread thread : threads) {
            thread.join();
        }

        System.out.println("主线程：所有任务执行完毕，继续执行后续操作");
    }

    /**
     * CountDownLatch 带超时的等待
     * 避免无限期等待，更加健壮
     */
    private static void testCountDownLatchWithTimeout() throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(3);

        for (int i = 1; i <= 3; i++) {
            final int taskId = i;
            new Thread(() -> {
                try {
                    System.out.println(Thread.currentThread().getName() + " - 任务 " + taskId + " 开始执行");
                    // 模拟某个任务执行时间过长
                    if (taskId == 2) {
                        Thread.sleep(5000);
                    } else {
                        Thread.sleep(1000);
                    }
                    System.out.println(Thread.currentThread().getName() + " - 任务 " + taskId + " 执行完成");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    latch.countDown();
                }
            }, "Timeout-Task-" + i).start();
        }

        // 设置超时时间为3秒
        boolean completed = latch.await(3, TimeUnit.SECONDS);
        if (completed) {
            System.out.println("主线程：所有任务在超时时间内完成");
        } else {
            System.out.println("主线程：等待超时，部分任务未完成，但主线程继续执行");
        }
    }

    /**
     * 模拟并发测试场景
     * 多个线程同时启动，测试接口的并发安全性
     */
    private static void testConcurrentSimulation() throws InterruptedException {
        int threadCount = 5;
        CountDownLatch startLatch = new CountDownLatch(1); // 控制同时启动
        CountDownLatch endLatch = new CountDownLatch(threadCount); // 等待所有线程完成

        for (int i = 1; i <= threadCount; i++) {
            final int taskId = i;
            new Thread(() -> {
                try {
                    startLatch.await(); // 等待统一启动信号
                    System.out.println(Thread.currentThread().getName() + " - 模拟业务处理");
                    Thread.sleep((long) (Math.random() * 1000));
                    System.out.println(Thread.currentThread().getName() + " - 业务处理完成");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    endLatch.countDown();
                }
            }, "Concurrent-" + i).start();
        }

        System.out.println("准备就绪，3秒后同时启动所有线程...");
        Thread.sleep(3000);
        startLatch.countDown(); // 释放启动信号，所有线程同时开始

        endLatch.await(); // 等待所有线程执行完毕
        System.out.println("并发测试完成，所有线程执行结束");
    }
}
```
