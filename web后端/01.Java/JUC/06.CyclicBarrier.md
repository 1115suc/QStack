# CyclicBarrier 学习笔记

## 一、核心概念

**CyclicBarrier（循环栅栏）**：JUC 包中的同步辅助类，让一组线程互相等待，直到所有线程都到达某个屏障点后再同时继续执行。

### 关键特性
- **可循环使用**：屏障点后自动重置，可重复使用
- **互相等待**：多个线程彼此等待，而非单向等待
- **屏障动作**：支持在所有线程到达后执行一个回调任务

---

## 二、核心 API

```java
// 1. 创建屏障（parties: 参与线程数, barrierAction: 屏障动作）
CyclicBarrier barrier = new CyclicBarrier(int parties);
CyclicBarrier barrier = new CyclicBarrier(int parties, Runnable barrierAction);

// 2. 等待其他线程到达屏障点
barrier.await();

// 3. 带超时的等待
barrier.await(long timeout, TimeUnit unit);

// 4. 获取当前等待的线程数
int waiting = barrier.getNumberWaiting();

// 5. 重置屏障（手动重置，慎用）
barrier.reset();
```

## 三、典型应用场景

### 场景 1：多线程协同计算

```java
public class MatrixCalculation {
    public static void main(String[] args) {
        int threadCount = 4;
        CyclicBarrier barrier = new CyclicBarrier(threadCount, () -> {
            System.out.println("所有子任务完成，开始汇总结果");
        });

        for (int i = 0; i < threadCount; i++) {
            final int taskId = i;
            new Thread(() -> {
                try {
                    System.out.println("线程 " + taskId + " 开始计算");
                    Thread.sleep((long) (Math.random() * 3000));
                    System.out.println("线程 " + taskId + " 计算完成，等待其他线程");

                    barrier.await(); // 等待其他线程

                    System.out.println("线程 " + taskId + " 继续执行后续任务");
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}
```

                    // 第三阶段：结果输出
                    System.out.println("线程 " + taskId + " - 阶段3: 输出结果");

                    e.printStackTrace();
                }
            }).start();
        }
    }
}

## 四、CyclicBarrier vs CountDownLatch

| 对比项 | CyclicBarrier | CountDownLatch |
|--------|---------------|----------------|
| **使用场景** | 多线程互相等待 | 一个线程等待多个线程 |
| **可重用性** | ✅ 可循环使用 | ❌ 一次性使用 |
| **计数器变化** | 自动重置 | 只能递减，不可重置 |
| **屏障动作** | ✅ 支持回调任务 | ❌ 不支持 |
| **失败处理** | 一个线程失败，其他线程抛出 BrokenBarrierException | 不受影响 |
| **底层实现** | ReentrantLock + Condition | AQS |

### 代码对比

```java
// CyclicBarrier：多线程协同，可重复使用
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有线程到达屏障点");
});

for (int round = 0; round < 2; round++) {
    for (int i = 0; i < 3; i++) {
        new Thread(() -> {
            barrier.await(); // 第一轮使用后，自动重置，第二轮可继续使用
        }).start();
    }
}

// CountDownLatch：一次性使用
CountDownLatch latch = new CountDownLatch(3);
for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        latch.countDown();
    }).start();
}
latch.await();
// latch 已失效，无法再次使用

```

## 五、注意事项

### ⚠️ 常见陷阱

1. **线程数不匹配**
```java
// ❌ 错误：创建了 3 个线程，但屏障要求 5 个
CyclicBarrier barrier = new CyclicBarrier(5);
for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        barrier.await(); // 永远等待，因为只有 3 个线程
    }).start();
}

// ✅ 正确：线程数与屏障数一致
CyclicBarrier barrier = new CyclicBarrier(3);

```

2. **未处理异常导致屏障损坏**

```java
// ❌ 危险：异常会导致其他线程抛出 BrokenBarrierException
CyclicBarrier barrier = new CyclicBarrier(3);
new Thread(() -> {
    try {
        doSomething();
        barrier.await();
    } catch (Exception e) {
        // 未处理异常，屏障损坏
    }
}).start();

// ✅ 安全：捕获并处理异常
try {
    barrier.await();
} catch (BrokenBarrierException e) {
    log.error("屏障已损坏", e);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
}

```

3. **超时后屏障状态**

```java
// 超时会导致屏障损坏，其他线程会抛出 BrokenBarrierException
try {
    barrier.await(5, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    log.warn("等待超时，屏障已损坏");
    // 此时其他 await() 的线程会抛出 BrokenBarrierException
}

```

## 六、底层原理

### 6.1 核心机制

```

CyclicBarrier 内部结构：
┌──────────────────────────┐
│  ReentrantLock lock      │  ← 独占锁
│  Condition trip          │  ← 条件队列
│  int count               │  ← 剩余线程数
│  int parties             │  ← 总线程数
│  Runnable barrierAction  │  ← 屏障动作
└──────────────────────────┘
```
### 6.2 工作流程

1. **线程调用 await()**：
   - 获取锁，count--
   - 如果 count > 0，进入条件队列等待
   - 如果 count == 0，执行屏障动作，唤醒所有等待线程

2. **屏障重置**：
   - 所有线程唤醒后，count 重置为 parties
   - 屏障可再次使用

3. **屏障损坏**：
   - 任一线程超时或中断，设置 broken = true
   - 其他等待线程抛出 BrokenBarrierException

### 6.3 源码简化版

```java
public class CyclicBarrier {
private final ReentrantLock lock = new ReentrantLock();
private final Condition trip = lock.newCondition();
private final int parties;
private final Runnable barrierAction;
private int count;
private boolean broken;

    public CyclicBarrier(int parties, Runnable barrierAction) {
        this.parties = parties;
        this.count = parties;
        this.barrierAction = barrierAction;
    }

    public int await() throws InterruptedException, BrokenBarrierException {
        lock.lock();
        try {
            if (broken) throw new BrokenBarrierException();
            
            int index = --count;
            if (index == 0) { // 最后一个线程到达
                boolean ranAction = false;
                try {
                    if (barrierAction != null) {
                        barrierAction.run(); // 执行屏障动作
                    }
                    ranAction = true;
                    nextGeneration(); // 重置屏障
                    return 0;
                } finally {
                    if (!ranAction) breakBarrier(); // 失败则损坏屏障
                }
            }

            // 非最后一个线程，进入等待
            while (!broken && count > 0) {
                trip.await();
            }
            
            if (broken) throw new BrokenBarrierException();
            return index;
        } finally {
            lock.unlock();
        }
    }

    private void nextGeneration() {
        trip.signalAll(); // 唤醒所有等待线程
        count = parties;  // 重置计数器
        broken = false;
    }

    private void breakBarrier() {
        broken = true;
        count = parties;
        trip.signalAll(); // 唤醒所有线程，让它们感知屏障损坏
    }
}
```

## 八、面试高频问题

### Q1: CyclicBarrier 和 CountDownLatch 的核心区别？

**A:** 
- CyclicBarrier：多线程互相等待，可循环使用，基于 ReentrantLock
- CountDownLatch：一对多等待，一次性使用，基于 AQS

### Q2: CyclicBarrier 的屏障动作何时执行？

**A:** 最后一个线程到达屏障点时，在执行 `await()` 返回前执行，由最后一个到达的线程执行。

### Q3: 如果一个线程在 await() 时超时，其他线程会怎样？

**A:** 屏障会被标记为损坏（broken），其他正在等待的线程会抛出 `BrokenBarrierException`。

### Q4: CyclicBarrier 如何实现可重用？

**A:** 所有线程到达屏障点后，调用 `nextGeneration()` 方法重置 count 为 parties，并唤醒所有线程，屏障恢复初始状态。

## 九、总结

**CyclicBarrier 核心价值：**
- ✅ 多线程协同计算
- ✅ 支持多阶段并行处理
- ✅ 可循环使用，减少对象创建
- ✅ 提供屏障动作回调

**使用口诀：**
> 创建屏障 → 线程 await → 全部到达 → 执行动作 → 自动重置

**选型建议：**
- 多线程协同、多阶段处理 → CyclicBarrier
- 等待多个任务完成 → CountDownLatch
- 控制并发访问数 → Semaphore
