# 第一章：Lambda 表达式

## 1. 函数式编程思想


| 思想                    | 核心理念                         | 举例                                   |
| ----------------------- | -------------------------------- | -------------------------------------- |
| 面向对象思想            | 强调**找对象**、由对象来完成任务 | 去北京 → 关注怎么去（高铁/飞机/汽车） |
| 函数式编程思想（JDK8+） | 强调**结果**，不关注过程         | 去北京 → 只关注是否到达               |

## 2. Lambda 表达式定义格式

```
(参数列表) -> { 方法体 }
```


| 部分 | 含义                     |
| ---- | ------------------------ |
| `()` | 重写方法的**参数**列表   |
| `->` | 将参数**传递**到方法体中 |
| `{}` | 重写方法的**方法体**     |

### 示例：简化匿名内部类

```java
// 传统匿名内部类写法
new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("我执行了");
    }
}).start();

// Lambda 表达式写法
new Thread(() -> System.out.println("我执行了")).start();
```

## 3. Lambda 表达式使用前提

1. **必须是函数式接口**作为方法参数传递
2. **函数式接口**：有且只有一个抽象方法的接口，使用 `@FunctionalInterface` 注解检测

```java
@FunctionalInterface
public interface USB {
    void open(); // 有且只有一个抽象方法
}
```

> ⚠️ 补充：Lambda 表达式本质上是函数式接口的匿名实现，编译器会自动进行类型推断。`@FunctionalInterface` 注解可以让编译器帮助检查接口是否符合函数式接口要求，如果不符合则编译报错。

## 4. Lambda 表达式省略规则

**新手使用步骤：**

1. 观察方法参数是否为函数式接口
2. 先以匿名内部类形式传递实参
3. 从 `new 接口名` 到重写方法名结束处删除，再删除右半个大括号 `}`
4. 在参数后、方法体 `{` 前加上 `->`

**省略规则（可选）：**


| 省略内容               | 条件                                         |
| ---------------------- | -------------------------------------------- |
| 参数类型               | 可以省略参数类型                             |
| 小括号`()`             | 有且只有**一个参数**时，`()` 可以省略        |
| 大括号`{}` 及分号      | 方法体**只有一条语句**时可省略               |
| `{}` 、`;` 和 `return` | 方法体只有一条**带 return 的语句**时均可省略 |

### 示例：集合排序

```java
// 完整 Lambda
Collections.sort(list, (Person o1, Person o2) -> {
    return o1.getAge() - o2.getAge();
});

// 省略后最简写法
Collections.sort(list, (o1, o2) -> o1.getAge() - o2.getAge());
```

# 第二章：函数式接口

JDK 内置了四大常用函数式接口，位于 `java.util.function` 包下：


| 接口            | 泛型     | 核心方法            | 语义                  |
| --------------- | -------- | ------------------- | --------------------- |
| `Supplier<T>`   | `<T>`    | `T get()`           | **供给型** - 提供数据 |
| `Consumer<T>`   | `<T>`    | `void accept(T t)`  | **消费型** - 操作数据 |
| `Function<T,R>` | `<T, R>` | `R apply(T t)`      | **功能型** - 类型转换 |
| `Predicate<T>`  | `<T>`    | `boolean test(T t)` | **判断型** - 条件判断 |

> ⚠️ 泛型 `<>` 中只能写引用数据类型，基本数据类型需使用对应包装类：
> `byte→Byte`，`short→Short`，`int→Integer`，`long→Long`，`float→Float`，`double→Double`，`char→Character`，`boolean→Boolean`

## 1. Supplier&lt;T&gt; 供给型接口

- 核心方法：`T get()` —— 无参，有返回值
- 用途：需要**提供/获取**某类型数据时使用

```java
public static void method(Supplier<Integer> supplier) {
    Integer max = supplier.get();
    System.out.println("max = " + max);
}

// 调用：用 Lambda 求数组最大值
method(() -> {
    int[] arr = {4, 3, 4, 6, 7};
    Arrays.sort(arr);
    return arr[arr.length - 1];
});
```

![1706074451114-1712308401534-187.png](img/1706074451114-1712308401534-187.png)

## 2. Consumer&lt;T&gt; 消费型接口

- 核心方法：`void accept(T t)` —— 有参，无返回值
- 用途：需要**操作/消费**某类型数据时使用
- 扩展方法：`andThen(Consumer after)` —— 链式消费（先执行当前 accept，再执行 after 的 accept）

```java
public static void method(Consumer<String> consumer, String s) {
    consumer.accept(s);
}

// 调用：打印字符串长度
method(s -> System.out.println(s.length()), "abcdefg");
```

![1706074799342-1712308406897-189.png](img/1706074799342-1712308406897-189.png)

## 3. Function&lt;T, R&gt; 功能型接口

- 核心方法：`R apply(T t)` —— 根据 T 类型参数，返回 R 类型结果
- 用途：**类型转换**场景

```java
public static void method(Function<Integer, String> function, Integer number) {
    String s = function.apply(number);
    System.out.println("s = " + (s + 1));
}

// 调用：Integer 转 String
method(integer -> integer + "", 200);
```

![1706075113406-1712308413000-191.png](img/1706075113406-1712308413000-191.png)

> ⚠️ 补充：`Function` 接口还提供了 `andThen(Function after)` 和 `compose(Function before)` 方法，可以实现多个 Function 的链式组合，实现函数管道（Pipeline）效果。

## 4. Predicate&lt;T&gt; 判断型接口

- 核心方法：`boolean test(T t)` —— 有参，返回布尔值
- 用途：**条件判断**，常用于 Stream 流的 `filter` 操作

```java
public static void method(Predicate<String> predicate, String s) {
    boolean test = predicate.test(s);
    System.out.println("test = " + test);
}

// 调用：判断字符串长度是否为 7
method(s -> s.length() == 7, "abcdefg");
```

> ⚠️ 补充：`Predicate` 接口提供了三个默认方法：
>
> - `and(Predicate other)` —— 逻辑与（&&）
> - `or(Predicate other)` —— 逻辑或（||）
> - `negate()` —— 逻辑非（!）

# 第三章：Stream 流

## 1. 概述

Stream 流是 JDK 8 引入的**流式编程**风格，可以看作一条"数据流水线"，通过链式调用对集合数据进行过滤、转换、统计等操作，代码更简洁。

> ⚠️ 注意：Stream 流中的"流"**不是 IO 流**，它是一种**流式计算**（数据处理方式）。

![1706078145010-1712308418974-193.png](img/1706078145010-1712308418974-193.png)

### 传统方式 vs Stream 流

```java
// 需求：筛选姓张的、且名字为3个字的人并打印

// 传统方式（繁琐）
ArrayList<String> listZhang = new ArrayList<>();
for (String s : list) {
    if (s.startsWith("张")) listZhang.add(s);
}
ArrayList<String> listThree = new ArrayList<>();
for (String s : listZhang) {
    if (s.length() == 3) listThree.add(s);
}
for (String s : listThree) System.out.println(s);

// Stream 流写法（简洁）
list.stream()
    .filter(s -> s.startsWith("张"))
    .filter(s -> s.length() == 3)
    .forEach(s -> System.out.println(s));
```

## 2. Stream 流的获取

```java
// 方式一：针对集合（Collection 中的方法）
Stream<E> stream()

// 方式二：针对数组（Stream 接口的静态方法）
static <T> Stream<T> of(T... values)
```

```java
// 集合获取 Stream
ArrayList<String> list = new ArrayList<>();
Stream<String> stream1 = list.stream();

// 数组获取 Stream
Stream<String> stream2 = Stream.of("金莲", "三上", "松下");
```

## 3. Stream 的常用方法

### 3.1 中间方法（返回新的 Stream，可链式调用）


| 方法                                | 说明                               |
| ----------------------------------- | ---------------------------------- |
| `filter(Predicate<T> predicate)`    | 按条件**过滤**元素                 |
| `limit(long maxSize)`               | 获取**前 n 个**元素                |
| `skip(long n)`                      | **跳过**前 n 个元素                |
| `distinct()`                        | **去重**（依赖 hashCode + equals） |
| `map(Function<T,R> mapper)`         | **转换**元素类型                   |
| `sorted()`                          | **排序**（自然顺序）               |
| `sorted(Comparator<T> comparator)`  | **自定义排序**                     |
| `static concat(Stream a, Stream b)` | **合并**两个流                     |

### 3.2 终结方法（结束流操作，流不可再用）


| 方法                               | 说明                |
| ---------------------------------- | ------------------- |
| `void forEach(Consumer<T> action)` | **遍历**每个元素    |
| `long count()`                     | **统计**元素个数    |
| `collect(Collectors.toList())`     | 转换为**List 集合** |
| `collect(Collectors.toSet())`      | 转换为**Set 集合**  |
| `collect(Collectors.toMap(...))`   | 转换为**Map 集合**  |

> ⚠️ 注意：**终结方法**执行后，Stream 流就不能再使用了，否则会抛出 `IllegalStateException`。

### 3.3 各方法代码示例

```java
Stream<String> stream = Stream.of("金莲", "三上", "松下", "柳岩", "张无忌");

// forEach：遍历
stream.forEach(s -> System.out.println(s));

// count：统计
long count = Stream.of("金莲", "三上", "松下", "柳岩").count(); // 4

// filter：过滤（保留长度为2的元素）
Stream.of("金莲", "三上", "松下", "柳岩", "张无忌")
      .filter(s -> s.length() == 2)
      .forEach(s -> System.out.println(s));

// limit：取前3个
Stream.of("金莲", "三上", "松下", "柳岩", "张无忌")
      .limit(3).forEach(s -> System.out.println(s));

// skip：跳过前2个
Stream.of("金莲", "三上", "松下", "柳岩", "张无忌")
      .skip(2).forEach(s -> System.out.println(s));

// distinct：去重（自定义类需重写 hashCode + equals）
Stream.of("a", "b", "a").distinct().forEach(System.out::println);

// map：类型转换（Integer → String）
Stream.of(1, 2, 3, 4, 5)
      .map(i -> i + "")
      .forEach(s -> System.out.println(s + "!"));

// concat：合并两个流
Stream<String> s1 = Stream.of("a", "b");
Stream<String> s2 = Stream.of("c", "d");
Stream.concat(s1, s2).forEach(System.out::println);

// collect：流转集合
List<String> result = Stream.of("金莲", "三上", "松下")
                             .collect(Collectors.toList());
```

## 4. Stream 综合练习

```java
// 需求：
// 队伍A：筛选3个字的名字，取前3人
// 队伍B：筛选姓张的，跳过前2人
// 合并两队，打印所有人姓名

Stream.concat(
    teamA.stream().filter(s -> s.length() == 3).limit(3),
    teamB.stream().filter(s -> s.startsWith("张")).skip(2)
).forEach(s -> System.out.println(s));
```

# 第四章：方法引用

## 1. 概述

**方法引用**（Method Reference）是 Lambda 表达式的进一步简化，用于直接引用已有方法。

**使用前提：**

- 被引用的方法要写在重写方法体里面
- 被引用的方法在**参数**、**返回值**上要与重写方法一致
- 被引用的方法**操作**的是重写方法的参数值

**简化步骤：**

```
Lambda 写法：s -> System.out.println(s)
方法引用写法：System.out::println
```

> 规律：干掉参数，干掉 `->`，干掉被引用方法的参数，把 `.` 改成 `::`

## 2. 四种方法引用格式


| 引用类型       | 格式                 | 示例           |
| -------------- | -------------------- | -------------- |
| 对象::成员方法 | `对象名::成员方法名` | `"abc"::trim`  |
| 类名::静态方法 | `类名::静态方法名`   | `Math::random` |
| 类名::构造方法 | `类名::new`          | `Person::new`  |
| 数组::构造引用 | `数据类型[]::new`    | `int[]::new`   |

### 2.1 对象名引用成员方法

```java
// Lambda 写法
method(() -> " abc ".trim());

// 方法引用写法（" abc " 是对象，trim 是成员方法）
method(" abc "::trim);
```

### 2.2 类名引用静态方法

```java
// Lambda 写法
method(() -> Math.random());

// 方法引用写法
method(Math::random);
```

### 2.3 类名引用构造方法

```java
// Lambda 写法
method(s -> new Person(s), "金莲");

// 方法引用写法（通过参数构造 Person 对象）
method(Person::new, "三上");
```

### 2.4 数组构造引用

```java
// Lambda 写法
method(integer -> new int[integer], 10);

// 方法引用写法（创建长度为 10 的 int 数组）
method(int[]::new, 10);
```

### 综合示例

```java
// 对象成员方法引用（System.out 是对象，println 是成员方法）
Stream<String> stream = Stream.of("明日", "三上", "金莲");
stream.forEach(System.out::println); // 等价于 s -> System.out.println(s)
```

## 5.2 接口私有方法（Java 9）

**接口方法演进历程：**


| JDK 版本     | 接口新增成员               |
| ------------ | -------------------------- |
| JDK 7 及之前 | 只有抽象方法和常量         |
| JDK 8        | 公共默认方法、公共静态方法 |
| **JDK 9**    | **私有方法（实例/静态）**  |

**引入原因：** 当多个默认方法或静态方法存在公共代码时，需要抽取出来，但又不希望对外暴露，因此引入了私有方法。

```java
public interface USB {
    // 私有实例方法（只能在接口内部调用）
    private void open() {
        System.out.println("私有非静态方法");
    }

    // 私有静态方法
    private static void close() {
        System.out.println("私有静态方法");
    }

    // 默认方法中调用私有方法
    public default void methodDef() {
        open();
        close();
    }
}
```

> ⚠️ 私有方法不能被实现类继承或调用，只能在接口内部的 `default` 或 `static` 方法中被调用。

## 5.3 钻石操作符与匿名内部类结合（Java 9）

**JDK 8 及之前**：钻石操作符 `<>` **不能**与匿名内部类一起使用。

**JDK 9+**：匿名内部类支持**类型自动推断**，`<>` 可与匿名内部类结合使用。

```java
// JDK 8 会报错：'<>' cannot be used with anonymous classes
// JDK 9+ 正常编译运行
Collections.sort(list, new Comparator<>() {
    @Override
    public int compare(Person o1, Person o2) {
        return o1.getAge() - o2.getAge();
    }
});
```

## 5.4 try-with-resources 升级（Java 9）

**JDK 7 引入**的 try-with-resources 要求在 `try()` 内部声明资源对象：

```java
// JDK 7 写法：资源必须在 try() 内部声明
try (FileWriter fw = new FileWriter("io.txt")) {
    fw.write("你好");
} catch (Exception e) {
    e.printStackTrace();
}
```

**JDK 9 升级**：资源对象可以在外部声明，在 `try()` 中直接引用变量名（变量必须是 effectively final）：

```java
// JDK 9 写法：先在外面创建，再放进 try()
FileWriter fw = new FileWriter("io.txt");
try (fw) {
    fw.write("你好");
} catch (Exception e) {
    e.printStackTrace();
}
```

> ⚠️ 两种写法的资源都会在 `try` 块结束时**自动关闭**（调用 `close()` 方法），该资源必须实现 `java.io.Closeable` 接口。

## 5.5 局部变量类型推断 var（Java 10）

**JDK 10** 引入 `var` 关键字，编译器根据右侧初始值**自动推断**局部变量类型。

```java
var 变量名 = 值; // 编译器自动推断类型
```

```java
var i = 10;              // 推断为 int
var j = "helloworld";   // 推断为 String
var arr = new int[]{1, 2, 3, 4, 5}; // 推断为 int[]

for (var element : arr) {
    System.out.println(element);
}
```

**使用限制：**


| 场景                 | 是否支持              |
| -------------------- | --------------------- |
| 局部变量（有初始值） | ✅ 支持               |
| 方法参数             | ❌ 不支持             |
| 成员变量             | ❌ 不支持             |
| 返回值类型           | ❌ 不支持             |
| `var x = null`       | ❌ 不支持（无法推断） |

> ⚠️ `var` 不是关键字，而是一个**保留类型名**（reserved type name），可以作为变量名使用（不推荐）。`var` 声明的变量依然是**强类型**的，只是让编译器来推断，与 JavaScript 的 `var` 有本质区别。

## 5.6 switch 表达式升级（Java 12 / 13 / 14）

### 传统 switch 的痛点

1. 忘写 `break` 会导致 **case 穿透**
2. 不同 case 中**不能定义同名变量**
3. 单个 case **不支持多个匹配值**
4. 整个 switch **不能作为表达式**返回值

### Java 12：switch 表达式预览（箭头语法）

```java
// JDK 12 之前（传统写法）
switch (month) {
    case 3: case 4: case 5:
        System.out.println("春季"); break;
    // ...
}

// JDK 12+ 箭头语法（多值合并、自动 break）
switch (month) {
    case 12, 1, 2 -> System.out.println("冬季");
    case 3, 4, 5  -> System.out.println("春季");
    case 6, 7, 8  -> System.out.println("夏季");
    case 9, 10, 11 -> System.out.println("秋季");
    default -> System.out.println("月份有误");
}
```

### Java 13：yield 返回值

`yield` 语句用于在 switch 表达式的代码块中**返回值**（类似 `return`）：

```java
// JDK 13+：switch 表达式可以有返回值，用 var 或明确类型接收
var season = switch (month) {
    case 12, 1, 2 -> { yield "冬季"; }
    case 3, 4, 5  -> { yield "春季"; }
    case 6, 7, 8  -> { yield "夏季"; }
    case 9, 10, 11 -> { yield "秋季"; }
    default -> { yield "月份有误"; }
};
System.out.println("season = " + season);
```

> **注意区分：**
>
> - `switch 语句`（不返回值）：使用 `break`
> - `switch 表达式`（有返回值）：使用 `yield`
> - 箭头语法 `->` 后跟单个表达式时，无需 yield；跟代码块时，需要 `yield` 返回值

## 5.7 文本块 Text Blocks（Java 13 预览 → Java 15 正式）

**背景**：Java 中编写多行 HTML/XML/JSON/SQL 字符串需要大量转义字符，可读性差。

**语法**：使用 `"""` 作为开始和结束分隔符：

```java
// 传统写法
String html = "<html>\n" +
              "  <body>\n" +
              "    <p>Hello, world</p>\n" +
              "  </body>\n" +
              "</html>\n";

// 文本块写法（JDK 13+）
String html = """
    <html>
      <body>
        <p>Hello, world</p>
      </body>
    </html>
    """;
```

**使用规则：**

1. 开始分隔符 `"""` 后必须紧跟**换行**，不能写在同一行 `""""""` 是错误的
2. 文本内容可自由使用双引号（无需转义）
3. 支持转义字符 `\n`、`\t`、`\b`、`\f`、`\r` 等
4. 结束 `"""` 的位置决定了缩进的基准

```java
// ❌ 错误：开始符后没有换行
String err = """""";

// ✅ 正确：空字符串的文本块写法
String emp = """
    """;

// ✅ 正确：在文本块中使用双引号
String story = """
    He said, "Hello World!"
    She said, "Hi there!"
    """;
```

> ⚠️ Java 15 正式版中文本块已稳定，**不再是预览特性**，生产环境可放心使用。

---

## 5.8 instanceof 模式匹配（Java 14 预览 → Java 16 正式）

**传统写法**：先 `instanceof` 判断，再强制类型转换

```java
// JDK 14 之前
if (animal instanceof Dog) {
    Dog dog = (Dog) animal; // 需要手动强转
    dog.lookDoor();
}
```

**JDK 14+ 新写法**：`instanceof` 判断与类型转换**合二为一**

```java
// JDK 14+（模式匹配）
if (animal instanceof Dog dog) {
    dog.lookDoor(); // dog 变量在 if 块内直接可用，无需强转
}
```

> ⚠️ `instanceof` 模式匹配在 JDK 14/15 是预览特性，**JDK 16 正式转正**。绑定变量 `dog` 的作用域仅在条件为 `true` 的代码块内。

## 5.9 Record 类（Java 14 预览 → Java 16 正式）

`record` 是一种全新的**不可变数据载体**类型，本质是一个 `final` 类，所有属性也是 `final` 的。

**自动生成：** 全参构造、`getXxx()` 访问器（注意不是 `getXxx` 而是**同名方法**）、`equals()`（比较所有属性值）、`hashCode()`、`toString()`

**语法：**

```java
public record Person(String name, Integer age) {
    // 自动拥有：全参构造、name()、age()、equals、hashCode、toString
}
```

**限制条件：**


| 限制                     | 说明                         |
| ------------------------ | ---------------------------- |
| 不能声明实例变量         | 只能使用构造参数中声明的字段 |
| 不能显式声明空参构造     | 只有全参构造                 |
| 类不能是 abstract        | record 本身是 final 类       |
| 不能显式继承其他类       | 默认继承`java.lang.Record`   |
| 不能被继承               | 因为 record 是 final 类      |
| 可以有静态字段和静态方法 | ✅                           |
| 可以有实例方法（非静态） | ✅                           |
| 可以实现接口             | ✅                           |

```java
public record Person(String name) {
    static int count;          // ✅ 可以声明静态变量
    // int age;                // ❌ 不能声明实例变量

    public static void method() { }  // ✅ 静态方法
    public void method01() { }       // ✅ 实例方法
}

// 使用
Person p1 = new Person("张三");
Person p2 = new Person("张三");
System.out.println(p1);             // Person[name=张三]
System.out.println(p1.equals(p2)); // true（比较所有字段值）
System.out.println(p1.name());     // 张三（通过同名方法访问）
```

> ⚠️ Record 的访问器方法名与属性名相同（如 `name()`），而非传统的 `getName()`，这是与普通 JavaBean 的重要区别。

## 5.10 密封类 Sealed Class（Java 15 预览 → Java 17 正式）

密封类用于**限制哪些类可以继承/实现**某个类或接口，比 `final`（完全禁止继承）更灵活。

**语法：**

```java
// 密封类声明
[修饰符] sealed class 父类 [extends ...] [implements ...] permits 子类1, 子类2 {
}

// 密封接口声明
[修饰符] sealed interface 接口 [extends ...] permits 实现类1, 实现类2 {
}
```

**子类/实现类必须**用以下三种修饰符之一修饰：


| 子类修饰符   | 说明                                 |
| ------------ | ------------------------------------ |
| `final`      | 该子类不能再被继承                   |
| `sealed`     | 该子类也是密封类，需要再指定 permits |
| `non-sealed` | 该子类放开继承限制，任何类都可以继承 |

```java
// 密封类示例
public sealed class Animal permits Dog, Cat { }

public non-sealed class Dog extends Animal { }  // 放开，Dog 可以被任意继承
public non-sealed class Cat extends Animal { }

// 密封接口示例
sealed interface Flyable permits Bird { }
non-sealed class Bird implements Flyable { }
```

> ⚠️ `sealed` 类/接口**必须至少有一个** `permits` 的子类/实现类，否则编译报错。`permits` 列表中的类必须与密封类在**同一包**（或同一模块）中。

## 5.11 Java 9 模块化系统（Module System）

**引入原因：**

Java 平台越来越庞大，存在以下问题：

1. **资源浪费**：运行 Hello World 也需要整个 JRE（rt.jar 约 60.5MB）
2. **JAR Hell**：classpath 中存在多个版本的类，运行时歧义
3. **封装性差**：所有 public 类都可以被其他任意类访问，意外暴露内部 API
4. **依赖不明确**：JAR 之间没有明确的依赖声明

**模块化优势：**

- 减少内存开销，按需加载模块
- 更好的封装性：通过声明控制哪些包对外暴露
- 更可靠的依赖关系：在编译期/启动时就能发现缺失依赖
- 可适配不同大小的计算设备（嵌入式等）

**API 结构变化：**

- Java 8：`包（package）`是顶级封装
- Java 9+：`模块（module）`是顶级封装，模块下才是包

# 总结

## 一、Lambda 表达式总结


| 知识点   | 核心内容                                                    |
| -------- | ----------------------------------------------------------- |
| 定义格式 | `(参数) -> {方法体}`                                        |
| 使用前提 | 必须是**函数式接口**作为参数（有且只有一个抽象方法）        |
| 省略规则 | 参数类型可省、单参数括号可省、单语句大括号可省、return 可省 |
| 本质     | 函数式接口的**匿名实现对象**                                |

## 二、四大函数式接口总结


| 接口             | 方法签名            | 使用场景          |
| ---------------- | ------------------- | ----------------- |
| `Supplier<T>`    | `T get()`           | 提供/生产数据     |
| `Consumer<T>`    | `void accept(T t)`  | 消费/操作数据     |
| `Function<T, R>` | `R apply(T t)`      | 类型转换/数据映射 |
| `Predicate<T>`   | `boolean test(T t)` | 条件判断/数据筛选 |

## 三、Stream 流总结


| 分类     | 方法                                                             | 特点                      |
| -------- | ---------------------------------------------------------------- | ------------------------- |
| 获取流   | `集合.stream()`、`Stream.of(...)`                                | 不消耗流                  |
| 中间操作 | `filter`、`map`、`limit`、`skip`、`distinct`、`sorted`、`concat` | 返回新 Stream，可链式调用 |
| 终结操作 | `forEach`、`count`、`collect`                                    | 结束流，**流用完即废**    |

> 💡 **最佳实践**：Stream 流是**惰性求值**的，中间操作不会立即执行，只有遇到终结操作时才会触发整条流水线的计算。

## 四、方法引用总结


| 引用类型       | 格式                  | 对应 Lambda                  |
| -------------- | --------------------- | ---------------------------- |
| 对象::成员方法 | `obj::method`         | `() -> obj.method()`         |
| 类名::静态方法 | `Class::staticMethod` | `() -> Class.staticMethod()` |
| 类名::构造方法 | `Class::new`          | `(args) -> new Class(args)`  |
| 数组::构造引用 | `Type[]::new`         | `n -> new Type[n]`           |

| 特性                      | 引入版本 | 正式确定 | 核心价值                                      |
| ------------------------- | -------- | -------- | --------------------------------------------- |
| 接口私有方法              | Java 9   | Java 9   | 接口内部代码复用                              |

## 六、重要补充说明

> #### ① Lambda vs 匿名内部类
>
> Lambda 表达式中 `this` 指向**外部类**，而匿名内部类中 `this` 指向**匿名内部类本身**。Lambda 没有自己的 `this`，更没有自己的作用域，这是两者的本质区别。

> #### ② Stream 流注意事项
>
> - Stream 是**一次性**的，终结操作后不可复用
> - `distinct()` 去重依赖 `hashCode()` 和 `equals()`，自定义类必须重写这两个方法
> - Stream 是**惰性求值**，链式中间操作不会立即执行
> - `parallel()` 可以开启并行流，利用多核 CPU 提升大数据量处理性能

> #### ③ Record 类 vs Lombok @Data
>
> - Lombok `@Data`：生成 `get/set`，可变对象，需要插件支持
> - `record`：原生 Java 支持，不可变对象，访问器方法名与字段名相同，更安全

> #### ④ 密封类使用场景
>
> 密封类非常适合配合 **switch 模式匹配**（Java 17 预览）使用，编译器可以检查所有 `permits` 的子类是否都被处理，类似 Kotlin 的 `when` + `sealed class` 组合。
