# Spring AOP 切点表达式

---

## 一、什么是切点表达式？

在 Spring AOP 中，切点表达式用于**定义"在哪里"织入增强逻辑**。
通俗地说，就是告诉 Spring：**我要拦截哪些类的哪些方法**。

切点表达式写在 `@Pointcut` 注解中，或直接写在通知注解（`@Before`、`@Around` 等）中。

```java
@Aspect
@Component
public class MyAspect {

    // 方式一：单独定义切点
    @Pointcut("execution(* com.example.service.*.*(..))")
    public void myPointcut() {}

    @Before("myPointcut()")
    public void before() { ... }

    // 方式二：直接写在通知注解中
    @Before("execution(* com.example.service.*.*(..))")
    public void before2() { ... }
}
```

---

## 二、切点指示符分类总览

Spring AOP 支持的切点指示符可以分为**四大类**：

| 分类         | 指示符                                       | 匹配依据         |
| ------------ | -------------------------------------------- | ---------------- |
| **方法匹配** | `execution`                                  | 方法的签名（最常用） |
| **类型匹配** | `within`、`this`、`target`                   | 类的类型         |
| **注解匹配** | `@annotation`、`@within`、`@target`、`@args` | 注解的位置       |
| **参数匹配** | `args`                                       | 方法参数类型     |
| **Bean匹配** | `bean`                                       | Spring Bean 的名称 |

---

## 三、逐一详解

---

### 1. `execution` — 匹配方法签名（最核心、最常用）

#### 语法格式

```
execution(修饰符? 返回类型 包名.类名.方法名(参数类型) 异常?)
```

> 其中 `?` 表示可选项

#### 通配符说明

| 符号 | 含义                                                                     |
| ---- | ------------------------------------------------------------------------ |
| `*`  | 匹配任意**单个**部分（一层包、一个类名、一个方法名、一种返回类型）       |
| `..` | 在**包路径**中表示当前包及所有子包；在**参数列表**中表示任意个数任意类型 |

#### 示例

```java
// 匹配 com.example.service 包下所有类的所有方法
@Pointcut("execution(* com.example.service.*.*(..))")

// 匹配 com.example.service 包及其子包下所有类的所有方法
@Pointcut("execution(* com.example.service..*.*(..))")

// 匹配所有 public 方法
@Pointcut("execution(public * *(..))")

// 匹配所有以 get 开头的方法
@Pointcut("execution(* get*(..))")

// 匹配 UserService 类中返回值为 String 的所有方法
@Pointcut("execution(String com.example.service.UserService.*(..))")

// 匹配只有一个 String 参数的方法
@Pointcut("execution(* *(String))")

// 匹配第一个参数为 String，后面任意的方法
@Pointcut("execution(* *(String, ..))")
```

#### 完整拆解示例

```
execution(public String com.example.service.UserService.getUserName(Long))
            │       │              │              │           │        │
          修饰符  返回类型         包名           类名       方法名    参数类型
```

---

### 2. `within` — 匹配指定类型内的所有方法

#### 作用

匹配**某个类（或某个包下所有类）内部的所有方法**，不关心方法签名。

#### 与 `execution` 的区别

- `execution`：可以精确到**方法级别**（方法名、参数、返回值）
- `within`：只能精确到**类级别**，匹配该类中的所有方法

#### 示例

```java
// 匹配 UserService 类中的所有方法
@Pointcut("within(com.example.service.UserService)")

// 匹配 service 包下所有类的所有方法
@Pointcut("within(com.example.service.*)")

// 匹配 service 包及其子包下所有类的所有方法
@Pointcut("within(com.example.service..*)")
```

#### 等价关系

```java
// 以下两种写法效果相同
within(com.example.service.UserService)
execution(* com.example.service.UserService.*(..))
```

---

### 3. `@annotation` — 匹配被指定注解标注的**方法**

#### 作用

拦截所有**被某个注解标注的方法**。注解打在哪个方法上，就拦截哪个方法。

#### 这是自定义注解 + AOP 最常用的方式

#### 示例

**第一步：自定义注解**

```java
@Target(ElementType.METHOD)           // 注解用在方法上
@Retention(RetentionPolicy.RUNTIME)   // 运行时保留
public @interface MyLog {
    String value() default "";
}
```

**第二步：编写切面**

```java
@Aspect
@Component
public class LogAspect {

    // 拦截所有被 @MyLog 标注的方法
    @Around("@annotation(com.example.annotation.MyLog)")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        System.out.println("方法执行前...");
        Object result = joinPoint.proceed();
        System.out.println("方法执行后...");
        return result;
    }
}
```

**第三步：使用注解**

```java
@Service
public class UserService {

    @MyLog("查询用户")    // ← 这个方法会被拦截
    public User getUser(Long id) { ... }

    public User findUser(Long id) { ... }  // ← 这个方法不会被拦截
}
```

#### 获取注解中的参数

```java
// 通过参数绑定的方式获取注解实例
@Around("@annotation(myLog)")   // 注意：这里写的是参数名，不是全类名
public Object around(ProceedingJoinPoint joinPoint, MyLog myLog) throws Throwable {
    String value = myLog.value();  // 获取注解的 value 值
    System.out.println("日志描述：" + value);
    return joinPoint.proceed();
}
```

---

### 4. `@within` — 匹配被指定注解标注的**类**中的所有方法

#### 作用

如果一个**类**上面有某个注解，那么这个类中的**所有方法**都会被拦截。

#### `@annotation` vs `@within` 的核心区别

| 指示符        | 注解标注在哪里 | 拦截范围                 |
| ------------- | -------------- | ------------------------ |
| `@annotation` | **方法**上     | 只拦截被标注的那个方法   |
| `@within`     | **类**上       | 拦截该类中的**所有方法** |

#### 示例

**自定义注解**

```java
@Target(ElementType.TYPE)             // 注解用在类上
@Retention(RetentionPolicy.RUNTIME)
public @interface Monitorable {
}
```

**切面**

```java
@Aspect
@Component
public class MonitorAspect {

    // 拦截所有被 @Monitorable 标注的类中的所有方法
    @Around("@within(com.example.annotation.Monitorable)")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        System.out.println("监控开始: " + joinPoint.getSignature().getName());
        Object result = joinPoint.proceed();
        System.out.println("监控结束");
        return result;
    }
}
```

**使用**

```java
@Monitorable   // ← 注解打在类上
@Service
public class OrderService {

    public void createOrder() { ... }   // ✅ 会被拦截
    public void cancelOrder() { ... }   // ✅ 会被拦截
    public void getOrder() { ... }      // ✅ 都会被拦截
}

@Service
public class UserService {
    public void getUser() { ... }       // ❌ 不会被拦截（类上没有 @Monitorable）
}
```

---

### 5. `@target` — 匹配运行时目标对象的类上有指定注解

#### 作用

与 `@within` 非常相似，区别在于：

| 指示符    | 判断时机               | 是否考虑继承                 |
| --------- | ---------------------- | ---------------------------- |
| `@within` | **编译时**（静态类型） | ❌ 只看声明类本身             |
| `@target` | **运行时**（实际对象类型） | ✅ 看实际运行时的对象        |

#### 示例

```java
@Around("@target(com.example.annotation.Monitorable)")
public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
    return joinPoint.proceed();
}
```

#### `@within` vs `@target` 区别场景

```java
@Monitorable
public class ParentService {
    public void parentMethod() { }
}

public class ChildService extends ParentService {
    public void childMethod() { }
}
```

| 调用                             | `@within` | `@target` |
| -------------------------------- | --------- | --------- |
| `parentService.parentMethod()`   | ✅ 拦截    | ✅ 拦截    |
| `childService.parentMethod()`    | ✅ 拦截（方法声明在 Parent 中） | ❌ 不拦截（ChildService 上没有注解） |
| `childService.childMethod()`     | ❌ 不拦截  | ❌ 不拦截  |

> **实际开发中**：90% 的场景用 `@within` 就够了，`@target` 很少使用。

---

### 6. `this` — 匹配代理对象的类型

#### 作用

当 Spring 生成的 **AOP 代理对象** 是指定类型（或其子类型）时，拦截其所有方法。

```java
// 拦截代理对象实现了 UserService 接口的所有方法
@Pointcut("this(com.example.service.UserService)")
```

---

### 7. `target` — 匹配目标对象的类型

#### 作用

当被代理的 **原始目标对象** 是指定类型（或其子类型）时，拦截其所有方法。

```java
// 拦截目标对象是 UserService 类型的所有方法
@Pointcut("target(com.example.service.UserService)")
```

#### `this` vs `target` 的区别

|            | `this`               | `target`                 |
| ---------- | -------------------- | ------------------------ |
| 匹配的是   | **代理对象**         | **被代理的原始对象**     |
| JDK动态代理 | 代理对象类型是接口   | 目标对象类型是实现类     |
| CGLIB代理  | 两者基本相同         | 两者基本相同             |

> **实际开发中**：大部分情况下两者效果一样，不需要刻意区分。

---

### 8. `args` — 匹配方法参数类型

#### 作用

根据方法的**运行时参数类型**进行匹配。

```java
// 拦截只有一个 String 参数的方法
@Pointcut("args(String)")

// 拦截第一个参数为 Long，后面任意的方法
@Pointcut("args(Long, ..)")
```

#### 绑定参数值

```java
@Before("execution(* com.example.service.*.*(..)) && args(id, ..)")
public void before(Long id) {
    System.out.println("传入的ID参数值: " + id);
}
```

---

### 9. `@args` — 匹配参数的类上有指定注解

#### 作用

如果方法的**参数对象**所属的类被指定注解标注，则匹配。

```java
// 拦截方法参数类型上标注了 @MyEntity 注解的方法
@Pointcut("@args(com.example.annotation.MyEntity)")
```

```java
@MyEntity           // ← 注解打在参数类型上
public class User { ... }

// 以下方法会被拦截，因为参数 User 类上有 @MyEntity
public void saveUser(User user) { ... }
```

> **使用频率**：非常低，了解即可。

---

### 10. `bean` — 匹配 Spring Bean 名称（Spring AOP 独有）

#### 作用

根据 Spring 容器中 Bean 的**名称（ID）**进行匹配。

```java
// 拦截名为 userService 的 Bean 中的所有方法
@Pointcut("bean(userService)")

// 拦截所有名称以 Service 结尾的 Bean 中的所有方法
@Pointcut("bean(*Service)")
```

## 四、组合使用（逻辑运算符）

多个切点表达式可以用逻辑运算符组合：

| 运算符 | 含义             |
| ------ | ---------------- |
| `&&`   | 且（同时满足）   |
| `\|\|` | 或（满足任一）   |
| `!`    | 非（取反）       |

```java
// 拦截 service 包下被 @MyLog 注解标注的方法
@Pointcut("within(com.example.service..*) && @annotation(com.example.annotation.MyLog)")
public void serviceLog() {}

// 拦截 service 包或 controller 包下的所有方法
@Pointcut("within(com.example.service..*) || within(com.example.controller..*)")
public void allBusiness() {}

// 拦截除 UserService 外的所有 service 包下的方法
@Pointcut("within(com.example.service..*) && !within(com.example.service.UserService)")
public void exceptUser() {}
```

---

## 五、实战中最常用的组合（速查表）

| 场景                           | 推荐写法                                                   |
| ------------------------------ | ---------------------------------------------------------- |
| 拦截某个包下所有方法           | `execution(* com.example.service..*.*(..))`                |
| 拦截某个类的所有方法           | `within(com.example.service.UserService)`                  |
| 拦截自定义注解标注的方法       | `@annotation(com.example.annotation.MyLog)`                |
| 拦截自定义注解标注的类的所有方法 | `@within(com.example.annotation.Monitorable)`              |
| 拦截某个 Bean 的所有方法       | `bean(userService)`                                        |
| 组合：某包下 + 带注解的方法    | `within(com.example.service..*) && @annotation(xx)`        |

## 六、完整实战案例：接口耗时统计

```java
// 1. 自定义注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface TimeCost {
    String desc() default "";
}

// 2. 切面类
@Aspect
@Component
@Slf4j
public class TimeCostAspect {

    @Around("@annotation(timeCost)")
    public Object around(ProceedingJoinPoint joinPoint, TimeCost timeCost) throws Throwable {
        String methodName = joinPoint.getSignature().getName();
        String desc = timeCost.desc();

        long startTime = System.currentTimeMillis();
        log.info("【{}】方法 {} 开始执行", desc, methodName);

        Object result = joinPoint.proceed();

        long costTime = System.currentTimeMillis() - startTime;
        log.info("【{}】方法 {} 执行完毕，耗时 {} ms", desc, methodName, costTime);

        return result;
    }
}

// 3. 使用
@Service
public class OrderService {

    @TimeCost(desc = "创建订单")
    public Order createOrder(OrderDTO dto) {
        // 业务逻辑...
    }
}

// 4. 输出日志
// 【创建订单】方法 createOrder 开始执行
// 【创建订单】方法 createOrder 执行完毕，耗时 126 ms
```

## 七、总结

### 记忆口诀

```
execution   看方法签名，最常用
within      看类，不管方法
@annotation 注解在方法上，拦截那一个
@within     注解在类上，拦截一整个
args        看参数类型
bean        看Bean名字
this/target 看代理/目标对象类型
```

---

### `within` 与 `@within` 的区别对比

这两个名字相似但本质完全不同，是最容易混淆的一对：

| 对比项           | `within`                                 | `@within`                                      |
| ---------------- | ---------------------------------------- | ---------------------------------------------- |
| **匹配依据**     | 类的**全限定名 / 包路径**                | 类上是否有**指定注解**                         |
| **参数内容**     | 填写的是**类名或包路径**                 | 填写的是**注解的全类名**                       |
| **支持通配符**   | ✅ 支持 `*` 和 `..`                      | ❌ 不支持，只能写完整注解类名                   |
| **拦截粒度**     | 类级别（该类所有方法）                   | 类级别（该类所有方法）                         |
| **使用场景**     | 按**包结构**批量拦截                     | 按**注解标记**选择性拦截                       |
| **耦合度**       | 与包结构强耦合，重构包名后需要同步修改   | 与注解绑定，移动包路径不影响                   |
| **灵活度**       | 固定拦截某个包/类，不够灵活             | 哪个类想被拦截，加个注解即可，非常灵活         |

#### 示例对比

```java
// within：按包路径匹配，service 包下所有类的所有方法都会被拦截
@Pointcut("within(com.example.service..*)")

// @within：按注解匹配，只有类上标注了 @Monitorable 的才会被拦截
@Pointcut("@within(com.example.annotation.Monitorable)")
```

#### 选择建议

```
┌──────────────────────────────────────────────────────┐
│  需求：拦截某个包下的所有类？                          │
│  → 用 within                                         │
│                                                      │
│  需求：灵活地选择"某些类"进行拦截，且不受包结构限制？   │
│  → 用 @within + 自定义注解                            │
│                                                      │
│  需求：只拦截某个具体方法？                            │
│  → 用 @annotation + 自定义注解                        │
└──────────────────────────────────────────────────────┘
```

> **总结一句话**：`within` 是"我指定拦截谁"，`@within` 是"谁身上有标记我就拦截谁"。
