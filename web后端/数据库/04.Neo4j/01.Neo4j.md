# 🌟 Neo4j 图数据库

## 一、Neo4j 基础介绍

### 1.1 什么是图数据库？🔍

#### 1.1.1 概念定义
图数据库是一种使用图结构进行语义查询的数据库，它使用节点、边和属性来表示和存储数据。与传统的关系型数据库（RDBMS）相比，图数据库专门设计用于存储和导航关系。

#### 1.1.2 图数据库 vs 关系型数据库

| 特性 | 图数据库（Neo4j） | 关系型数据库（MySQL） |
|------|-------------------|----------------------|
| 数据结构 | 图（节点和关系） | 表格 |
| 关系处理 | 存储为一级公民 | 通过外键关联 |
| 查询性能 | 关系查询速度快 | 复杂连接查询慢 |
| 扩展性 | 水平扩展困难 | 垂直扩展为主 |
| 适用场景 | 社交网络、推荐系统 | 事务处理、报表 |

### 1.2 Neo4j 核心优势 ⚡

#### 1.2.1 性能优势
- **关系查询性能**：传统数据库的JOIN操作复杂度为O(n²)，而Neo4j为O(1)
- **深度查询**：6度关系查询在毫秒级别完成
- **实时推荐**：适合实时推荐系统和欺诈检测

#### 1.2.2 功能特性
- **ACID事务**：支持完整的事务特性
- **高可用性**：企业版支持集群部署
- **完整生态系统**：提供可视化工具、ETL工具等
- **社区活跃**：拥有庞大的开发者社区

### 1.3 Neo4j 数据模型 🧩

#### 1.3.1 节点（Node）
```cypher
// 节点示例
(:Person {name: '张三', age: 30})
(:City {name: '北京', population: 21540000})
```


#### 1.3.2 关系（Relationship）
```cypher
// 关系示例
(:Person)-[:LIVES_IN]->(:City)
(:Person)-[:FRIEND]->(:Person)
```


#### 1.3.3 属性（Property）
- 节点和关系都可以有属性
- 属性是键值对形式
- 支持多种数据类型：字符串、数字、布尔、日期等

#### 1.3.4 标签（Label）
```cypher
// 标签示例
:Person         // 人员标签
:City           // 城市标签
:Movie          // 电影标签
```


### 1.4 Neo4j 适用场景 🎯

#### 1.4.1 典型应用场景
1. **社交网络**：好友推荐、社交关系分析
2. **推荐系统**：商品推荐、内容推荐
3. **知识图谱**：实体关系查询、语义搜索
4. **路径规划**：物流路线、导航系统
5. **反欺诈**：关联交易分析、风险控制

#### 1.4.2 不适用场景
1. **大量事务处理**：如银行交易系统
2. **大规模数据仓库**：如数据仓库分析
3. **简单的CRUD应用**：没有复杂关系查询

## 三、Cypher 查询语言详解 🔍

### 3.1 Cypher 语法基础

#### 3.1.1 基本结构
```cypher
MATCH (n:Person {name: '张三'})    -- 匹配模式
WHERE n.age > 25                   -- 条件过滤
RETURN n.name, n.age              -- 返回结果
ORDER BY n.age DESC               -- 排序
LIMIT 10                          -- 限制数量
```


#### 3.1.2 注释语法
```cypher
// 单行注释

/*
 * 多行注释
 * 可以跨越多行
 */

-- SQL风格注释
```


### 3.2 数据创建（CREATE） ✨

#### 3.2.1 创建节点
```cypher
-- 创建简单节点
CREATE (n {name: '张三'})

-- 创建带标签的节点
CREATE (p:Person {name: '李四', age: 25})

-- 创建多个节点
CREATE (a:Person {name: '王五'}),
       (b:Person {name: '赵六'}),
       (c:Company {name: '阿里巴巴'})
```


#### 3.2.2 创建关系
```cypher
-- 创建节点和关系（一步完成）
CREATE (a:Person {name: '张三'})-[:FRIEND]->(b:Person {name: '李四'})

-- 为现有节点创建关系
MATCH (a:Person {name: '张三'}), (b:Person {name: '李四'})
CREATE (a)-[:FRIEND]->(b)

-- 创建带属性的关系
CREATE (a:City {name: '北京'})-[:DISTANCE {km: 1200, time: 2}]->(b:City {name: '上海'})

-- 创建双向关系
CREATE (a:Person)-[:FRIEND]->(b:Person),
       (b)-[:FRIEND]->(a)
```


#### 3.2.3 MERGE 创建或更新
```cypher
-- MERGE：存在则返回，不存在则创建
MERGE (p:Person {name: '张三'})
RETURN p

-- ON CREATE：创建时执行
-- ON MATCH：匹配时执行
MERGE (p:Person {name: '张三'})
ON CREATE SET p.created = timestamp(), p.version = 1
ON MATCH SET p.updated = timestamp(), p.version = p.version + 1
RETURN p

-- MERGE 关系
MATCH (a:Person {name: '张三'}), (b:Person {name: '李四'})
MERGE (a)-[r:FRIEND]->(b)
ON CREATE SET r.since = date()
ON MATCH SET r.lastMet = date()
RETURN r
```


### 3.3 数据查询（MATCH） 🔍

#### 3.3.1 基础查询
```cypher
-- 查询所有节点
MATCH (n) RETURN n

-- 查询特定标签的节点
MATCH (p:Person) RETURN p

-- 查询特定属性的节点
MATCH (p:Person {name: '张三'}) RETURN p

-- 查询节点属性
MATCH (p:Person) 
RETURN p.name AS 姓名, p.age AS 年龄

-- 查询关系
MATCH ()-[r:FRIEND]->() RETURN r

-- 查询完整路径
MATCH (a:Person)-[r:FRIEND]->(b:Person) 
RETURN a.name AS 朋友1, b.name AS 朋友2, type(r) AS 关系类型
```


#### 3.3.2 条件过滤（WHERE）
```cypher
-- 基础条件
MATCH (p:Person) 
WHERE p.age > 18 
RETURN p

-- 多条件组合
MATCH (p:Person) 
WHERE p.age > 18 AND p.name CONTAINS '张'
RETURN p

-- 正则表达式
MATCH (p:Person) 
WHERE p.name =~ '张.*'
RETURN p

-- IN 条件
MATCH (p:Person) 
WHERE p.name IN ['张三', '李四', '王五']
RETURN p

-- 属性存在性检查
MATCH (p:Person) 
WHERE EXISTS(p.email)
RETURN p

-- NULL 检查
MATCH (p:Person) 
WHERE p.email IS NULL
RETURN p

-- 字符串函数
MATCH (p:Person)
WHERE p.name STARTS WITH '张'
RETURN p

MATCH (p:Person)
WHERE p.name ENDS WITH '三'
RETURN p
```


#### 3.3.3 关系查询
```cypher
-- 查询单层关系
MATCH (a:Person)-[:FRIEND]->(b:Person)
RETURN a, b

-- 查询多层关系
MATCH (a:Person)-[:FRIEND*2]->(c:Person)
RETURN a, c

-- 查询指定深度的关系
MATCH (a:Person)-[:FRIEND*1..3]->(b:Person)
RETURN a, b

-- 查询最短路径
MATCH path = shortestPath(
  (a:Person {name: '张三'})-[*..10]-(b:Person {name: '王五'})
)
RETURN path

-- 查询所有路径
MATCH path = (a:Person {name: '张三'})-[*..5]->(b:Person {name: '赵六'})
RETURN path, length(path) AS 路径长度
```


#### 3.3.4 聚合查询
```cypher
-- 计数
MATCH (p:Person) 
RETURN count(p) AS 总人数

-- 分组统计
MATCH (p:Person)-[:LIVES_IN]->(c:City)
RETURN c.name AS 城市, count(p) AS 人数, avg(p.age) AS 平均年龄

-- 聚合函数
MATCH (p:Person)
RETURN 
  count(p) AS 总数,
  sum(p.age) AS 年龄总和,
  avg(p.age) AS 平均年龄,
  min(p.age) AS 最小年龄,
  max(p.age) AS 最大年龄

-- 收集为列表
MATCH (c:City)<-[:LIVES_IN]-(p:Person)
RETURN c.name, collect(p.name) AS 居民列表

-- 展开列表
UNWIND ['张三', '李四', '王五'] AS name
MATCH (p:Person {name: name})
RETURN p
```


### 3.4 数据更新（SET） 🔄

#### 3.4.1 更新属性
```cypher
-- 更新单个属性
MATCH (p:Person {name: '张三'})
SET p.age = 30
RETURN p

-- 更新多个属性
MATCH (p:Person {name: '张三'})
SET p.age = 30, p.city = '北京', p.updated = timestamp()
RETURN p

-- 基于现有值更新
MATCH (p:Person {name: '张三'})
SET p.age = p.age + 1
RETURN p

-- 添加标签
MATCH (p:Person {name: '张三'})
SET p:Employee:Manager
RETURN p
```


#### 3.4.2 更新关系
```cypher
-- 更新关系属性
MATCH (a:Person {name: '张三'})-[r:FRIEND]->(b:Person {name: '李四'})
SET r.since = '2020-01-01', r.close = true
RETURN r
```


### 3.5 数据删除（DELETE/REMOVE） ❌

#### 3.5.1 删除节点
```cypher
-- 删除节点（必须先删除关系）
MATCH (p:Person {name: '测试用户'})
DETACH DELETE p

-- 删除多个节点
MATCH (p:Person)
WHERE p.age > 100
DETACH DELETE p
```


#### 3.5.2 删除关系
```cypher
-- 删除关系
MATCH (a:Person {name: '张三'})-[r:FRIEND]->(b:Person {name: '李四'})
DELETE r

-- 删除所有关系
MATCH (a:Person)-[r]->()
DELETE r
```


#### 3.5.3 删除属性
```cypher
-- 删除属性
MATCH (p:Person {name: '张三'})
REMOVE p.tempProperty
RETURN p

-- 删除标签
MATCH (p:Person {name: '张三'})
REMOVE p:OldLabel
RETURN p
```


### 3.6 高级查询技巧 🧠

#### 3.6.1 路径查询优化
```cypher
-- 使用变量路径
MATCH path = (a:Person)-[:FRIEND*..5]->(b:Person)
RETURN path, length(path) AS depth

-- 路径过滤
MATCH path = (a:Person)-[:FRIEND*..5]->(b:Person)
WHERE ALL(node IN nodes(path) WHERE node.age > 18)
RETURN path

-- 排除环状路径
MATCH path = (a:Person)-[:FRIEND*..5]->(b:Person)
WHERE a <> b AND NONE(node IN tail(nodes(path)) WHERE node = a)
RETURN path
```


#### 3.6.2 集合操作
```cypher
-- 集合交集
MATCH (p:Person)
WHERE p.name IN ['张三', '李四', '王五']
RETURN p

-- 集合差集
MATCH (p:Person)
WHERE NOT p.name IN ['张三', '李四']
RETURN p

-- 集合操作函数
MATCH (p:Person)
RETURN collect(p.name) AS 所有姓名
```


#### 3.6.3 分页查询
```cypher
-- 基础分页
MATCH (p:Person)
RETURN p
ORDER BY p.name
SKIP 0
LIMIT 10

-- 分页参数化
MATCH (p:Person)
RETURN p
ORDER BY p.name
SKIP $skip
LIMIT $limit

-- 获取总数和分页数据
MATCH (p:Person)
WITH count(p) AS total, collect(p) AS persons
RETURN total, persons[0..10] AS page1, persons[10..20] AS page2
```


### 3.7 索引与约束 📚

#### 3.7.1 创建索引
```cypher
-- 创建单属性索引
CREATE INDEX person_name_index FOR (p:Person) ON (p.name)

-- 创建复合索引
CREATE INDEX person_name_age_index FOR (p:Person) ON (p.name, p.age)

-- 创建全文索引
CREATE FULLTEXT INDEX person_fulltext FOR (p:Person) ON EACH [p.name, p.bio, p.email]

-- 创建文本索引（Neo4j 4.x）
CREATE TEXT INDEX person_text_index FOR (p:Person) ON (p.name)

-- 查看索引
SHOW INDEXES

-- 删除索引
DROP INDEX person_name_index
```


#### 3.7.2 创建约束
```cypher
-- 创建唯一约束
CREATE CONSTRAINT person_email_unique FOR (p:Person) REQUIRE p.email IS UNIQUE

-- 创建节点键约束
CREATE CONSTRAINT person_id_unique FOR (p:Person) REQUIRE (p.id) IS NODE KEY

-- 创建存在性约束
CREATE CONSTRAINT person_name_exists FOR (p:Person) REQUIRE p.name IS NOT NULL

-- 查看约束
SHOW CONSTRAINTS

-- 删除约束
DROP CONSTRAINT person_email_unique
```


### 3.8 性能优化技巧 ⚡

#### 3.8.1 查询性能优化
```cypher
-- 1. 使用参数化查询
MATCH (p:Person {name: $name}) RETURN p

-- 2. 限制返回结果数量
MATCH (p:Person) RETURN p LIMIT 100

-- 3. 使用索引提示
MATCH (p:Person) USING INDEX p:Person(name)
WHERE p.name = '张三' RETURN p

-- 4. 避免全节点扫描
MATCH (p:Person) WHERE p.name STARTS WITH '张' RETURN p

-- 5. 使用投影减少数据传输
MATCH (p:Person) RETURN p.name, p.age
```


#### 3.8.2 批量操作优化
```cypher
-- 1. 批量创建节点（使用UNWIND）
UNWIND $persons AS person
CREATE (p:Person {name: person.name, age: person.age})
RETURN count(p)

-- 2. 批量创建关系
MATCH (a:Person) WHERE a.name IN $names1
MATCH (b:Person) WHERE b.name IN $names2
CREATE (a)-[:FRIEND]->(b)

-- 3. 批量更新
MATCH (p:Person)
WHERE p.name IN $names
SET p.updated = timestamp()
```
