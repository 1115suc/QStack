# Neo4j 性能优化

## 一、性能优化与最佳实践 🚀

### 1.1 查询性能优化 ⚡

#### 1.1.1 索引优化 🔍

```cypher
-- 创建合适索引
CREATE INDEX person_name_index FOR (p:Person) ON (p.name)
CREATE INDEX person_email_index FOR (p:Person) ON (p.email)
CREATE INDEX city_name_index FOR (c:City) ON (c.name)

-- 复合索引
CREATE INDEX person_name_age_index FOR (p:Person) ON (p.name, p.age)

-- 全文索引
CREATE FULLTEXT INDEX person_fulltext FOR (p:Person) ON EACH [p.name, p.bio, p.email]
```


#### 1.1.2 查询优化技巧 💡
```cypher
-- 1. 使用参数化查询
MATCH (p:Person {name: $name}) RETURN p

-- 2. 限制返回结果数量
MATCH (p:Person) RETURN p LIMIT 100

-- 3. 使用投影减少数据传输
MATCH (p:Person) RETURN p.name, p.age, p.email

-- 4. 避免全节点扫描
-- 不好：MATCH (p:Person) WHERE p.name STARTS WITH '张' RETURN p
-- 好：MATCH (p:Person) USING INDEX p:Person(name) WHERE p.name STARTS WITH '张' RETURN p

-- 5. 使用EXPLAIN分析查询计划
EXPLAIN MATCH (p:Person)-[:FRIEND]->(f:Person) RETURN p, f

-- 6. 使用PROFILE查看性能
PROFILE MATCH (p:Person)-[:FRIEND]->(f:Person) RETURN p, f
```


#### 1.1.3 批量操作优化 📦
```java
// 1. 使用UNWIND进行批量操作
@Query("UNWIND $persons AS person " +
       "MERGE (p:Person {name: person.name}) " +
       "SET p.age = person.age, p.email = person.email")
void batchUpsertPersons(@Param("persons") List<Map<String, Object>> persons);

// 2. 分批处理大数据量
public void batchProcess(List<PersonEntity> persons, int batchSize) {
    for (int i = 0; i < persons.size(); i += batchSize) {
        List<PersonEntity> batch = persons.subList(i, Math.min(i + batchSize, persons.size()));
        personRepository.saveAll(batch);
    }
}

// 3. 使用事务批量操作
@Transactional
public void batchCreateInTransaction(List<PersonEntity> persons) {
    persons.forEach(personRepository::save);
}
```

### 2.2 性能问题 🐢

#### 2.2.1 查询慢 🕐

```cypher
-- 1. 检查索引
SHOW INDEXES

-- 2. 分析查询计划
EXPLAIN MATCH (p:Person)-[:FRIEND*3]->(f:Person) RETURN p, f

-- 3. 优化查询
-- 避免深度过大的查询
MATCH (p:Person)-[:FRIEND*1..5]->(f:Person) RETURN p, f

-- 使用最短路径
MATCH path = shortestPath((p:Person)-[*..10]-(f:Person)) RETURN path
```


#### 2.2.2 内存不足 💾

```properties
# 调整内存配置
dbms.memory.heap.initial_size=4g
dbms.memory.heap.max_size=8g
dbms.memory.pagecache.size=2g
```


### 2.3 数据一致性问题 🔄

#### 2.3.1 事务管理 📜
```java
@Service
@Transactional
public class PersonService {
    
    // 只读事务
    @Transactional(readOnly = true)
    public List<PersonEntity> getAllPersons() {
        return personRepository.findAll();
    }
    
    // 写事务，设置超时
    @Transactional(timeout = 30)
    public PersonEntity createPerson(PersonEntity person) {
        return personRepository.save(person);
    }
    
    // 手动事务控制
    public void complexOperation() {
        TransactionTemplate transactionTemplate = new TransactionTemplate(transactionManager);
        transactionTemplate.setTimeout(30);
        
        transactionTemplate.execute(status -> {
            // 复杂操作
            return null;
        });
    }
}
```
