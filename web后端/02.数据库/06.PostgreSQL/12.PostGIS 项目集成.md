## 三、数据库表设计

### 3.1 建表 SQL

```sql
-- 启用 PostGIS 扩展（只需执行一次）
CREATE EXTENSION IF NOT EXISTS postgis;

-- 方式一：使用 geometry 类型字段（推荐）
CREATE TABLE poi (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(200) NOT NULL,
    description TEXT,
    -- ⭐ geometry 类型字段，指定为 Point 类型，SRID=4326
    geom        geometry(Point, 4326),
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建空间索引（必须！）
CREATE INDEX idx_poi_geom ON poi USING GIST(geom);

-- 插入测试数据
INSERT INTO poi (name, description, geom) VALUES
('天安门', '北京地标', ST_SetSRID(ST_MakePoint(116.397428, 39.90923), 4326)),
('故宫', '紫禁城', ST_SetSRID(ST_MakePoint(116.403414, 39.924091), 4326)),
('颐和园', '皇家园林', ST_SetSRID(ST_MakePoint(116.275045, 39.999932), 4326));


-- 方式二：使用 x, y 分开存储的传统方式
CREATE TABLE road_section (
    id    SERIAL PRIMARY KEY,
    name  VARCHAR(200),
    pos_x DOUBLE PRECISION,   -- X坐标（经度或投影X）
    pos_y DOUBLE PRECISION    -- Y坐标（纬度或投影Y）
);

-- 方式三：存储面（多边形区域）
CREATE TABLE district (
    id   SERIAL PRIMARY KEY,
    name VARCHAR(100),
    geom geometry(MultiPolygon, 4326)
);

CREATE INDEX idx_district_geom ON district USING GIST(geom);
```

## 四、PGgeometry 类型处理器（核心）

### 4.1 为什么需要类型处理器？

text

```
┌────────────────────────────────────────────────────────────┐
│  问题：                                                     │
│  数据库中 geometry 类型的字段，                               │
│  MyBatis 默认不认识，无法自动映射到 Java 对象。               │
│                                                            │
│  解决方案：                                                  │
│  自定义 TypeHandler，告诉 MyBatis：                          │
│  "遇到 geometry 类型时，请用我这个处理器来做 Java ↔ DB 转换"  │
└────────────────────────────────────────────────────────────┘
```

### 4.2 PGgeometry 类型处理器实现

```java
/**
 * PostGIS PGgeometry 类型处理器
 * 处理 Java PGgeometry 对象 与 数据库 geometry 类型的双向转换
 */
@MappedTypes(PGgeometry.class)
public class PGgeometryTypeHandler extends BaseTypeHandler<PGgeometry> {

    /**
     * Java → 数据库（插入/更新时调用）
     * 将 Java 的 PGgeometry 对象设置到 PreparedStatement 中
     */
    @Override
    public void setNonNullParameter(PreparedStatement ps, int i,
                                     PGgeometry parameter, JdbcType jdbcType)
            throws SQLException {
        // 直接将 PGgeometry 对象设置进去，PostgreSQL 驱动会自动处理
        ps.setObject(i, parameter);
    }

    /**
     * 数据库 → Java（查询时，通过列名获取）
     */
    @Override
    public PGgeometry getNullableResult(ResultSet rs, String columnName)
            throws SQLException {
        String value = rs.getString(columnName);
        if (value == null || value.isEmpty()) {
            return null;
        }
        try {
            return new PGgeometry(value);
        } catch (Exception e) {
            throw new SQLException("无法解析 PGgeometry: " + value, e);
        }
    }

    /**
     * 数据库 → Java（查询时，通过列索引获取）
     */
    @Override
    public PGgeometry getNullableResult(ResultSet rs, int columnIndex)
            throws SQLException {
        String value = rs.getString(columnIndex);
        if (value == null || value.isEmpty()) {
            return null;
        }
        try {
            return new PGgeometry(value);
        } catch (Exception e) {
            throw new SQLException("无法解析 PGgeometry: " + value, e);
        }
    }

    /**
     * 数据库 → Java（存储过程调用时获取）
     */
    @Override
    public PGgeometry getNullableResult(CallableStatement cs, int columnIndex)
            throws SQLException {
        String value = cs.getString(columnIndex);
        if (value == null || value.isEmpty()) {
            return null;
        }
        try {
            return new PGgeometry(value);
        } catch (Exception e) {
            throw new SQLException("无法解析 PGgeometry: " + value, e);
        }
    }
}
```

### 4.3 String 类型处理器（简化方案）

如果你不想用 PGgeometry 对象，也可以直接用 **String** 来接收 geometry 字段：

```java
/**
 * 将数据库 geometry 类型直接映射为 Java String（WKT/WKB 文本）
 */
@MappedTypes(String.class)
@MappedJdbcTypes(JdbcType.OTHER)
public class GeometryStringTypeHandler extends BaseTypeHandler<String> {

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i,
                                     String parameter, JdbcType jdbcType)
            throws SQLException {
        ps.setString(i, parameter);
    }

    @Override
    public String getNullableResult(ResultSet rs, String columnName)
            throws SQLException {
        return rs.getString(columnName);
    }

    @Override
    public String getNullableResult(ResultSet rs, int columnIndex)
            throws SQLException {
        return rs.getString(columnIndex);
    }

    @Override
    public String getNullableResult(CallableStatement cs, int columnIndex)
            throws SQLException {
        return cs.getString(columnIndex);
    }
}
```

## 七、Mapper XML 文件（核心重点）

### 7.1 PoiMapper.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.mapper.PoiMapper">

    <!--
    ============================================================
    ⭐⭐⭐ XML 文件中使用 PostGIS 的注意事项 ⭐⭐⭐
    ============================================================

    1. 参数占位符使用 #{} 而非 ${}
       #{} → PreparedStatement，防SQL注入，自动类型匹配
       ${} → 字符串拼接，存在SQL注入风险

    2. 小于号 < 必须使用转义或 CDATA
       XML中 < 是特殊字符，直接写会报解析错误！
       方式一：用 &lt; 替代 <
       方式二：用 <![CDATA[ ... ]]> 包裹整段SQL
       方式三：使用 ST_DWithin 替代 ST_Distance < xxx（推荐）

    3. geometry 字段的 resultMap 必须指定 typeHandler
       否则 MyBatis 不知道如何将数据库的 geometry 转为 Java 对象

    4. 强制类型转换 ::geometry 在XML中正常使用，不需要转义

    5. PostGIS 函数可以直接在 XML 的 SQL 中使用，和普通 SQL 函数用法一样
    ============================================================
    -->

    <!-- ⭐ 包含 geometry 字段的 resultMap，必须指定 typeHandler -->
    <resultMap id="poiResultMap" type="com.example.entity.Poi">
        <id column="id" property="id"/>
        <result column="name" property="name"/>
        <result column="description" property="description"/>
        <!-- ⭐ 关键：geometry 字段必须指定 typeHandler -->
        <result column="geom" property="geom"
                typeHandler="com.example.handler.PGgeometryTypeHandler"/>
        <result column="create_time" property="createTime"/>
    </resultMap>


    <!-- ========== 查询附近的POI ========== -->
    <select id="findNearby" resultType="com.example.vo.NearbyResultVO">
        SELECT
            p.id,
            p.name,
            -- ⭐ 提取经度
            ST_X(p.geom) AS longitude,
            -- ⭐ 提取纬度
            ST_Y(p.geom) AS latitude,
            -- ⭐ 计算距离（转为geography计算球面距离，单位：米）
            ROUND(
                ST_Distance(
                    p.geom::geography,
                    ST_SetSRID(ST_MakePoint(#{lng}, #{lat}), 4326)::geography
                )::numeric, 2
            ) AS distance
        FROM poi p
        WHERE ST_DWithin(
            p.geom::geography,
            ST_SetSRID(ST_MakePoint(#{lng}, #{lat}), 4326)::geography,
            #{distance}
        )
        ORDER BY distance
    </select>


    <!-- ========== 判断点是否在行政区内 ========== -->
    <select id="isPointInDistrict" resultType="boolean">
        SELECT ST_Contains(
            d.geom,
            ST_SetSRID(ST_MakePoint(#{lng}, #{lat}), 4326)
        )
        FROM district d
        WHERE d.name = #{districtName}
        LIMIT 1
    </select>


    <!-- ========== 插入POI（含geometry字段） ========== -->
    <insert id="insertPoi">
        INSERT INTO poi (name, description, geom)
        VALUES (
            #{name},
            #{description},
            <!--
            ⭐ 在 XML 中构建 geometry
            直接使用 PostGIS 函数构建点几何体
            -->
            ST_SetSRID(ST_MakePoint(#{lng}, #{lat}), 4326)
        )
    </insert>


    <!-- ========== 使用 PGgeometry 对象查询（配合 resultMap） ========== -->
    <select id="selectById" resultMap="poiResultMap" parameterType="int">
        SELECT id, name, description, geom, create_time
        FROM poi
        WHERE id = #{id}
    </select>


    <!-- ========== 使用 PGgeometry 对象插入 ========== -->
    <insert id="insertWithGeom" parameterType="com.example.entity.Poi">
        INSERT INTO poi (name, description, geom)
        VALUES (
            #{name},
            #{description},
            <!--
            ⭐ 使用 typeHandler 将 Java PGgeometry 对象写入数据库
            -->
            #{geom, typeHandler=com.example.handler.PGgeometryTypeHandler}
        )
    </insert>


    <!--
    ========== ⚠️ 常见错误示范与正确写法 ==========
    -->

    <!-- ❌ 错误！XML中直接写 < 会报解析错误 -->
    <!--
    <select id="wrong">
        SELECT * FROM poi
        WHERE ST_Distance(geom::geography, ...) < 1000
    </select>
    -->

    <!-- ✅ 正确方式一：使用转义符 &lt; -->
    <!--
    <select id="correct1">
        SELECT * FROM poi
        WHERE ST_Distance(geom::geography, ...) &lt; 1000
    </select>
    -->

    <!-- ✅ 正确方式二：使用 CDATA 包裹 -->
    <!--
    <select id="correct2">
        <![CDATA[
            SELECT * FROM poi
            WHERE ST_Distance(geom::geography, ...) < 1000
        ]]>
    </select>
    -->

    <!-- ✅ 正确方式三（推荐）：直接用 ST_DWithin 替代，避免 < 号 -->
    <!--
    <select id="correct3">
        SELECT * FROM poi
        WHERE ST_DWithin(geom::geography, ..., 1000)
    </select>
    -->

</mapper>
```

### 7.2 RoadSectionMapper.xml（4547坐标系优化版）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.mapper.RoadSectionMapper">

    <!--
    ⭐ 性能优化核心：
    数据库存储 4547（投影坐标/米），不对表中数据做 ST_Transform，
    只对用户传入的 1 个 4326 坐标转换为 4547，
    然后在 4547 下用 geometry 平面计算（勾股定理，极快）
    -->
    <select id="findNearbyRoads" resultType="com.example.vo.NearbyResultVO">
        SELECT
            s.id,
            s.name,
            s.pos_x AS longitude,
            s.pos_y AS latitude,
            ROUND(
                ST_Distance(
                    <!-- 表中的点：直接贴4547标签，不做任何转换 -->
                    ST_SetSRID(ST_MakePoint(s.pos_x, s.pos_y), 4547),

                    <!-- 用户的点：4326 → 4547（只执行1次！） -->
                    ST_Transform(
                        ST_SetSRID(ST_MakePoint(#{x}, #{y}), 4326),
                        4547
                    )
                )::numeric, 2
            ) AS distance
        FROM road_section s
        WHERE ST_DWithin(
            ST_SetSRID(ST_MakePoint(s.pos_x, s.pos_y), 4547),
            ST_Transform(
                ST_SetSRID(ST_MakePoint(#{x}, #{y}), 4326),
                4547
            ),
            #{distance}
        )
        ORDER BY distance
    </select>

</mapper>
```

## 八、Service 层

```java
@Service
@RequiredArgsConstructor
public class PoiService {

    private final PoiMapper poiMapper;

    /**
     * 查询附近的POI
     */
    public List<NearbyResultVO> findNearby(NearbySearchDTO dto) {
        return poiMapper.findNearby(
                dto.getLongitude(),
                dto.getLatitude(),
                dto.getDistance()
        );
    }

    /**
     * 判断点是否在某个行政区内
     */
    public boolean isInDistrict(Double lng, Double lat, String districtName) {
        return poiMapper.isPointInDistrict(lng, lat, districtName);
    }

    /**
     * 通过经纬度插入POI（在SQL中构建geometry）
     */
    public void addPoi(String name, String description, Double lng, Double lat) {
        poiMapper.insertPoi(name, description, lng, lat);
    }

    /**
     * ⭐ 通过 PGgeometry 对象插入POI
     * 展示如何在 Java 中手动构建 PGgeometry
     */
    public void addPoiWithGeom(String name, String description,
                                Double lng, Double lat) throws SQLException {
        Poi poi = new Poi();
        poi.setName(name);
        poi.setDescription(description);

        // ⭐ 在 Java 中构建 PGgeometry 对象
        Point point = new Point(lng, lat);
        point.setSrid(4326);    // 设置坐标系
        PGgeometry pgGeometry = new PGgeometry(point);

        poi.setGeom(pgGeometry);
        poiMapper.insert(poi);
    }

    /**
     * ⭐ 查询POI并解析 PGgeometry 对象
     */
    public void parseGeometry(Integer id) {
        Poi poi = poiMapper.selectById(id);

        if (poi != null && poi.getGeom() != null) {
            PGgeometry pgGeom = poi.getGeom();

            // 获取底层几何对象
            org.postgis.Geometry geometry = pgGeom.getGeometry();

            // 判断几何类型
            switch (geometry.getType()) {
                case org.postgis.Geometry.POINT:
                    Point point = (Point) geometry;
                    System.out.println("经度: " + point.getX());
                    System.out.println("纬度: " + point.getY());
                    System.out.println("SRID: " + point.getSrid());
                    break;
                case org.postgis.Geometry.POLYGON:
                    System.out.println("这是一个多边形");
                    break;
                case org.postgis.Geometry.MULTIPOLYGON:
                    System.out.println("这是一个多面集合");
                    break;
                default:
                    System.out.println("其他类型: " + geometry.getTypeString());
            }
        }
    }
}
```

## 十、XML 中 PostGIS 注意事项总结

```text
┌────────────────────────────────────────────────────────────────────┐
│                  XML 中使用 PostGIS 的 6 大注意事项                  │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  1️⃣  < 号必须转义                                                  │
│     ❌  ST_Distance(...) < 1000                                    │
│     ✅  ST_Distance(...) &lt; 1000                                 │
│     ✅  <![CDATA[ ST_Distance(...) < 1000 ]]>                      │
│     ✅  ST_DWithin(..., 1000)    ← 最推荐，完全避免 < 号            │
│                                                                    │
│  2️⃣  参数使用 #{} 不要用 ${}                                       │
│     ❌  ST_MakePoint(${lng}, ${lat})    → SQL注入风险               │
│     ✅  ST_MakePoint(#{lng}, #{lat})    → PreparedStatement安全     │
│                                                                    │
│  3️⃣  geometry 字段必须在 resultMap 中指定 typeHandler               │
│     <result column="geom" property="geom"                          │
│             typeHandler="...PGgeometryTypeHandler"/>                │
│                                                                    │
│  4️⃣  插入 geometry 参数时需指定 typeHandler                         │
│     #{geom, typeHandler=...PGgeometryTypeHandler}                   │
│                                                                    │
│  5️⃣  ::geography 和 ::geometry 等类型转换在 XML 中可以直接使用       │
│     不需要额外转义，PostgreSQL驱动能正常解析                          │
│                                                                    │
│  6️⃣  PostGIS函数（ST_Distance等）直接写在SQL中即可                  │
│     和普通SQL函数用法完全一样                                        │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 十一、MyBatis-Plus 直接使用（不写 XML 的方式）

如果你想使用 MyBatis-Plus 的 Wrapper 方式查询，可以通过 `apply` 方法注入 PostGIS SQL 片段：

```java
@Service
@RequiredArgsConstructor
public class PoiMpService {

    private final PoiMapper poiMapper;

    /**
     * 使用 MyBatis-Plus Wrapper 的方式查询附近POI
     */
    public List<Poi> findNearby(Double lng, Double lat, Double distance) {
        QueryWrapper<Poi> wrapper = new QueryWrapper<>();

        // ⭐ 使用 apply 方法注入 PostGIS 的 WHERE 条件
        wrapper.apply(
            "ST_DWithin(geom::geography, " +
            "ST_SetSRID(ST_MakePoint({0}, {1}), 4326)::geography, " +
            "{2})",
            lng, lat, distance
        );

        // ⭐ 使用 select 添加自定义计算列
        wrapper.select(
            "id",
            "name",
            "description",
            "ST_X(geom) AS longitude",
            "ST_Y(geom) AS latitude",
            "ROUND(ST_Distance(geom::geography, " +
                "ST_SetSRID(ST_MakePoint(" + lng + ", " + lat + "), 4326)::geography" +
            ")::numeric, 2) AS distance"
        );

        wrapper.orderByAsc("distance");

        return poiMapper.selectList(wrapper);
    }
}
```

> ⚠️ `apply` 中使用 `{0}`, `{1}`, `{2}` 作为参数占位符，MyBatis-Plus 会自动转为 PreparedStatement 的 `?`，是安全的。

## 十三、常见问题排查

### 问题速查表

|错误信息|原因|解决方案|
|---|---|---|
|`column "geom" is of type geometry but expression is of type character varying`|插入geometry时类型不匹配|URL加 `?stringtype=unspecified`，或使用 typeHandler|
|`Unknown geometry type`|PGgeometry无法解析|检查postgis-jdbc版本，确保与PostGIS版本兼容|
|`No typehandler found for property geom`|未注册类型处理器|yml中配置 `type-handlers-package`，或在字段上指定 `typeHandler`|
|`元素内容必须由格式正确的字符数据或标记组成`|XML中使用了 `<` 号|用 `&lt;` 转义，或 `CDATA` 包裹，或改用 `ST_DWithin`|
|`ERROR: Operation on mixed SRID geometries`|两个几何体SRID不一致|确保参与运算的几何体SRID统一|
|`transform: couldn't project point`|ST_Transform坐标越界|检查坐标值是否和SRID匹配（如4326的值不能超过180/90）|

### 调试技巧

```sql
-- 1. 验证PostGIS是否安装
SELECT PostGIS_Full_Version();

-- 2. 验证SRID是否存在
SELECT * FROM spatial_ref_sys WHERE srid = 4547;

-- 3. 查看geometry字段的SRID
SELECT ST_SRID(geom) FROM poi LIMIT 1;

-- 4. 查看geometry字段的类型
SELECT GeometryType(geom) FROM poi LIMIT 1;

-- 5. 将geometry转为可读文本（调试用）
SELECT ST_AsText(geom) FROM poi LIMIT 5;
```
