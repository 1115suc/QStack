## 二、实体类配置

### 2.1 基础实体（含 MyBatis-Plus 注解）

```java
import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;
import java.time.LocalDateTime;

@Data
@TableName("users")  // 对应 PostgreSQL 表名（小写）
public class User {

    @TableId(type = IdType.AUTO)  // 对应 BIGSERIAL / GENERATED ALWAYS AS IDENTITY
    private Long id;

    private String username;

    private String email;

    private Boolean isActive;

    private LocalDateTime createdAt;

    private LocalDateTime updatedAt;

    @TableLogic  // 逻辑删除（配合 global-config 使用）
    private LocalDateTime deletedAt;
}
```

### 2.2 关于自动填充（创建时间 / 更新时间）

```java
import com.baomidou.mybatisplus.core.handlers.MetaObjectHandler;
import org.apache.ibatis.reflection.MetaObject;
import org.springframework.stereotype.Component;
import java.time.LocalDateTime;

@Component
public class MyMetaObjectHandler implements MetaObjectHandler {

    @Override
    public void insertFill(MetaObject metaObject) {
        this.strictInsertFill(metaObject, "createdAt", LocalDateTime.class, LocalDateTime.now());
        this.strictInsertFill(metaObject, "updatedAt", LocalDateTime.class, LocalDateTime.now());
    }

    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updatedAt", LocalDateTime.class, LocalDateTime.now());
    }
}
```

实体类字段加上注解：

```java
@TableField(fill = FieldFill.INSERT)
private LocalDateTime createdAt;

@TableField(fill = FieldFill.INSERT_UPDATE)
private LocalDateTime updatedAt;
```

        .like(User::getEmail, "@example.com")
        .orderByDesc(User::getCreatedAt)

    new LambdaQueryWrapper<User>()
        .eq(User::getIsActive, true)
);

System.out.println(result.getTotal());   // 总条数

// 更新
userService.update(
    new LambdaUpdateWrapper<User>()
        .eq(User::getId, 1L)
        .set(User::getEmail, "new@example.com")
);

// 逻辑删除（自动填充 deleted_at）
userService.removeById(1L);

## 四、处理 PostgreSQL 特有类型

### 4.1 JSONB 映射

**方式一：映射为 String（最简单）**

```java
@TableField("meta")
private String meta;  // 直接用 String 存 JSON，取出后手动解析

// 使用 Jackson 解析
ObjectMapper mapper = new ObjectMapper();
Map<String, Object> metaMap = mapper.readValue(user.getMeta(), Map.class);
```

**方式二：自定义 TypeHandler（推荐）**

```java
import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.ibatis.type.BaseTypeHandler;
import org.apache.ibatis.type.JdbcType;
import org.postgresql.util.PGobject;
import java.sql.*;

public class JsonbTypeHandler<T> extends BaseTypeHandler<T> {

    private final Class<T> type;
    private static final ObjectMapper objectMapper = new ObjectMapper();

    public JsonbTypeHandler(Class<T> type) {
        this.type = type;
    }

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType)
            throws SQLException {
        PGobject pgObject = new PGobject();
        pgObject.setType("jsonb");
        try {
            pgObject.setValue(objectMapper.writeValueAsString(parameter));
        } catch (Exception e) {
            throw new SQLException("Failed to convert to JSONB", e);
        }
        ps.setObject(i, pgObject);
    }

    @Override
    public T getNullableResult(ResultSet rs, String columnName) throws SQLException {
        return parse(rs.getString(columnName));
    }

    @Override
    public T getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        return parse(rs.getString(columnIndex));
    }

    @Override
    public T getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        return parse(cs.getString(columnIndex));
    }

    private T parse(String json) {
        if (json == null) return null;
        try {
            return objectMapper.readValue(json, type);
        } catch (Exception e) {
            throw new RuntimeException("Failed to parse JSONB", e);
        }
    }
}
```

实体类使用：

```java
@Data
@TableName("products")
public class Product {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String name;

    // 映射为 Map，自动序列化/反序列化
    @TableField(typeHandler = JsonbTypeHandler.class)
    private Map<String, Object> meta;
}
```

注册 TypeHandler（全局）：

```yaml
mybatis-plus:
  type-handlers-package: com.example.handler
```

### 4.2 数组类型映射

```java
import org.apache.ibatis.type.BaseTypeHandler;
import org.apache.ibatis.type.JdbcType;
import java.sql.*;

public class StringArrayTypeHandler extends BaseTypeHandler<String[]> {

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, String[] parameter, JdbcType jdbcType)
            throws SQLException {
        Array array = ps.getConnection().createArrayOf("text", parameter);
        ps.setArray(i, array);
    }

    @Override
    public String[] getNullableResult(ResultSet rs, String columnName) throws SQLException {
        Array array = rs.getArray(columnName);
        return array == null ? null : (String[]) array.getArray();
    }

    @Override
    public String[] getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        Array array = rs.getArray(columnIndex);
        return array == null ? null : (String[]) array.getArray();
    }

    @Override
    public String[] getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        Array array = cs.getArray(columnIndex);
        return array == null ? null : (String[]) array.getArray();
    }
}
```

实体类使用：

```java
@TableField(typeHandler = StringArrayTypeHandler.class)
private String[] tags;
```

## 五、XML Mapper 编写（复杂 SQL）

### 5.1 XML 配置

```yaml
mybatis-plus:
  mapper-locations: classpath*:mapper/**/*.xml
```

### 5.2 RETURNING 用法（获取插入后的值）

MyBatis-Plus 的 `save()` 通过 `useGeneratedKeys` 自动回填 ID，但如果需要 RETURNING 多个字段，需要自定义 XML：

```xml
<!-- UserMapper.xml -->
<mapper namespace="com.example.mapper.UserMapper">

    <!-- 插入并返回 id 和 created_at -->
    <insert id="insertAndReturn" parameterType="com.example.entity.User">
        INSERT INTO users (username, email, is_active)
        VALUES (#{username}, #{email}, #{isActive})
        RETURNING id, created_at
    </insert>

</mapper>
```

```java
// Mapper 接口
@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 注意：RETURNING 需要用 @Select 或 XML 的 select 标签
    @Select("INSERT INTO users(username, email) VALUES(#{username}, #{email}) RETURNING id, created_at")
    @ResultType(User.class)
    User insertAndReturn(User user);
}
```

> ⚠️ MyBatis 的 `<insert>` 标签不支持返回结果集，RETURNING 多列时需要用 `<select>` 标签包裹。

### 5.3 UPSERT 写法

```xml
<insert id="upsertUser" parameterType="com.example.entity.User">
    INSERT INTO users (username, email, is_active)
    VALUES (#{username}, #{email}, #{isActive})
    ON CONFLICT (username)
    DO UPDATE SET
        email = EXCLUDED.email,
        updated_at = NOW()
</insert>
```

### 5.4 JSONB 条件查询

```xml
<select id="findByMeta" resultType="com.example.entity.Product">
    SELECT * FROM products
    <where>
        <if test="brand != null">
            AND meta @> CAST(#{brand} AS JSONB)
        </if>
        <if test="minPrice != null">
            AND (meta ->> 'price')::NUMERIC >= #{minPrice}
        </if>
    </where>
</select>
```

```java
// 在 Mapper 中定义
List<Product> findByMeta(@Param("brand") String brand,
                          @Param("minPrice") BigDecimal minPrice);

// 调用时 brand 传 JSON 字符串
List<Product> products = productMapper.findByMeta(
    "{\"brand\": \"Apple\"}",
    new BigDecimal("500")
);
```

### 5.5 数组条件查询

```xml
<!-- 查询 tags 包含某些标签的记录 -->
<select id="findByTags" resultType="com.example.entity.Article">
    SELECT * FROM articles
    WHERE tags @> CAST(#{tags} AS TEXT[])
</select>
```

```java
// Mapper
List<Article> findByTags(@Param("tags") String tags);

// 调用
List<Article> articles = articleMapper.findByTags("{java,spring}");
```

### 5.6 分页 + 窗口函数

```xml
<select id="getSalesRanking" resultType="com.example.vo.SalesRankVO">
    SELECT
        salesperson,
        department,
        amount,
        RANK() OVER (PARTITION BY department ORDER BY amount DESC) AS rnk
    FROM sales
    WHERE department = #{department}
    ORDER BY rnk
    LIMIT #{pageSize} OFFSET #{offset}
</select>
```

## 六、纯 MyBatis（不用 Plus）的写法

如果项目只用 MyBatis（无 Plus），以下是对应配置。

### 6.1 application.yml

```yaml
mybatis:
  mapper-locations: classpath:mapper/*.xml
  configuration:
    map-underscore-to-camel-case: true
  type-handlers-package: com.example.handler
```

### 6.2 Mapper 接口 + XML

```java
@Mapper
public interface UserMapper {
    void insert(User user);
    User selectById(Long id);
    List<User> selectByCondition(@Param("username") String username,
                                  @Param("isActive") Boolean isActive);
    void updateById(User user);
    void deleteById(Long id);
}
```

```xml
<mapper namespace="com.example.mapper.UserMapper">

    <resultMap id="userResultMap" type="com.example.entity.User">
        <id column="id" property="id"/>
        <result column="username" property="username"/>
        <result column="email" property="email"/>
        <result column="is_active" property="isActive"/>
        <result column="created_at" property="createdAt"/>
        <!-- JSONB 列使用自定义 TypeHandler -->
        <result column="meta" property="meta" typeHandler="com.example.handler.JsonbTypeHandler"/>
    </resultMap>

    <insert id="insert" parameterType="com.example.entity.User" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO users (username, email, is_active, created_at, updated_at)
        VALUES (#{username}, #{email}, #{isActive}, NOW(), NOW())
    </insert>

    <select id="selectById" resultMap="userResultMap">
        SELECT * FROM users WHERE id = #{id} AND deleted_at IS NULL
    </select>

    <select id="selectByCondition" resultMap="userResultMap">
        SELECT * FROM users
        <where>
            deleted_at IS NULL
            <if test="username != null and username != ''">
                AND username ILIKE CONCAT('%', #{username}, '%')
            </if>
            <if test="isActive != null">
                AND is_active = #{isActive}
            </if>
        </where>
        ORDER BY created_at DESC
    </select>

</mapper>
```

## 七、事务管理

### 7.1 Spring 声明式事务

```java
@Service
@Transactional(rollbackFor = Exception.class)
public class OrderService {

    @Autowired
    private OrderMapper orderMapper;

    @Autowired
    private InventoryMapper inventoryMapper;

    public void createOrder(Order order) {
        // 1. 扣减库存
        inventoryMapper.decreaseStock(order.getProductId(), order.getQuantity());

        // 2. 创建订单
        orderMapper.insert(order);

        // 如果抛出异常，以上操作自动回滚
    }

    // 只读事务（查询优化，PostgreSQL 会跳过不必要的写操作检查）
    @Transactional(readOnly = true)
    public User getUserById(Long id) {
        return userMapper.selectById(id);
    }

    // 指定隔离级别（对应 PG 的 REPEATABLE READ）
    @Transactional(
        isolation = Isolation.REPEATABLE_READ,
        rollbackFor = Exception.class
    )
    public void sensitiveOperation() {
        // ...
    }
}
```

### 7.2 编程式事务

```java
@Autowired
private TransactionTemplate transactionTemplate;

public void doWithTransaction() {
    transactionTemplate.execute(status -> {
        try {
            userMapper.insert(user);
            orderMapper.insert(order);
            return true;
        } catch (Exception e) {
            status.setRollbackOnly();  // 标记回滚
            return false;
        }
    });
}
```

## 八、常见问题与解决方案

### 8.1 字段名大小写问题

PostgreSQL 默认将标识符转为小写，而 Java 驼峰命名需要做映射：

```yaml
# application.yml 开启下划线转驼峰（推荐）
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
```

表字段用 `snake_case`（`user_name`），Java 用驼峰（`userName`），自动映射。

### 8.2 布尔类型映射

PostgreSQL 的 `BOOLEAN` 与 Java 的 `Boolean`/`boolean` 可以直接映射，无需特殊处理。

```java
private Boolean isActive;  // 对应 PG 的 BOOLEAN
```

### 8.3 时间类型映射

```java
// 推荐使用 Java 8+ 时间类型
private LocalDate date;           // 对应 PG DATE
private LocalDateTime createdAt;  // 对应 PG TIMESTAMP
private OffsetDateTime ts;        // 对应 PG TIMESTAMPTZ（带时区）
```

确保 PostgreSQL 驱动版本 >= 42.2.x，支持 Java 8 时间类型直接映射。

### 8.4 批量插入性能优化

```java
// MyBatis-Plus 批量插入（内部分批执行，默认每批 1000 条）
userService.saveBatch(userList, 500);

// 更高性能：使用 COPY 命令（需要自定义实现）
// 或者使用 reWriteBatchedInserts=true 驱动参数
```

```
jdbc:postgresql://localhost:5432/mydb?reWriteBatchedInserts=true
```

加上 `reWriteBatchedInserts=true` 后，JDBC 会将多个 INSERT 合并为一次批量发送，性能提升显著。

### 8.5 连接池配置建议

```yaml
spring:
  datasource:
    hikari:
      minimum-idle: 5
      maximum-pool-size: 20
      # PostgreSQL 默认最大连接数 100，注意别超过
      connection-test-query: SELECT 1
      # 连接泄漏检测（开发环境推荐开启）
      leak-detection-threshold: 60000
```

查看 PostgreSQL 当前最大连接数：

```sql
SHOW max_connections;  -- 默认 100

-- 查看当前连接数
SELECT COUNT(*) FROM pg_stat_activity;
```

### 8.6 Schema 不匹配问题

如果使用了非 public Schema，需要在 JDBC URL 或连接后设置：

```
jdbc:postgresql://localhost:5432/mydb?currentSchema=order_service
```

或通过 HikariCP 的 `connectionInitSql`：

```yaml
spring:
  datasource:
    hikari:
      connection-init-sql: SET search_path TO order_service, public
```

## 九、完整示例：JSONB 字段的 CRUD

以下是一个完整的 Product（含 JSONB meta 字段）操作示例。

```java
// ProductService.java
@Service
public class ProductService extends ServiceImpl<ProductMapper, Product> {

    // 按品牌查询
    public List<Product> findByBrand(String brand) {
        // 使用 apply 传入原生 SQL 片段
        return list(new LambdaQueryWrapper<Product>()
            .apply("meta @> CAST({0} AS JSONB)", "{\"brand\":\"" + brand + "\"}")
        );
    }

    // 更新 meta 中的 price
    public void updatePrice(Long id, BigDecimal price) {
        // 使用 XML 或原生 SQL 更新 JSONB 字段
        baseMapper.updatePrice(id, price.toPlainString());
    }
}
```

```xml
<!-- ProductMapper.xml -->
<update id="updatePrice">
    UPDATE products
    SET meta = jsonb_set(meta, '{price}', #{price}::JSONB)
    WHERE id = #{id}
</update>
```

## 小结

| 知识点       | 要点                                                                 |
| --------- | ------------------------------------------------------------------ |
| 驱动        | `org.postgresql:postgresql`，JDBC URL 格式不同于 MySQL                   |
| 主键策略      | `IdType.AUTO` + `BIGSERIAL` / `IDENTITY`                           |
| JSONB 映射  | 自定义 `TypeHandler`，用 `PGobject` 设置类型                                |
| 数组映射      | 自定义 `TypeHandler`，用 `connection.createArrayOf`                     |
| UPSERT    | XML 中写 `ON CONFLICT ... DO UPDATE`                                 |
| RETURNING | 用 `<select>` 标签替代 `<insert>`                                       |
| 时间类型      | 推荐 `LocalDateTime` / `OffsetDateTime`                              |
| 性能        | JDBC URL 加 `reWriteBatchedInserts=true` 提升批量插入速度                   |
| Schema    | JDBC URL 指定 `currentSchema` 或 `connectionInitSql` 设置 `search_path` |
