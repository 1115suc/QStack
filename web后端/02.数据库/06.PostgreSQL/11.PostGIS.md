# # PostgreSQL PostGIS

## 一、PostGIS 简介

```text
PostgreSQL  →  关系型数据库（存储普通数据）
    +
PostGIS     →  空间数据扩展插件（让数据库能存储和计算地理信息）
    =
具备 GIS 能力的数据库（存储点/线/面，计算距离/面积/相交等）
```

### 启用 PostGIS

```sql
-- 在数据库中启用PostGIS扩展
CREATE EXTENSION postgis;

-- 验证是否安装成功
SELECT PostGIS_Version();
```

## 二、核心概念

### 2.1 SRID（空间参考标识符）

SRID 是坐标系的唯一编号，告诉数据库"这个坐标是在哪个坐标系下的"。

|SRID|名称|坐标单位|适用范围|常见用途|
|---|---|---|---|---|
|**4326**|WGS84|度（经纬度）|全球|GPS、Google地图|
|**4547**|CGCS2000 3度带|米（投影）|中国局部区域|国内工程测量|
|**3857**|Web墨卡托|米（投影）|全球|瓦片地图/互联网地图|
|**4490**|CGCS2000|度（经纬度）|中国|国内经纬度坐标|

### 2.2 Geometry 与 Geography 类型区别

```text
┌─────────────────────────────────────────────────────────┐
│  Geometry（平面几何）                                     │
│  · 基于平面坐标系计算（勾股定理）                          │
│  · 计算速度极快                                           │
│  · 适合投影坐标系（如4547，单位：米）                       │
│  · 距离单位取决于坐标系                                    │
├─────────────────────────────────────────────────────────┤
│  Geography（球面地理）                                    │
│  · 基于地球椭球体计算（三角函数）                          │
│  · 计算速度较慢（3~5倍）                                   │
│  · 适合经纬度坐标系（如4326，单位：度）                     │
│  · 距离单位始终是米                                        │
└─────────────────────────────────────────────────────────┘
```

### 2.3 几何类型家族

```text
PostGIS 几何类型体系
│
├── 基础类型
│   ├── Point            → 点     (1个坐标位置)
│   ├── LineString       → 线     (多个点连成的线)
│   └── Polygon          → 面     (封闭的线围成的区域)
│
├── 集合类型（Multi系列）
│   ├── MultiPoint       → 多点集合
│   ├── MultiLineString  → 多线集合
│   └── MultiPolygon     → 多面集合
│
└── 混合类型
    └── GeometryCollection → 任意几何体混合集合
```

## 三、几何类型详解

### 3.1 ST_Point —— 点

**概念**：空间中的一个坐标位置，0维，无面积无长度。

**现实映射**：GPS坐标、加油站位置、摄像头位置

```sql
-- 创建点的三种方式
SELECT ST_SetSRID(ST_MakePoint(116.397, 39.909), 4326);   -- 推荐方式
SELECT ST_SetSRID(ST_Point(116.397, 39.909), 4326);
SELECT ST_GeomFromText('POINT(116.397 39.909)', 4326);

-- 建表示例
CREATE TABLE cameras (
    id   SERIAL PRIMARY KEY,
    name VARCHAR(100),
    geom geometry(Point, 4326)
);
```

```text
图示：
Y轴
 |
 |    *(116.397, 39.909)   ← 一个点
 |
 └──────────────── X轴
```

### 3.2 ST_Polygon —— 单个多边形面

**概念**：首尾相连的坐标围成的封闭区域，2维，有面积有周长。

**现实映射**：湖泊轮廓、行政区边界、建筑占地

```sql
-- 创建简单矩形面（首尾坐标必须相同！）
SELECT ST_SetSRID(
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))'),
    4547
);

-- 创建带洞的面（外环 - 内环）
SELECT ST_SetSRID(
    ST_GeomFromText(
        'POLYGON(
            (0 0, 100 0, 100 100, 0 100, 0 0),
            (30 30, 70 30, 70 70, 30 70, 30 30)
        )'
    ),
    4547
);
```

```text
图示（带洞的面，类比甜甜圈）：
(0,100)___________(100,100)
  |                   |
  |  (30,70)_(70,70)  |
  |    |  洞  |       |
  |  (30,30)_(70,30)  |
  |                   |
(0,0)_____________(100,0)

实际面积 = 外环面积 - 内环面积（洞）
```

### 3.3 ST_MultiPolygon —— 多个多边形集合

**概念**：多个彼此独立、互不相连的Polygon组合为一个整体对象。

**现实映射**：省级行政区（含岛屿）、群岛、企业多个不相邻厂区

```sql
-- 创建两个独立的矩形面
SELECT ST_SetSRID(
    ST_GeomFromText(
        'MULTIPOLYGON(
            ((0 0, 10 0, 10 10, 0 10, 0 0)),
            ((20 20, 30 20, 30 30, 20 30, 20 20))
        )'
    ),
    4547
);

-- 将多个Polygon合并为MultiPolygon
SELECT ST_Collect(geom) FROM district WHERE province = '广东省';
SELECT ST_Union(geom)   FROM district WHERE province = '广东省';
```

```text
图示：
Y轴
 |
30|    ┌──────┐
  |    │  面B │   ← 独立的第二块面
20|    └──────┘
  |
10|┌──────┐
  |│  面A │       ← 独立的第一块面
 0|└──────┘
  └────────────── X轴
   0  10  20  30

面A 和 面B 互不相连，但同属一个 MultiPolygon 对象
```

### 3.4 三种类型核心对比

| 特性        | ST_Point     | ST_Polygon           | ST_MultiPolygon             |
| --------- | ------------ | -------------------- | --------------------------- |
| **几何维度**  | 0维           | 2维                   | 2维                          |
| **面积**    | ❌            | ✅                    | ✅（各子面之和）                    |
| **连续性**   | 单一坐标         | 一块连续区域               | 多块不连续区域                     |
| **子元素**   | 1个坐标         | 1个（可含洞）              | N个独立Polygon                 |
| **现实案例**  | GPS坐标        | 一个湖泊                 | 群岛/省级区划                     |
| **WKT格式** | `POINT(x y)` | `POLYGON((x y,...))` | `MULTIPOLYGON(((x y,...)))` |

## 四、核心方法详解

### 4.1 构建几何体

#### ST_MakePoint —— 创建点

```sql
-- 语法
ST_MakePoint(x, y)
ST_MakePoint(x, y, z)       -- 含高程
ST_MakePoint(x, y, z, m)    -- 含高程和测量值

-- 示例
SELECT ST_MakePoint(116.397, 39.909);         -- 创建2D点（无SRID）
SELECT ST_MakePoint(116.397, 39.909, 100.0);  -- 创建3D点（含海拔100米）
```

> ⚠️ 注意：ST_MakePoint 创建的点没有坐标系，需配合 ST_SetSRID 使用

#### ST_SetSRID —— 设置坐标系（贴标签）

```sql
-- 语法
ST_SetSRID(geometry, srid)

-- 示例
SELECT ST_SetSRID(ST_MakePoint(116.397, 39.909), 4326);
SELECT ST_SetSRID(ST_MakePoint(500000, 3000000), 4547);
```

> ⚠️ 关键：ST_SetSRID 只是声明坐标系，不改变坐标数值！

#### ST_Transform —— 坐标系转换（真正转换）

```sql
-- 语法
ST_Transform(geometry, target_srid)

-- 示例：将4547坐标转换为4326
SELECT ST_Transform(
    ST_SetSRID(ST_MakePoint(500000, 3000000), 4547),
    4326
);
-- 坐标数值会真正发生变化（米 → 经纬度度数）
```

> 🔑 核心区别：
> ```text
> ST_SetSRID  →  "贴标签"  只声明，坐标数值不变
> ST_Transform → "做转换"  坐标数值真正发生改变
> ```

#### ST_GeomFromText —— 从WKT文本创建几何体

```sql
-- 语法
ST_GeomFromText(wkt_text, srid)

-- 示例
SELECT ST_GeomFromText('POINT(116.397 39.909)', 4326);
SELECT ST_GeomFromText('LINESTRING(0 0, 5 5, 10 0)', 4547);
SELECT ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))', 4547);
```

#### ST_MakeLine —— 创建线

```sql
-- 由两点创建线
SELECT ST_MakeLine(ST_MakePoint(0, 0), ST_MakePoint(10, 10));

-- 由多点创建线
SELECT ST_MakeLine(ARRAY[
    ST_MakePoint(0, 0),
    ST_MakePoint(5, 5),
    ST_MakePoint(10, 0)
]);
```

#### ST_MakePolygon —— 由线环创建面

```sql
SELECT ST_MakePolygon(
    ST_GeomFromText('LINESTRING(0 0, 10 0, 10 10, 0 10, 0 0)')
);
```

### 4.2 距离计算

#### ST_Distance —— 计算两几何体间距离

```sql
-- 语法
ST_Distance(geometry1, geometry2)    -- 返回坐标系单位的距离
ST_Distance(geography1, geography2)  -- 返回球面距离（米）

-- 示例1：4547坐标系下（geometry），结果单位为米
SELECT ST_Distance(
    ST_SetSRID(ST_MakePoint(0, 0), 4547),
    ST_SetSRID(ST_MakePoint(3, 4), 4547)
);
-- 返回 5.0（米）（勾股定理：√(3²+4²)=5）

-- 示例2：4326坐标系下（geography），球面距离，结果单位为米
SELECT ST_Distance(
    ST_SetSRID(ST_MakePoint(116.397, 39.909), 4326)::geography,
    ST_SetSRID(ST_MakePoint(121.473, 31.230), 4326)::geography
);
-- 返回约 1067198（米）≈ 1067公里（北京到上海）
```

#### ST_DWithin —— 判断是否在指定距离范围内

```sql
-- 语法
ST_DWithin(geometry1, geometry2, distance)
ST_DWithin(geography1, geography2, distance_meters)

-- 示例
SELECT ST_DWithin(
    ST_SetSRID(ST_MakePoint(0, 0), 4547),
    ST_SetSRID(ST_MakePoint(500, 0), 4547),
    1000  -- 1000米范围内
);
-- 返回 true（500米 < 1000米）
```

> 🚀 **为什么用 ST_DWithin 而不用 ST_Distance < xxx？**
> 
> ```text
> ST_DWithin(geom, point, 1000)      → ✅ 能利用空间索引，极快
> ST_Distance(geom, point) < 1000    → ❌ 不能用索引，全表扫描，极慢
> ```

#### ST_ClosestPoint —— 求最近点（垂足）

```sql
-- 求线上距离指定点最近的点
SELECT ST_AsText(
    ST_ClosestPoint(
        ST_GeomFromText('LINESTRING(0 0, 10 0)'),  -- 线（X轴）
        ST_MakePoint(5, 8)                          -- 点
    )
);
-- 返回 POINT(5 0)（垂足）
```

```text
图示：
     *(5,8)  ← 原始点
     |
     | 8米
     |
─────*─────  ← 线（X轴）
   (5,0) ← 垂足（最近点）
```

### 4.3 空间关系判断

#### ST_Intersects —— 是否有交集（最常用）

```sql
-- 判断两几何体是否有任何交集（点/线/面均适用）
SELECT ST_Intersects(
    ST_GeomFromText('LINESTRING(0 0, 10 10)'),
    ST_GeomFromText('LINESTRING(0 10, 10 0)')
);
-- 返回 true（两条对角线相交）
```

#### ST_Intersection —— 返回交集几何体

```sql
-- 返回两几何体的交集部分
SELECT ST_AsText(
    ST_Intersection(
        ST_GeomFromText('LINESTRING(-5 5, 15 5)'),
        ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))')
    )
);
-- 返回 LINESTRING(0 5, 10 5)（线被面裁剪后的部分）
```

#### ST_Contains —— A 完全包含 B

```sql
SELECT ST_Contains(
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))'),  -- 面 A
    ST_MakePoint(5, 5)                                           -- 点 B
);
-- 返回 true（面包含点）
```

#### ST_Within —— A 完全在 B 内

```sql
-- ST_Within 是 ST_Contains 的反向
SELECT ST_Within(
    ST_MakePoint(5, 5),                                          -- 点 A
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))') -- 面 B
);
-- 返回 true
```

> 判断点在线内，若是在线的两头端点返回 false

#### ST_Crosses —— 是否穿越（有进有出）

```sql
SELECT ST_Crosses(
    ST_GeomFromText('LINESTRING(-5 5, 5 5)'),                    -- 线（从面外穿入面内）
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))')
);
-- 返回 true（线一部分在面内，一部分在面外）
```

#### ST_Touches —— 仅在边界接触

```sql
SELECT ST_Touches(
    ST_MakePoint(10, 5),                                          -- 点在边界上
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))')
);
-- 返回 true
```

#### ST_Overlaps —— 部分重叠（同维度）

```sql
SELECT ST_Overlaps(
    ST_GeomFromText('POLYGON((0 0, 6 0, 6 6, 0 6, 0 0))'),
    ST_GeomFromText('POLYGON((3 3, 10 3, 10 10, 3 10, 3 3))')
);
-- 返回 true（两个面部分重叠）
```

#### ST_Disjoint —— 完全不相交

```sql
SELECT ST_Disjoint(
    ST_MakePoint(100, 100),
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))')
);
-- 返回 true（点在面外，完全不相交）
```

#### 空间关系方法对比速查

|点的位置|ST_Intersects|ST_Contains|ST_Within|ST_Touches|ST_Disjoint|
|---|---|---|---|---|---|
|面**内部**|✅|✅|✅|❌|❌|
|面**边界上**|✅|❌|❌|✅|❌|
|面**外部**|❌|❌|❌|❌|✅|

### 4.4 几何体属性获取

#### ST_Area —— 计算面积

```sql
-- geometry类型（单位取决于坐标系，4547为平方米）
SELECT ST_Area(ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))', 4547));
-- 返回 100（平方米）

-- geography类型（始终返回平方米）
SELECT ST_Area(ST_GeomFromText('POLYGON((0 0, 1 0, 1 1, 0 1, 0 0))', 4326)::geography);
```

#### ST_Length —— 计算长度

```sql
-- 计算线的长度
SELECT ST_Length(ST_GeomFromText('LINESTRING(0 0, 3 4)', 4547));
-- 返回 5（米）（勾股定理：√(3²+4²)=5）
```

#### ST_Perimeter —— 计算周长

```sql
SELECT ST_Perimeter(ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))', 4547));
-- 返回 40（米）
```

#### ST_Centroid —— 计算重心（中心点）

```sql
SELECT ST_AsText(
    ST_Centroid(ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))'))
);
-- 返回 POINT(5 5)
```

#### ST_AsText —— 转换为WKT文本

```sql
SELECT ST_AsText(ST_MakePoint(116.397, 39.909));
-- 返回 POINT(116.397 39.909)
```

#### ST_AsGeoJSON —— 转换为GeoJSON格式

```sql
SELECT ST_AsGeoJSON(ST_MakePoint(116.397, 39.909));
-- 返回 {"type":"Point","coordinates":[116.397,39.909]}
```

#### ST_X / ST_Y —— 获取点的坐标值

```sql
SELECT ST_X(ST_MakePoint(116.397, 39.909));  -- 返回 116.397
SELECT ST_Y(ST_MakePoint(116.397, 39.909));  -- 返回 39.909
```

### 4.5 几何体处理

#### ST_Buffer —— 缓冲区分析

```sql
-- 以点为中心创建500米半径的圆形缓冲区
SELECT ST_Buffer(
    ST_SetSRID(ST_MakePoint(500000, 3000000), 4547),
    500   -- 500米
);

-- 以线为中心创建100米缓冲区（道路扩展）
SELECT ST_Buffer(
    ST_GeomFromText('LINESTRING(0 0, 100 0)', 4547),
    100
);
```

```text
图示（点的缓冲区）：
        ___
       /   \
      |  *  |   ← 圆形缓冲区（半径500米）
       \___/
         ↑
       中心点
```

#### ST_Union —— 合并几何体

```sql
-- 合并两个面（会消除重叠部分，合并相邻边界）
SELECT ST_AsText(ST_Union(
    ST_GeomFromText('POLYGON((0 0, 5 0, 5 5, 0 5, 0 0))'),
    ST_GeomFromText('POLYGON((3 3, 8 3, 8 8, 3 8, 3 3))')
));
```

#### ST_Collect —— 收集几何体（不合并）

```sql
-- 将多个几何体收集为集合（保留各自独立性，不消除重叠）
SELECT ST_Collect(geom) FROM road_section WHERE city = '北京市';
```

#### ST_Difference —— 几何体差集

```sql
-- A 减去 A与B的交集部分（A中不属于B的部分）
SELECT ST_AsText(ST_Difference(
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))'),   -- A
    ST_GeomFromText('POLYGON((5 0, 15 0, 15 10, 5 10, 5 0))')    -- B
));
-- 返回 A 中不与 B 重叠的部分
```

#### ST_SymDifference —— 对称差集

```sql
-- 返回两个几何体中，不属于交集的部分（排除公共区域）
SELECT ST_AsText(ST_SymDifference(
    ST_GeomFromText('POLYGON((0 0, 10 0, 10 10, 0 10, 0 0))'),
    ST_GeomFromText('POLYGON((5 0, 15 0, 15 10, 5 10, 5 0))')
));
```

## 五、坐标系选择与性能优化

### 5.1 坐标系选择原则

```text
┌─────────────────────────────────────────────────────────┐
│  业务范围在单个城市/省份     → 使用投影坐标系（如4547）   │
│  · 单位：米，计算速度快                                  │
│  · 直接用 geometry 类型                                  │
│  · 适合高精度局部计算                                    │
├─────────────────────────────────────────────────────────┤
│  业务范围跨省/全国/全球      → 使用地理坐标系（4326）     │
│  · 单位：度，需转geography                              │
│  · 球面计算，精度全局统一                                │
│  · 适合大范围距离计算                                    │
└─────────────────────────────────────────────────────────┘
```

### 5.2 性能优化核心原则

#### ❌ 错误写法（性能极差）

```sql
-- 对表中每一行都执行 ST_Transform，10万行执行10万次！
WHERE ST_DWithin(
    ST_Transform(ST_SetSRID(ST_MakePoint(pos_x, pos_y), 4547), 4326)::geography,
    ST_Transform(ST_SetSRID(ST_MakePoint(#{x}, #{y}), 4326), 4326)::geography,
    #{distance}
)
```

#### ✅ 正确写法（性能最优）

```sql
-- 只对用户传入的1个点做转换，其余数据直接在原坐标系下计算
WHERE ST_DWithin(
    ST_SetSRID(ST_MakePoint(pos_x::numeric, pos_y::numeric), 4547),    -- 直接贴标签，不转换
    ST_Transform(ST_SetSRID(ST_MakePoint(#{x}, #{y}), 4326), 4547),    -- 只转换1次！
    #{distance}
)
```

**性能提升逻辑**：

```text
❌ 错误写法：表中N条数据 × 1次ST_Transform = N次转换
✅ 正确写法：用户输入1个点 × 1次ST_Transform = 1次转换

性能提升 = N倍（N为表中数据量）
```

### 5.3 建立空间索引

```sql
-- 建立GiST空间索引（必须！否则ST_DWithin无法走索引）
CREATE INDEX idx_road_section_geom
ON road_section
USING GIST (
    ST_SetSRID(ST_MakePoint(pos_x::numeric, pos_y::numeric), 4547)
);

-- 如果表中有geometry类型的列，直接建索引
CREATE INDEX idx_road_geom ON road_section USING GIST(geom);
```

> 🔑 **空间索引对性能的影响**：
> 
> ```text
> 无索引 + ST_DWithin  → 全表扫描，O(N)
> 有索引 + ST_DWithin  → 索引扫描，O(log N)
> 有索引 + ST_Distance < xxx → ❌ 索引失效，全表扫描
> ```

## 六、综合业务示例

### 6.1 查询附近路段（最优写法）

```sql
-- 查询距用户位置 #{distance} 米内的路段，按距离排序
SELECT s1.id,
       s1.name,
       s1.pos_x,
       s1.pos_y,
       ROUND(
           ST_Distance(
               ST_SetSRID(ST_MakePoint(s1.pos_x::numeric, s1.pos_y::numeric), 4547),
               ST_Transform(ST_SetSRID(ST_MakePoint(#{x}, #{y}), 4326), 4547)
           )::numeric, 2
       ) AS distance
FROM road_section s1
WHERE ST_DWithin(
          ST_SetSRID(ST_MakePoint(s1.pos_x::numeric, s1.pos_y::numeric), 4547),
          ST_Transform(ST_SetSRID(ST_MakePoint(#{x}, #{y}), 4326), 4547),
          #{distance}
      )
ORDER BY distance;
```

### 6.2 查询某行政区内的所有路段

```sql
-- 查询完全在行政区内的路段
SELECT r.id, r.name
FROM road_section r
JOIN district d ON ST_Within(r.geom, d.geom)
WHERE d.name = '海淀区';

-- 查询穿越行政区的路段（部分在内部，部分在外部）
SELECT r.id, r.name
FROM road_section r
JOIN district d ON ST_Crosses(r.geom, d.geom)
WHERE d.name = '海淀区';

-- 裁剪路段，只保留行政区内的线段部分
SELECT r.id,
       r.name,
       ST_AsText(ST_Intersection(r.geom, d.geom)) AS clipped_geom
FROM road_section r
JOIN district d ON ST_Intersects(r.geom, d.geom)
WHERE d.name = '海淀区';
```

### 6.3 以用户位置为中心创建缓冲区，查询覆盖范围内的设施

```sql
SELECT f.id, f.name, f.type
FROM facilities f
WHERE ST_Within(
    f.geom,
    ST_Buffer(
        ST_Transform(ST_SetSRID(ST_MakePoint(#{x}, #{y}), 4326), 4547),
        1000   -- 1000米缓冲区
    )
);
```
