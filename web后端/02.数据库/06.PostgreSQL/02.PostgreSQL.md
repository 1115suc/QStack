## 一、JSONB —— 最实用的进阶特性

### 1.1 JSON vs JSONB

| | JSON | JSONB |
|--|------|-------|
| 存储方式 | 原始文本 | 二进制解析后存储 |
| 写入速度 | 稍快 | 稍慢（需解析）|
| 读取速度 | 慢（每次解析）| **快** |
| 支持索引 | ❌ | ✅ GIN 索引 |
| 保留字段顺序 | ✅ | ❌（但实际开发无影响）|
| 推荐使用 | 几乎不用 | ✅ **生产环境首选** |

### 1.2 建表与插入

```sql
CREATE TABLE products (
    id      BIGSERIAL PRIMARY KEY,
    name    VARCHAR(100) NOT NULL,
    meta    JSONB   -- 存储扩展属性
);

INSERT INTO products (name, meta) VALUES
('iPhone 16', '{"brand": "Apple", "color": ["black","white"], "price": 999, "specs": {"ram": 8, "storage": 256}}'),
('Galaxy S25', '{"brand": "Samsung", "color": ["blue"], "price": 899, "specs": {"ram": 12, "storage": 256}}');
```

### 1.3 查询操作符

```sql
-- -> 返回 JSON 对象（保留 JSON 类型）
SELECT meta -> 'brand' FROM products;          -- "Apple"

-- ->> 返回文本（常用于 WHERE 条件）
SELECT meta ->> 'brand' FROM products;         -- Apple

-- 嵌套访问
SELECT meta -> 'specs' ->> 'ram' FROM products;  -- 8

-- #> 通过路径数组访问
SELECT meta #> '{specs, ram}' FROM products;

-- #>> 路径访问，返回文本
SELECT meta #>> '{specs, ram}' FROM products;  -- 8

-- 过滤：找到 brand 为 Apple 的产品
SELECT * FROM products WHERE meta ->> 'brand' = 'Apple';

-- @> 包含操作符（检查是否包含某个子 JSON）
SELECT * FROM products WHERE meta @> '{"brand": "Apple"}';

-- ? 检查某个 key 是否存在
SELECT * FROM products WHERE meta ? 'color';

-- ?& 多个 key 都存在
SELECT * FROM products WHERE meta ?& ARRAY['brand', 'price'];
```

### 1.4 修改 JSONB

```sql
-- jsonb_set 修改某个字段
UPDATE products
SET meta = jsonb_set(meta, '{price}', '1099')
WHERE id = 1;

-- || 合并（覆盖同名 key）
UPDATE products
SET meta = meta || '{"discount": true}'
WHERE id = 1;

-- 删除某个 key
UPDATE products
SET meta = meta - 'discount'
WHERE id = 1;

-- 删除嵌套路径
UPDATE products
SET meta = meta #- '{specs, storage}'
WHERE id = 1;
```

### 1.5 JSONB GIN 索引

```sql
-- 对整个 JSONB 列建 GIN 索引（支持 @>、?、?& 等操作）
CREATE INDEX idx_products_meta ON products USING GIN(meta);

-- 对特定路径建索引（更精准，占用空间小）
CREATE INDEX idx_products_brand ON products USING GIN((meta -> 'brand'));
```

### 1.6 展开 JSONB 数组

```sql
-- jsonb_array_elements 展开数组
SELECT id, jsonb_array_elements_text(meta -> 'color') AS color
FROM products;

-- 结果：
-- 1 | black
-- 1 | white
-- 2 | blue
```

## 二、数组类型

### 2.1 基本使用

```sql
CREATE TABLE tags_demo (
    id    BIGSERIAL PRIMARY KEY,
    name  VARCHAR(50),
    tags  TEXT[]   -- 字符串数组
);

INSERT INTO tags_demo (name, tags) VALUES
('文章A', ARRAY['java', 'spring', 'backend']),
('文章B', '{python, django, backend}');  -- 另一种写法
```

### 2.2 查询与操作

```sql
-- 访问第 1 个元素（PostgreSQL 数组下标从 1 开始！）
SELECT tags[1] FROM tags_demo;

-- @> 包含
SELECT * FROM tags_demo WHERE tags @> ARRAY['java'];

-- && 有交集
SELECT * FROM tags_demo WHERE tags && ARRAY['java', 'python'];

-- 追加元素
UPDATE tags_demo SET tags = array_append(tags, 'mybatis') WHERE id = 1;

-- 删除元素
UPDATE tags_demo SET tags = array_remove(tags, 'spring') WHERE id = 1;

-- 数组长度
SELECT array_length(tags, 1) FROM tags_demo;

-- 展开数组（类似 JSONB 的 jsonb_array_elements）
SELECT id, UNNEST(tags) AS tag FROM tags_demo;
```

### 2.3 数组 GIN 索引

```sql
CREATE INDEX idx_tags ON tags_demo USING GIN(tags);
```

## 三、窗口函数

窗口函数是处理排名、累计统计等需求的利器，不需要 GROUP BY 却能进行分组计算。

### 3.1 语法结构

```sql
函数名() OVER (
    PARTITION BY 分组列
    ORDER BY 排序列
    ROWS/RANGE BETWEEN ... AND ...
)
```

### 3.2 排名函数

```sql
-- 准备数据
CREATE TABLE sales (
    id          SERIAL PRIMARY KEY,
    salesperson VARCHAR(50),
    department  VARCHAR(50),
    amount      NUMERIC(10,2),
    sale_date   DATE
);

-- ROW_NUMBER：每行唯一编号（即使金额相同）
SELECT
    salesperson,
    department,
    amount,
    ROW_NUMBER() OVER (PARTITION BY department ORDER BY amount DESC) AS row_num
FROM sales;

-- RANK：相同值同排名，下一名跳过（1,1,3）
SELECT
    salesperson,
    amount,
    RANK() OVER (ORDER BY amount DESC) AS rnk
FROM sales;

-- DENSE_RANK：相同值同排名，下一名不跳过（1,1,2）
SELECT
    salesperson,
    amount,
    DENSE_RANK() OVER (ORDER BY amount DESC) AS dense_rnk
FROM sales;

-- NTILE：分成 N 组
SELECT
    salesperson,
    amount,
    NTILE(4) OVER (ORDER BY amount DESC) AS quartile
FROM sales;
```

### 3.3 聚合窗口函数

```sql
-- 计算每个部门的累计销售额（按日期排序）
SELECT
    salesperson,
    department,
    sale_date,
    amount,
    SUM(amount) OVER (
        PARTITION BY department
        ORDER BY sale_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS cumulative_sales
FROM sales;

-- 部门内每人销售额 vs 部门总销售额
SELECT
    salesperson,
    department,
    amount,
    SUM(amount) OVER (PARTITION BY department) AS dept_total,
    ROUND(amount / SUM(amount) OVER (PARTITION BY department) * 100, 2) AS pct
FROM sales;
```

### 3.4 LAG / LEAD —— 访问前后行

```sql
-- 与上一行比较（环比）
SELECT
    sale_date,
    amount,
    LAG(amount, 1) OVER (ORDER BY sale_date) AS prev_amount,
    amount - LAG(amount, 1) OVER (ORDER BY sale_date) AS diff
FROM sales;

-- LEAD：访问下一行
SELECT
    sale_date,
    amount,
    LEAD(amount, 1) OVER (ORDER BY sale_date) AS next_amount
FROM sales;
```

### 3.5 FIRST_VALUE / LAST_VALUE

```sql
-- 每个部门最高销售额的人
SELECT
    salesperson,
    department,
    amount,
    FIRST_VALUE(salesperson) OVER (
        PARTITION BY department ORDER BY amount DESC
    ) AS top_salesperson
FROM sales;
```

## 四、CTE（公用表表达式）

CTE（`WITH` 子句）让复杂查询更加清晰易读。

### 4.1 基本 CTE

```sql
WITH active_users AS (
    SELECT id, username, email
    FROM users
    WHERE is_active = TRUE
),
recent_orders AS (
    SELECT user_id, COUNT(*) AS order_count
    FROM orders
    WHERE created_at > NOW() - INTERVAL '30 days'
    GROUP BY user_id
)
SELECT
    u.username,
    u.email,
    COALESCE(o.order_count, 0) AS order_count
FROM active_users u
LEFT JOIN recent_orders o ON u.id = o.user_id
ORDER BY order_count DESC;
```

### 4.2 递归 CTE（树形结构）

适合处理菜单、组织架构、分类树等场景：

```sql
CREATE TABLE categories (
    id        BIGSERIAL PRIMARY KEY,
    name      VARCHAR(100),
    parent_id BIGINT REFERENCES categories(id)
);

-- 递归查询所有子分类
WITH RECURSIVE category_tree AS (
    -- 基础查询（根节点）
    SELECT id, name, parent_id, 0 AS level, name::TEXT AS path
    FROM categories
    WHERE parent_id IS NULL

    UNION ALL

    -- 递归部分
    SELECT c.id, c.name, c.parent_id, ct.level + 1, ct.path || ' > ' || c.name
    FROM categories c
    JOIN category_tree ct ON c.parent_id = ct.id
)
SELECT * FROM category_tree ORDER BY path;
```

## 五、全文搜索

### 5.1 基础全文搜索

```sql
-- tsvector：文档向量
-- tsquery：查询向量

-- 简单示例
SELECT to_tsvector('english', 'The quick brown fox jumps over the lazy dog')
    @@ to_tsquery('english', 'fox & dog');  -- true

-- 在表上使用
ALTER TABLE articles ADD COLUMN search_vector TSVECTOR;

-- 生成向量（中文需要安装 zhparser 等扩展）
UPDATE articles
SET search_vector = to_tsvector('english', title || ' ' || content);

-- 创建 GIN 索引
CREATE INDEX idx_articles_search ON articles USING GIN(search_vector);

-- 查询
SELECT title FROM articles
WHERE search_vector @@ to_tsquery('english', 'postgresql & java')
ORDER BY ts_rank(search_vector, to_tsquery('english', 'postgresql & java')) DESC;
```

### 5.2 自动更新 search_vector（触发器）

```sql
CREATE OR REPLACE FUNCTION update_search_vector()
RETURNS TRIGGER AS $$
BEGIN
    NEW.search_vector = to_tsvector('english', NEW.title || ' ' || NEW.content);
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER articles_search_vector_update
BEFORE INSERT OR UPDATE ON articles
FOR EACH ROW EXECUTE FUNCTION update_search_vector();
```

## 六、常用内置函数

### 6.1 日期时间函数

```sql
-- 当前时间
SELECT NOW();                          -- 2024-01-15 10:30:00.123456+08
SELECT CURRENT_TIMESTAMP;             -- 同上
SELECT CURRENT_DATE;                  -- 2024-01-15

-- 时间加减
SELECT NOW() + INTERVAL '1 day';
SELECT NOW() - INTERVAL '30 minutes';

-- 时间截断（常用于按天/月/年分组）
SELECT DATE_TRUNC('day', NOW());       -- 2024-01-15 00:00:00
SELECT DATE_TRUNC('month', NOW());     -- 2024-01-01 00:00:00

-- 提取部分
SELECT EXTRACT(YEAR FROM NOW());       -- 2024
SELECT EXTRACT(DOW FROM NOW());        -- 0=周日 ... 6=周六

-- 时间差（返回 interval）
SELECT AGE(NOW(), '2000-01-01');

-- 格式化
SELECT TO_CHAR(NOW(), 'YYYY-MM-DD HH24:MI:SS');
```

### 6.2 字符串函数

```sql
SELECT LENGTH('hello');                     -- 5
SELECT UPPER('hello');                      -- HELLO
SELECT LOWER('HELLO');                      -- hello
SELECT TRIM('  hello  ');                   -- hello
SELECT LTRIM('  hello');                    -- hello
SELECT REPLACE('hello world', 'world', 'pg'); -- hello pg
SELECT SPLIT_PART('a,b,c', ',', 2);        -- b
SELECT SUBSTRING('hello world', 7, 5);     -- world
SELECT REGEXP_REPLACE('hello123', '[0-9]+', ''); -- hello
SELECT LEFT('hello', 3);                   -- hel
SELECT RIGHT('hello', 3);                  -- llo
SELECT LPAD('5', 3, '0');                  -- 005
```

### 6.3 聚合函数

```sql
-- 标准聚合
SELECT COUNT(*), SUM(amount), AVG(amount), MAX(amount), MIN(amount) FROM sales;

-- 字符串聚合（MySQL 的 GROUP_CONCAT 对应）
SELECT
    department,
    STRING_AGG(salesperson, ', ' ORDER BY salesperson) AS members
FROM sales
GROUP BY department;

-- 数组聚合
SELECT
    department,
    ARRAY_AGG(salesperson ORDER BY amount DESC) AS sorted_members
FROM sales
GROUP BY department;

-- FILTER（条件聚合）
SELECT
    COUNT(*) AS total,
    COUNT(*) FILTER (WHERE amount > 1000) AS high_sales,
    AVG(amount) FILTER (WHERE department = 'IT') AS it_avg
FROM sales;
```

## 七、PostgreSQL 特有的实用功能

### 7.1 GENERATE_SERIES —— 生成序列

```sql
-- 生成数字序列
SELECT GENERATE_SERIES(1, 5);           -- 1,2,3,4,5

-- 生成日期序列（报表场景非常实用）
SELECT GENERATE_SERIES(
    '2024-01-01'::DATE,
    '2024-01-31'::DATE,
    '1 day'::INTERVAL
)::DATE AS day;

-- 实用场景：生成日历后 LEFT JOIN 订单表，补全没有数据的日期
SELECT
    d.day,
    COALESCE(SUM(o.amount), 0) AS daily_revenue
FROM GENERATE_SERIES('2024-01-01', '2024-01-31', '1 day') AS d(day)
LEFT JOIN orders o ON o.created_at::DATE = d.day
GROUP BY d.day
ORDER BY d.day;
```

### 7.2 物化视图（Materialized View）

```sql
-- 创建物化视图（查询结果会物理存储）
CREATE MATERIALIZED VIEW mv_department_stats AS
SELECT
    department,
    COUNT(*) AS employee_count,
    AVG(salary) AS avg_salary
FROM employees
GROUP BY department;

-- 建索引（物化视图支持索引）
CREATE INDEX idx_mv_dept ON mv_department_stats(department);

-- 刷新数据
REFRESH MATERIALIZED VIEW mv_department_stats;

-- 不锁表刷新（CONCURRENTLY 要求有唯一索引）
REFRESH MATERIALIZED VIEW CONCURRENTLY mv_department_stats;
```

### 7.3 表继承（了解即可）

```sql
-- 子表继承父表结构
CREATE TABLE logs (
    id         BIGSERIAL,
    message    TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE error_logs (
    error_code VARCHAR(20)
) INHERITS (logs);
```

### 7.4 advisory lock（应用级锁）

```sql
-- 获取锁（阻塞式）
SELECT pg_advisory_lock(12345);

-- 尝试获取锁（非阻塞，失败返回 false）
SELECT pg_try_advisory_lock(12345);

-- 释放锁
SELECT pg_advisory_unlock(12345);
```

> 📌 适合分布式场景中替代 Redis 分布式锁（如定时任务防重复执行）。

## 八、性能调优要点

### 8.1 VACUUM & ANALYZE

PostgreSQL 的 MVCC 机制会产生「死元组」（dead tuples），需要定期清理：

```sql
-- 手动执行（通常由 autovacuum 自动处理）
VACUUM ANALYZE users;

-- 查看表的死元组数量
SELECT relname, n_dead_tup, n_live_tup, last_vacuum, last_autovacuum
FROM pg_stat_user_tables
ORDER BY n_dead_tup DESC;
```

### 8.2 常用性能诊断 SQL

```sql
-- 查看当前活跃连接
SELECT pid, usename, application_name, state, query, query_start
FROM pg_stat_activity
WHERE state != 'idle';

-- 查看表大小
SELECT
    relname AS table_name,
    pg_size_pretty(pg_total_relation_size(relid)) AS total_size,
    pg_size_pretty(pg_relation_size(relid)) AS data_size,
    pg_size_pretty(pg_indexes_size(relid)) AS index_size
FROM pg_stat_user_tables
ORDER BY pg_total_relation_size(relid) DESC;

-- 查看未使用的索引
SELECT
    schemaname, tablename, indexname,
    idx_scan AS index_scans
FROM pg_stat_user_indexes
WHERE idx_scan = 0
ORDER BY tablename;

-- 查看慢查询（需开启 pg_stat_statements 扩展）
SELECT query, calls, total_exec_time, mean_exec_time, rows
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;
```

## 小结

| 特性 | 使用场景 |
|------|---------|
| JSONB | 存储灵活的扩展字段，替代多余的关联表 |
| 数组类型 | 标签、权限列表等一对多但量少的场景 |
| 窗口函数 | 排名、累计、环比等统计分析 |
| 递归 CTE | 树形结构（菜单、分类、组织架构）|
| 全文搜索 | 站内搜索（中文需额外扩展）|
| 物化视图 | 复杂报表的查询加速 |
| GENERATE_SERIES | 补全时间序列（报表零数据填充）|
