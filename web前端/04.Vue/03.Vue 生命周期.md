# Vue 生命周期与组件化开发

---

## 一、Vue 生命周期

### 什么是生命周期？

**Vue 生命周期：** 一个 Vue 实例从**创建**到**销毁**的完整过程，就像人从出生到死亡的一生。

Vue 在这个过程中会自动经历 **4 个阶段**，每个阶段前后都提供了"钩子函数"，让我们可以在特定时刻执行自己的代码。

**四个阶段：**

![生命周期四阶段](img/1682065937815.png)

| 阶段 | 说明 | 此阶段能做什么 |
|------|------|----------------|
| **创建阶段** | 初始化响应式数据 | 可访问 data、methods，但 DOM 尚未生成 |
| **挂载阶段** | 渲染模板，生成真实 DOM | DOM 已就绪，可操作 DOM 元素 |
| **更新阶段** | 数据变化，重新渲染视图 | 每次数据更新后触发 |
| **销毁阶段** | 销毁 Vue 实例 | 清理定时器、取消事件监听等 |

---

### 生命周期钩子函数

钩子函数就是 Vue 在各个阶段自动调用的函数，我们可以在里面写自己的逻辑。

![生命周期钩子](img/1682066040295.png)

```js
const app = new Vue({
  el: '#app',
  data: { count: 0 },

  // ===== 创建阶段 =====
  beforeCreate() {
    // 数据和方法还未初始化，this.xxx 访问不到
    // 几乎不使用
  },
  created() {
    // ✅ 数据和方法已初始化完成，但 DOM 还没渲染
    // 最佳用途：发送网络请求获取初始数据（越早越好）
  },

  // ===== 挂载阶段 =====
  beforeMount() {
    // DOM 还未完成渲染，几乎不使用
  },
  mounted() {
    // ✅ DOM 渲染完毕，页面已显示
    // 最佳用途：操作 DOM、初始化第三方库（如 echarts）、让输入框自动聚焦
  },

  // ===== 更新阶段 =====
  beforeUpdate() {
    // 数据已改变，但 DOM 还未同步更新
  },
  updated() {
    // DOM 已更新完毕
  },

  // ===== 销毁阶段 =====
  beforeDestroy() {
    // ✅ 实例销毁前，还能访问数据和方法
    // 最佳用途：清除定时器、取消事件监听、释放内存
  },
  destroyed() {
    // 实例已销毁完毕
  }
})
```

**常用钩子速记（最重要的 3 个）：**

| 钩子 | 触发时机 | 典型用途 |
|------|----------|----------|
| `created` | 数据初始化完成，DOM 未生成 | 发送网络请求、初始化数据 |
| `mounted` | DOM 渲染完成 | 操作 DOM、初始化图表、自动聚焦 |
| `beforeDestroy` | 实例销毁前 | 清除定时器、取消订阅 |

**完整案例：在 `created` 发请求，在 `mounted` 操作 DOM**

```js
const app = new Vue({
  el: '#app',
  data: { list: [], words: '' },

  async created() {
    // 越早发请求越好，created 是第一个能发请求的时机
    const res = await axios.get('http://hmajax.itheima.net/api/news')
    this.list = res.data.data
  },

  mounted() {
    // DOM 已渲染完毕，可以操作 DOM 元素
    document.querySelector('#inp').focus()
  }
})
```

> 💡 **为什么要在 `created` 发请求而不是 `mounted`？**  
> 因为 `created` 比 `mounted` 更早执行，可以让网络请求更早发出，数据更快返回，用户看到内容的时间更短。

## 二、工程化开发与脚手架

### 两种开发方式对比

| 方式 | 说明 | 优缺点 |
|------|------|--------|
| 传统方式 | 直接在 HTML 中用 `<script>` 引入 Vue | 简单快速，但不支持模块化、Less 等现代语法 |
| 工程化方式 | 基于 webpack 构建工具开发 | 功能强大，支持组件化开发，但需配置环境 |

![工程化开发](img/1682090039070.png)

### Vue CLI 脚手架

> Vue CLI 是 Vue 官方提供的**全局命令工具**，一行命令就能生成配置好 webpack 的标准项目结构，省去繁琐配置。

**安装与使用：**

```bash
# 第1步：全局安装 Vue CLI（只需安装一次）
npm install -g @vue/cli

# 第2步：查看安装是否成功
vue --version

# 第3步：创建项目（进入你想存放项目的目录，项目名不能含中文）
vue create my-project

# 第4步：进入项目并启动开发服务器
cd my-project
npm run serve
```

---

### 项目目录结构

![项目目录](img/1682092148521.png)

```
my-project/
├── public/
│   └── index.html        # 模板文件（提供 #app 挂载点）
├── src/
│   ├── assets/           # 静态资源（图片、样式等）
│   ├── components/       # 子组件存放目录
│   ├── App.vue           # 根组件（整个应用的起点）
│   └── main.js           # 入口文件（启动整个应用）
├── package.json          # 项目依赖配置
└── vue.config.js         # Vue CLI 项目配置文件
```

**三个核心文件的职责：**

| 文件 | 职责 |
|------|------|
| `index.html` | 提供 `<div id="app">` 挂载点，是最终输出的 HTML |
| `main.js` | 程序入口，创建 Vue 实例，将根组件挂载到 `#app` |
| `App.vue` | 根组件，所有页面内容都从这里展开 |

### 运行流程

![运行流程](img/1682094032876.png)

```
① index.html  →  提供 <div id="app"> 容器
        ↓
② main.js     →  new Vue()，将 App.vue 渲染并挂载到 #app
        ↓
③ App.vue     →  根组件，包含整体页面布局
        ↓
④ 各子组件    →  HmHeader、HmMain、HmFooter...
```

**main.js 完整示例：**

```js
import Vue from 'vue'
import App from './App.vue'

Vue.config.productionTip = false  // 关闭生产环境提示

new Vue({
  render: h => h(App),  // 将 App 根组件渲染到页面
}).$mount('#app')        // 挂载到 index.html 中的 #app
```

## 三、组件化开发

### 什么是组件？为什么要组件化？

**核心思想：** 把一个完整的页面拆分成若干个**独立的、可复用的小块**，每个小块就是一个"组件"，各自负责自己的结构、样式和逻辑。

![组件化拆分](img/1682168852372.png)

**用一个类比来理解：**

> 🏠 盖房子时，我们不会把整栋楼一次性浇筑，而是预制好门、窗、墙等标准化"构件"，再组装起来。组件就是网页的"预制构件"。

**组件化的优势：**

| 优势 | 说明 |
|------|------|
| **便于维护** | 每个组件只负责一块功能，修改时不影响其他部分 |
| **利于复用** | 相同的 UI 模块只写一次，到处使用 |
| **提升效率** | 多人可以同时开发不同组件，互不干扰 |
| **易于测试** | 可以单独测试每个组件的功能 |

**组件分类：**

| 类型 | 说明 |
|------|------|
| 根组件（`App.vue`） | 整个应用最顶层的组件，是所有组件的"祖先" |
| 子组件（`.vue` 文件） | 可复用的局部功能模块，如 Header、Footer、Button 等 |

## 四、单文件组件（.vue 文件）

### 什么是 .vue 单文件组件？

在工程化开发中，每个组件被写成一个独立的 `.vue` 文件，这就是**单文件组件（SFC，Single File Component）**。

每个 `.vue` 文件由三个区块组成：

```
┌─────────────────────────────┐
│  <template>                 │  ← 结构（HTML）
│    组件的 HTML 内容          │
│  </template>                │
│                             │
│  <script>                   │  ← 逻辑（JavaScript）
│    组件的 JS 代码            │
│  </script>                  │
│                             │
│  <style>                    │  ← 样式（CSS/Less/Sass）
│    组件的样式                │
│  </style>                   │
└─────────────────────────────┘
```

### 根组件 App.vue 详解

![根组件结构](img/1682169131688.png)

```vue
<template>
  <!--
    结构层（必须有）
    ⚠️ template 内部有且只能有一个根元素（通常是 <div>）
       多个平级元素需要包裹在一个父元素中
  -->
  <div class="app">
    <h1>{{ title }}</h1>
  </div>
</template>

<script>
// 逻辑层（可选）
// 必须用 export default 导出组件配置对象
export default {
  // ⚠️ 在组件中，data 必须是一个函数，返回一个对象
  // （原因：每个组件实例都应该有自己独立的数据，函数能保证每次返回新对象）
  data() {
    return {
      title: 'Hello Vue!'
    }
  },
  methods: {
    // 组件内的方法
  },
  computed: {
    // 计算属性
  }
}
</script>

<style lang="less">
/*
  样式层（可选）
  lang="less" 表示使用 Less 预处理器（需安装 less 和 less-loader）
  不加 lang 默认就是普通 CSS
*/
.app {
  color: red;
  h1 {
    font-size: 24px;  /* Less 支持嵌套写法 */
  }
}
</style>
```

**三个区块说明：**

| 区块 | 是否必须 | 说明 |
|------|----------|------|
| `<template>` | ✅ 必须 | 组件的 HTML 结构，**只能有一个根元素** |
| `<script>` | 否 | JS 逻辑，用 `export default {}` 导出配置 |
| `<style>` | 否 | 样式，`lang="less"` 支持 Less 嵌套写法 |

> 💡 **data 为什么要写成函数？**
> 
> 假设同一个组件在页面上使用了两次。如果 `data` 是对象，两个实例会**共享同一个对象**，一个变了另一个也跟着变，造成混乱。
> 写成函数后，每次创建组件实例都会调用函数**返回新对象**，各自独立，互不影响。

> 📌 推荐安装 **Vetur** 插件（VSCode），可以获得 `.vue` 文件的语法高亮和智能提示。

## 五、局部注册组件

### 什么是局部注册？

局部注册的组件只能在**注册它的父组件内部**使用，其他组件用不了。

> 💡 **类比**：就像公司的内部工具，只有本部门的人能用，外部人员不行。

### 使用步骤（4步）

1. **创建**子组件的 `.vue` 文件
2. 在父组件中 **`import` 导入**
3. 在 `components` 配置项中**注册**
4. 在父组件的模板中**使用**（作为标签）

**完整示例：**

**第1步：创建子组件文件**

```vue
<!-- src/components/HmHeader.vue -->
<template>
  <div class="hm-header">
    <h1>网站头部</h1>
  </div>
</template>

<script>
export default {
  // 组件名称（可选，但建议写上，调试时有用）
  name: 'HmHeader'
}
</script>

<style>
.hm-header { background: #333; color: #fff; }
</style>
```

**第2、3、4步：在父组件中导入、注册、使用**

```vue
<!-- src/App.vue -->
<template>
  <div class="app">
    <!-- 第4步：像 HTML 标签一样使用注册好的组件 -->
    <HmHeader></HmHeader>
    <!-- 也可以写成自闭合标签 -->
    <HmMain />
    <HmFooter></HmFooter>
  </div>
</template>

<script>
// 第2步：导入组件
import HmHeader from './components/HmHeader.vue'
import HmMain from './components/HmMain.vue'
import HmFooter from './components/HmFooter.vue'

export default {
  // 第3步：在 components 中注册
  components: {
    HmHeader,   // 等同于 HmHeader: HmHeader（ES6 属性简写）
    HmMain,
    HmFooter
  }
}
</script>
```

![局部注册布局](img/1682227902873.png)

**组件注册后，父组件的结构相当于：**

```
App（父）
├── HmHeader（子组件1）
├── HmMain（子组件2）
└── HmFooter（子组件3）
```

> 📌 **命名规范：** 组件名使用**大驼峰命名法（PascalCase）**，如 `HmHeader`、`MyButton`。
> 在模板中两种写法均可：`<HmHeader>` 或 `<hm-header>`（Vue 会自动识别）。

## 六、全局注册组件

### 什么是全局注册？

全局注册的组件，**整个项目中任意组件**都可以直接使用，无需再次 import 和注册。

> 💡 **类比**：就像公司所有人都能用的公共设施（厕所、食堂），全局可访问。

### 适合全局注册的组件

并非所有组件都应该全局注册，只有**通用的、频繁使用**的组件才适合：
- 通用按钮（`MyButton`）
- 通用弹窗（`MyDialog`）
- 通用加载动画（`MyLoading`）
- 全局提示条（`MyToast`）

### 使用步骤（2步）

1. 创建 `.vue` 组件文件
2. 在 `main.js` 中导入并调用 `Vue.component()` 注册

```js
// src/main.js

import Vue from 'vue'
import App from './App.vue'

// 第1步：导入要全局注册的组件
import HmButton from './components/HmButton.vue'

// 第2步：全局注册（必须在 new Vue 之前调用）
// 参数1：组件名（字符串），参数2：组件配置
Vue.component('HmButton', HmButton)

new Vue({
  render: h => h(App)
}).$mount('#app')
```

```vue
<!-- 任意组件中，无需 import，直接使用 -->
<template>
  <div>
    <HmButton></HmButton>
  </div>
</template>
```

---

### 局部注册 vs 全局注册

| 对比项 | 局部注册 | 全局注册 |
|--------|----------|----------|
| 注册位置 | 组件内的 `components` 选项 | `main.js` 中的 `Vue.component()` |
| 可用范围 | 仅在注册它的组件内可用 | 项目中任意组件均可使用 |
| 适用场景 | 专属于某个页面/模块的组件 | 通用基础组件（按钮、弹窗等） |
| 按需加载 | ✅ 只在需要时加载 | ❌ 始终打包，可能增大体积 |

> 💡 **建议**：
> - 业务组件（只在某个页面用到的）→ 局部注册
> - 基础 UI 组件（全项目通用的）→ 全局注册

## 七、组件化开发总结

### 组件与 Vue 实例的关系

| 对比项 | Vue 实例（`new Vue()`） | 组件（`.vue` 文件） |
|--------|------------------------|---------------------|
| `data` 写法 | 对象 `data: {}` | 函数 `data() { return {} }` |
| 挂载方式 | 有 `el` 配置 | 无 `el`，通过父组件使用 |
| 数量 | 整个应用只有一个 | 可以有无数个 |

### 组件嵌套示意图

```
App.vue（根组件）
├── HmHeader.vue（头部组件）
│   └── HmLogo.vue（Logo 子组件）
├── HmMain.vue（主内容组件）
│   ├── HmBanner.vue（轮播图组件）
│   └── HmList.vue（列表组件）
└── HmFooter.vue（底部组件）
```

### 组件开发完整流程

```
① 规划页面结构，确定哪些部分可以抽成组件
        ↓
② 创建对应的 .vue 文件，编写组件内容
        ↓
③ 在需要使用该组件的父组件中 import 导入
        ↓
④ 在 components 配置项中注册
        ↓
⑤ 在模板中像 HTML 标签一样使用
```

## 本篇重点总结

| 知识点 | 核心要点 |
|--------|----------|
| 生命周期 | 创建→挂载→更新→销毁，共 8 个钩子 |
| `created` | 发网络请求的最佳时机 |
| `mounted` | 操作 DOM 或初始化第三方库的最佳时机 |
| `beforeDestroy` | 清理定时器和事件监听的最佳时机 |
| 单文件组件 | `.vue` 文件 = `<template>` + `<script>` + `<style>` |
| `data` 函数写法 | 组件中 `data` 必须是函数，保证每个实例数据独立 |
| 局部注册 | `components` 配置项，仅当前组件可用 |
| 全局注册 | `main.js` 中 `Vue.component()`，全项目可用 |

### 🔑 重点难点提示（全篇汇总）

1. **v-show vs v-if 的选择** — 频繁切换用 `v-show`（只改 CSS），条件复杂/初始不渲染用 `v-if`（操作 DOM）

2. **v-for 必须加 :key** — 使用稳定唯一的 `id` 作为 key，避免用 `index`，否则列表增删时会出现渲染错乱

3. **computed 缓存机制** — 依赖数据不变则不重新计算，直接返回缓存结果；`methods` 无论如何每次都执行

4. **watch 深度监视** — 监视整个对象需加 `deep: true`，否则对象内部属性变化不会触发；监视对象某属性用 `'obj.key'` 字符串形式

5. **生命周期钩子选择** — 请求数据用 `created`（更早），操作 DOM 和初始化第三方库用 `mounted`（DOM 已就绪）

6. **组件中 data 是函数** — 保证多个组件实例之间数据隔离，互不影响

7. **组件注册命名规范** — 统一使用**大驼峰（PascalCase）**命名，在模板中 `<MyComp>` 或 `<my-comp>` 均可
