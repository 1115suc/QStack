# Vue 基础与常用指令

---

## 一、什么是 Vue

> Vue（读音 /vjuː/，类似 view）是一套**构建用户界面**的**渐进式框架**。

### 核心概念拆解

**① 构建用户界面**

简单来说，就是基于数据，渲染出用户可见的页面内容。你只需要专注于数据本身，Vue 会帮你把数据显示在页面上。

![构建用户界面](img/1681875887026.png)

**② 渐进式**

Vue 不强迫你一次性学会所有内容，可以循序渐进，学一点用一点：

| 开发方式 | 说明 | 适用场景 |
|----------|------|----------|
| Vue 核心包 | 只引入 Vue 核心包 | 局部模块改造 |
| Vue 核心包 + 插件 + 工程化 | 完整工程化体系 | 整站开发 |

**③ 框架 vs 库**

![框架与库对比](img/1681876620277.png)

| 对比项 | 库（Library） | 框架（Framework） |
|--------|--------------|------------------|
| 定义 | 工具箱，一堆方法集合 | 完整的解决方案 |
| 代表 | axios、lodash、echarts | Vue、React、Angular |
| 使用方式 | 按需调用 | 遵守框架规则编写代码 |

> 💡 **类比理解**：库就像一个工具箱，你想用哪把锤子就拿哪把；框架就像一条流水线，你必须按流水线的规则工作，但效率更高。

## 二、创建 Vue 实例

**核心步骤（4步）：**

1. 准备容器（HTML 挂载点）
2. 引入 Vue 包（开发版 / 生产版）
3. `new Vue()` 创建实例
4. 指定配置项渲染数据

![创建Vue实例](img/1681877405007.png)

```html
<!-- 第1步：准备容器，id="app" 是 Vue 的"管理区域" -->
<div id="app">
  {{ msg }}
</div>

<!-- 第2步：引包（通过 CDN 在线引入） -->
<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
<script>
  // 第3步：创建实例，第4步：指定配置项
  const app = new Vue({
    el: '#app',    // 挂载点：告诉 Vue 管理哪个 DOM 容器
    data: {        // 提供数据：这里的数据会自动渲染到页面
      msg: 'Hello Vue!'
    }
  })
</script>
```

**配置项说明：**

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `el` | String | 挂载点，CSS 选择器形式，指定 Vue 管理的容器 |
| `data` | Object | 提供渲染数据，数据变化视图自动更新 |
| `methods` | Object | 提供方法（事件处理函数等） |

> 💡 **类比理解**：创建 Vue 实例就像雇了一个"数据管家"，`el` 告诉他管哪个房间，`data` 是他管理的物品清单，有任何变动他都会立刻更新房间内的展示。

## 三、插值表达式

**语法：** `{{ 表达式 }}`

用于在模板（HTML）中将 `data` 中的数据渲染到页面。`{{}}` 里面可以写 JS 表达式。

![插值表达式](img/1681886494417.png)

```html
<h3>{{ title }}</h3>
<p>{{ nickName.toUpperCase() }}</p>        <!-- 调用字符串方法 -->
<p>{{ age >= 18 ? '成年' : '未成年' }}</p>  <!-- 三元表达式 -->
<p>{{ obj.name }}</p>                       <!-- 访问对象属性 -->
<p>{{ fn() }}</p>                           <!-- 调用函数 -->
```

**✅ 支持的表达式：**

```js
money + 100                    // 算术运算
price >= 100 ? 'A' : 'B'      // 三元表达式
obj.name                       // 对象属性访问
arr[0]                         // 数组索引
fn()                           // 函数调用
```

**❌ 错误用法：**

```html
<!-- 1. data 中未声明的变量 -->
<p>{{ hobby }}</p>

<!-- 2. 语句（if/for 是语句不是表达式） -->
<p>{{ if (true) {} }}</p>

<!-- 3. 标签属性中不能用插值表达式（要用 v-bind） -->
<p title="{{ username }}">错误示范</p>
```

> ✅ 插值表达式**只能**用在**标签内容**中，不能用于标签属性。绑定属性要使用 `v-bind`。

## 四、响应式特性

**响应式：数据变 → 视图自动更新**

`data` 中的数据最终会被代理到 Vue 实例上，可以直接通过实例访问和修改。

```js
const app = new Vue({
  el: '#app',
  data: { count: 0 }
})

// 直接通过实例访问数据
console.log(app.count)  // 0

// 修改数据 → 视图自动更新（不需要手动操作 DOM！）
app.count = 10
```

![响应式演示](img/1681888539340.png)

> 💡 **类比理解**：就像 Excel 表格，你改了某个单元格的值，所有引用了它的地方都会自动更新。Vue 的响应式原理和这个类似。

## 五、常用指令总览

> 指令（Directives）是 Vue 提供的带有 **`v-` 前缀**的特殊标签属性。
> 普通的 HTML 属性控制静态内容，而 Vue 指令能让属性与数据绑定，动态控制页面。

| 分类 | 指令 | 作用 |
|------|------|------|
| 内容渲染 | `v-text` | 渲染纯文本内容（类似 innerText） |
| 内容渲染 | `v-html` | 渲染 HTML 内容（类似 innerHTML） |
| 条件渲染 | `v-show` | 控制元素显示隐藏（切换 display） |
| 条件渲染 | `v-if` / `v-else-if` / `v-else` | 条件性创建/移除元素 |
| 事件绑定 | `v-on` / `@` | 绑定事件监听 |
| 属性绑定 | `v-bind` / `:` | 动态绑定 HTML 属性 |
| 双向绑定 | `v-model` | 表单元素双向数据绑定 |
| 列表渲染 | `v-for` | 基于数组循环渲染列表 |

## 六、内容渲染指令

### v-text

```html
<!-- 将 uname 的值渲染到 p 标签中，会覆盖原有内容 -->
<p v-text="uname">这里会被覆盖</p>
```

### v-html

```html
<!-- 能解析 HTML 标签，会覆盖原有内容 -->
<p v-html="intro"></p>
```

```js
const app = new Vue({
  el: '#app',
  data: {
    uname: '张三',
    intro: '<h2>这是一个<strong>非常优秀</strong>的boy</h2>'
  }
})
```

**三种渲染方式对比：**

| 指令 | 是否解析 HTML | 说明 |
|------|--------------|------|
| `v-text` | ❌ 不解析 | 渲染纯文本，同 innerText |
| `v-html` | ✅ 解析 | 渲染 HTML，同 innerHTML |
| `{{ }}` | ❌ 不解析 | 灵活，可与其他文字混合使用 |

> ⚠️ `v-html` 存在 XSS 安全风险，**不要**将用户输入的内容传给 `v-html`，只在信任的内容上使用。

## 七、条件渲染指令

### v-show

```html
<div v-show="isShow">我是通过 display 控制显示隐藏的</div>
```

- 原理：切换元素的 `display: none`，**DOM 始终存在于页面中**
- 适用：**频繁切换**显示隐藏的场景（如 Tab 切换、折叠面板）

![v-show原理](img/1681891228284.png)

### v-if / v-else-if / v-else

```html
<p v-if="gender === 1">♂ 男</p>
<p v-else>♀ 女</p>

<p v-if="score >= 90">A：奖励电脑一台</p>
<p v-else-if="score >= 75">B：奖励周末郊游</p>
<p v-else-if="score >= 60">C：奖励零食礼包</p>
<p v-else>D：惩罚一周不能玩手机</p>
```

- 原理：条件为 `false` 时**直接从 DOM 中删除节点**，条件为 `true` 时重新创建
- 适用：**不频繁切换**，或初始不需要渲染的场景

![v-if原理](img/1681891237750.png)

**v-show vs v-if 对比：**

| 对比项 | v-show | v-if |
|--------|--------|------|
| 控制方式 | CSS display | 创建/移除 DOM |
| 初始渲染消耗 | 较高（都渲染） | 较低（false 不渲染） |
| 切换消耗 | 低 | 高（重新创建销毁） |
| 适用场景 | 频繁切换 | 不频繁切换 |

> 💡 **选择口诀**：频繁切换用 `v-show`，条件复杂或很少变用 `v-if`。

## 八、事件绑定指令

**语法：**
```html
<button v-on:事件名="处理逻辑">按钮</button>
<!-- 简写（推荐） -->
<button @事件名="处理逻辑">按钮</button>
```

### 三种使用方式

**① 内联语句（适合简单逻辑）**

```html
<button @click="count++">+1</button>
<button @click="count--">-1</button>
<span>{{ count }}</span>
```

**② 方法名（推荐，逻辑复杂时使用）**

```html
<button @click="toggle">切换显示</button>
```

```js
const app = new Vue({
  el: '#app',
  data: { isShow: true },
  methods: {
    toggle() {
      this.isShow = !this.isShow  // methods 中 this 指向 Vue 实例
    }
  }
})
```

**③ 方法调用传参**

```html
<!-- 传递自定义参数 -->
<button @click="buy(5)">可乐 5元</button>
<button @click="buy(10)">咖啡 10元</button>

<!-- 既传参数，又需要原生事件对象时，用 $event 占位 -->
<button @click="buy(5, $event)">可乐 5元</button>
```

```js
methods: {
  buy(price, e) {
    this.money -= price
    console.log(e)  // 原生事件对象（MouseEvent）
  }
}
```

> 📌 `methods` 中的函数内部 `this` 始终指向 **Vue 实例**，可以通过 `this.xxx` 访问 `data` 中的数据。

## 九、属性绑定指令

**为什么需要 v-bind？**

普通 HTML 属性是静态的，例如 `<img src="./a.png">` 中图片路径是固定的。
如果图片路径存放在 `data` 中，需要动态绑定，就要用 `v-bind`。

**语法：**
```html
<标签 v-bind:属性名="表达式"></标签>
<!-- 简写（推荐） -->
<标签 :属性名="表达式"></标签>
```

```html
<img :src="imgUrl" :title="msg" :alt="desc">
<a :href="link">跳转</a>
<input :disabled="isDisabled">
```

```js
const app = new Vue({
  el: '#app',
  data: {
    imgUrl: './imgs/banner.png',
    msg: '图片标题',
    isDisabled: true  // 为 true 时输入框被禁用
  }
})
```

> 📌 `v-bind` 可以绑定**任意 HTML 属性**，引号中的值会被当作 **JS 表达式**执行，而不是普通字符串。

## 十、列表渲染指令 v-for

**作用：** 基于数组或对象，循环渲染一组相同结构的元素。

**语法：**
```html
<li v-for="(item, index) in arr" :key="item.id">
  {{ index }} - {{ item.name }}
</li>
```

```html
<!-- 遍历对象 -->
<div v-for="(value, key, index) in obj">{{ key }}: {{ value }}</div>

<!-- 遍历数字（从1开始） -->
<p v-for="n in 10">第 {{ n }} 项</p>
```

**删除列表项案例（书架）：**

```html
<ul>
  <li v-for="(item, index) in booksList" :key="item.id">
    <span>{{ item.name }}</span>
    <span>{{ item.author }}</span>
    <button @click="del(item.id)">删除</button>
  </li>
</ul>
```

```js
methods: {
  del(id) {
    // filter 返回一个不包含该 id 的新数组，Vue 检测到数据变化后自动更新页面
    this.booksList = this.booksList.filter(item => item.id !== id)
  }
}
```

## 十一、v-for 中的 key

**语法：** `:key="唯一值"`

**作用：** 给列表项添加**唯一标识**，帮助 Vue 在数据变化时**正确地复用和排序 DOM 节点**，避免渲染错乱。

```html
<li v-for="(item, index) in list" :key="item.id">
  {{ item.name }}
</li>
```

**为什么不能用 index 作为 key？**

| 情况 | index 作为 key 的问题 |
|------|----------------------|
| 列表**中间删除**一项 | 后续所有项的 index 变化，Vue 误以为大量节点变了，导致错误复用 |
| 列表**头部插入**一项 | 所有项 index +1，渲染会错乱 |

**key 的使用规则：**

| 规则 | 说明 |
|------|------|
| 值类型 | 只能是**字符串**或**数字** |
| 唯一性 | 必须在当前列表中**唯一** |
| 推荐用 id | 稳定不变，不推荐用 index |

> ⚠️ 不加 key 或使用 index 作为 key，在列表有增删操作时可能导致**渲染错乱**和**性能下降**。

## 十二、双向绑定指令 v-model

**作用：** 将表单元素的值与 `data` 中的数据**双向同步**：
- **数据 → 视图**：data 变化时，表单内容自动更新
- **视图 → 数据**：用户输入时，data 自动更新

**语法：** `v-model="data中的变量"`

**本质：** `:value="变量"` + `@input="变量 = $event.target.value"` 的语法糖

```html
<div id="app">
  账户：<input type="text" v-model="username"><br>
  密码：<input type="password" v-model="password"><br>
  <button @click="login">登录</button>
  <button @click="reset">重置</button>
</div>
```

```js
const app = new Vue({
  el: '#app',
  data: {
    username: '',
    password: ''
  },
  methods: {
    login() {
      console.log(this.username, this.password)
    },
    reset() {
      // 直接修改数据，表单内容自动清空
      this.username = ''
      this.password = ''
    }
  }
})
```

![双向绑定](img/1681913125738.png)

**v-model 在各类表单元素的使用：**

| 表单元素 | 绑定属性 | 说明 |
|----------|----------|------|
| `input[text]` / `textarea` | `value` | 文本内容 |
| `input[checkbox]` | `checked` | 是否勾选（布尔值） |
| `input[radio]` | `checked` | 选中的值（需配合 `value`） |
| `select` | `value` | 选中项的 value |

```html
<!-- 复选框 -->
是否单身：<input type="checkbox" v-model="isSingle">

<!-- 单选框（需设置 value 才能区分选了哪个） -->
性别：
<input type="radio" v-model="gender" value="male">男
<input type="radio" v-model="gender" value="female">女

<!-- 下拉框 -->
城市：
<select v-model="city">
  <option value="bj">北京</option>
  <option value="sh">上海</option>
</select>
```

## 本篇重点总结

| 知识点 | 核心要点 |
|--------|----------|
| 插值表达式 `{{ }}` | 只能写表达式，只能用在标签内容中 |
| `v-show` vs `v-if` | 频繁切换用 `v-show`，不频繁用 `v-if` |
| `v-on` / `@` | methods 中 `this` 指向 Vue 实例 |
| `v-bind` / `:` | 动态绑定属性，值是 JS 表达式 |
| `v-for` + `:key` | key 必须唯一，推荐用 id 不用 index |
| `v-model` | 双向绑定，表单专用，是语法糖 |
