# Vue2 指令、计算属性与侦听器

---

## 一、指令修饰符

通过 `.` 后缀为指令添加特殊的处理行为，让代码更简洁。

> 💡 **理解**：修饰符就像给指令"加个帽子"，让它在原有功能基础上多做一件事。

### 按键修饰符

监听特定按键触发的事件：

```html
<!-- 仅在按下 Enter 键时触发 submit 方法 -->
<input @keyup.enter="submit" v-model="username">
```

常见按键修饰符：`.enter`、`.esc`、`.tab`、`.up`、`.down`

### v-model 修饰符

```html
<!-- .trim：自动去除输入内容的首尾空格 -->
<input v-model.trim="username">

<!-- .number：自动将输入值转换为数字类型（原本是字符串） -->
<input v-model.number="age">
```

### 事件修饰符

```html
<!-- .stop：阻止事件冒泡（防止子元素的事件触发父元素的事件） -->
<div @click="fatherFn">
  <div @click.stop="sonFn">点我只触发 son，不触发 father</div>
</div>

<!-- .prevent：阻止默认行为（如阻止 a 标签跳转） -->
<a @click.prevent href="http://www.baidu.com">点击不会跳转</a>

<!-- 修饰符可以链式组合 -->
<a @click.stop.prevent href="#">阻止冒泡 + 阻止默认行为</a>
```

**修饰符汇总：**

| 类型 | 修饰符 | 说明 |
|------|--------|------|
| 按键 | `.enter` `.esc` `.tab` | 监听特定按键 |
| v-model | `.trim` | 去除首尾空格 |
| v-model | `.number` | 转换为数字类型 |
| 事件 | `.stop` | 阻止事件冒泡 |
| 事件 | `.prevent` | 阻止默认行为 |
| 事件 | `.once` | 只触发一次 |

## 二、v-bind 样式控制增强

除了绑定普通属性，`v-bind` 在绑定 `class` 和 `style` 上有更强大的语法支持。

### 动态绑定 class（对象语法）

**适用场景：** 一个或多个类名来回切换（如高亮、选中效果）

```html
<!-- 键为类名，值为布尔值；true 则添加该类，false 则移除 -->
<div :class="{ active: isActive, disabled: isDisabled }"></div>
```

**Tab 栏切换案例：**

```html
<li v-for="(item, index) in list" :key="item.id">
  <!-- 当前点击的 index 与循环 index 相等时，添加 active 类 -->
  <a :class="{ active: activeIndex === index }" @click="activeIndex = index">
    {{ item.name }}
  </a>
</li>
```

> 💡 **理解**：对象语法就像开关，`:class="{ 类名: 布尔值 }"`，布尔值为 `true` 开灯（加类），`false` 关灯（去类）。

### 动态绑定 class（数组语法）

**适用场景：** 一次性添加多个类名

```html
<!-- 数组中所有类名都会添加到元素上，支持三元表达式 -->
<div :class="['box', 'pink', isBig ? 'big' : '']"></div>
```

### 动态绑定 style

```html
<!-- 对象语法：CSS 属性名用驼峰命名法 -->
<div :style="{ backgroundColor: bgColor, fontSize: size + 'px' }"></div>
```

**进度条案例：**

```html
<div class="inner" :style="{ width: percent + '%' }">
  <span>{{ percent }}%</span>
</div>
<button @click="percent = 25">25%</button>
<button @click="percent = 50">50%</button>
<button @click="percent = 75">75%</button>
<button @click="percent = 100">100%</button>
```

## 三、computed 计算属性

### 什么是计算属性？

**概念：** 基于**现有数据**计算出的新属性，只要依赖的数据发生变化，计算属性会**自动重新计算**。

> 💡 **类比理解**：就像 Excel 中的公式单元格，`=A1+B1` 会在 A1 或 B1 变化时自动更新结果，你不需要手动计算。

**语法：**

```js
const app = new Vue({
  el: '#app',
  data: {
    list: [
      { id: 1, name: '篮球', num: 3 },
      { id: 2, name: '玩具', num: 2 },
      { id: 3, name: '铅笔', num: 5 }
    ]
  },
  computed: {
    // 计算属性：写成函数，但在模板中当属性用（不加括号）
    totalCount() {
      return this.list.reduce((sum, item) => sum + item.num, 0)
    }
  }
})
```

```html
<!-- 模板中像普通属性一样使用，不加 () -->
<p>礼物总数：{{ totalCount }} 个</p>
```

![计算属性案例](img/1682039327858.png)

**注意事项：**

- `computed` 与 `data` 是**同级**配置项
- 计算属性是**属性**不是方法，模板中**不加 `()`**
- 计算属性名**不能**与 `data` 中的属性**同名**
- 内部 `this` 指向 **Vue 实例**

---

### computed vs methods 的核心区别

![计算属性vs方法](img/1682041822963.png)

**最关键的区别：有无缓存**

```js
computed: {
  // 有缓存：依赖数据（list）不变时，多次访问只计算一次
  totalCount() {
    console.log('计算属性执行了')  // 依赖不变时只打印一次
    return this.list.reduce((sum, item) => sum + item.num, 0)
  }
},
methods: {
  // 无缓存：每次调用都重新执行，即使数据没变
  getTotalCount() {
    console.log('方法执行了')  // 每次调用都打印
    return this.list.reduce((sum, item) => sum + item.num, 0)
  }
}
```

| 对比项 | computed 计算属性 | methods 方法 |
|--------|-----------------|-------------|
| 缓存 | ✅ 有缓存，性能更好 | ❌ 无缓存 |
| 调用方式 | 当属性使用 `{{ total }}` | 当方法调用 `{{ getTotal() }}` |
| 适用场景 | 基于数据推算出的结果 | 事件处理、业务逻辑 |

> 💡 **选择建议**：只要是"根据现有数据算出新数据"的需求，优先用计算属性，性能更好。

---

### 计算属性的完整写法（getter + setter）

默认的简写只支持**读取**。如果需要**修改**计算属性，必须使用完整的 `get` + `set` 写法。

```js
computed: {
  // 简写（只读，大多数情况用这个）
  // fullName() { return this.firstName + ' ' + this.lastName }

  // 完整写法（可读可写）
  fullName: {
    // 读取 fullName 时调用
    get() {
      return this.firstName + ' ' + this.lastName
    },
    // 修改 fullName 时调用（如 this.fullName = '刘 备'）
    set(newValue) {
      const parts = newValue.split(' ')
      this.firstName = parts[0]
      this.lastName = parts[1]
    }
  }
}
```

```html
姓：<input v-model="firstName">
名：<input v-model="lastName">
全名：<span>{{ fullName }}</span>
<button @click="fullName = '刘 备'">改名</button>
```

## 四、watch 侦听器

### 什么是侦听器？

**作用：** 监视某个数据的变化，当数据改变时，执行对应的业务逻辑或异步操作（如发请求、操作 DOM 等）。

> 💡 **computed vs watch 的选择**：
> - 需要**计算出新数据**用 `computed`
> - 数据变化时需要**执行某些操作**（尤其是异步操作）用 `watch`

### 简单写法（监视基本类型 / 对象的某个属性）

```js
watch: {
  // 监视 data 中的 words 属性
  words(newValue, oldValue) {
    console.log('words 变化了', newValue, oldValue)
    // 可在此发起异步请求
  },

  // 监视对象中的某个属性，用引号包裹（点语法）
  'obj.words'(newValue) {
    // 带防抖的异步请求（300ms 内重复触发只执行最后一次）
    clearTimeout(this.timer)
    this.timer = setTimeout(async () => {
      const res = await axios({ url: '...', params: { words: newValue } })
      this.result = res.data.data
    }, 300)
  }
}
```

### 完整写法（深度监视 / 立即执行）

当需要监视**整个对象**（对象内部任意属性变化都触发）时，必须使用完整写法并开启 `deep: true`。

```js
watch: {
  obj: {
    deep: true,       // 深度监视：对象内部任意属性变化都会触发 handler
    immediate: true,  // 立即执行：页面初始化时立即触发一次 handler（不需要等数据变化）
    handler(newValue) {
      clearTimeout(this.timer)
      this.timer = setTimeout(async () => {
        const res = await axios({
          url: 'https://applet-base-api-t.itheima.net/api/translate',
          params: newValue
        })
        this.result = res.data.data
      }, 300)
    }
  }
}
```

**两种写法对比：**

| 特性 | 简单写法 | 完整写法 |
|------|----------|----------|
| 语法复杂度 | 简单 | 较复杂 |
| 深度监视 | ❌ | ✅ `deep: true` |
| 立即执行 | ❌ | ✅ `immediate: true` |
| 适用场景 | 监视基础类型或对象某个属性 | 监视整个对象或需要立即执行 |

> ⚠️ 如果用简单写法监视一个对象（如 `obj(newVal){}`），修改对象内部的属性**不会触发**，必须加 `deep: true`。

## 本篇重点总结

| 知识点 | 核心要点 |
|--------|----------|
| 指令修饰符 | `.stop` 阻止冒泡、`.prevent` 阻止默认、`.trim` 去空格、`.number` 转数字 |

| `computed` 完整写法 | 需要修改计算属性时用 `get` + `set` |
| `watch` 简单写法 | 监视基础类型或对象某个属性 |
| `watch` 完整写法 | `deep: true` 深度监视，`immediate: true` 立即执行 |
