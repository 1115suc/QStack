# CSS盒子模型完整学习笔记

## 一、CSS三大特性深度解析

### 1.1 继承性 (Inheritance)

**定义**: 子元素默认继承父元素的某些样式属性

**核心概念**:
```
父元素样式 → 自动传递 → 子元素
```

**可继承的属性**:
```css
/* 文字相关 */
color
font-size
font-weight
font-style
font-family
font

/* 文本相关 */
text-align
text-indent
line-height

/* 列表相关 */
list-style

/* 其他 */
visibility
cursor
```

**代码示例**:
```html
<div class="parent">
    <p>我会继承父元素的样式</p>
    <span>我也会继承</span>
</div>
```

```css
.parent {
    color: red;
    font-size: 16px;
    font-family: "微软雅黑";
}

/* 子元素自动继承这些样式,无需单独设置 */
```

**继承失效的特殊情况**:
```html
<div style="color: red;">
    <a href="#">我不会继承红色</a>
    <h1>我的字号不会继承</h1>
</div>
```

**原因**: 浏览器默认样式优先级更高

**解决方法**:
```css
a {
    color: inherit;  /* 强制继承父元素颜色 */
}

h1 {
    font-size: inherit;  /* 强制继承父元素字号 */
}
```

### 1.2 层叠性 (Cascading)

**定义**: 样式的覆盖与叠加

**两种情况**:

#### 情况1: 不同属性 → 层叠叠加
```css
div {
    color: red;
    font-size: 20px;
    background-color: pink;
}
/* 三个属性都生效,共同作用 */
```

#### 情况2: 相同属性 → 层叠覆盖
```css
div {
    color: red;
    font-size: 20px;
    color: blue;  /* blue生效,覆盖red */
}
/* 最终: color为blue, font-size为20px */
```

**层叠规则**:
```
不同属性 → 叠加(共存)
相同属性 → 覆盖(后面生效)
前提条件 → 选择器优先级相同
```

### 1.3 优先级 (Specificity) ⭐⭐⭐

**定义**: 不同选择器具有不同的优先级,优先级高的样式会覆盖优先级低的样式

#### 1.3.1 基础优先级公式

```
继承 < 通配符选择器 < 标签选择器 < 类选择器 < id选择器 < 行内样式 < !important
```

**权重值对照表**:

| 选择器 | 权重值 | 示例 |
|--------|--------|------|
| **继承** | 0,0,0,0 | `继承自父元素` |
| **通配符** | 0,0,0,0 | `*` |
| **标签选择器** | 0,0,0,1 | `div`、`p` |
| **类选择器** | 0,0,1,0 | `.box` |
| **id选择器** | 0,1,0,0 | `#header` |
| **行内样式** | 1,0,0,0 | `style=""` |
| **!important** | ∞ | `color: red !important` |

**代码示例**:
```html
<div class="box" id="container" style="color: yellow;">文字</div>
```

```css
/* 继承: 0,0,0,0 */
body {
    color: gray;
}

/* 通配符: 0,0,0,0 */
* {
    color: pink;
}

/* 标签选择器: 0,0,0,1 */
div {
    color: red;
}

/* 类选择器: 0,0,1,0 */
.box {
    color: blue;
}

/* id选择器: 0,1,0,0 */
#container {
    color: green;
}

/* 行内样式: 1,0,0,0 */
/* style="color: yellow;" */

/* !important: ∞ (最高) */
div {
    color: pink !important;
}

/* 最终生效: pink (!important最高) */
```

#### 1.3.2 !important 使用规则

**语法**:
```css
选择器 {
    属性: 值 !important;
}
```

**代码示例**:
```css
p {
    color: red !important;  /* ✅ 正确 */
}

p {
    color: !important red;  /* ❌ 错误 */
}
```

**注意事项**:

1. **位置**: 写在属性值后面,分号前面
```css
color: red !important;  /* ✅ 正确 */
color: red; !important  /* ❌ 错误 */
```

2. **继承失效**: `!important`不能提升继承的优先级
```css
/* ❌ 无效 */
.parent {
    color: red !important;
}

/* 子元素的直接选择器优先级更高 */
.child {
    color: blue;  /* blue生效 */
}
```

3. **实际开发**: 不推荐使用
```css
/* ❌ 不推荐 */
p {
    color: red !important;
}

/* ✅ 推荐: 使用更具体的选择器 */
.container p {
    color: red;
}
```

#### 1.3.3 权重叠加计算 ⭐⭐⭐⭐⭐

**定义**: 复合选择器的优先级需要通过权重叠加计算

**计算公式**:
```
(行内样式, id选择器, 类选择器, 标签选择器)
```

**每一级的含义**:
- **第一级**: 行内样式的个数
- **第二级**: id选择器的个数
- **第三级**: 类选择器/伪类选择器/属性选择器的个数
- **第四级**: 标签选择器/伪元素选择器的个数

**重要规则**:
```
⚠️ 每一级之间不存在进位
⚠️ 10个类选择器 ≠ 1个id选择器
⚠️ 10个标签选择器 ≠ 1个类选择器
```

**计算示例**:

```css
/* 示例1 */
div {
    color: red;
}
/* 权重: (0, 0, 0, 1) */

/* 示例2 */
.box {
    color: blue;
}
/* 权重: (0, 0, 1, 0) */

/* 示例3 */
#header {
    color: green;
}
/* 权重: (0, 1, 0, 0) */

/* 示例4 */
div.box {
    color: yellow;
}
/* 权重: (0, 0, 1, 1) 
   = 0个行内 + 0个id + 1个类 + 1个标签 */

/* 示例5 */
.nav .menu li {
    color: pink;
}
/* 权重: (0, 0, 2, 1)
   = 0个行内 + 0个id + 2个类 + 1个标签 */

/* 示例6 */
#header .nav a {
    color: orange;
}
/* 权重: (0, 1, 1, 1)
   = 0个行内 + 1个id + 1个类 + 1个标签 */
```

**比较规则**:

```
1. 从左往右逐级比较
2. 先比较第一级,谁大谁优先
3. 第一级相同,比较第二级
4. 以此类推
5. 所有级别都相同,看层叠性(后面的生效)
```

**实战案例**:

```html
<div class="father">
    <p class="son">文字</p>
</div>
```

**案例1**: 基础对比
```css
/* 选择器1 */
div p {
    color: red;
}
/* 权重: (0, 0, 0, 2) */

/* 选择器2 */
.son {
    color: blue;
}
/* 权重: (0, 0, 1, 0) */

/* 比较:
   第一级: 0 = 0
   第二级: 0 = 0  
   第三级: 0 < 1 ✓
   结果: blue生效
*/
```

**案例2**: 复杂对比
```css
/* 选择器1 */
.father .son {
    color: red;
}
/* 权重: (0, 0, 2, 0) */

/* 选择器2 */
div .son {
    color: blue;
}
/* 权重: (0, 0, 1, 1) */

/* 比较:
   第一级: 0 = 0
   第二级: 0 = 0
   第三级: 2 > 1 ✓
   结果: red生效
*/
```

**案例3**: 不进位证明
```css
/* 选择器1: 10个类选择器 */
.a.b.c.d.e.f.g.h.i.j {
    color: red;
}
/* 权重: (0, 0, 10, 0) */

/* 选择器2: 1个id选择器 */
#header {
    color: blue;
}
/* 权重: (0, 1, 0, 0) */

/* 比较:
   第一级: 0 = 0
   第二级: 0 < 1 ✓
   结果: blue生效 (id优先级更高)
   证明: 10个类 ≠ 1个id
*/
```

**案例4**: 继承的优先级
```html
<div id="father">
    <p>文字</p>
</div>
```

```css
/* 选择器1: 通过id选择父元素 */
#father {
    color: red;
}
/* p继承的权重: (0, 0, 0, 0) - 继承最低 */

/* 选择器2: 直接选择p */
p {
    color: blue;
}
/* p的权重: (0, 0, 0, 1) */

/* 结果: blue生效
   原因: 直接选择 > 继承
*/
```

**案例5**: 层叠性
```css
/* 选择器1 */
.box {
    color: red;
}
/* 权重: (0, 0, 1, 0) */

/* 选择器2 */
.container {
    color: blue;
}
/* 权重: (0, 0, 1, 0) */

/* 权重相同,看层叠性
   结果: blue生效(后面的覆盖前面的)
*/
```

#### 1.3.4 权重计算练习题

**练习1**:
```html
<div class="nav">
    <ul>
        <li><a href="#">链接</a></li>
    </ul>
</div>
```

```css
/* 选择器1 */

    color: red;
}
/* 权重: (0, 0, 1, 3) */

/* 选择器2 */

    color: blue;

/* 答案: red生效 */
```

**练习2**:
```html
<div id="header" class="box">
    <p>文字</p>
</div>
```

```css
/* 选择器1 */
#header p {
    color: red;
}
/* 权重: (0, 1, 0, 1) */

/* 选择器2 */
.box p {
    color: blue;
}
/* 权重: (0, 0, 1, 1) */

/* 答案: red生效 */
```

**练习3**:
```html
<div class="father">
    <div class="son">
        <p>文字</p>
    </div>
</div>
```

```css
/* 选择器1 */
.father .son p {
    color: red;
}
/* 权重: (0, 0, 2, 1) */

/* 选择器2 */
div div p {
    color: blue;
}
/* 权重: (0, 0, 0, 3) */

/* 答案: red生效 */
```

#### 1.3.5 权重计算解题步骤

```css
步骤1: 判断选择器是否直接选中标签
  └─ 是继承 → 优先级最低 → 直接pass
  └─ 直接选中 → 进入步骤2

步骤2: 统计权重
  └─ 统计(行内, id, 类, 标签)的个数

步骤3: 从左往右比较
  └─ 先比第一级,再比第二级...

步骤4: 所有级别相同
  └─ 看层叠性(后面的生效)
```

**实战技巧**:
```css
✅ 选择器要精准,避免冲突
✅ 尽量使用类选择器
✅ 避免使用!important
✅ 避免过深的嵌套
```

## 三、盒子模型核心知识

### 3.1 盒子模型概念

#### 3.1.1 什么是盒子

**定义**: 页面中的每个标签都可以看作是一个"盒子"

**为什么要有盒子概念**:
- 方便页面布局
- 统一理解元素
- 便于计算尺寸
- 更好地控制间距

**可视化理解**:
```css
┌─────────────────────────────────┐
│        外边距 margin             │
│  ┌───────────────────────────┐  │
│  │     边框 border            │  │
│  │  ┌─────────────────────┐  │  │
│  │  │  内边距 padding      │  │  │
│  │  │  ┌───────────────┐  │  │  │
│  │  │  │  内容 content │  │  │  │
│  │  │  └───────────────┘  │  │  │
│  │  └─────────────────────┘  │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

#### 3.1.2 盒子模型的组成

**四个部分**:

1. **内容区域** (content)
    - 作用: 显示文字、图片等内容
    - 属性: `width`、`height`

2. **内边距区域** (padding)
    - 作用: 内容与边框之间的距离
    - 属性: `padding`

3. **边框区域** (border)
    - 作用: 盒子的边框线
    - 属性: `border`

4. **外边距区域** (margin)
    - 作用: 盒子与盒子之间的距离
    - 属性: `margin`

**记忆方法**:
```css
联想现实中的快递盒:
  内容 = 商品本身
  内边距 = 填充物(泡沫)
  边框 = 纸盒
  外边距 = 盒子之间的距离
```

### 3.2 内容区域 (Content)

**作用**: 设置盒子内容区域的大小

**属性**:
```css
width: 宽度;
height: 高度;
```

**取值**: 数字 + px

**代码示例**:
```css
.box {
    width: 300px;   /* 内容区域宽度 */
    height: 200px;  /* 内容区域高度 */
}
```

**注意事项**:
```
⚠️ width和height默认设置的是内容区域的大小
⚠️ 不包括padding、border、margin
⚠️ 添加padding和border会撑大盒子
```

**实际应用**:
```css
/* 固定尺寸盒子 */
.card {
    width: 300px;
    height: 400px;
    background-color: #fff;
}

/* 百分比宽度 */
.container {
    width: 100%;
    height: 500px;
}
```

### 3.3 边框 (Border)

#### 3.3.1 边框单个属性

**三个属性**:

```css
border-width: 边框粗细;
border-style: 边框样式;
border-color: 边框颜色;
```

**border-width** (边框粗细):
```css
border-width: 1px;
border-width: 5px;
border-width: thin;   /* 细线 */
border-width: medium; /* 中等 */
border-width: thick;  /* 粗线 */
```

**border-style** (边框样式):

| 值 | 效果 | 说明 |
|----|------|------|
| **solid** | 实线 | 最常用 |
| **dashed** | 虚线 | 常用 |
| **dotted** | 点线 | 常用 |
| **double** | 双实线 | 少用 |
| **none** | 无边框 | 默认值 |

```css
border-style: solid;  /* 实线(最常用) */
border-style: dashed; /* 虚线 */
border-style: dotted; /* 点线 */
border-style: double; /* 双线 */
border-style: none;   /* 无边框 */
```

**border-color** (边框颜色):
```css
border-color: red;
border-color: #ff6700;
border-color: rgb(255, 0, 0);
border-color: rgba(0, 0, 0, 0.5);
```

**代码示例**:
```css
.box {
    width: 200px;
    height: 200px;
    border-width: 5px;
    border-style: solid;
    border-color: red;
}
```

#### 3.3.2 边框复合属性 ⭐推荐

**语法**:
```css
border: width style color;
```

**Emmet**: `bd` + `Tab`

**代码示例**:
```css
/* 完整写法 */
border: 10px solid red;

/* 等同于 */
border-width: 10px;
border-style: solid;
border-color: red;
```

**常用边框**:
```css
/* 1像素实线黑色边框(最常用) */

/* 2像素虚线灰色边框 */

/* 5像素实线蓝色边框 */

border: none;
```

#### 3.3.3 单方向设置边框

**属性名**:
```css
border-top: 上边框;

```

**代码示例**:

```css
    border-left: 3px solid #007bff;

/* 上下边框 */
.box {
    border-top: 1px solid #e5e5e5;
    border-bottom: 1px solid #e5e5e5;

```

**实际应用**:

**场景1: 导航栏下划线**
```css
.nav {
    border-bottom: 2px solid #ff6700;
}
```

**场景2: 卡片边框**
```css

```

**场景3: 左侧强调边框**
```css

    padding-left: 15px;

```

**场景4: 表格单元格边框**
```css

    border: 1px solid #ddd;
}

    border: 1px solid #ddd;
}
```

#### 3.3.4 盒子实际大小计算(初级)

**问题**: 设置border会撑大盒子

**案例**:
```css
/* 需求: 盒子400×400,边框10px实线黑色 */

/* ❌ 错误写法 */
.box {
    width: 400px;
    height: 400px;
    background-color: green;
    border: 10px solid black;
}
/* 实际大小: 420×420 (被撑大了) */
```

**计算公式**:
```css
盒子宽度 = 左边框 + 内容宽度 + 右边框
盒子高度 = 上边框 + 内容高度 + 下边框
```

**计算过程**:
```
盒子宽度 = 10px + 400px + 10px = 420px
盒子高度 = 10px + 400px + 10px = 420px
```

**解决方法: 手动内减**
```css
/* ✅ 正确写法 */
.box {
    width: 380px;   /* 400 - 10×2 = 380 */
    height: 380px;  /* 400 - 10×2 = 380 */
    background-color: green;
    border: 10px solid black;
}
/* 实际大小: 400×400 ✓ */
```

**计算步骤**:
```css
1. 确定目标大小: 400×400
2. 计算边框占用: 10×2 = 20px
3. 内容区域 = 目标大小 - 边框
   width = 400 - 20 = 380px
   height = 400 - 20 = 380px
```

### 3.4 内边距 (Padding)

#### 3.4.1 内边距的作用

**定义**: 设置边框与内容区域之间的距离

**效果**:
```css
增加内边距 → 内容离边框更远 → 盒子被撑大
```

#### 3.4.2 内边距取值方式

**属性名**: `padding`

**四种取值方式**:

| 取值个数 | 示例 | 含义 |
|---------|------|------|
| **1个值** | `padding: 10px;` | 上下左右都是10px |
| **2个值** | `padding: 10px 20px;` | 上下10px,左右20px |
| **3个值** | `padding: 10px 20px 30px;` | 上10px,左右20px,下30px |
| **4个值** | `padding: 10px 20px 30px 40px;` | 上右下左(顺时针) |

**记忆规则**:
```css
从上开始赋值,顺时针方向
缺少的看对面
```

**代码示例**:

**1个值**:
```css
padding: 20px;

/* 等同于 */
padding-top: 20px;
padding-right: 20px;
padding-bottom: 20px;
padding-left: 20px;
```

**2个值**:
```css
padding: 10px 20px;

/* 等同于 */
padding-top: 10px;     /* 上 */
padding-right: 20px;   /* 右 */
padding-bottom: 10px;  /* 下(看对面的上) */
padding-left: 20px;    /* 左(看对面的右) */
```

**3个值**:
```css
padding: 10px 20px 30px;

/* 等同于 */
padding-top: 10px;     /* 上 */
padding-right: 20px;   /* 右 */
padding-bottom: 30px;  /* 下 */
padding-left: 20px;    /* 左(看对面的右) */
```

**4个值**:
```css
padding: 10px 20px 30px 40px;

/* 等同于 */
padding-top: 10px;     /* 上 */
padding-right: 20px;   /* 右 */
padding-bottom: 30px;  /* 下 */
padding-left: 40px;    /* 左 */
```

**顺时针记忆**:
```css
     上(top)
      ↑
左 ← □ → 右
(left)  (right)
      ↓
    下(bottom)

顺序: 上 → 右 → 下 → 左 (顺时针)
```

#### 3.4.3 单方向设置内边距

**属性名**:
```css
padding-top: 上内边距;
padding-right: 右内边距;
padding-bottom: 下内边距;
padding-left: 左内边距;
```

**代码示例**:
```css
/* 只设置左内边距 */
.box {
    padding-left: 20px;
}

/* 只设置上下内边距 */
.box {
    padding-top: 15px;
    padding-bottom: 15px;
}

/* 组合使用 */
.box {
    padding-top: 20px;
    padding-left: 30px;
}
```

#### 3.4.4 盒子实际大小计算(终极)

**问题**: padding和border都会撑大盒子

**案例**:
```css
/* 需求: 盒子300×300,边框10px,内边距20px */

/* ❌ 错误写法 */
.box {
    width: 300px;
    height: 300px;
    background-color: pink;
    border: 10px solid black;
    padding: 20px;
}
/* 实际大小: 360×360 (被撑大了) */
```

**终极计算公式**:
```css
盒子宽度 = 左边框 + 左padding + 内容宽度 + 右padding + 右边框
盒子高度 = 上边框 + 上padding + 内容高度 + 下padding + 下边框
```

**计算过程**:
```css
盒子宽度 = 10 + 20 + 300 + 20 + 10 = 360px
盒子高度 = 10 + 20 + 300 + 20 + 10 = 360px
```

**解决方法1: 手动内减**
```css
/* ✅ 正确写法 */
.box {
    width: 240px;   /* 300 - (10+20)×2 = 240 */
    height: 240px;  /* 300 - (10+20)×2 = 240 */
    background-color: pink;
    border: 10px solid black;
    padding: 20px;
}
/* 实际大小: 300×300 ✓ */
```

**计算步骤**:
```css
1. 目标大小: 300×300
2. 边框: 10×2 = 20px
3. 内边距: 20×2 = 40px
4. 总共占用: 20 + 40 = 60px
5. 内容区域 = 300 - 60 = 240px
```

**解决方法2: CSS3自动内减** ⭐⭐⭐推荐

**语法**:
```css
box-sizing: border-box;
```

**代码示例**:
```css
/* ✅ 推荐写法 */
.box {
    width: 300px;
    height: 300px;
    background-color: pink;
    border: 10px solid black;
    padding: 20px;
    box-sizing: border-box;  /* 自动内减 */
}
/* 实际大小: 300×300 ✓ */
/* 浏览器自动计算内容区域 */
```

**box-sizing属性**:

| 值 | 说明 | 计算方式 |
|----|------|----------|
| **content-box** | 默认值 | width/height = 内容区域 |
| **border-box** | CSS3 | width/height = 内容+padding+border |

**对比示例**:
```css
/* content-box (默认) */
.box1 {
    width: 300px;
    padding: 20px;
    border: 10px solid #000;
    box-sizing: content-box;
}
/* 实际宽度 = 300 + 20×2 + 10×2 = 360px */

/* border-box (推荐) */
.box2 {
    width: 300px;
    padding: 20px;
    border: 10px solid #000;
    box-sizing: border-box;
}
/* 实际宽度 = 300px (自动内减) */
```

**全局设置**:
```css
/* 推荐: 全局设置border-box */
* {
    box-sizing: border-box;
}

/* 或者 */
html {
    box-sizing: border-box;
}

*, *::before, *::after {
    box-sizing: inherit;
}
```

**优势**:
- ✅ 无需手动计算
- ✅ 更符合直觉
- ✅ 便于维护
- ✅ 响应式布局友好

#### 3.4.5 不会撑大盒子的特殊情况

**规则**: 块级元素没有设置width时,padding不会撑大盒子

**案例**:
```html
<div class="father">
    <div class="son">子盒子</div>
</div>
```

```css
.father {
    width: 300px;
    height: 300px;
    background-color: pink;
}

.son {
    /* 不设置width,默认是父盒子的宽度 */
    height: 100px;
    background-color: skyblue;
    padding: 0 20px;  /* 左右内边距不会撑大盒子 */
}
/* 子盒子宽度仍然是300px */
```

**原理**:
```css
子盒子没有width → 默认width = 100%
设置padding → 自动从100%中减去padding
结果 → 盒子不会撑大
```

**注意**:
```css
⚠️ 只适用于块级元素
⚠️ 只适用于左右padding
⚠️ 上下padding仍会撑大盒子高度
⚠️ 有width时,padding会撑大盒子
```

### 3.5 外边距 (Margin)

#### 3.5.1 外边距的作用

**定义**: 设置边框以外,盒子与盒子之间的距离

**效果**:
```css
增加外边距 → 盒子之间距离增大 → 不会撑大盒子
```

#### 3.5.2 外边距取值方式

**属性名**: `margin`

**取值方式**: 与padding完全相同

| 取值个数 | 示例 | 含义 |
|---------|------|------|
| **1个值** | `margin: 10px;` | 上下左右都是10px |
| **2个值** | `margin: 10px 20px;` | 上下10px,左右20px |
| **3个值** | `margin: 10px 20px 30px;` | 上10px,左右20px,下30px |
| **4个值** | `margin: 10px 20px 30px 40px;` | 上右下左(顺时针) |

**代码示例**:
```css
/* 1个值 */
margin: 20px;  /* 上下左右都是20px */

/* 2个值(最常用) */
margin: 20px 30px;  /* 上下20px,左右30px */

/* 3个值 */
margin: 10px 20px 30px;  /* 上10px,左右20px,下30px */

/* 4个值 */
margin: 10px 20px 30px 40px;  /* 上右下左 */
```

#### 3.5.3 单方向设置外边距

**属性名**:
```css
margin-top: 上外边距;
margin-right: 右外边距;
margin-bottom: 下外边距;
margin-left: 左外边距;
```

**代码示例**:
```css
/* 只设置下外边距 */
.box {
    margin-bottom: 20px;
}

/* 只设置左外边距 */
.box {
    margin-left: 30px;
}

/* 组合使用 */
.box {
    margin-top: 20px;
    margin-bottom: 30px;
}
```

#### 3.5.4 margin的典型应用

**应用1: 块级元素水平居中** ⭐⭐⭐
```css
.box {
    width: 800px;
    margin: 0 auto;  /* 上下0,左右自动 */
}
```

**应用2: 清除默认边距**
```css
/* 方法一: 通配符(简单粗暴) */
* {
    margin: 0;
    padding: 0;
}

/* 方法二: 具体标签(推荐) */
body, h1, h2, h3, h4, h5, h6, p, ul, ol, li {
    margin: 0;
    padding: 0;
}

/* 方法三: normalize.css */
```

**应用3: 元素间距**
```css
/* 列表项间距 */
li {
    margin-bottom: 15px;
}

/* 段落间距 */
p {
    margin-bottom: 20px;
}

/* 卡片间距 */
.card {
    margin-bottom: 30px;
}
```

#### 3.5.5 清除默认内外边距

**问题**: 浏览器默认给某些标签设置了margin和padding

**默认有margin/padding的标签**:
```css
body → margin: 8px
p → margin: 16px 0
h1~h6 → margin上下
ul/ol → margin上下 + padding-left: 40px
...
```

**解决方案**:

**方案1: 通配符(简单但性能差)**
```css
* {
    margin: 0;
    padding: 0;
}
```

**方案2: 列出具体标签(推荐)** ⭐
```css
body,
h1, h2, h3, h4, h5, h6,
p,
ul, ol, li,
dl, dt, dd {
    margin: 0;
    padding: 0;
}

ul, ol {
    list-style: none;  /* 去除列表符号 */
}
```

**方案3: CSS Reset(完整重置)**
```css
/* 淘宝的CSS Reset */
body, h1, h2, h3, h4, h5, h6, hr, p, blockquote,
dl, dt, dd, ul, ol, li,
pre, form, fieldset, legend, button, input, textarea,
th, td {
    margin: 0;
    padding: 0;
}

body, button, input, select, textarea {
    font: 12px/1.5 "Microsoft YaHei", tahoma, arial, "Hiragino Sans GB", "\5b8b\4f53", sans-serif;
}

h1, h2, h3, h4, h5, h6 {
    font-size: 100%;
}

ul, ol {
    list-style: none;
}

a {
    text-decoration: none;
}

...
```

**方案4: Normalize.css(现代化重置)** ⭐⭐⭐
```html
<!-- 引入normalize.css -->
<link rel="stylesheet" href="normalize.css">
```

**对比**:
| 方案 | 优点 | 缺点 | 使用场景 |
|------|------|------|----------|
| 通配符 | 简单 | 性能差 | 小项目 |
| 具体标签 | 性能好 | 需维护 | 中型项目 |
| CSS Reset | 完整 | 代码多 | 兼容老浏览器 |
| Normalize | 现代 | 需引入 | 推荐★★★ |

#### 3.5.6 外边距的问题

##### 问题1: 外边距正常情况(左右margin叠加)

**场景**: 水平布局的盒子

**现象**: 左右margin正常叠加

**代码示例**:
```html
<div class="box1">盒子1</div>
<div class="box2">盒子2</div>
```

```css
.box1 {
    display: inline-block;
    width: 100px;
    height: 100px;
    background-color: pink;
    margin-right: 30px;
}

.box2 {
    display: inline-block;
    width: 100px;
    height: 100px;
    background-color: skyblue;
    margin-left: 20px;
}

/* 两个盒子之间的距离 = 30 + 20 = 50px */
```

**结论**: 水平方向margin正常叠加 ✓

##### 问题2: 外边距折叠-合并现象 ⭐⭐⭐

**场景**: 垂直布局的块级元素

**现象**: 上下margin会合并,取最大值

**代码示例**:
```html
<div class="box1">盒子1</div>
<div class="box2">盒子2</div>
```

```css
.box1 {
    width: 100px;
    height: 100px;
    background-color: pink;
    margin-bottom: 50px;
}

.box2 {
    width: 100px;
    height: 100px;
    background-color: skyblue;
    margin-top: 30px;
}

/* 两个盒子之间的距离 = max(50, 30) = 50px */
/* 不是50 + 30 = 80px */
```

**规则**:
```
垂直方向相邻的margin会合并
取其中的最大值
```

**解决方法**: 避免即可
```css
/* ✅ 只给一个盒子设置margin */
.box1 {
    margin-bottom: 50px;
}

/* 或者 */
.box2 {
    margin-top: 50px;
}
```

##### 问题3: 外边距折叠-塌陷现象 ⭐⭐⭐⭐⭐

**场景**: 嵌套的块级元素,子元素的margin-top

**现象**: 子元素的margin-top会作用在父元素上

**代码示例**:
```html
<div class="father">
    <div class="son">子盒子</div>
</div>
```

```css
.father {
    width: 300px;
    height: 300px;
    background-color: pink;
}

.son {
    width: 100px;
    height: 100px;
    background-color: skyblue;
    margin-top: 50px;  /* 想让子盒子向下移动 */
}

/* ❌ 结果: 父盒子和子盒子一起向下移动了50px */
```

**问题原因**:
```css
父盒子没有border-top
父盒子没有padding-top
↓
子盒子的margin-top会传递给父盒子
↓
父盒子整体向下移动
```

**解决方案**:

**方案1: 给父元素设置border-top** (不推荐)
```css
.father {
    width: 300px;
    height: 300px;
    background-color: pink;
    border-top: 1px solid transparent;  /* 透明边框 */
}
```

**方案2: 给父元素设置padding-top** (不推荐)
```css
.father {
    width: 300px;
    height: 300px;
    background-color: pink;
    padding-top: 1px;
}
```

**方案3: 给父元素设置overflow: hidden** ⭐推荐
```css
.father {
    width: 300px;
    height: 300px;
    background-color: pink;
    overflow: hidden;  /* 触发BFC */
}
```

**方案4: 转换成行内块元素**
```css
.father {
    display: inline-block;
    width: 300px;
    height: 300px;
    background-color: pink;
}
```

**方案5: 设置浮动** (后续学习)
```css
.father {
    float: left;
    width: 300px;
    height: 300px;
    background-color: pink;
}
```

**方案对比**:

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| border-top | 简单 | 增加边框 | ⭐ |
| padding-top | 简单 | 改变尺寸 | ⭐ |
| overflow: hidden | 不影响布局 | 溢出内容隐藏 | ⭐⭐⭐⭐⭐ |
| inline-block | 彻底解决 | 改变显示模式 | ⭐⭐⭐ |
| float | 彻底解决 | 脱离文档流 | ⭐⭐⭐ |

**推荐方案**: `overflow: hidden` (最简单有效)

**更好的方案**: 用父元素的padding代替子元素的margin
```css
/* ❌ 不推荐 */
.father {
    width: 300px;
    height: 300px;
}

.son {
    margin-top: 50px;  /* 会塌陷 */
}

/* ✅ 推荐 */
.father {
    width: 300px;
    height: 300px;
    padding-top: 50px;  /* 用padding代替margin */
    box-sizing: border-box;
}

.son {
    /* 不设置margin-top */
}
```

#### 3.5.7 行内元素的margin和padding

**规则**:
```css
水平方向(左右) → margin和padding都有效 ✓
垂直方向(上下) → margin和padding布局无效 ✗
```

**代码示例**:
```html
<span class="text">行内元素</span>
```

```css
.text {
    /* 水平方向有效 */
    margin-left: 20px;     /* ✓ 生效 */
    margin-right: 20px;    /* ✓ 生效 */
    padding-left: 10px;    /* ✓ 生效 */
    padding-right: 10px;   /* ✓ 生效 */
    
    /* 垂直方向无效 */
    margin-top: 20px;      /* ✗ 无效(不占位) */
    margin-bottom: 20px;   /* ✗ 无效(不占位) */
    padding-top: 10px;     /* ⚠️ 有显示但不占位 */
    padding-bottom: 10px;  /* ⚠️ 有显示但不占位 */
}
```

**解决方法**: 转换成块级或行内块元素
```css
.text {
    display: inline-block;  /* 或 block */
    margin: 20px;
    padding: 10px;
}
```

### 4.3 外边距问题汇总

```css
场景: 垂直相邻的块级元素
```

#### 4.3.2 外边距塌陷 ⭐⭐⭐
```css
场景: 嵌套的块级元素
现象: 子元素margin-top作用在父元素
解决: overflow: hidden (推荐)
```

#### 4.3.3 行内元素margin/padding
```css
水平方向: 有效 ✓
垂直方向: 无效 ✗
解决: 转换成块级或行内块
```

### 4.8 常见错误与解决

#### 错误1: 盒子被撑大
```css
/* ❌ 问题 */
.box {
    width: 300px;
    padding: 20px;
}

/* ✅ 解决 */
.box {
    width: 300px;
    padding: 20px;
    box-sizing: border-box;
}
```

#### 错误2: margin不生效
```css
/* ❌ 问题: 行内元素垂直margin无效 */
span {
    margin-top: 20px;
}

/* ✅ 解决 */
span {
    display: inline-block;
    margin-top: 20px;
}
```

#### 错误3: 子盒子塌陷
```css
/* ❌ 问题 */
.father {
    width: 300px;
    height: 300px;
}

.son {
    margin-top: 50px;  /* 塌陷 */
}

/* ✅ 解决 */
.father {
    width: 300px;
    height: 300px;
    overflow: hidden;
}
```

#### 错误4: 居中不生效
```css
/* ❌ 问题: 没有宽度 */
.box {
    margin: 0 auto;
}

/* ✅ 解决 */
.box {
    width: 800px;
    margin: 0 auto;
}
```
