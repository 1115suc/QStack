## 一、TypeScript 简介

### 1.1 什么是 TypeScript？

TypeScript（简称 TS）是由微软开发的开源编程语言，是 **JavaScript 的超集**，在 JS 的基础上添加了：

- **静态类型系统**：编译时发现类型错误，而不是运行时
- **ES6+ 新特性支持**：类、模块、箭头函数等
- **IDE 智能提示**：代码补全、重构更友好

```
TypeScript → 编译（tsc） → JavaScript → 运行在浏览器 / Node.js
```

### 1.2 为什么使用 TypeScript？

| 对比维度       | JavaScript         | TypeScript              |
| -------------- | ------------------ | ----------------------- |
| 类型检查       | 运行时             | 编译时                  |
| IDE 提示       | 较弱               | 强（类型推断 + 补全）   |
| 大型项目维护   | 容易出错           | 可读性、安全性更高      |
| 学习成本       | 低                 | 需额外学习类型系统      |

### 1.3 安装与配置

```bash
# 全局安装
npm install -g typescript

# 初始化配置文件
tsc --init

# 编译单文件
tsc index.ts

# 实时监听
tsc --watch

npm install -g ts-node
# 节省自己手动编译的过程
ts-node index.ts
```

**tsconfig.json 常用配置：**

```json
{
  "compilerOptions": {
    "target": "ES6",           // 编译目标版本
    "module": "commonjs",      // 模块规范
    "strict": true,            // 开启严格模式（推荐）
    "outDir": "./dist",        // 输出目录
    "rootDir": "./src",        // 源码目录
    "sourceMap": true,         // 生成 source map
    "esModuleInterop": true    // 兼容 CommonJS/ESM 混用
  }
}
```

## 二、基础类型

### 2.1 原始类型

```typescript
let isDone: boolean = false;
let count: number = 42;
let name: string = "TypeScript";
let n: null = null;
let u: undefined = undefined;

// ES2020 BigInt
let big: bigint = 100n;

// Symbol
let sym: symbol = Symbol("key");
```

### 2.2 数组类型

```typescript
// 方式一：类型[]
let list1: number[] = [1, 2, 3];

// 方式二：泛型 Array<类型>
let list2: Array<string> = ["a", "b", "c"];

// 只读数组（防止修改）
let readonlyArr: ReadonlyArray<number> = [1, 2, 3];
// readonlyArr.push(4); // ❌ 报错
```

### 2.3 元组（Tuple）

元组是**固定长度、固定类型**的数组：

```typescript
let point: [number, number] = [10, 20];
let user: [string, number] = ["Alice", 25];

// 访问元素
console.log(user[0]); // "Alice"
console.log(user[1]); // 25

// 可选元素（TS 4.0+）
let optionalTuple: [string, number?] = ["hello"];
```

### 2.4 枚举（Enum）

```typescript
// 数字枚举（默认从 0 开始）
enum Direction {
  Up,    // 0
  Down,  // 1
  Left,  // 2
  Right, // 3
}
console.log(Direction.Up);    // 0
console.log(Direction[0]);    // "Up"（反向映射）

// 字符串枚举
enum Status {
  Pending = "PENDING",
  Active = "ACTIVE",
  Inactive = "INACTIVE",
}

// 常量枚举（编译后内联，不生成对象）
const enum Color {
  Red,
  Green,
  Blue,
}
let c = Color.Red; // 编译为：let c = 0;
```

### 2.5 特殊类型

```typescript
// any：关闭类型检查（尽量避免）
let anything: any = 42;
anything = "hello";
anything = true;

// unknown：比 any 更安全，使用前需要类型收窄
let val: unknown = "hello";
if (typeof val === "string") {
  console.log(val.toUpperCase()); // 安全使用
}

// void：函数无返回值
function logMessage(msg: string): void {
  console.log(msg);
}

// never：永远不会有返回值（抛出错误/死循环）
function throwError(msg: string): never {
  throw new Error(msg);
}

// object：非原始类型
let obj: object = { key: "value" };
```

## 三、类型注解与类型推断

### 3.1 类型注解

**显式**声明变量、参数、返回值的类型：

```typescript
let age: number = 18;
function greet(name: string): string {
  return `Hello, ${name}`;
}
```

### 3.2 类型推断

TypeScript 会根据赋值**自动推断**类型，不需要每次都手写注解：

```typescript
let message = "hello"; // 推断为 string
let num = 42;           // 推断为 number

// 最佳通用类型推断
let arr = [1, "hello", true]; // 推断为 (number | string | boolean)[]
```

> **原则**：能让 TS 自动推断的，就不用手动写注解；推断不准确时再手动标注。

### 3.3 类型断言

当你比 TS 更清楚某个值的类型时，可以使用断言：

```typescript
// as 语法（推荐）
let someValue: unknown = "this is a string";
let strLength: number = (someValue as string).length;

// 尖括号语法（在 JSX 中不可用）
let strLength2: number = (<string>someValue).length;

// 非空断言（! 操作符）
const input = document.getElementById("input")!; // 断言不为 null
```

⚠️ **断言只影响编译时，不做任何运行时转换，滥用会绕过类型检查！**

## 四、接口（Interface）

### 4.1 基本用法

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

function getUser(user: User): string {
  return `${user.name} <${user.email}>`;
}

getUser({ id: 1, name: "Alice", email: "alice@example.com" }); // ✅
```

### 4.2 可选属性与只读属性

```typescript
interface Config {
  host: string;
  port?: number;          // 可选属性
  readonly apiKey: string; // 只读属性
}

const config: Config = { host: "localhost", apiKey: "abc123" };
// config.apiKey = "new"; // ❌ 报错：只读
```

### 4.3 索引签名

```typescript
interface StringMap {
  [key: string]: string;
}

const dict: StringMap = {
  name: "Alice",
  city: "Beijing",
};
```

### 4.4 接口继承

```typescript
interface Animal {
  name: string;
}

interface Dog extends Animal {
  breed: string;
}

const dog: Dog = { name: "Rex", breed: "Labrador" };
```

### 4.5 接口合并（声明合并）

同名接口会自动合并：

```typescript
interface Window {
  myLib: string;
}

interface Window {
  myVersion: number;
}

// 最终 Window 包含两个属性
```

### 4.6 接口 vs 类型别名（type）

| 特性           | interface         | type                    |
| -------------- | ----------------- | ----------------------- |
| 对象类型定义   | ✅                | ✅                      |
| 可扩展（继承） | extends           | 交叉类型 `&`            |
| 声明合并       | ✅                | ❌                      |
| 联合类型       | ❌                | ✅                      |
| 元组           | ❌                | ✅                      |
| 推荐使用场景   | 对象/类的契约定义 | 联合类型、工具类型等    |

## 五、函数类型

### 5.1 函数类型定义

```typescript
// 参数类型 + 返回值类型
function add(a: number, b: number): number {
  return a + b;
}

// 函数类型表达式
const multiply: (a: number, b: number) => number = (a, b) => a * b;

// 用 interface 描述函数类型
interface MathFn {
  (a: number, b: number): number;
}
const subtract: MathFn = (a, b) => a - b;
```

### 5.2 可选参数与默认参数

```typescript
// 可选参数（必须在必填参数之后）
function greet(name: string, greeting?: string): string {
  return `${greeting ?? "Hello"}, ${name}!`;
}

// 默认参数
function createUser(name: string, role: string = "user"): object {
  return { name, role };
}
```

### 5.3 剩余参数

```typescript
function sum(...numbers: number[]): number {
  return numbers.reduce((acc, n) => acc + n, 0);
}

console.log(sum(1, 2, 3, 4)); // 10
```

### 5.4 函数重载

```typescript
// 重载签名
function format(value: string): string;
function format(value: number): string;

// 实现签名
function format(value: string | number): string {
  if (typeof value === "string") {
    return value.trim();
  }
  return value.toFixed(2);
}

console.log(format("  hello  ")); // "hello"
console.log(format(3.14159));     // "3.14"
```

### 5.5 this 类型

```typescript
interface Counter {
  count: number;
  increment(this: Counter): void;
}

const counter: Counter = {
  count: 0,
  increment() {
    this.count++;
  },
};
```

## 六、类型收窄（Type Narrowing）

TypeScript 通过**控制流分析**在不同分支自动收窄类型：

```typescript
// typeof 收窄
function padLeft(value: string, padding: string | number) {
  if (typeof padding === "number") {
    return " ".repeat(padding) + value; // 这里 padding 是 number
  }
  return padding + value; // 这里 padding 是 string
}

// instanceof 收窄
class Cat { meow() {} }
class Dog { bark() {} }

function makeSound(animal: Cat | Dog) {
  if (animal instanceof Cat) {
    animal.meow();
  } else {
    animal.bark();
  }
}

// in 操作符收窄
interface Bird { fly(): void; }
interface Fish { swim(): void; }

function move(animal: Bird | Fish) {
  if ("fly" in animal) {
    animal.fly();
  } else {
    animal.swim();
  }
}

// 类型谓词（自定义类型守卫）
function isFish(animal: Bird | Fish): animal is Fish {
  return (animal as Fish).swim !== undefined;
}
```
