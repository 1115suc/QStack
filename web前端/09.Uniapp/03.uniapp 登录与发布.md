## 1. 微信小程序登录

### 1.1 登录流程图

```
用户点击登录
    ↓
wx.login() 获取 code（临时凭证，5分钟有效）
    ↓
将 code 发送给后端
    ↓
后端用 code + AppID + AppSecret 换取 openid 和 session_key
    ↓
后端返回自定义 token
    ↓
前端存储 token，完成登录
```

> ⚠️ **安全要求**：AppSecret 绝对不能放在前端！必须由后端完成 code → openid 的换取。

### 1.2 完整登录实现

```typescript
// src/api/auth.ts
import { http } from '@/utils/request'

interface WxLoginParams {
  code: string
  userInfo?: {
    nickName: string
    avatarUrl: string
    gender: number
  }
}

interface LoginResult {
  token: string
  isNewUser: boolean
  userInfo: {
    id: number
    openid: string
    nickname: string
    avatar: string
  }
}

export const authApi = {
  wxLogin(params: WxLoginParams) {
    return http.post<LoginResult>('/auth/wx-login', params)
  },
  refreshToken() {
    return http.post<{ token: string }>('/auth/refresh')
  },
}
```

```typescript
// src/composables/useLogin.ts
import { ref } from 'vue'
import { useUserStore } from '@/stores/user'
import { authApi } from '@/api/auth'

export function useWxLogin() {
  const loading = ref(false)
  const userStore = useUserStore()

  async function login() {
    loading.value = true
    try {
      // Step 1: 获取 code
      const loginRes = await uni.login({ provider: 'weixin' })
      if (!loginRes.code) throw new Error('获取 code 失败')

      // Step 2: 发送 code 到后端
      const result = await authApi.wxLogin({ code: loginRes.code })

      // Step 3: 存储登录状态
      userStore.setToken(result.token)
      userStore.setUserInfo(result.userInfo)

      uni.showToast({ title: '登录成功', icon: 'success' })

      // Step 4: 跳转首页
      uni.switchTab({ url: '/pages/index/index' })
    } catch (error) {
      uni.showToast({ title: '登录失败，请重试', icon: 'none' })
    } finally {
      loading.value = false
    }
  }

  return { login, loading }
}
```

### 1.3 登录态维护与 Token 刷新

```typescript
// src/utils/request.ts 中添加 Token 刷新逻辑

let isRefreshing = false
let pendingRequests: Array<() => void> = []

async function handleTokenExpired() {
  if (isRefreshing) {
    // 等待刷新完成后重试
    return new Promise<void>((resolve) => {
      pendingRequests.push(resolve)
    })
  }

  isRefreshing = true
  try {
    const { token } = await authApi.refreshToken()
    uni.setStorageSync('token', token)
    pendingRequests.forEach(resolve => resolve())
    pendingRequests = []
  } catch {
    // 刷新失败，跳转登录
    uni.reLaunch({ url: '/pages/login/index' })
    throw new Error('Token refresh failed')
  } finally {
    isRefreshing = false
  }
}
```

## 4. 微信支付

### 4.1 支付流程

```
用户点击支付
    ↓
前端调用后端下单接口（传入订单信息）
    ↓
后端调用微信统一下单API，获取预支付数据（prepay_id）
    ↓
后端签名后返回支付参数给前端
    ↓
前端调用 uni.requestPayment 唤起支付
    ↓
用户完成/取消支付
    ↓
前端查询订单状态（不能依赖支付回调！）
```

### 4.2 前端代码

```typescript
// src/api/order.ts
import { http } from '@/utils/request'

interface CreateOrderParams {
  goodsId: number
  quantity: number
  addressId: number
}

interface PayParams {
  timeStamp: string
  nonceStr: string
  package: string
  signType: 'MD5' | 'RSA'
  paySign: string
}

interface OrderResult {
  orderId: string
  payParams: PayParams
}

export const orderApi = {
  // 创建订单
  createOrder(params: CreateOrderParams) {
    return http.post<OrderResult>('/order/create', params)
  },

  // 查询订单状态
  queryOrder(orderId: string) {
    return http.get<{ status: 'pending' | 'paid' | 'failed' }>(`/order/${orderId}/status`)
  },
}
```

```typescript
// src/composables/usePay.ts
import { ref } from 'vue'
import { orderApi } from '@/api/order'

export function usePay() {
  const paying = ref(false)

  async function pay(goodsId: number, quantity: number, addressId: number) {
    paying.value = true
    let orderId = ''

    try {
      // Step 1: 创建订单
      const { orderId: oid, payParams } = await orderApi.createOrder({
        goodsId,
        quantity,
        addressId,
      })
      orderId = oid

      // Step 2: 唤起微信支付
      await uni.requestPayment({
        provider: 'wxpay',
        timeStamp: payParams.timeStamp,
        nonceStr: payParams.nonceStr,
        package: payParams.package,
        signType: payParams.signType,
        paySign: payParams.paySign,
      })

      // Step 3: 支付成功，查询订单状态（重要！不能只依赖回调）
      await pollOrderStatus(orderId)

      uni.showToast({ title: '支付成功', icon: 'success' })
      uni.redirectTo({ url: `/pages/order/success?orderId=${orderId}` })
    } catch (err: any) {
      if (err?.errMsg?.includes('cancel')) {
        uni.showToast({ title: '已取消支付', icon: 'none' })
      } else {
        uni.showToast({ title: '支付失败，请重试', icon: 'none' })
      }
    } finally {
      paying.value = false
    }
  }

  // 轮询订单状态（最多查询 5 次，每次间隔 1 秒）
  async function pollOrderStatus(orderId: string, maxTry = 5) {
    for (let i = 0; i < maxTry; i++) {
      await new Promise(resolve => setTimeout(resolve, 1000))
      const { status } = await orderApi.queryOrder(orderId)
      if (status === 'paid') return true
      if (status === 'failed') throw new Error('支付失败')
    }
    throw new Error('查询超时')
  }

  return { pay, paying }
}
```

## 7. 常见问题排查

### 7.1 样式问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 样式只在 H5 生效，小程序不生效 | 使用了小程序不支持的 CSS | 检查是否用了 `*`、`:last-child`、属性选择器 |
| 图片不显示 | 路径问题 | 确认用 `/static/` 开头的绝对路径 |
| `scoped` 样式无法穿透子组件 | Vue scoped 隔离 | 使用 `:deep(.class)` 穿透 |
| 底部被 iPhoneX 安全区遮挡 | 没处理安全区 | 底部加 `padding-bottom: env(safe-area-inset-bottom)` |

### 7.2 网络请求问题

| 现象 | 解决 |
|------|------|
| 开发环境跨域 | `vite.config.ts` 配置 `proxy`，`VITE_BASE_URL` 设为 `/api` |
| 小程序 request 不合法域名 | 开发时勾选"不校验合法域名"，上线前在公众平台配置 |
| H5 生产环境跨域 | Nginx 配置 `proxy_pass` 或后端设置 CORS 响应头 |
| 上传文件失败 | 检查 `uploadFile` 域名是否配置，文件大小是否超限（微信限制 10MB）|
