# ⚡ Vite 笔记

## 一、 什么是 Vite？

**Vite**（法语意为 "快速"）是由 Vue 之父尤雨溪开发的新一代前端构建工具。

### 1. 核心组成

*   **开发阶段**：基于浏览器原生 **ESM** (ES Modules) 的开发服务器。
*   **生产阶段**：使用 **Rollup** 进行打包，构建极度优化的静态资源。

### 2. 为什么比 Webpack 快？

*   **冷启动快**：Webpack 需要先打包整个项目才能启动；Vite 直接启动服务器，根据浏览器请求**按需编译**。
*   **热更新 (HMR) 快**：无论项目多大，HMR 始终保持极快的响应速度，因为它不需要重新构建整个 bundle。
*   **预构建**：使用 Go 编写的 **esbuild** 预构建依赖（如 lodash, axios），速度比 JS 编写的打包器快 10-100 倍。

---

## 二、 快速上手

### 1. 创建项目

```bash
npm create vite@latest my-vue-app --template vue
```

### 2. 目录结构要点

*   `index.html`：在 Vite 中，`index.html` 是项目的入口文件（位于根目录而非 public），Vite 会解析其中的 `<script type="module" src="...">`。
*   `public/`：存放纯静态资源，不会被编译，直接复制到 dist 根目录。

## 三、 常用配置 (`vite.config.js`)

这是 Vite 项目的心脏。

```javascript
import { defineConfig, loadEnv } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig(({ command, mode }) => {
  // 根据当前工作目录中的 `mode` 加载 .env 文件
  const env = loadEnv(mode, process.cwd())

  return {
    // 1. 插件配置
    plugins: [vue()],

    // 2. 路径别名 (需要安装 @types/node)
    resolve: {
      alias: {
        '@': path.resolve(__dirname, './src'),
      },
    },

    // 3. 开发服务器配置
    server: {
      host: '0.0.0.0', // 允许局域网访问
      port: 3000,      // 端口号
      open: true,      // 自动打开浏览器
      proxy: {         // 反向代理，解决跨域
        '/api': {
          target: 'http://backend-api.com',
          changeOrigin: true,
          rewrite: (path) => path.replace(/^\/api/, '')
        }
      }
    },

    // 4. 构建配置
    build: {
      outDir: 'dist',     // 打包输出目录
      assetsDir: 'assets', // 静态资源目录
      sourcemap: false,    // 是否生成 sourceMap
      minify: 'esbuild',   // 混淆压缩方式
      rollupOptions: {     // 分包策略
        output: {
          manualChunks: {
            vender: ['vue', 'vue-router', 'pinia']
          }
        }
      }
    }
  }
})
```

## 四、 静态资源处理

### 1. 导入资源

*   **图片/媒体**：直接 `import imgUrl from './img.png'`，返回解析后的 URL。
*   **JSON**：支持具名导入 `import { name } from './data.json'`。
*   **CSS**：支持模块化 `.module.css` 和预处理器（Sass/Less 仅需安装对应的编译器即可，无需配置 loader）。

### 2. `public` 目录

任何放入 `public` 的资源都可以通过根路径 `/` 直接访问（例如 `public/logo.png` 在代码中通过 `/logo.png` 访问）。

## 五、 环境变量与模式

Vite 使用 `import.meta.env` 暴露环境变量。

### 1. 文件命名

*   `.env`：所有模式都会加载
*   `.env.development`：开发模式加载
*   `.env.production`：生产模式加载

### 2. 变量定义

变量必须以 **`VITE_`** 开头才能暴露给客户端。
```env
# .env.development
VITE_API_URL = http://localhost:8080
```

### 3. 代码中使用

```javascript
console.log(import.meta.env.VITE_API_URL)
```

## 六、 性能优化技巧

### 1. 依赖预构建 (Dependency Pre-Bundling)

Vite 会自动将 CommonJS/UMD 依赖转换为 ESM。如果某些动态引入的依赖未被预构建，可以在配置中强制加入：
```javascript
optimizeDeps: {
  include: ['my-lib/components']
}
```

### 2. 分包策略 (Code Splitting)

在 `rollupOptions` 中将大型第三方库单独打包，避免单个 JS 文件过大。

### 3. 开启 Gzip/Brotli 压缩

使用插件 `vite-plugin-compression`：
```javascript
import compression from 'vite-plugin-compression'
plugins: [compression()]
```

### 4. 图片压缩

使用 `vite-plugin-imagemin` 优化图片体积。

## 七、 常用插件生态

Vite 的强大部分来自于其丰富的插件系统：

1.  **@vitejs/plugin-vue**: 提供 Vue 3 支持。
2.  **unplugin-auto-import**: 自动导入 Vue/Router/Pinia API（不需要再写 `import { ref } from 'vue'`）。
3.  **unplugin-vue-components**: 自动引入组件库（如 Element Plus/Ant Design）。
4.  **vite-plugin-svg-icons**: 快速制作 SVG Sprite 图标。
5.  **vite-plugin-mock**: 模拟后端接口数据。

---

## 八、 避坑指南

1.  **Node.js 版本**：Vite 需要 Node.js 版本 18+ 或 20+。
2.  **CommonJS 问题**：Vite 核心基于 ESM。如果你使用的第三方库只支持 CommonJS 且没有导出 ESM，Vite 会尝试转换，但偶尔会失败，需通过 `optimizeDeps` 解决。
3.  **动态导入图片**：不能像 Webpack 那样直接使用 `require('@/assets/img.png')`。在 Vite 中应使用：
    ```javascript
    const imgUrl = new URL('./assets/img.png', import.meta.url).href
    ```
4.  **浏览器兼容性**：默认只支持原生支持 ESM 的浏览器。如果需要兼容旧版浏览器（如 Chrome < 80），需要使用 `@vitejs/plugin-legacy`。

---

## 九、 总结：Vite 工作流

1.  **开发环境**：`vite` -> 利用浏览器 ESM 直接运行 -> 极速 HMR。
2.  **生产环境**：`vite build` -> Rollup 打包 -> 生成优化后的静态资源。
3.  **预览**：`vite preview` -> 本地模拟生产环境运行 dist 目录。
