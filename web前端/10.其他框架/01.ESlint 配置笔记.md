# Vue 框架 ESLint 配置笔记

在 Vue 项目中，ESLint 是保障代码质量、统一代码风格以及捕获潜在错误的重要工具。合理的配置能够极大提升团队协作的效率。以下是 Vue 项目中 ESLint 配置的核心要点梳理。

## 1. 核心解析与验证：`eslint-plugin-vue`

官方提供的 `eslint-plugin-vue` 是 Vue 项目配置 ESLint 的基石。它赋予了 ESLint 检查 `.vue` 单文件组件的能力，不仅能校验 `<script>` 中的 JavaScript/TypeScript，还能解析并校验 `<template>` 中的 HTML 语法和 Vue 特有的指令（如 `v-if`、`v-for`）。

## 2. 基础配置结构 (`.eslintrc.js`)

一个典型的 Vue 项目 ESLint 配置文件通常包含以下几个关键部分：

JavaScript

```javascript
module.exports = {
  root: true, // 标识为根目录，停止 ESLint 向上级目录查找配置文件
  env: {
    node: true, // 启用 Node.js 全局变量
    browser: true, // 启用浏览器全局变量 (如 window, document)
    es2021: true, // 启用最新的 ES 语法
  },
  // 继承现有的规则集合，数组后面的配置会覆盖前面的
  extends: [
    'plugin:vue/vue3-essential', // 针对 Vue 3 的基础防错规则
    'eslint:recommended' // ESLint 官方推荐的基础规则
  ],
  parserOptions: {
    // 对于 Vue 项目，外层通常由 vue-eslint-parser 处理，此处配置针对内部 JS/TS 的解析器
    parser: '@babel/eslint-parser', 
    ecmaVersion: 'latest', 
    sourceType: 'module'
  },
  rules: {
    // 自定义规则覆盖，优先级最高
    'no-console': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    'no-debugger': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    'vue/no-unused-components': 'warn', // 未使用的组件触发警告
  }
}
```

## 3. Vue 规则集的优先级梯队

`eslint-plugin-vue` 提供了不同严格等级的规则集，你可以根据项目需求在 `extends` 中引入：

- **Base:** `plugin:vue/base` - 仅包含基本的解析设置和核心规则。
    
- **Essential (必需):** `plugin:vue/vue3-essential` (Vue 2 为 `plugin:vue/essential`) - 包含防止常见错误的规则，**强烈建议启用**。
    
- **Strongly Recommended (强烈推荐):** `plugin:vue/vue3-strongly-recommended` - 在 Essential 基础上，增加改善代码可读性的规则（如属性换行、自闭合标签等）。
    
- **Recommended (推荐):** `plugin:vue/vue3-recommended` - 最严格的级别，在 Strongly Recommended 基础上，增加强制团队统一主观代码风格的规则（如组件属性排序）。
    

## 4. 与 TypeScript 和 Prettier 的生态集成

现代前端工程化中，ESLint 通常不是孤立存在的：

- **集成 TypeScript:** 需要引入 `@typescript-eslint/parser` 和 `@typescript-eslint/eslint-plugin`。
    
- **集成 Prettier:** Prettier 负责代码格式化，ESLint 负责代码质量。为了防止两者对格式规则的冲突，需要使用 `eslint-config-prettier` 关闭所有可能与 Prettier 冲突的 ESLint 规则。**关键点：Prettier 的配置必须放在 `extends` 数组的最后一位。**

## 5. 常用的高频自定义规则

在 `rules` 节点中，你可能会经常根据团队习惯微调以下规则：

- **允许单文件组件名为单个单词：**
    
    （默认规则要求组件名必须是多词组合以防与 HTML 原生标签冲突）
    
    `'vue/multi-word-component-names': 'off'`
    
- **配置 HTML 自闭合风格：**
    
    `'vue/html-self-closing': ['error', { 'html': { 'void': 'always', 'normal': 'never', 'component': 'always' } }]`
    
- **允许使用 v-html（需警惕 XSS 风险）：**
    
    `'vue/no-v-html': 'off'`
    

## 6. `.eslintignore` 文件

与 `.gitignore` 类似，你应该在项目根目录创建一个 `.eslintignore` 文件，告诉 ESLint 检查时跳过哪些编译后的产物或第三方依赖，以提升校验速度：

Plaintext

```
build/
dist/
node_modules/
public/
*.config.js
```
