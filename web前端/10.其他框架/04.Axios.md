# Axios 笔记

> 基于 Promise 的 HTTP 客户端，适用于浏览器和 Node.js

---

## 一、简介 & 安装

### 核心特性

| 特性 | 说明 |
|------|------|
| Promise 驱动 | 支持 async/await，避免回调地狱 |
| 双端运行 | 浏览器（XMLHttpRequest）和 Node.js（http 模块）均可使用 |
| 请求/响应拦截 | 统一处理 token 注入、错误弹窗等逻辑 |
| 自动转换 JSON | 请求体自动序列化，响应自动 JSON.parse |
| 取消请求 | 支持 AbortController 和 CancelToken |
| XSRF 防护 | 自动读取 cookie 中的 XSRF-TOKEN |

### 安装

```bash
# npm
npm install axios

# yarn
yarn add axios

# pnpm
pnpm add axios
```

### CDN 引入

```html
<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>
```

### 导入方式

```js
// ESModule
import axios from 'axios'

// CommonJS
const axios = require('axios')

// TypeScript 按需导入类型
import axios, { AxiosInstance, InternalAxiosRequestConfig, AxiosResponse } from 'axios'
```

## 二、基础请求

### GET 请求

```js
// 方式一：Promise 链式
axios.get('/user', { params: { id: 1 } })
  .then(res => console.log(res.data))
  .catch(err => console.error(err))

// 方式二：async/await（推荐）
async function getUser() {
  try {
    const { data } = await axios.get('/user', {
      params: { id: 1 }
    })
    return data
  } catch (error) {
    console.error(error)
  }
}
```

### POST 请求

```js
// 发送 JSON（默认）
await axios.post('/login', {
  username: 'admin',
  password: '123456'
})

// 发送表单数据
const form = new FormData()
form.append('file', file)
await axios.post('/upload', form, {
  headers: { 'Content-Type': 'multipart/form-data' }
})
```

### 其他请求方法

```js
axios.put('/user/1', { name: 'Tom' })      // 全量更新
axios.patch('/user/1', { name: 'Tom' })    // 部分更新
axios.delete('/user/1')                    // 删除
axios.head('/user')                        // 仅获取响应头
axios.options('/user')                     // 预检请求

// 并发请求
const [users, posts] = await Promise.all([
  axios.get('/users'),
  axios.get('/posts')
])
```

### 响应结构

```js
{
  data: {},          // 服务器返回的数据
  status: 200,       // HTTP 状态码
  statusText: 'OK',  // 状态描述
  headers: {},       // 响应头
  config: {},        // 请求配置
  request: {}        // 请求实例（XMLHttpRequest）
}
```

## 三、配置项

### 常用配置项说明

```js
axios({
  url: '/user',                    // 请求地址
  method: 'GET',                   // 请求方法，默认 GET
  baseURL: 'https://api.xxx.com',  // 基础 URL，会自动拼接到 url 前面
  headers: {                       // 请求头
    'Authorization': 'Bearer token',
    'Content-Type': 'application/json'
  },
  params: { id: 1 },              // URL 查询参数（拼接在 URL 后）
  data: { name: 'Tom' },          // 请求体数据（POST / PUT / PATCH）
  timeout: 10000,                  // 超时时间（ms），0 表示不限制
  responseType: 'json',            // 响应类型：json / blob / arraybuffer / text / stream
  withCredentials: false,          // 跨域请求是否携带 cookie，默认 false
  maxContentLength: 2000,          // 响应内容最大字节数
})
```

### 错误处理

```js
try {
  await axios.get('/user')
} catch (error) {
  if (error.response) {
    // 服务器有响应，但状态码超出 2xx 范围
    console.log(error.response.status)   // 400 / 401 / 500...
    console.log(error.response.data)     // 响应体中的错误信息
  } else if (error.request) {
    // 请求已发出，但未收到响应（断网 / 超时）
    console.log('网络异常，请检查网络连接')
  } else {
    // 请求配置本身有问题
    console.log('请求配置错误:', error.message)
  }
}
```

> ⚠️ **注意**：只有 2xx 范围的状态码才会进入 `.then()`，其余状态码均触发 `.catch()`。

## 四、实例 & 默认值

### 全局默认值

```js
axios.defaults.baseURL = 'https://api.example.com'
axios.defaults.timeout = 10000
axios.defaults.headers.common['Authorization'] = 'Bearer token'
axios.defaults.headers.post['Content-Type'] = 'application/json'
```

> ⚠️ **不推荐**修改全局实例的默认值，建议通过 `axios.create()` 创建独立实例隔离配置。

### 创建自定义实例（推荐）

```js
const request = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 用法与全局 axios 完全一致
const { data } = await request.get('/user')
```

> ✅ **多后端场景**下，可以创建多个实例，分别配置不同的 `baseURL` 和 `token`。

### 配置优先级

优先级从高到低：**请求时的 config** > **实例默认值** > **全局默认值**

```js
const instance = axios.create({ timeout: 5000 })

// 此次请求的 timeout 会被覆盖为 3000
instance.get('/user', { timeout: 3000 })
```

## 五、拦截器

### 请求拦截器

```js
instance.interceptors.request.use(
  config => {
    // 请求发出前：注入 token、开启 loading...
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config          // 必须返回 config，否则请求不会发出
  },
  error => {
    // 请求配置阶段的错误（极少出现）
    return Promise.reject(error)
  }
)
```

### 响应拦截器

```js
instance.interceptors.response.use(
  response => {
    // 统一处理响应数据
    const { data } = response
    if (data.code !== 0) {
      // 业务错误统一提示（如 token 失效、权限不足）
      showToast(data.message)
      return Promise.reject(data)
    }
    return data   // 直接返回业务数据，调用处无需再 .data 取值
  },
  error => {
    // HTTP 层面的错误统一处理
    const status = error.response?.status
    if (status === 401) router.push('/login')
    if (status === 403) showToast('无权限')
    if (status === 500) showToast('服务器错误')
    return Promise.reject(error)
  }
)
```

### 移除拦截器

```js
// 添加时保存返回的 id
const id = instance.interceptors.request.use(handler)

// 在合适时机（如组件卸载）移除
instance.interceptors.request.eject(id)
```

## 六、取消请求

### AbortController（推荐，axios v0.22+）

```js
const controller = new AbortController()

axios.get('/slow-api', {
  signal: controller.signal
})

// 取消请求（如：组件卸载、路由跳转时调用）
controller.abort()
```

**Vue 3 中配合 `onUnmounted` 自动取消：**

```js
import { onUnmounted } from 'vue'

const controller = new AbortController()

axios.get('/data', { signal: controller.signal })

onUnmounted(() => controller.abort())
```

> ✅ `AbortController` 是 Web 标准，同样适用于 `fetch`，推荐优先使用。

### CancelToken（旧版，v0.22 已废弃）

```js
const source = axios.CancelToken.source()

axios.get('/user', {
  cancelToken: source.token
})

// 取消请求，可传入取消原因
source.cancel('用户主动取消了请求')

// 在 catch 中判断是否为取消引起的错误
if (axios.isCancel(error)) {
  console.log('请求被取消:', error.message)
}
```

> ⚠️ `CancelToken` 已在 v0.22 中废弃，新项目建议统一使用 `AbortController`。

## 七、封装实践

### 完整封装模板（TypeScript）

```ts
// src/utils/request.ts
import axios from 'axios'
import type { AxiosInstance, InternalAxiosRequestConfig, AxiosResponse } from 'axios'

// 统一接口返回结构
interface Result<T = any> {
  code: number
  message: string
  data: T
}

const request: AxiosInstance = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 15000,
  headers: { 'Content-Type': 'application/json' }
})

// 请求拦截：注入 token
request.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = localStorage.getItem('access_token')
    if (token) config.headers.Authorization = `Bearer ${token}`
    return config
  },
  err => Promise.reject(err)
)

// 响应拦截：统一处理业务错误和 HTTP 错误
request.interceptors.response.use(
  (response: AxiosResponse<Result>) => {
    const { code, message, data } = response.data
    if (code === 0) return data
    ElMessage.error(message)
    return Promise.reject(new Error(message))
  },
  err => {
    const status = err.response?.status
    const msgMap: Record<number, string> = {
      400: '请求参数错误',
      401: '登录已过期，请重新登录',
      403: '没有操作权限',
      404: '请求资源不存在',
      500: '服务器内部错误'
    }
    ElMessage.error(msgMap[status] || '网络异常，请稍后重试')
    if (status === 401) router.push('/login')
    return Promise.reject(err)
  }
)

export default request
```

### API 模块化组织

```ts
// src/api/user.ts
import request from '@/utils/request'

interface UserInfo { id: number; name: string; email: string }
interface LoginParams { username: string; password: string }

export const userApi = {
  login: (data: LoginParams) =>
    request.post<string>('/auth/login', data),

  getUserInfo: (id: number) =>
    request.get<UserInfo>(`/user/${id}`),

  updateUser: (id: number, data: Partial<UserInfo>) =>
    request.patch(`/user/${id}`, data),

  deleteUser: (id: number) =>
    request.delete(`/user/${id}`)
}

// 使用时直接调用，类型完整推导
const info = await userApi.getUserInfo(1)
```

## 八、Vue 集成

### Vue 3 + Composition API（封装 useRequest）

```ts
// src/composables/useRequest.ts
import { ref } from 'vue'

export function useRequest<T>(apiFn: () => Promise<T>) {
  const data = ref<T | null>(null)
  const loading = ref(false)
  const error = ref<Error | null>(null)

  const execute = async () => {
    loading.value = true
    error.value = null
    try {
      data.value = await apiFn()
    } catch (e) {
      error.value = e as Error
    } finally {
      loading.value = false
    }
  }

  return { data, loading, error, execute }
}
```

**在组件中使用：**

```vue
<script setup lang="ts">
import { onMounted } from 'vue'
import { useRequest } from '@/composables/useRequest'
import { userApi } from '@/api/user'

const { data: user, loading, execute } = useRequest(() => userApi.getUserInfo(1))

onMounted(execute)
</script>

<template>
  <div v-if="loading">加载中...</div>
  <div v-else>{{ user?.name }}</div>
</template>
```

### Vue 2 挂载到原型

```js
// main.js
import axios from 'axios'
import request from './utils/request'

Vue.prototype.$axios = axios      // 全局 axios（不推荐）
Vue.prototype.$request = request  // 封装后的实例（推荐）

```

### 搭配 Pinia 管理请求状态（Vue 3）

```ts
// src/stores/user.ts
import { ref } from 'vue'
import { defineStore } from 'pinia'
import { userApi } from '@/api/user'

export const useUserStore = defineStore('user', () => {
  const userInfo = ref(null)
  const loading = ref(false)

  async function fetchUser(id: number) {
    loading.value = true
    try {
      userInfo.value = await userApi.getUserInfo(id)
    } finally {
      loading.value = false
    }
  }

  return { userInfo, loading, fetchUser }
})

// 组件中使用
const userStore = useUserStore()
await userStore.fetchUser(1)
```

## 附：常见问题速查

| 问题          | 解决方案                                                                       |
| ----------- | -------------------------------------------------------------------------- |
| 跨域携带 cookie | 设置 `withCredentials: true`，服务端也需配置 `Access-Control-Allow-Credentials`      |
| 上传进度监听      | 配置 `onUploadProgress: e => percent = Math.round(e.loaded / e.total * 100)` |
| 下载文件        | 设置 `responseType: 'blob'`，然后创建 `<a>` 标签触发下载                                |
| 请求重试        | 使用 `axios-retry` 插件，或在响应拦截器中手动实现                                           |
| token 刷新    | 响应拦截器捕获 401 后，调用刷新接口，成功后重放原始请求                                             |
| 防止重复提交      | 请求拦截器中用 `Map` 记录进行中的请求，完成后删除                                               |
