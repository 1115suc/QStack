> 目标：能用工程语言讲清楚 RAG、Agent、部署、性能和稳定性。

---

AI 应用开发岗通常重点考：

- RAG 流程

- 幻觉

## 二、RAG 高频问题

### 1. RAG 的完整流程是什么？

回答结构：

```text
文档加载 -> 文本分块 -> embedding -> 向量入库 -> 用户问题向量化 -> 检索 top_k -> 构造 Prompt -> LLM 生成 -> 返回答案和引用来源
```

补充工程点：

- 分块要保留 metadata
- 检索结果要记录日志
- 生产环境要做评估和引用来源

---

### 2. RAG 回答不准确怎么排查？

按链路排查：

1. 文档是否解析正确
2. chunk 是否切得太碎或太大
3. embedding 是否适合中文或业务语料
4. top_k 是否过小
5. 是否需要混合检索
6. 是否需要 rerank
7. Prompt 是否要求只能基于上下文回答
8. 模型是否忽略引用资料

---

### 3. 如何降低 RAG 幻觉？

可回答：

- Prompt 明确要求“无法确定就回答资料中没有提到”
- 返回引用来源
- 控制上下文只包含高相关片段
- 使用 rerank 提升上下文质量
- 对答案做 faithfulness 评估
- 对高风险场景加入人工审核

---

## 三、Agent 高频问题

### 1. Agent 和普通 ChatBot 的区别？

普通 ChatBot 主要生成文本。

Agent 可以根据任务调用工具，例如查数据库、查知识库、调用 API，然后根据工具结果继续推理。

---

### 2. 工具调用怎么保证安全？

可以回答：

- 工具参数用 Pydantic Schema 校验
- 工具内部做权限校验
- 高风险写操作需要人工确认
- 设置 timeout 和 retry
- 日志记录每次工具调用
- 不把密钥暴露给模型

---

### 3. Agent 调错工具怎么办？

处理方式：

- 工具名称和描述写清楚
- 工具粒度不要太大
- 返回结构化错误信息
- 限制可用工具集合
- 使用 LangGraph 做流程控制
- 对关键节点加入人工确认

---

## 四、工程化高频问题

### 1. AI 应用如何做性能优化？

方向：

- Redis 缓存高频问题和 embedding
- 异步调用外部 API
- embedding 批处理
- 控制 top_k 和上下文长度
- 选择更合适的小模型处理简单任务
- 对模型调用设置 timeout
- 使用流式输出改善体感速度

---

### 2. 如何做可观测性？

需要记录：

- request_id
- 模型名称
- prompt_tokens
- completion_tokens
- 总成本
- 模型耗时
- 工具耗时
- 检索命中文档
- 错误信息

工具：

- LangSmith
- Langfuse
- Prometheus
- Grafana
- 结构化日志
