# DrissionPage 请求监听与实战篇：SessionPage、模式切换、接口监听和项目套路

> 版本说明：本文基于 DrissionPage 稳定版 v4.1.1.2 整理，重点讲请求模式、网络监听和实战组合。

## 1. 为什么要学请求和监听

浏览器自动化能解决复杂交互，但它不是所有采集任务的最佳方式。

如果你用浏览器一页一页点击、滚动、解析 DOM，通常会遇到：

- 速度慢；
- 内存占用高；
- 元素定位不稳定；
- 页面结构变化导致脚本失效；
- 动态加载数据不好判断是否加载完成。

很多现代网页的数据其实来自接口。

更高效的方案是：

```text
浏览器只负责登录和触发
监听器负责发现接口
Session 负责批量请求
```

这也是 DrissionPage 很适合采集和自动化的原因。

## 2. 三种工作方式

| 方式 | 使用对象 | 是否启动浏览器 | 适合场景 |
|---|---|---|---|
| 浏览器自动化 | `Chromium` / `ChromiumPage` | 是 | 登录、点击、JS 渲染、动态交互 |
| 请求模式 | `SessionPage` | 否 | 静态页面、接口请求、批量采集 |
| 混合模式 | `WebPage` | 可切换 | 登录后请求、复杂站点采集 |

建议判断顺序：

```text
数据能不能直接请求？
能：优先 SessionPage
不能：是否可以监听接口？
能：用浏览器触发 + listen 抓接口
还不能：再用浏览器解析 DOM
```

## 3. SessionPage 基础

`SessionPage` 是纯请求页面对象，不启动浏览器。

```python
from DrissionPage import SessionPage

page = SessionPage()
page.get('https://www.example.com')

print(page.title)
print(page.url)
print(page.html)
```

它的好处是：即使不打开浏览器，也可以像页面一样查找元素。

```python
links = page.eles('tag:a')

for link in links:
    print(link.text, link.link)
```

## 4. SessionPage 发送 GET 请求

```python
from DrissionPage import SessionPage

page = SessionPage()
page.get('https://httpbin.org/get')

print(page.html)
```

带参数：

```python
page.get(
    'https://example.com/search',
    params={'keyword': 'DrissionPage', 'page': 1}
)
```

## 5. SessionPage 发送 POST 请求

```python
from DrissionPage import SessionPage

page = SessionPage()

page.post(
    'https://example.com/api/login',
    data={
        'username': 'admin',
        'password': '123456'
    }
)

print(page.html)
```

JSON 请求：

```python
page.post(
    'https://example.com/api/data',
    json={
        'page': 1,
        'size': 20
    }
)
```

## 6. SessionOptions 配置

```python
from DrissionPage import SessionOptions, SessionPage

so = SessionOptions()

so.set_headers({
    'User-Agent': 'Mozilla/5.0',
    'Accept': 'application/json, text/plain, */*',
    'Referer': 'https://example.com/'
})

page = SessionPage(so)
page.get('https://example.com/api/list')
```

常见配置：

```python
so.set_headers({...})
so.set_cookies({...})
so.set_proxies({...})
```

经验：

- 接口请求失败时，先检查 headers；
- 登录态异常时，检查 cookies；
- 403 时，检查 Referer、User-Agent、token；
- 接口返回空数据时，检查参数和分页字段。

## 7. 静态页面采集示例

```python
from DrissionPage import SessionPage

page = SessionPage()
page.get('https://gitee.com/explore/all')

items = page.eles('tag:h3')

for item in items:
    a = item.ele('tag:a', timeout=1)
    if a:
        print(a.text, a.link)
```

这类页面如果 HTML 里已经有数据，用 SessionPage 比浏览器快很多。

## 8. 请求模式适用与不适用

适合：

- 页面源码里直接包含数据；
- 接口参数简单；
- 登录态可以通过 cookies 维持；
- 需要大量分页请求；
- 不需要点击、滚动、JS 执行。

不适合：

- 页面必须 JS 渲染；
- 接口参数强加密；
- 请求依赖浏览器环境变量；
- 需要滑块、验证码、人机交互；
- 登录流程复杂。

遇到不适合的情况，不要硬写 requests，先用浏览器模式或监听器。

## 9. WebPage 混合模式

`WebPage` 可以在 d 模式和 s 模式之间切换。

d 模式：

```text
Driver 模式，控制浏览器
```

s 模式：

```text
Session 模式，发送请求
```

示例：

```python
from DrissionPage import WebPage

page = WebPage()

page.get('https://example.com')
print(page.mode)

page.change_mode('s')
print(page.mode)
```

## 10. 登录后切换请求模式

这是 DrissionPage 最重要的实战套路之一。

```python
from DrissionPage import WebPage

page = WebPage()

page.get('https://example.com/login')

page('#username').input('admin')
page('#password').input('123456')
page('#login').click()

page.wait.url_change('/home')

page.change_mode('s')
page.get('https://example.com/api/profile')

print(page.html)
```

流程解释：

```text
1. d 模式打开登录页
2. 输入用户名密码
3. 点击登录
4. 等待进入首页
5. 切换到 s 模式
6. 复用 cookies 请求接口
```

## 11. cookies 同步

手动从浏览器同步到 Session：

```python
page.cookies_to_session()
```

从 Session 同步到浏览器：

```python
page.cookies_to_browser()
```

常见场景：

```python
page.change_mode('s')
page.cookies_to_session()
```

注意：

- cookies 通常能同步；
- headers 不一定完整同步；
- token 可能藏在 localStorage、sessionStorage 或接口返回里；
- 必要时要自己提取并设置 headers。

## 12. 提取 localStorage token

很多网站登录后 token 不在 cookies，而在 localStorage。

```python
token = page.run_js('return localStorage.getItem("token");')
print(token)
```

切到请求模式后设置 headers：

```python
page.change_mode('s')

page.session.headers.update({
    'Authorization': f'Bearer {token}'
})
```

如果 token 在 sessionStorage：

```python
token = page.run_js('return sessionStorage.getItem("token");')
```

## 13. 网络监听基础

监听器用于抓取浏览器发出的请求和响应。

关键原则：

```text
先 start，再触发请求
```

错误思路：

```text
先点击，再监听
```

正确思路：

```text
先监听，再点击
```

示例：

```python
from DrissionPage import Chromium

tab = Chromium().latest_tab

tab.listen.start('/api/list')
tab.get('https://example.com/list')

packet = tab.listen.wait()

print(packet.url)
print(packet.response.body)
```

## 14. 监听点击触发的接口

```python
from DrissionPage import Chromium

tab = Chromium().latest_tab

tab.get('https://example.com/list')
tab.listen.start('/api/page')

tab('#next').click()

packet = tab.listen.wait()
data = packet.response.body

print(data)
```

分页采集：

```python
tab.listen.start('/api/page')

for page_no in range(1, 6):
    tab('#next').click()
    packet = tab.listen.wait()
    data = packet.response.body
    print(page_no, data)
```

## 15. DataPacket 常用属性

```python
packet.url
packet.method
packet.request
packet.response
```

请求信息：

```python
packet.request.headers
packet.request.postData
```

响应信息：

```python
packet.response.status
packet.response.headers
packet.response.body
```

如果响应是 JSON，`packet.response.body` 通常可以直接作为 Python 对象使用。

```python
body = packet.response.body

if isinstance(body, dict):
    print(body.keys())
```

## 16. 持续监听

监听多个包：

```python
tab.listen.start('/api')
tab.get('https://example.com')

for packet in tab.listen.steps():
    print(packet.url)
```

根据业务条件停止：

```python
tab.listen.start('/api')

for packet in tab.listen.steps():
    print(packet.url)

    if 'target_api' in packet.url:
        print(packet.response.body)
        break
```

暂停和恢复：

```python
tab.listen.pause()
tab.listen.resume()
```

停止监听：

```python
tab.listen.stop()
```

## 17. 监听接口后改成 Session 请求

实战里通常先监听接口，拿到 URL、headers、payload，再改成 Session 批量请求。

第一步：监听接口。

```python
from DrissionPage import Chromium

tab = Chromium().latest_tab

tab.listen.start('/api/list')
tab.get('https://example.com/list')

packet = tab.listen.wait()

print(packet.url)
print(packet.method)
print(packet.request.headers)
print(packet.request.postData)
print(packet.response.body)
```

第二步：分析参数，改成请求模式。

```python
from DrissionPage import SessionPage

page = SessionPage()

headers = {
    'User-Agent': 'Mozilla/5.0',
    'Referer': 'https://example.com/list'
}

for page_no in range(1, 11):
    page.get(
        'https://example.com/api/list',
        params={'page': page_no, 'size': 20},
        headers=headers
    )
    print(page.html)
```

## 18. 下载文件

请求模式下载：

```python
from DrissionPage import SessionPage

page = SessionPage()
page.download('https://example.com/file.zip')
```

指定保存路径：

```python
page.download(
    'https://example.com/file.zip',
    save_path=r'D:\downloads',
    rename='demo.zip'
)
```

浏览器点击下载：

```python
from DrissionPage import Chromium

tab = Chromium().latest_tab
tab.set.download_path(r'D:\downloads')

tab('#download').click()
tab.wait.download_begin()
tab.wait.downloads_done()
```

更适合稳定处理的写法：

```python
mission = tab('#download').click.to_download(r'D:\downloads', 'demo.zip')
mission.wait()
```

## 21. 登录态持久化

使用用户数据目录保存浏览器登录态：

```python
from DrissionPage import Chromium, ChromiumOptions

co = ChromiumOptions()
co.set_user_data_path(r'D:\chrome_profile\my_site')

tab = Chromium(co).latest_tab
tab.get('https://example.com')
```

第一次手动或自动登录后，后续打开同一个用户数据目录通常可以复用登录状态。

适合：

- 不想每次登录；
- 登录有验证码；
- 需要长期保持 cookies；
- 多账号隔离。

多账号建议：

```text
账号 A -> profile_a
账号 B -> profile_b
账号 C -> profile_c
```

## 22. 反爬和稳定性注意事项

技术上能自动化，不代表可以随意请求。

应注意：

- 遵守目标网站服务条款；
- 控制请求频率；
- 不绕过访问控制；
- 不抓取敏感个人数据；
- 不影响目标站点正常运行。

脚本稳定性建议：

- 每个关键点击后加等待；
- 监听接口优先于解析复杂 DOM；
- 请求失败要重试；
- 分页采集要记录进度；
- 文件下载要检查大小和后缀；
- 登录态失效要能重新登录。

## 27. 实战五：采集结果保存为 CSV

```python
import csv
from DrissionPage import SessionPage

page = SessionPage()

rows = []

for page_no in range(1, 6):
    page.get(
        'https://example.com/list',
        params={'page': page_no}
    )

    cards = page.eles('.card')

    for card in cards:
        title = card.ele('.title', timeout=1)
        price = card.ele('.price', timeout=1)

        rows.append({
            'title': title.text if title else '',
            'price': price.text if price else ''
        })

with open('result.csv', 'w', encoding='utf-8-sig', newline='') as f:
    writer = csv.DictWriter(f, fieldnames=['title', 'price'])
    writer.writeheader()
    writer.writerows(rows)
```

## 28. 项目结构建议

小型项目：

```text
project/
  main.py
  config.py
  requirements.txt
  data/
  downloads/
```

中型项目：

```text
project/
  main.py
  config.py
  browser.py
  session_client.py
  parser.py
  storage.py
  data/
  logs/
  downloads/
```

职责划分：

| 文件 | 职责 |
|---|---|
| `browser.py` | 浏览器启动、登录、监听 |
| `session_client.py` | 请求接口、同步 cookies |
| `parser.py` | 解析 HTML 或 JSON |
| `storage.py` | 保存 CSV、Excel、数据库 |
| `config.py` | 账号、路径、代理、URL |
| `main.py` | 主流程编排 |

## 29. 常见问题排查

### 29.1 元素找不到

检查：

- 定位语法是否正确；
- 元素是否在 iframe；
- 元素是否在 shadow-root；
- 页面是否加载完成；
- 是否需要先滚动；
- 是否登录态失效。

### 29.2 点击无效

检查：

- 元素是否可见；
- 是否被遮挡；
- 是否需要滚动到可见；
- 是否需要 JS 点击；
- 点击后是否需要等待。

### 29.3 请求模式返回 403

检查：

- cookies 是否同步；
- headers 是否完整；
- Referer 是否正确；
- token 是否在 localStorage；
- 请求频率是否太高；
- 接口参数是否缺失。

### 29.4 监听不到接口

检查：

- 是否先 `listen.start()` 再触发请求；
- 过滤条件是否过窄；
- 页面是否真的发出了请求；
- 接口是否走缓存；
- 是否监听了错误的 tab。

### 29.5 下载文件失败

检查：

- 下载路径是否存在；
- 文件名是否合法；
- 是否等待下载完成；
- 是否登录态失效；
- URL 是否需要鉴权。

## 30. 最佳实践总结

1. 能直接请求的页面，优先用 `SessionPage`；
2. 需要登录和交互时，用浏览器模式；
3. 动态数据优先监听接口，不要只解析 DOM；
4. 登录后优先考虑切换 s 模式；
5. cookies 能同步，但 token 和 headers 要重点检查；
6. 所有关键动作后都加等待；
7. 采集脚本要有异常处理和重试；
8. 批量任务要保存进度，避免失败后从头开始；
9. 调试先有界面，稳定后再考虑无头；
10. 遵守网站规则，控制访问频率。

## 31. 请求监听篇总结

DrissionPage 的高级用法可以总结为一句话：

```text
用浏览器解决人机交互，用监听器发现数据接口，用 Session 批量拿数据。
```

实战时不要一开始就决定用哪种方式，而是先分析数据来源：

```text
HTML 里有数据 -> SessionPage
接口里有数据 -> listen + SessionPage
必须页面渲染 -> 浏览器自动化
登录复杂 -> 浏览器登录后切请求模式
```

掌握这套思路后，DrissionPage 就不只是 Selenium 替代品，而是一个完整的网页自动化和数据采集工具箱。
