# DrissionPage 浏览器自动化篇：定位、点击、输入、等待与页面控制

> 版本说明：本文基于 DrissionPage 稳定版 v4.1.1.2 整理，重点讲浏览器自动化场景。

## 1. 浏览器自动化基本结构

最小代码：

```python
from DrissionPage import Chromium

tab = Chromium().latest_tab

tab.get('https://www.baidu.com')
tab('#kw').input('DrissionPage')
tab('#su').click()
```

浏览器自动化的核心是四类操作：

```text
打开页面
定位元素
操作元素
等待结果
```

对应代码：

```python
tab.get(url)
ele = tab.ele(locator)
ele.click()
tab.wait.xxx()
```

## 4. 元素定位总览

DrissionPage 的定位语法非常重要，建议优先使用简洁定位语法，再考虑 CSS 和 XPath。

获取一个元素：

```python
ele = tab.ele('#kw')
```

简写：

```python
ele = tab('#kw')
```

获取多个元素：

```python
eles = tab.eles('tag:a')
```

元素内继续查找：

```python
box = tab('.list')
items = box.eles('tag:li')
```

## 5. 常用定位语法

| 写法 | 含义 | 示例 |
|---|---|---|
| `#id` | 根据 id 查找 | `tab('#kw')` |
| `.class` | 根据 class 查找 | `tab('.btn')` |
| `tag:标签名` | 根据标签名查找 | `tab('tag:input')` |
| `text=文本` | 文本精确匹配 | `tab('text=登录')` |
| `text:文本` | 文本模糊匹配 | `tab('text:登录')` |
| `@属性=值` | 属性精确匹配 | `tab('@name=username')` |
| `@属性:值` | 属性包含匹配 | `tab('@href:login')` |
| `css:选择器` | CSS selector | `tab('css:.list > li')` |
| `xpath:路径` | XPath | `tab('xpath://div[@id="main"]')` |

示例：

```python
tab('#username')
tab('.submit-btn')
tab('tag:button')
tab('text=提交')
tab('text:提交')
tab('@placeholder=请输入用户名')
tab('@href:/login')
```

## 6. 属性匹配规则

| 符号 | 含义 | 示例 |
|---|---|---|
| `=` | 精确匹配 | `@id=row1` |
| `:` | 包含 | `@id:row` |
| `^` | 以某值开头 | `@id^row` |
| `$` | 以某值结尾 | `@id$1` |

示例：

```python
tab('@id=row1')
tab('@id:row')
tab('@id^row')
tab('@id$1')
```

## 7. 多条件定位

同时满足多个条件：

```python
tab('@@class=item@@text():标题')
```

满足任意条件：

```python
tab('@|id=login@|class=login-btn')
```

排除条件：

```python
tab('@!class=disabled')
```

实际例子：

```python
submit = tab('tag:button@@text():提交@@class:primary')
submit.click()
```

## 8. CSS 与 XPath

CSS selector：

```python
tab('css:.nav > li:first-child a')
```

XPath：

```python
tab('xpath://div[@id="main"]//a')
```

建议：

- 简单场景优先用 `#id`、`.class`、`text`、`@attr`；
- 层级复杂时用 CSS；
- 需要按文本、父子、兄弟复杂关系查找时用 XPath；
- 不建议直接复制浏览器生成的超长绝对 XPath。

## 16. 等待机制

自动化脚本最常见的问题是页面还没加载完就开始操作。

不要滥用：

```python
time.sleep(3)
```

优先用等待条件。

### 16.1 等待页面加载

```python
tab.wait.load_start()
tab.wait.doc_loaded()
```

### 16.2 等待元素加载

```python
tab.wait.eles_loaded('#content')
```

### 16.3 等待元素显示

```python
tab.wait.ele_displayed('#login')
```

### 16.4 等待元素隐藏

```python
tab.wait.ele_hidden('#loading')
```

### 16.5 等待元素删除

```python
tab.wait.ele_deleted('#dialog')
```

### 16.6 等待 URL 或标题变化

```python
tab.wait.url_change('/dashboard')
tab.wait.title_change('控制台')
```

典型登录等待：

```python
tab('#login').click()
tab.wait.url_change('/home')
```

典型搜索等待：

```python
tab('#kw').input('DrissionPage')
tab('#su').click()
tab.wait.eles_loaded('tag:h3')
```


## 21. 下载文件

设置下载路径：

```python
tab.set.download_path(r'D:\downloads')
```

点击下载：

```python
tab('#download').click()
tab.wait.download_begin()
tab.wait.downloads_done()
```

指定文件名：

```python
tab.set.download_file_name('demo.zip')
tab('#download').click()
tab.wait.downloads_done()
```

点击后转下载任务：

```python
mission = tab('#download').click.to_download(r'D:\downloads', 'demo.zip')
mission.wait()
```

注意：

- 如果要重命名下载文件，必须等下载完成；
- 下载路径建议使用绝对路径；
- 批量下载时要做好文件名冲突处理。

## 22. iframe 操作

如果元素在 iframe 中，普通 `tab('#id')` 可能找不到。

先获取 iframe：

```python
iframe = tab.get_frame('tag:iframe')
```

在 iframe 中查找元素：

```python
iframe('#username').input('admin')
iframe('#password').input('123456')
```

按属性找 iframe：

```python
iframe = tab.get_frame('@id=login-frame')
```

实战建议：

- 页面元素找不到时，先检查是否在 iframe 内；
- iframe 对象可以像普通页面一样使用；
- 跨域 iframe 的能力以 v4.1.1.2 文档为准，不要套用 v4.2 测试版写法。

## 23. shadow-root 操作

部分现代网页使用 Shadow DOM，普通定位可能找不到内部元素。

通常流程：

```python
host = tab('#shadow-host')
sr = host.shadow_root
ele = sr('#inner-button')
ele.click()
```

实际脚本中，应先用浏览器开发者工具确认目标元素是否位于 shadow-root 内。

## 24. 异常处理

```python
from DrissionPage.errors import ElementNotFoundError

try:
    tab('#login').click()
except ElementNotFoundError:
    print('没有找到登录按钮')
```

或者设置超时并判断：

```python
ele = tab.ele('#login', timeout=3)

if ele:
    ele.click()
else:
    print('登录按钮不存在')
```

更适合采集脚本的写法：

```python
title = card.ele('.title', timeout=1)
price = card.ele('.price', timeout=1)

data = {
    'title': title.text if title else '',
    'price': price.text if price else ''
}
```

## 27. 动态列表采集模板

```python
from DrissionPage import Chromium

tab = Chromium().latest_tab
tab.get('https://example.com/list')

for _ in range(5):
    tab.scroll.to_bottom()
    tab.wait(1)

cards = tab.eles('.card')

for card in cards:
    title = card.ele('.title', timeout=1)
    price = card.ele('.price', timeout=1)

    print({
        'title': title.text if title else '',
        'price': price.text if price else ''
    })
```

## 28. 浏览器自动化调试建议

1. 先用有界面浏览器，不要一开始就无头模式；
2. 每一步操作后打印 `tab.url`、`tab.title`；
3. 元素找不到时检查是否在 iframe 或 shadow-root；
4. 点击无效时检查元素是否被遮挡；
5. 多用等待，少用固定 sleep；
6. 定位语法尽量写短、稳定、可读；
7. 优先定位业务属性，例如 `id`、`name`、`placeholder`、文本；
8. 不要依赖容易变化的 class 哈希值；
9. 抓数据时优先考虑接口监听，而不是硬解析复杂 DOM；
10. 登录态需要保留时配置用户数据目录。

## 29. 浏览器自动化篇总结

DrissionPage 浏览器自动化最重要的能力是：

```text
定位元素
操作元素
等待页面变化
处理特殊结构
```

写脚本时建议形成固定思路：

```text
先确认页面结构
再写稳定定位
再补等待条件
最后处理异常分支
```

如果页面数据来自接口，不要只盯着 DOM，下一步应该学习网络监听和请求模式。
