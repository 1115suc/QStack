# DrissionPage 基础篇：框架定位、安装配置与核心概念

> 版本说明：本文基于 DrissionPage 稳定版 v4.1.1.2 整理，不包含 v4.2 测试版新增特性。

## 1. DrissionPage 是什么

DrissionPage 是一个 Python 网页自动化框架，它把浏览器自动化和 HTTP 请求能力整合到同一套使用方式里。

它的名字来自：

```text
Drission = Driver + Session
Page = 页面对象
```

也就是说，它既能像 Selenium 一样控制浏览器，又能像 requests 一样直接收发数据包。

常见用途：

- 自动打开网页、点击、输入、滚动、上传、下载；
- 自动登录网站；
- 抓取动态网页数据；
- 监听浏览器接口请求；
- 登录后复用 cookies 请求接口；
- 静态页面快速采集；
- 替代部分 Selenium + requests + lxml 的组合代码。

## 2. 为什么学习 DrissionPage

传统网页自动化通常有两条路线：

| 路线 | 常用工具 | 优点 | 缺点 |
|---|---|---|---|
| 浏览器自动化 | Selenium、Playwright | 能处理 JS、登录、点击、验证码、人机交互 | 速度慢，占用资源多 |
| 请求采集 | requests、httpx | 快、轻量、适合大规模采集 | 遇到 JS、登录态、加密参数会变复杂 |

DrissionPage 的特点是把这两条路线合并：

```text
浏览器负责解决复杂交互
Session 负责高速请求数据
监听器负责抓接口
```

典型思路：

```text
用浏览器登录 -> 同步 cookies -> 切换到请求模式 -> 高速抓接口或页面
```

这就是 DrissionPage 的核心价值。

## 3. 环境要求

DrissionPage v4.1.1.2 的常见运行环境：

- Python 3.6+
- Windows、macOS、Linux
- Chromium 内核浏览器，例如 Chrome、Edge
- 推荐浏览器版本 100+

安装：

```bash
pip install DrissionPage==4.1.1.2
```

如果已经安装过其他版本，建议指定版本重装：

```bash
pip install DrissionPage==4.1.1.2 --force-reinstall
```

查看版本：

```bash
pip show DrissionPage
```

Python 中也可以查看：

```python
import DrissionPage

print(DrissionPage.__version__)
```

## 4. 浏览器路径配置

如果 DrissionPage 找不到浏览器，需要指定浏览器路径。

代码方式：

```python
from DrissionPage import ChromiumOptions

co = ChromiumOptions()
co.set_browser_path(r'C:\Program Files\Google\Chrome\Application\chrome.exe')
co.save()
```

命令行方式：

```bash
dp -p "C:\Program Files\Google\Chrome\Application\chrome.exe"
```

Edge 示例：

```python
from DrissionPage import ChromiumOptions

co = ChromiumOptions()
co.set_browser_path(r'C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe')
co.save()
```

## 5. 第一个程序

```python
from DrissionPage import Chromium

browser = Chromium()
tab = browser.latest_tab

tab.get('https://www.baidu.com')
tab.ele('#kw').input('DrissionPage')
tab.ele('#su').click()
```

简写：

```python
from DrissionPage import Chromium

tab = Chromium().latest_tab

tab.get('https://www.baidu.com')
tab('#kw').input('DrissionPage')
tab('#su').click()
```

核心流程：

```text
创建浏览器 -> 获取标签页 -> 打开页面 -> 查找元素 -> 操作元素
```

## 7. 核心对象

### 7.1 浏览器对象 Chromium

```python
from DrissionPage import Chromium

browser = Chromium()
```

它代表一个 Chromium 浏览器，可以：

- 获取当前标签页；
- 新建标签页；
- 管理多个标签页；
- 关闭浏览器；
- 接管已有浏览器。

获取最新标签页：

```python
tab = browser.latest_tab
```

新建标签页：

```python
tab = browser.new_tab('https://www.drissionpage.cn')
```

### 7.2 标签页对象 Tab

标签页对象是日常操作最常用的对象。

```python
tab.get('https://www.example.com')
print(tab.title)
print(tab.url)
```

它可以：

- 打开网址；
- 查找元素；
- 点击输入；
- 等待页面；
- 监听网络；
- 执行 JS；
- 滚动页面。

### 7.3 元素对象 Element

查找到的页面节点就是元素对象。

```python
ele = tab.ele('#kw')
```

元素对象可以：

- 获取文本；
- 获取属性；
- 输入内容；
- 点击；
- 悬停；
- 拖拽；
- 滚动到可见；
- 执行 JS。

### 7.4 SessionPage

`SessionPage` 不启动浏览器，直接请求网页。

```python
from DrissionPage import SessionPage

page = SessionPage()
page.get('https://www.example.com')

print(page.title)
print(page.html)
```

适合：

- 静态 HTML 页面；
- 接口直接返回数据；
- 大量分页采集；
- 不需要浏览器渲染的任务。

## 8. d 模式和 s 模式

DrissionPage 的核心概念是两种模式：

| 模式 | 全称 | 含义 | 适合场景 |
|---|---|---|---|
| d 模式 | Driver 模式 | 控制真实浏览器 | 登录、点击、JS 渲染、复杂交互 |
| s 模式 | Session 模式 | 使用 requests 请求 | 高速采集、接口请求、静态页面 |

d 模式优点：

- 能执行 JavaScript；
- 能处理真实页面交互；
- 能看到浏览器画面；
- 对登录、验证码、动态页面更友好。

d 模式缺点：

- 慢；
- 占用资源；
- 不适合大量并发。

s 模式优点：

- 快；
- 轻量；
- 适合批量请求；
- 不需要浏览器。

s 模式缺点：

- 不能执行 JS；
- 遇到加密参数要自己分析；
- 登录态和 headers 可能要手动处理。

## 9. WebPage 的模式切换

`WebPage` 可以在 d 模式和 s 模式之间切换。

```python
from DrissionPage import WebPage

page = WebPage()
page.get('https://www.example.com')

print(page.mode)

page.change_mode('s')
print(page.mode)
```

典型流程：

```python
from DrissionPage import WebPage

page = WebPage()

page.get('https://example.com/login')
page('#username').input('admin')
page('#password').input('123456')
page('#login').click()

page.wait.url_change('/home')

page.change_mode('s')
page.get('https://example.com/api/user')

print(page.html)
```

要点：

- 登录、点击、复杂交互用 d 模式；
- 采集、接口、分页请求用 s 模式；
- 切换模式时会同步 cookies；
- headers、token 等不一定总能自动补齐，必要时要手动处理。

## 10. ChromiumOptions 常用配置

### 10.1 指定浏览器路径

```python
from DrissionPage import ChromiumOptions, Chromium

co = ChromiumOptions()
co.set_browser_path(r'C:\Program Files\Google\Chrome\Application\chrome.exe')

browser = Chromium(co)
```

### 10.2 设置用户数据目录

用户数据目录可以保存登录态、cookies、插件等。

```python
from DrissionPage import ChromiumOptions, Chromium

co = ChromiumOptions()
co.set_user_data_path(r'D:\chrome_user_data\dp_profile')

browser = Chromium(co)
```

### 10.3 设置代理

```python
from DrissionPage import ChromiumOptions, Chromium

co = ChromiumOptions()
co.set_proxy('127.0.0.1:7890')

browser = Chromium(co)
```

### 10.4 无头模式

```python
from DrissionPage import ChromiumOptions, Chromium

co = ChromiumOptions()
co.headless()

browser = Chromium(co)
```

无头模式适合服务器运行，但调试阶段建议先用有界面模式。

## 11. SessionOptions 常用配置

```python
from DrissionPage import SessionOptions, SessionPage

so = SessionOptions()
so.set_headers({
    'User-Agent': 'Mozilla/5.0',
    'Accept': 'text/html,application/xhtml+xml'
})

page = SessionPage(so)
page.get('https://www.example.com')
```

常见请求配置：

- headers；
- cookies；
- proxies；
- timeout；
- verify；
- auth。

## 12. 常用全局设置

```python
from DrissionPage.common import Settings

Settings.raise_when_ele_not_found = False
```

如果设置为 `False`，元素找不到时通常返回 `None`，而不是直接抛异常。

调试阶段可以保留默认抛异常，便于发现问题；写采集脚本时可以根据场景改成返回 `None`。

## 13. 基础异常处理

```python
from DrissionPage import Chromium
from DrissionPage.errors import ElementNotFoundError

tab = Chromium().latest_tab
tab.get('https://www.example.com')

try:
    tab('#login').click()
except ElementNotFoundError:
    print('没有找到登录按钮')
```

更常见的写法：

```python
ele = tab.ele('#login', timeout=3)

if ele:
    ele.click()
else:
    print('登录按钮不存在')
```

## 16. 基础篇总结

DrissionPage 的学习重点不是背 API，而是理解三个判断：

```text
页面是否必须用浏览器？
数据是否可以直接请求？
登录态能否从浏览器同步到请求模式？
```

如果必须交互，就用 d 模式。

如果可以请求，就用 s 模式。

如果登录复杂，就先浏览器登录，再切请求模式。
