# Agent进阶篇

## 1. 进阶开发的核心变化

基础 Agent 关注能不能调用工具；进阶 Agent 关注工具是否可控、流程是否稳定、结果是否可追踪。

进阶阶段要解决的问题：

- 工具越来越多，模型容易选错工具。
- 工具参数不完整，需要追问。
- 工具调用失败，需要重试或降级。
- 多轮对话需要保存 session 状态。
- RAG、数据库、HTTP API 需要组合使用。
- Agent 的每一步都要能记录和排查。

进阶 Agent 的核心是工具治理和执行链路管理。

## 3. 工具治理

工具不是越多越好。

工具数量过多时，模型更容易：

- 选错工具
- 参数填错
- 重复调用
- 忽略更合适的工具

工具治理策略：

| 策略 | 说明 |
| --- | --- |
| 工具分组 | 按场景拆分工具集合 |
| 动态加载 | 根据用户问题只暴露相关工具 |
| 强 Schema | 使用 Pydantic 明确参数 |
| 工具描述优化 | docstring 写清楚使用边界 |
| 工具结果压缩 | 避免返回过长内容 |
| 工具审计 | 记录工具名、参数、结果、耗时 |

## 4. 动态选择工具集合

不要每次都把所有工具交给 Agent。

```python
def select_tools(question: str):
    if "订单" in question:
        return [get_order_status, list_user_orders]
    if "文档" in question or "笔记" in question:
        return [search_knowledge_base]
    return [calculator]
```

创建 Agent：

```python
tools = select_tools(user_question)

agent = create_agent(
    model="openai:gpt-4.1-mini",
    tools=tools,
    system_prompt=system_prompt,
)
```

好处：

- 降低模型选择难度。
- 减少 prompt token。
- 降低误调用风险。
- 便于做权限控制。

## 5. RAG 工具

RAG 可以封装成 Agent 的一个工具。

```python
from pydantic import BaseModel, Field
from langchain_core.tools import tool

class SearchKnowledgeBaseInput(BaseModel):
    query: str = Field(description="需要检索的知识库问题")
    top_k: int = Field(default=4, ge=1, le=10, description="返回文档数量")

@tool(args_schema=SearchKnowledgeBaseInput)
def search_knowledge_base(query: str, top_k: int = 4) -> str:
    """当用户询问内部文档、技术笔记、知识库内容时使用。"""
    docs = rag_service.retrieve(query=query, top_k=top_k)
    return rag_service.format_docs(docs)
```

注意：

- RAG 工具只负责检索，不一定负责最终回答。
- 返回内容要带来源。
- 不要把几十个 chunk 全部返回给 Agent。

## 6. HTTP API 工具

业务系统通常通过 HTTP API 暴露能力。

```python
import httpx
from pydantic import BaseModel, Field
from langchain_core.tools import tool

class UserProfileInput(BaseModel):
    user_id: str = Field(description="用户 ID")

@tool(args_schema=UserProfileInput)
async def get_user_profile(user_id: str) -> str:
    """根据用户 ID 查询用户基础信息。"""
    try:
        async with httpx.AsyncClient(timeout=5) as client:
            response = await client.get(
                f"https://api.example.com/users/{user_id}"
            )
            response.raise_for_status()
            data = response.json()
            return {
                "user_id": data["id"],
                "name": data["name"],
                "level": data["level"],
            }
    except httpx.TimeoutException:
        return "查询用户信息超时"
    except httpx.HTTPStatusError as exc:
        return f"查询用户信息失败：HTTP {exc.response.status_code}"
```

HTTP 工具必须设置 timeout，不能无限等待。

## 7. Memory 多轮对话

Agent 需要知道上下文，例如用户上一轮提到的订单号。

```text
用户：帮我查订单 A1001
助手：订单 A1001 已发货
用户：那它什么时候到？
```

第二轮问题需要依赖第一轮上下文。

简单做法是把历史消息传给 Agent：

```python
messages = [
    {"role": "user", "content": "帮我查订单 A1001"},
    {"role": "assistant", "content": "订单 A1001 已发货"},
    {"role": "user", "content": "那它什么时候到？"},
]

result = agent.invoke({"messages": messages})
```

工程注意：

- 历史消息不能无限增长。
- 长对话要做摘要。
- session_id 要和用户身份绑定。
- 敏感信息要脱敏。

## 8. 结构化输出

Agent 最终结果最好是结构化响应，便于前端展示和日志记录。

```python
from pydantic import BaseModel, Field

class ToolCallRecord(BaseModel):
    name: str
    args: dict
    latency_ms: int | None = None

class AgentResponse(BaseModel):
    answer: str
    tool_calls: list[ToolCallRecord] = Field(default_factory=list)
    sources: list[dict] = Field(default_factory=list)
```

接口返回：

```json
{
  "answer": "订单 A1001 已支付，等待发货。",
  "tool_calls": [
    {
      "name": "get_order_status",
      "args": {"order_id": "A1001"},
      "latency_ms": 42
    }
  ],
  "sources": []
}
```

## 9. 工具错误处理

常见错误：

- 参数缺失
- 参数格式错误
- 业务数据不存在
- HTTP 超时
- 第三方服务失败
- 权限不足

处理原则：

- 参数错误：让 Agent 追问用户。
- 业务不存在：如实告知。
- 网络错误：可以重试一次。
- 权限不足：不能绕过权限。
- 高风险失败：记录审计日志。

工具不要抛出一堆底层异常给模型。

## 10. FastAPI 接口设计

请求模型：

```python
from pydantic import BaseModel, Field

class AgentChatRequest(BaseModel):
    question: str = Field(min_length=1)
    session_id: str | None = None
    user_id: str

class AgentChatResponse(BaseModel):
    answer: str
    tool_calls: list[dict] = []
    sources: list[dict] = []
    latency_ms: int
```

路由：

```python
from fastapi import APIRouter

router = APIRouter(prefix="/agent", tags=["Agent"])

@router.post("/chat", response_model=AgentChatResponse)
async def chat(request: AgentChatRequest):
    return await agent_service.chat(request)
```
