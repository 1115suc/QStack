# LangChain基础篇

## 1. LangChain 解决什么问题

LangChain 是面向 LLM 应用开发的编排框架。它把常见能力抽象为统一接口：

- **模型调用**：用统一的 ChatModel 接口调用 OpenAI、Anthropic、Ollama、本地推理服务等。
- **消息与提示词**：用 `SystemMessage`、`HumanMessage`、`AIMessage`、`ToolMessage` 等结构表达对话。
- **工具调用**：把 Python 函数注册成模型可调用的工具。
- **Agent**：通过 `create_agent` 构建能“调用模型 - 选择工具 - 执行工具 - 继续推理”的循环。
- **状态与运行时**：LangChain 的 agent 底层基于 LangGraph，适合接入状态管理、checkpoint、人工确认和观测。

官方推荐把 LangChain 用在“需要 LLM 与外部系统协作”的应用中，而不是只做一次简单补全。

## 2. 安装与包结构

基础安装：

```bash
pip install -U langchain
```

按模型供应商安装对应集成包，例如：

```bash
pip install -U langchain-openai
pip install -U langchain-anthropic
pip install -U langchain-community
```

常见包职责：

| 包 | 作用 |
| --- | --- |
| `langchain` | Agent、chains、RAG、通用编排入口 |
| `langchain-core` | 消息、Runnable、工具、模型基础协议 |
| `langchain-community` | 社区维护的 loader、vector store、tool 等集成 |
| `langchain-openai` / `langchain-anthropic` | 具体模型供应商适配 |
| `langgraph` | 状态图、checkpoint、human-in-the-loop、持久化 agent runtime |
| `langsmith` | tracing、评估、数据集、实验管理 |

建议生产项目显式固定版本，例如：

```txt
langchain>=1.0,<2.0
langchain-openai>=1.0,<2.0
langgraph>=1.0,<2.0
langsmith>=0.4
```

## 3. 最小模型调用

LangChain v1.x 推荐使用标准模型初始化接口：

```python
from langchain.chat_models import init_chat_model
import dotenv

dotenv.load_dotenv()

model = init_chat_model(  
    model="deepseek-chat",  
    model_provider="openai",  
    api_key=os.getenv("DEEPSEEK_API_KEY"),  
    base_url=os.getenv("DEEPSEEK_API_BASE"),  
)

response = model.invoke("用一句话解释 LangChain 是什么")
print(response.content)
```

环境变量通常由 provider 决定，例如：

```powershell
$env:DEEPSEEK_API_KEY="sk-..."
```

如果使用本地推理服务，例如 Ollama，通常需要安装对应集成并配置本地服务地址。核心思想是：业务层尽量依赖 LangChain 的标准模型接口，减少 provider 切换成本。

## 4. 消息模型

多轮对话不要拼接字符串，应该使用消息对象：

```python
from langchain_core.messages import SystemMessage, HumanMessage
from langchain.chat_models import init_chat_model

model = init_chat_model("openai:gpt-4.1-mini", temperature=0)

messages = [
    SystemMessage(content="你是一个严谨的 Python 后端工程师。"),
    HumanMessage(content="FastAPI 里依赖注入适合解决什么问题？"),
]

result = model.invoke(messages)
print(result.content)
```

这样做有几个好处：

- 角色边界清晰，方便插入 system / human / ai / tool 消息。
- 便于接入 memory、trace、replay。
- 便于把对话历史存储为结构化数据。

## 5. Prompt 模板

Prompt 模板适合把业务变量和系统指令分离：

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain.chat_models import init_chat_model

prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是一个{role}。输出要简洁、准确。"),
        ("human", "请解释：{question}"),
    ]
)

model = init_chat_model("openai:gpt-4.1-mini", temperature=0)

chain = prompt | model
result = chain.invoke(
    {
        "role": "LangChain 讲师",
        "question": "Runnable 接口的价值是什么？",
    }
)

print(result.content)
```

`|` 是 LangChain Expression Language 的组合方式，常用于把 prompt、model、parser、retriever 等组件串起来。

## 6. Runnable 基础

Runnable 是 LangChain 统一执行协议，常见方法：

| 方法 | 说明 |
| --- | --- |
| `invoke(input)` | 单次同步调用 |
| `ainvoke(input)` | 单次异步调用 |
| `batch(inputs)` | 批量同步调用 |
| `abatch(inputs)` | 批量异步调用 |
| `stream(input)` | 流式同步输出 |
| `astream(input)` | 流式异步输出 |

示例：

```python
result = chain.invoke({"role": "助手", "question": "什么是 RAG？"})

results = chain.batch(
    [
        {"role": "助手", "question": "什么是 embedding？"},
        {"role": "助手", "question": "什么是 vector store？"},
    ]
)
```

掌握 Runnable 后，后续组合 RAG、Agent、评估链会更自然。

## 7. 工具定义

工具是 Agent 连接外部系统的核心。一个 Python 函数可以通过 `@tool` 注册为模型可调用工具：

```python
from langchain_core.tools import tool

@tool
def get_order_status(order_id: str) -> str:
    """查询订单状态。order_id 是订单编号。"""
    fake_db = {
        "A1001": "已支付，等待发货",
        "A1002": "已发货，运输中",
    }
    return fake_db.get(order_id, "未找到订单")
```

工具设计要点：

- 函数名表达动作，例如 `get_order_status`、`search_docs`。
- docstring 写清楚什么时候用、参数是什么、返回什么。
- 参数尽量结构化，避免只传一个任意字符串。
- 工具内部要做权限、超时、异常处理，不要完全依赖模型自觉。

## 8. 最小 Agent

LangChain v1.x 的核心入口是 `create_agent`：

```python
from langchain.agents import create_agent
from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    """获取指定城市的天气。"""
    return f"{city} 今天多云，22 摄氏度。"

agent = create_agent(
    model="openai:gpt-4.1-mini",
    tools=[get_weather],
    system_prompt="你是一个简洁的天气助手。需要实时信息时先调用工具。",
)

result = agent.invoke(
    {
        "messages": [
            {"role": "user", "content": "上海今天适合跑步吗？"}
        ]
    }
)

print(result["messages"][-1].content)
```

Agent 输入通常是包含 `messages` 的字典。输出也会包含完整消息列表，其中可能包括：

- 用户消息
- 模型消息
- 工具调用消息
- 工具返回消息
- 最终回答

这对调试和审计很重要。

## 9. 基础流式输出

Agent 支持流式返回。常见模式：

- `updates`：按 agent step 输出，例如模型调用、工具调用、工具结果。
- `messages`：按 token 或消息块输出。
- `custom`：输出自定义进度事件。

示例：

```python
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "查一下上海天气"}]},
    stream_mode="updates",
):
    print(chunk)
```

在 Web 服务中，`stream_mode="messages"` 更适合接 SSE 或 WebSocket；`updates` 更适合调试 agent 执行过程。

## 10. FastAPI 最小集成

```python
from fastapi import FastAPI
from pydantic import BaseModel
from langchain.agents import create_agent

app = FastAPI()

agent = create_agent(
    model="openai:gpt-4.1-mini",
    tools=[],
    system_prompt="你是一个 API 助手。",
)

class ChatRequest(BaseModel):
    message: str

@app.post("/chat")
def chat(req: ChatRequest):
    result = agent.invoke(
        {"messages": [{"role": "user", "content": req.message}]}
    )
    return {"answer": result["messages"][-1].content}
```

生产化时至少补齐：

- 请求 ID / 用户 ID / session ID
- 超时控制
- 限流
- 错误码
- LangSmith trace
- 工具权限控制
- 输入输出审计与脱敏

## 11. 基础调试与观测

LangSmith 是 LangChain 官方推荐的 tracing 和评估平台。开发阶段建议开启：

```powershell
$env:LANGSMITH_TRACING="true"
$env:LANGSMITH_API_KEY="lsv2_..."
$env:LANGSMITH_PROJECT="langchain-basic-demo"
```

打开 tracing 后，可以看到：

- 每次模型调用
- prompt 和 messages
- 工具调用参数
- token 用量
- latency
- 异常堆栈

对 Agent 应用来说，trace 不是锦上添花，而是排查“为什么模型调用了这个工具”的基础设施。
