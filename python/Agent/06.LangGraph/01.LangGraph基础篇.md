# LangGraph基础篇

## 1. LangGraph 解决什么问题

LangChain 适合快速组合模型、Prompt、工具和 RAG。

但当 Agent 变复杂后，只靠线性 Chain 会遇到几个问题：

- 多步骤任务需要保存状态。
- 工具调用后要根据结果决定下一步。
- 某些节点需要人工确认。
- 执行失败后希望从中间步骤恢复。
- 多个 Agent 或多个流程节点需要协作。
- 需要更清楚地控制循环次数、分支和结束条件。

LangGraph 的核心思想是：把 LLM 应用或 Agent 工作流建模成一张状态图。

```text
State -> Node -> Edge -> Next Node -> END
```

可以把 LangGraph 理解为 Agent 的工作流引擎。

## 2. LangGraph 和 LangChain 的关系

LangChain 更关注组件组合，LangGraph 更关注状态和流程控制。

| 对比项 | LangChain | LangGraph |
| --- | --- | --- |
| 核心能力 | Prompt、Model、Tool、Retriever 组合 | 状态图、分支、循环、持久化 |
| 适合场景 | 简单 Chain、RAG、工具调用 | 多步骤 Agent、可恢复工作流 |
| 流程控制 | 相对线性 | 图结构，可分支可循环 |
| 状态管理 | 需要自行维护 | State 统一维护 |
| 人工介入 | 可以做，但不够直接 | 原生适合 human-in-the-loop |

实际项目中通常是一起使用：

```text
LangChain 负责模型、Prompt、工具
LangGraph 负责状态、路由、循环、恢复
```

## 4. 核心概念

| 概念 | 说明 |
| --- | --- |
| State | 图运行时共享的状态数据 |
| Node | 一个处理步骤，可以是普通函数、LLM 调用、工具调用 |
| Edge | 节点之间的连接关系 |
| Conditional Edge | 根据状态决定下一步走向 |
| START | 图的开始节点 |
| END | 图的结束节点 |
| Compile | 把图编译成可运行对象 |
| Invoke | 执行图 |

LangGraph 的关键不是“画图”，而是用 State 把每一步的输入输出统一管理。

## 5. 最小状态图

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class AgentState(TypedDict):
    question: str
    answer: str

def answer_node(state: AgentState):
    return {"answer": f"你的问题是：{state['question']}"}

graph = StateGraph(AgentState)

graph.add_node("answer", answer_node)
graph.add_edge(START, "answer")
graph.add_edge("answer", END)

app = graph.compile()

result = app.invoke({"question": "什么是 RAG？"})
print(result["answer"])
```

执行流程：

```text
START -> answer -> END
```

节点函数接收 state，返回要更新的字段。

## 6. State 状态设计

State 是 LangGraph 的核心。

```python
from typing import TypedDict

class AgentState(TypedDict):
    question: str
    rewritten_question: str
    documents: list[str]
    answer: str
```

状态设计建议：

- 字段名要表达业务含义。
- 不要把所有内容都塞进一个 `data` 字段。
- 重要中间结果要保留，方便排查。
- 状态里可以保存问题、检索结果、工具结果、最终答案。

常见字段：

| 字段 | 作用 |
| --- | --- |
| `messages` | 对话消息 |
| `question` | 用户问题 |
| `documents` | 检索到的文档 |
| `tool_results` | 工具调用结果 |
| `next_step` | 下一步路由 |
| `answer` | 最终答案 |

## 7. Node 节点

节点就是一个处理函数。

```python
def rewrite_node(state: AgentState):
    question = state["question"]
    return {"rewritten_question": f"请回答：{question}"}

def retrieve_node(state: AgentState):
    query = state["rewritten_question"]
    return {"documents": [f"和 {query} 相关的资料"]}

def generate_node(state: AgentState):
    docs = "\n".join(state["documents"])
    return {"answer": f"根据资料回答：{docs}"}
```

注册节点：

```python
graph.add_node("rewrite", rewrite_node)
graph.add_node("retrieve", retrieve_node)
graph.add_node("generate", generate_node)
```

## 8. Edge 边

普通边表示固定流程。

```python
graph.add_edge(START, "rewrite")
graph.add_edge("rewrite", "retrieve")
graph.add_edge("retrieve", "generate")
graph.add_edge("generate", END)
```

流程：

```text
START -> rewrite -> retrieve -> generate -> END
```

这种结构适合固定步骤的 RAG：

```text
问题改写 -> 检索 -> 生成答案
```

## 9. Conditional Edge 条件路由

条件边用于根据状态选择下一步。

```python
from typing import Literal

def route_question(state: AgentState) -> Literal["retrieve", "direct_answer"]:
    question = state["question"]
    if "知识库" in question or "文档" in question:
        return "retrieve"
    return "direct_answer"

graph.add_conditional_edges(
    START,
    route_question,
    {
        "retrieve": "retrieve",
        "direct_answer": "direct_answer",
    },
)
```

适合场景：

- 判断是否需要 RAG
- 判断是否需要调用工具
- 判断是否需要人工确认
- 判断是否继续循环
- 判断任务是否完成

## 10. 一个基础 RAG Graph

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class RagState(TypedDict):
    question: str
    documents: list[str]
    answer: str

def retrieve_node(state: RagState):
    question = state["question"]
    return {"documents": [f"和 {question} 相关的文档片段"]}

def generate_node(state: RagState):
    context = "\n".join(state["documents"])
    return {"answer": f"基于上下文回答：{context}"}

graph = StateGraph(RagState)
graph.add_node("retrieve", retrieve_node)
graph.add_node("generate", generate_node)

graph.add_edge(START, "retrieve")
graph.add_edge("retrieve", "generate")
graph.add_edge("generate", END)

app = graph.compile()

result = app.invoke({"question": "FastAPI 依赖注入有什么用？"})
print(result["answer"])
```

基础阶段先理解固定流程，不急着上复杂 Agent。

## 11. 什么时候使用 LangGraph

适合使用：

- 多步骤任务
- 有条件分支
- 有循环调用
- 需要保存中间状态
- 需要人工确认
- 需要失败恢复
- 需要把 Agent 做成稳定服务

不一定需要：

- 单次 LLM 问答
- 简单 prompt 模板
- 一次性 RAG 查询
- 没有状态和分支的脚本

## 12. 基础阶段实践任务

完成一个基础 LangGraph RAG：

- 定义 `RagState`
- 编写 `retrieve_node`
- 编写 `generate_node`
- 使用 `StateGraph` 串联流程
- 调用 `app.invoke`
- 输出 answer 和 documents

基础阶段目标是理解 State、Node、Edge 三件事。
