# LangGraph进阶篇

## 1. 进阶开发的核心变化

基础 LangGraph 关注图能不能跑通；进阶 LangGraph 关注流程是否可控。

进阶阶段要解决的问题：

- 如何让 Agent 根据问题类型走不同流程。
- 如何把 RAG、工具调用、普通回答组合到一张图里。
- 如何让模型决定是否继续调用工具。
- 如何限制循环次数，避免 Agent 死循环。
- 如何管理多轮对话消息。
- 如何把每一步结果记录下来，方便排查。

进阶 LangGraph 的核心是条件路由和状态更新。

分层职责：

| 层 | 职责 |
| --- | --- |
| graph | 定义状态图结构 |

| service | 复用业务能力 |
| schema | 请求响应结构 |

不要把节点函数、工具函数、API 路由全部写在一个文件里。

## 3. Messages 状态

对话型 Agent 通常需要保存消息列表。

LangGraph 可以使用 reducer 来合并消息。

```python
from typing import Annotated, TypedDict
from langgraph.graph.message import add_messages
from langchain_core.messages import BaseMessage

class ChatState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
```

`add_messages` 的作用是：节点返回新消息时，自动追加到已有 messages 中。

```python
from langchain_core.messages import HumanMessage, AIMessage

def chat_node(state: ChatState):
    last_message = state["messages"][-1]
    return {
        "messages": [
            AIMessage(content=f"收到你的问题：{last_message.content}")
        ]
    }
```

## 4. 路由节点

路由节点用于判断用户问题应该走哪条链路。

```python
from typing import Literal

class AgentState(TypedDict):
    question: str
    route: str
    documents: list[str]
    tool_result: str
    answer: str

def classify_node(state: AgentState):
    question = state["question"]
    if "订单" in question:
        return {"route": "tool"}
    if "笔记" in question or "文档" in question:
        return {"route": "rag"}
    return {"route": "chat"}

def route_after_classify(state: AgentState) -> Literal["rag", "tool", "chat"]:
    return state["route"]
```

注册条件边：

```python
graph.add_node("classify", classify_node)

graph.add_conditional_edges(
    "classify",
    route_after_classify,
    {
        "rag": "retrieve",
        "tool": "call_tool",
        "chat": "chat",
    },
)
```

流程：

```text
START -> classify -> rag / tool / chat
```

## 5. RAG 分支

RAG 分支可以复用已有 RAG service。

```python
def retrieve_node(state: AgentState):
    docs = rag_service.retrieve(
        query=state["question"],
        top_k=4,
    )
    return {"documents": docs}

def rag_generate_node(state: AgentState):
    answer = rag_service.generate_answer(
        question=state["question"],
        documents=state["documents"],
    )
    return {"answer": answer}
```

边：

```python
graph.add_edge("retrieve", "rag_generate")
graph.add_edge("rag_generate", END)
```

## 6. 工具分支

工具分支适合订单查询、天气查询、数据库查询等外部能力。

```python
def call_tool_node(state: AgentState):
    question = state["question"]

    if "订单" in question:
        result = tool_service.get_order_status(question)
    else:
        result = "没有匹配到合适工具"

    return {"tool_result": result}

def tool_answer_node(state: AgentState):
    return {"answer": f"工具查询结果：{state['tool_result']}"}
```

边：

```python
graph.add_edge("call_tool", "tool_answer")
graph.add_edge("tool_answer", END)
```

## 7. 循环控制

Agent 常见流程：

```text
LLM 判断 -> 是否调用工具 -> 调用工具 -> LLM 继续判断 -> 最终回答
```

这需要循环，但必须控制最大次数。

```python
class ToolAgentState(TypedDict):
    question: str
    steps: int
    tool_result: str
    answer: str
    should_continue: bool

def agent_node(state: ToolAgentState):
    steps = state.get("steps", 0) + 1

    if steps >= 3:
        return {
            "steps": steps,
            "should_continue": False,
            "answer": "已达到最大工具调用次数，停止执行。",
        }

    if not state.get("tool_result"):
        return {"steps": steps, "should_continue": True}

    return {
        "steps": steps,
        "should_continue": False,
        "answer": f"根据工具结果回答：{state['tool_result']}",
    }

def should_continue(state: ToolAgentState):
    return "tool" if state["should_continue"] else "end"
```

条件边：

```python
graph.add_conditional_edges(
    "agent",
    should_continue,
    {
        "tool": "tool",
        "end": END,
    },
)

graph.add_edge("tool", "agent")
```

## 8. 节点返回值规范

节点应该只返回需要更新的字段。

推荐：

```python
def node(state):
    return {"answer": "处理完成"}
```

不推荐：

```python
def node(state):
    state["answer"] = "处理完成"
    return state
```

原因：

- 返回增量字段更清晰。
- 减少误覆盖状态。
- 更方便追踪每个节点修改了什么。

## 9. FastAPI 接口设计

请求模型：

```python
from pydantic import BaseModel, Field

class AgentRequest(BaseModel):
    question: str = Field(min_length=1)
    session_id: str | None = None

class AgentResponse(BaseModel):
    answer: str
    route: str | None = None
    steps: int | None = None
```

路由示例：

```python
from fastapi import APIRouter

router = APIRouter(prefix="/agent", tags=["Agent"])

@router.post("/chat", response_model=AgentResponse)
async def chat(request: AgentRequest):
    result = app.invoke({"question": request.question})
    return AgentResponse(
        answer=result["answer"],
        route=result.get("route"),
        steps=result.get("steps"),
    )
```

## 10. 进阶阶段实践任务

实现一个多分支 Agent Graph：

- `classify` 节点判断问题类型
- 文档问题走 RAG 分支
- 订单问题走工具分支
- 普通问题走 chat 分支
- 每个分支最终返回统一 `answer`
- 加入最大循环次数限制
- 用 FastAPI 暴露 `/agent/chat`

进阶阶段目标是掌握条件路由、分支组合和循环控制。
