# FastAPI 入门

> 目标：理解 FastAPI 核心概念，能够独立搭建一个可运行的高性能 Web API 服务

---

## 一、FastAPI 是什么

FastAPI 是一个基于 Python 3.8+ 的**现代、高性能 Web 框架**，核心特性：

- **极高性能**：基于 `Starlette`（ASGI）和 `Pydantic`，性能接近 Node.js 和 Go
- **自动生成文档**：内置 Swagger UI 和 ReDoc，开箱即用
- **类型安全**：基于 Python 类型注解，自动完成参数校验和序列化
- **原生异步支持**：天然支持 `async / await`，适合高并发 IO 密集型场景

与 Flask 的核心区别：

|对比项|Flask|FastAPI|
|---|---|---|
|运行模式|WSGI（同步）|ASGI（异步）|
|参数校验|手动|自动（Pydantic）|
|API 文档|需要插件|内置自动生成|
|性能|中等|极高|
|学习曲线|较平缓|稍陡（需了解类型注解）|

## 二、安装 FastAPI

```bash
# 激活 conda 虚拟环境
conda activate fastapi-env

pip install fastapi uvicorn[standard]

# 验证安装
python -c "import fastapi; print(fastapi.__version__)"
```

> `uvicorn` 是运行 FastAPI 的 ASGI 服务器，相当于 Flask 中的开发服务器。

## 三、第一个 FastAPI 应用

```python
# main.py
from fastapi import FastAPI

app = FastAPI(
    title='My API',       # 文档标题
    description='FastAPI 学习项目',
    version='1.0.0'
)

@app.get('/')
def index():
    return {'message': 'Hello, FastAPI!'}
```

运行：

```bash
uvicorn main:app --reload
# main    → 文件名 main.py
# app     → FastAPI 实例变量名
# --reload → 代码修改后自动重启（开发模式）

# 访问 http://127.0.0.1:8000
# 文档地址 http://127.0.0.1:8000/docs      （Swagger UI）
# 文档地址 http://127.0.0.1:8000/redoc     （ReDoc）
```

## 四、路由与 HTTP 方法

FastAPI 用装饰器直接对应 HTTP 方法：

```python
@app.get('/items')        # GET
@app.post('/items')       # POST
@app.put('/items/{id}')   # PUT
@app.delete('/items/{id}')# DELETE
@app.patch('/items/{id}') # PATCH
```

### 4.1 路径参数

```python
@app.get('/user/{user_id}')
def get_user(user_id: int):   # 类型注解自动校验和转换
    return {'user_id': user_id}

# 访问 /user/abc → 自动返回 422 参数校验错误
# 访问 /user/1   → 正常返回 {'user_id': 1}
```

### 4.2 查询参数

函数参数中**不在路径中**的参数，自动识别为查询参数：

```python
@app.get('/items')
def get_items(
    page:     int  = 1,      # 有默认值 → 可选参数
    size:     int  = 10,
    keyword:  str  = None,   # None 默认 → 可选
    is_active: bool = True
):
    return {'page': page, 'size': size, 'keyword': keyword}

# 访问 /items?page=2&size=5&keyword=flask
```

### 4.3 路径参数 + 查询参数混用

```python
@app.get('/user/{user_id}/orders')
def get_user_orders(user_id: int, page: int = 1, size: int = 10):
    return {'user_id': user_id, 'page': page, 'size': size}
```

## 五、Pydantic 模型（请求体）

FastAPI 使用 **Pydantic** 定义请求体结构，自动完成校验、类型转换和文档生成。

### 5.1 定义数据模型

```python
from pydantic import BaseModel, Field
from typing import Optional

class UserCreate(BaseModel):
    username: str                          # 必填字段
    email:    str
    password: str
    age:      Optional[int] = None         # 可选字段
    role:     str           = 'user'       # 有默认值

class UserUpdate(BaseModel):
    username: Optional[str] = None
    email:    Optional[str] = None
```

### 5.2 在路由中使用

```python
@app.post('/user', status_code=201)
def create_user(user: UserCreate):      # FastAPI 自动解析请求体 JSON
    # user.username, user.email 直接访问
    return {'message': '创建成功', 'data': user.model_dump()}
```

### 5.3 Field 添加校验规则

```python
from pydantic import BaseModel, Field, EmailStr

# pip install pydantic[email]
class UserCreate(BaseModel):
    username: str = Field(..., min_length=2, max_length=20, description='用户名')
    email:    EmailStr                      # 自动校验邮箱格式
    password: str = Field(..., min_length=6, description='密码至少6位')
    age:      int = Field(None, ge=0, le=150)  # ge: >=0, le: <=150
```

## 六、响应模型

### 6.1 定义响应模型

```python
from pydantic import BaseModel
from datetime import datetime

class UserResponse(BaseModel):
    id:         int
    username:   str
    email:      str
    created_at: datetime
    
    class Config:
        from_attributes = True   # 允许从 ORM 对象构建（SQLAlchemy 集成用）
```

### 6.2 指定响应模型

```python
@app.get('/user/{user_id}', response_model=UserResponse)
def get_user(user_id: int):
    # 即使返回的字典有多余字段，也只会输出 UserResponse 中定义的字段
    return {
        'id': 1,
        'username': 'Su',
        'email': 'su@example.com',
        'created_at': datetime.now(),
        'password': 'secret'    # ← 不会出现在响应中，被过滤掉
    }
```

### 6.3 自定义响应状态码

```python
from fastapi import status

@app.post('/user', status_code=status.HTTP_201_CREATED)
def create_user(user: UserCreate):
    return {'message': '创建成功'}

@app.delete('/user/{user_id}', status_code=status.HTTP_204_NO_CONTENT)
def delete_user(user_id: int):
    return  # 204 不返回内容
```

## 七、请求头与 Cookie

```python
from fastapi import Header, Cookie
from typing import Optional

@app.get('/info')
def get_info(
    user_agent:    Optional[str] = Header(None),   # 自动读取 User-Agent 请求头
    authorization: Optional[str] = Header(None),   # 读取 Authorization
    session_id:    Optional[str] = Cookie(None)    # 读取 Cookie
):
    return {'user_agent': user_agent, 'session': session_id}
```

## 八、错误处理

### 8.1 HTTPException

```python
from fastapi import HTTPException

@app.get('/user/{user_id}')
def get_user(user_id: int):
    user = fake_db.get(user_id)
    if not user:
        raise HTTPException(
            status_code=404,
            detail='用户不存在'   # detail 会出现在响应体中
        )
    return user
```

### 8.2 全局异常处理器

```python
from fastapi import Request
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError

# 覆盖默认的 422 参数校验错误格式
@app.exception_handler(RequestValidationError)
async def validation_exception_handler(request: Request, exc: RequestValidationError):
    return JSONResponse(
        status_code=400,
        content={
            'code':    400,
            'message': '参数校验失败',
            'errors':  exc.errors()
        }
    )

# 自定义异常
class BusinessException(Exception):
    def __init__(self, message: str, code: int = 400):
        self.message = message
        self.code = code

@app.exception_handler(BusinessException)
async def business_exception_handler(request: Request, exc: BusinessException):
    return JSONResponse(
        status_code=exc.code,
        content={'code': exc.code, 'message': exc.message}
    )
```

## 九、异步路由

FastAPI 同时支持同步和异步函数：

```python
import asyncio

# 同步路由（适合 CPU 密集型，或调用同步库）
@app.get('/sync')
def sync_route():
    return {'type': 'sync'}

# 异步路由（适合 IO 密集型：数据库、HTTP请求、文件读写）
@app.get('/async')
async def async_route():
    await asyncio.sleep(1)   # 模拟异步 IO
    return {'type': 'async'}
```

> **建议**：能写 `async` 就写 `async`，最大化利用 FastAPI 的异步性能。

## 十二、小结

|概念|关键点|
|---|---|
|路由|`@app.get/post/put/delete()` 直接对应 HTTP 方法|
|路径参数|函数参数类型注解，自动校验转换|
|查询参数|不在路径中的函数参数，自动识别为查询参数|
|请求体|`BaseModel` 定义 Pydantic 模型，自动解析 JSON|
|响应模型|`response_model=` 过滤并规范响应结构|
|错误处理|`HTTPException` + `@app.exception_handler`|
|异步|`async def` 路由函数，天然支持高并发|

> ✅ 完成入门篇后，你应该能独立搭建一个带参数校验、自动文档的 FastAPI 服务。
