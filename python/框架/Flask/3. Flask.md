## 一、JWT 身份认证

Session 认证依赖服务端存储，不适合分布式场景。**JWT（JSON Web Token）** 是无状态的令牌认证方案，更适合前后端分离和微服务架构。

```bash
pip install flask-jwt-extended
```

### 1.1 初始化配置

```python
# app/extensions.py
from flask_jwt_extended import JWTManager
jwt = JWTManager()

# config.py
class BaseConfig:
    JWT_SECRET_KEY        = 'your-jwt-secret'
    JWT_ACCESS_TOKEN_EXPIRES  = timedelta(hours=2)    # Access Token 2小时过期
    JWT_REFRESH_TOKEN_EXPIRES = timedelta(days=30)    # Refresh Token 30天过期
```

### 1.2 登录发放 Token

```python
from flask_jwt_extended import create_access_token, create_refresh_token

@auth_bp.route('/login', methods=['POST'])
def login():
    data = request.get_json()
    user = User.query.filter_by(email=data['email']).first()
    
    if not user or not check_password_hash(user.password, data['password']):
        return jsonify({'message': '邮箱或密码错误'}), 401
    
    # identity 通常传用户ID（字符串）
    access_token  = create_access_token(identity=str(user.id))
    refresh_token = create_refresh_token(identity=str(user.id))
    
    return jsonify({
        'access_token':  access_token,
        'refresh_token': refresh_token
    })
```

### 1.3 保护接口 & 获取当前用户

```python
from flask_jwt_extended import jwt_required, get_jwt_identity

@user_bp.route('/profile')
@jwt_required()       # 校验 Authorization: Bearer <token>
def profile():
    user_id = get_jwt_identity()      # 从 Token 中取出 identity
    user = User.query.get(int(user_id))
    return jsonify(user.to_dict())
```

### 1.4 刷新 Token

```python
from flask_jwt_extended import jwt_required, create_access_token, get_jwt_identity

@auth_bp.route('/refresh', methods=['POST'])
@jwt_required(refresh=True)   # 使用 Refresh Token 校验
def refresh():
    identity     = get_jwt_identity()
    access_token = create_access_token(identity=identity)
    return jsonify({'access_token': access_token})
```

### 1.5 Token 黑名单（主动吊销）

```python
# 需要 Redis 存储黑名单
from app.extensions import redis_client

@jwt.token_in_blocklist_loader
def check_if_token_revoked(jwt_header, jwt_payload):
    jti = jwt_payload['jti']
    return redis_client.get(f'blocklist:{jti}') is not None

@auth_bp.route('/logout', methods=['DELETE'])
@jwt_required()
def logout():
    jti = get_jwt()['jti']
    redis_client.setex(f'blocklist:{jti}', timedelta(hours=2), 'revoked')
    return jsonify({'message': '已退出登录'})
```

## 二、Redis 缓存

```bash
pip install redis flask-caching
```

### 2.1 初始化

```python
# app/extensions.py
import redis
from flask_caching import Cache

cache = Cache()
redis_client = redis.Redis(host='localhost', port=6379, db=0, decode_responses=True)

# config.py
class BaseConfig:
    CACHE_TYPE            = 'RedisCache'
    CACHE_REDIS_HOST      = 'localhost'
    CACHE_REDIS_PORT      = 6379
    CACHE_DEFAULT_TIMEOUT = 300  # 默认缓存 5 分钟
```

### 2.2 视图缓存（装饰器方式）

```python
from app.extensions import cache

@user_bp.route('/list')
@cache.cached(timeout=60, key_prefix='user_list')  # 缓存 60 秒
def get_user_list():
    users = User.query.filter_by(is_active=True).all()
    return jsonify([u.to_dict() for u in users])
```

### 2.3 手动缓存（精细控制）

```python
from app.extensions import cache

def get_user_by_id(user_id):
    cache_key = f'user:{user_id}'
    
    # 先查缓存
    user_data = cache.get(cache_key)
    if user_data:
        return user_data
    
    # 缓存未命中，查数据库
    user = User.query.get(user_id)
    if user:
        user_data = user.to_dict()
        cache.set(cache_key, user_data, timeout=300)
    
    return user_data

def update_user(user_id, data):
    user = User.query.get(user_id)
    # ... 更新操作
    db.session.commit()
    
    # 更新后删除缓存，避免数据不一致
    cache.delete(f'user:{user_id}')
```

### 2.4 直接操作 Redis

```python
from app.extensions import redis_client

# 字符串
redis_client.set('key', 'value', ex=3600)   # ex 单位秒
value = redis_client.get('key')

# 哈希（适合存对象）
redis_client.hset('user:1', mapping={'name': 'Su', 'role': 'admin'})
user = redis_client.hgetall('user:1')

# 计数器（如接口限流）
redis_client.incr('api:calls:today')
redis_client.expire('api:calls:today', 86400)

# 列表（消息队列简单场景）
redis_client.lpush('task_queue', 'task_data')
task = redis_client.rpop('task_queue')
```

## 三、接口限流

防止接口被恶意刷请求，常见策略是基于 Redis 实现滑动窗口限流。

```bash
pip install flask-limiter
```

```python
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address

limiter = Limiter(
    app,
    key_func=get_remote_address,    # 以客户端 IP 为限流 key
    default_limits=['200/day', '50/hour'],

)

# 针对特定接口单独设置
@auth_bp.route('/login', methods=['POST'])
@limiter.limit('5/minute')   # 登录接口每分钟最多 5 次

    ...
```

## 四、异步任务（Celery）

耗时操作（发邮件、生成报告、AI 推理等）不应阻塞 HTTP 请求，通过 **Celery** 异步处理。

```bash
pip install celery redis
```

### 4.1 配置 Celery

```python
# app/celery_app.py
from celery import Celery

def make_celery(app):
    celery = Celery(
        app.import_name,
        broker='redis://localhost:6379/1',    # 消息代理
        backend='redis://localhost:6379/2'   # 结果存储
    )
    celery.conf.update(app.config)
    
    class ContextTask(celery.Task):
        def __call__(self, *args, **kwargs):
            with app.app_context():           # 让任务运行在 Flask 上下文中
                return self.run(*args, **kwargs)
    
    celery.Task = ContextTask
    return celery
```

### 4.2 定义任务

```python
# app/tasks/email_tasks.py
from app.celery_app import celery

@celery.task(bind=True, max_retries=3)
def send_email_task(self, to_email, subject, content):
    try:
        # 发送邮件逻辑
        send_email(to_email, subject, content)
        return {'status': 'sent', 'to': to_email}
    except Exception as exc:
        # 失败后延迟重试（指数退避）
        raise self.retry(exc=exc, countdown=2 ** self.request.retries)
```

### 4.3 在接口中触发任务

```python
from app.tasks.email_tasks import send_email_task

@auth_bp.route('/register', methods=['POST'])
def register():
    # ... 注册逻辑
    db.session.commit()
    
    # 异步发送欢迎邮件，不阻塞接口响应
    send_email_task.delay(
        to_email=new_user.email,
        subject='欢迎注册',
        content='感谢您的注册...'
    )
    
    return jsonify({'message': '注册成功，请查收邮件'}), 201
```

### 4.4 启动 Celery Worker

```bash
# 启动 Worker
celery -A run.celery worker --loglevel=info

# 定时任务（使用 Celery Beat）
celery -A run.celery beat --loglevel=info
```

## 六、请求参数校验（marshmallow）

用 Schema 做请求体的自动校验和序列化，比手动 `if` 判断优雅得多。

```bash
pip install marshmallow flask-marshmallow
```

```python
# app/schemas/user_schema.py
from marshmallow import Schema, fields, validate, ValidationError

class UserRegisterSchema(Schema):
    username = fields.Str(required=True, validate=validate.Length(min=2, max=20))
    email    = fields.Email(required=True)
    password = fields.Str(required=True, validate=validate.Length(min=6))

register_schema = UserRegisterSchema()

# 在路由中使用
@auth_bp.route('/register', methods=['POST'])
def register():
    try:
        data = register_schema.load(request.get_json())   # 校验 + 反序列化
    except ValidationError as err:
        return jsonify({'code': 400, 'errors': err.messages}), 400
    
    # data 已经过校验，可以安全使用
    # ...
```

## 七、日志系统

```python
# app/logger.py
import logging
from logging.handlers import RotatingFileHandler
import os

def setup_logger(app):
    if not app.debug:
        # 生产环境写文件日志
        if not os.path.exists('logs'):
            os.mkdir('logs')
        
        handler = RotatingFileHandler(
            'logs/flask.log',
            maxBytes=10 * 1024 * 1024,   # 单文件最大 10MB
            backupCount=10               # 保留 10 个历史文件
        )
        handler.setLevel(logging.INFO)
        formatter = logging.Formatter(
            '[%(asctime)s] %(levelname)s %(module)s: %(message)s'
        )
        handler.setFormatter(formatter)
        app.logger.addHandler(handler)
    
    app.logger.setLevel(logging.INFO)

# 在路由中使用
from flask import current_app

@user_bp.route('/<int:user_id>')
def get_user(user_id):
    current_app.logger.info(f'查询用户 ID: {user_id}')
    # ...
```

## 八、生产部署

### 8.1 使用 Gunicorn 运行

Flask 内置服务器不适合生产，使用 **Gunicorn** 作为 WSGI 服务器：

```bash
pip install gunicorn

# 启动（4 个 worker 进程）
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app('production')"

# 推荐配置文件方式
gunicorn -c gunicorn.conf.py "app:create_app('production')"
```

```python
# gunicorn.conf.py
bind             = '0.0.0.0:5000'
workers          = 4               # 通常设为 CPU 核心数 × 2 + 1
worker_class     = 'sync'          # IO 密集型可选 gevent
timeout          = 120
accesslog        = 'logs/access.log'
errorlog         = 'logs/error.log'
loglevel         = 'info'
```

### 8.2 Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass         http://127.0.0.1:5000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态资源由 Nginx 直接处理，不经过 Flask
    location /static {
        alias /path/to/your/app/static;
        expires 7d;
    }
}
```

### 8.3 Docker 容器化

```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:5000", "app:create_app('production')"]
```

## 九、性能优化要点总结

|优化方向|具体措施|
|---|---|
|**数据库查询**|使用索引；避免 N+1 查询（用 `joinedload`）；只查需要的字段|
|**缓存策略**|热点数据 Redis 缓存；页面级缓存 `@cache.cached`|
|**连接池**|合理设置 `pool_size`、`pool_recycle`|
|**异步解耦**|耗时操作丢给 Celery，接口立即返回|
|**并发模型**|CPU 密集 → 多进程（Gunicorn sync worker）；IO 密集 → 协程（gevent worker）|
|**接口限流**|Flask-Limiter 防止恶意刷接口|
|**日志采集**|生产环境接入 ELK / Loki 做集中日志管理|
